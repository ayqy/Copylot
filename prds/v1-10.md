# V1-10 表格内选区复制一致性修复（整表优先 + 去除双重可见性裁剪）

## 目标
修复“表格内选区转换只输出局部碎片、丢失表格结构”的问题，并消除选区回退链路的双重可见性裁剪，确保 Popup/右键菜单的一键转换在表格场景下输出稳定可读（Markdown/CSV 按设置），同时补齐可执行的回归用例与文档，使之达到验收标准、可发布。

**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、表格内选区策略：在“精确选区”之前优先整表（修复 `review-issues.md` ISSUE-007）
  - 现状：`src/content/content.ts` 的 `CONVERT_PAGE_WITH_SELECTION` 优先 `getPreciseSelectedElement()`；用户在表格单元格内划词时，会只处理 fragment，导致输出缺表格结构。
  - 改造规则（必须可回归、尽量不改变非表格场景）：
    - 当存在有效选区，且选区的 `getSelectedElement()` 可命中 `getTableAncestor(...)` 时：
      - 以该 `HTMLTableElement` 作为处理根节点（等价于“整表复制/转换”），忽略 fragment。
    - 非表格场景：继续使用 `getPreciseSelectedElement()`（保留选区 HTML 结构，如 `<a>/<b>` 等）。
    - 需要统一应用到以下入口（保证行为一致）：
      - Popup “Convert Page”按钮 -> `CONVERT_PAGE_WITH_SELECTION`
      - 右键菜单 “Convert Page…” -> `CONVERT_PAGE_WITH_SELECTION`
      -（如仍保留）`PROCESS_SELECTION_WITH_PROMPT` 的选区处理入口
  - 代码组织要求：
    - 在 `src/content/content.ts` 新增一个集中函数（例如 `getSelectionRootElementForProcessing()`），封装“表格祖先优先 + 精确选区回退”的选择逻辑，避免多处复制判断。

2、去除选区回退链路的双重可见性裁剪（修复 `review-issues.md` ISSUE-010）
  - 现状：`src/shared/content-processor.ts` 的 `processElementWithTableDetection()` 会先 `createVisibleClone()`，随后又调用 `processContent()`（内部再次 `createVisibleClone()`），导致同一选区回退路径出现双重裁剪。
  - 改造要求：
    - `src/content/content.ts` 中不再调用 `processElementWithTableDetection()`；统一改为调用 `processContent()`。
    - `src/shared/content-processor.ts` 中将 `processElementWithTableDetection()` 收敛为对 `processContent()` 的薄封装（或直接废弃但保持编译/inline 不报错），保证全链路最多只做一次 `createVisibleClone()`。
    - 不引入新依赖；不改变表格 Markdown/CSV 输出规则（仍以 `Settings.tableOutputFormat` 为准）。

3、回归测试资产（便于 1 分钟复现 + 可长期维护）
  - 新增可复现页面：`test/cases/table-selection-inside-table.html`
    - 包含：2x2 以上表格 + 单元格内含内联元素（如 `<a>/<code>`）+ 表格前后普通段落文本（用于验证“表格内/外选区”差异）
  - 新增期望快照：`test/snapshots/table-selection-inside-table.expected.md`（用于 `public/test/index.html` 的 DOM 测试 runner；只校验整页 `processContent` 结果即可）
  - 更新测试清单：执行 `npm run build:tests` 生成最新 `test/test-manifest.json`
  - 验收前确保 `bash scripts/test.sh` 全量通过（lint/type-check/check-i18n/unit-tests/build:prod）

4、补齐测试用例文档（可执行、可回归）
  - 新增并维护测试/用例文档：`docs/test-cases/v1-10.md`
  - 文档必须覆盖（至少）：
    - 本地复现准备：用 `python3 -m http.server` 启动本地静态页（避免 file:// 权限差异），打开 `test/cases/table-selection-inside-table.html`
    - 用例 A（表格内选区）：在单元格内划词 -> 触发 “Convert Page（含选区）” -> 输出必须为整表 Markdown/CSV（按设置），不得仅输出局部碎片
    - 用例 B（表格外选区）：在表格外段落划词 -> 触发转换 -> 输出应仅包含选区内容（保持精确选区能力）
    - 负例：无选区 -> 触发转换 -> 仍为整页转换（与现有行为一致）
    - 回归：不影响 Magic Copy 主按钮复制、表格转 CSV、Prompt 处理整页、追加模式等既有路径
    - 自动化回归：记录一次 `bash scripts/test.sh` 执行结果（命令 + 结论）
    - 可选：记录一次 `npm run build:tests` + 打开 `public/test/index.html` 运行新增用例的结果（PASS/FAIL + 结论）
