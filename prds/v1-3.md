# V1-3 低打扰评价引导 MVP（触发条件计数 + 仅提示 1 次）

## 目标
在不影响现有复制体验、且不引入任何“匿名数据采集/联网埋点”的前提下，基于本地可审计的使用计数实现一次性“去评价”引导（可稍后/不再提示），提升商店评分与评价率；所有能力达到可验收、可测试、可发布状态。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、增长触发所需的本地使用计数（隐私合规、低开销、可审计）
  - 新增本地统计存储（建议使用 `chrome.storage.local`，避免 `sync` 高频写入限额）
    - 建议 Key：`copilot_growth_stats`
    - 字段（仅数字/枚举/时间戳；不得包含任何用户复制内容、页面内容、URL）：
      - `installedAt: number`（首次安装时间；由 `src/background.ts` 在 `onInstalled(reason=install)` 初始化）
      - `successfulCopyCount: number`（成功复制次数：Magic Copy 主按钮/Prompt 复制/Convert Page/Append Mode 成功写入剪贴板均计入；分享文案复制不计入）
      - `ratingPromptShownAt?: number`（第一次展示评价引导的时间；用于确保“仅提示 1 次”）
      - `ratingPromptAction?: 'rate' | 'later' | 'never'`（用户选择；用于后续分析/回归，不用于再次弹出）
      - `ratingPromptActionAt?: number`
  - 增量写入路径（需可回归、避免漏计/重复计）：
    - Content Script 侧在“确认写入剪贴板成功”后上报一次成功复制（建议通过 `chrome.runtime.sendMessage` 交由 `src/background.ts` 统一累加到 `chrome.storage.local`）
    - Append Mode（`copy-to-clipboard` / `copy-to-clipboard-from-background`）链路：以 content script 最终写入成功为准
    - 明确“成功复制”的判定与去重策略（例如：一次用户点击触发只累计 1 次；失败不累计）

2、Popup 内一次性评价引导（零打扰、可验证、可 i18n）
  - 触发条件（以 Popup 打开时评估为准）：
    - `installedAt` 距当前时间 >= 3 天（72 小时）
    - `successfulCopyCount >= 20`
    - `ratingPromptShownAt` 不存在（从未展示过）
  - 展示形态（最小可用，不阻断用户操作）：
    - Popup 内的轻量提示条/卡片（不新开 Tab、不使用系统 Notification）
    - 出现时即写入 `ratingPromptShownAt=Date.now()`，确保即使用户关闭 Popup 也不会重复提示
    - 需提供稳定 DOM 标识（便于用例与自动化扩展），建议：
      - 容器：`#rating-prompt`
      - 按钮：`#rating-prompt-rate`、`#rating-prompt-later`、`#rating-prompt-never`
  - 交互与落盘（必须可手工验证）：
    - “去评价”：打开 Chrome Web Store reviews 页面（复用 `src/shared/word-of-mouth.ts` 的 `buildChromeWebStoreReviewsUrl`）；落盘 `ratingPromptAction='rate'`
    - “稍后”：关闭提示；落盘 `ratingPromptAction='later'`（本版本不再自动提示）
    - “不再提示”：关闭提示；落盘 `ratingPromptAction='never'`
  - i18n（至少中英文）：
    - 新增/补齐 `_locales/en/messages.json`、`_locales/zh/messages.json` 对应文案 key
  - 建议实现位置（用于落地与 code review 对齐）：
    - UI：`src/popup/popup.html`、`src/popup/popup.css`
    - 逻辑：`src/popup/popup.ts`、`src/background.ts`（统计存储与消息处理）

3、测试与用例文档（可执行）
  - 新增并维护测试/用例文档：`docs/test-cases/v1-3.md`
  - 文档必须覆盖（至少）：
    - 清空本地存储后的初始化：`installedAt/successfulCopyCount` 的创建与默认值
    - “成功复制次数”累计：主复制/Prompt 复制/Convert Page/Append Mode 成功路径均会 +1；失败不会 +1；分享文案复制不会 +1
    - 满足触发条件后打开 Popup 展示评价引导；且“仅展示 1 次”（关闭 Popup 或点击任意按钮后都不再出现）
    - 三个按钮行为：reviews 跳转、稍后关闭、不再提示关闭；并校验本地落盘字段
    - 中英文 i18n 展示
    - 隐私校验：本地存储中不出现任何复制内容/页面内容/URL
  - 自动化/单测最小补齐：
    - 在 `scripts/unit-tests.ts` 增加对“触发条件判断”纯函数（例如 `shouldShowRatingPrompt(stats, now)`）的单测覆盖
  - 在验收前完成并记录：`bash scripts/test.sh` 通过

## 验收标准
* 修复上述所有问题，并测试通过，达到可发布状态
* 实现以上所有需求，并测试通过，达到可发布状态
* 当满足“安装满 3 天且成功复制 >= 20 次且从未展示过”条件时，打开 Popup 会展示评价引导；出现后即写入 `ratingPromptShownAt`，确保后续不再重复提示
* 评价引导提供“去评价/稍后/不再提示”三种操作：reviews 跳转可用、关闭可用、状态可持久化且可通过查看 `chrome.storage.local` 验证
* “成功复制次数”仅在复制成功时累计；分享文案复制不计入；本地统计仅包含时间戳/计数/枚举，不包含任何用户复制内容或页面内容
