# V1-6 商业化意向验证 MVP（升级 Pro / 候补名单入口）测试用例

## 目标
验证“升级 Pro / 加入候补名单”最小闭环在不新增权限、不影响现有免费功能、且符合“本地处理/隐私优先/零打扰”原则下可用且可回归：
- Popup/Options 存在稳定入口，可跳转/可复制
- `src/options/options.html#pro` hash 可定位到 Pro Tab
- 候补登记跳转到 GitHub `issues/new` 并预填 title/body（仅环境信息 + 占位提示）
- “匿名使用数据”默认关闭；开启后才记录本地匿名意向事件；关闭后立即清空且不再记录
- `bash scripts/test.sh` 全量通过，可发布

## 测试前置
1. 确保扩展已加载（建议加载开发构建或 `npm run build:prod` 产物）。
2. 打开 Options 页，找到“匿名使用数据”开关（默认应为关闭）。
3. 打开 Options 页 DevTools Console，用于执行下列校验脚本。

校验脚本（读取日志）：
```js
await chrome.storage.local.get('copilot_telemetry_events')
```

清理脚本（清空日志）：
```js
await chrome.storage.local.remove('copilot_telemetry_events')
```

## 用例 1：入口可见 + Popup 点击定位 Pro Tab
步骤：
1. 打开 Popup，确认存在“升级 Pro”入口（稳定 DOM：`#upgrade-pro-entry`）。
2. 点击“升级 Pro”。

期望：
- 自动打开 Options 页，URL 包含 `#pro`；
- Options 默认激活 Pro Tab（稳定选择器：`.tab-btn[data-tab="pro"]`，内容容器：`#pro-tab`）；
- 不影响 Popup 内现有“AI Convert/管理 Prompts/反馈/分享/去评价”等功能。

## 用例 2：候补登记跳转预填（仅环境信息 + 占位提示）
步骤：
1. 在 Popup 点击“加入候补名单”（按钮：`#popup-pro-waitlist`）或在 Options 的 Pro Tab 点击“加入候补名单”（按钮：`#pro-waitlist-button`）。
2. 在新打开的 GitHub `issues/new` 页面检查 Title/Body 预填内容。

期望：
- 跳转 URL 指向 GitHub `issues/new`；
- Title/Body 已预填；
- Body 仅包含环境信息与占位提示，至少包含：
  - 扩展版本（`chrome.runtime.getManifest().version`）
  - 扩展 ID（`chrome.runtime.id`）
  - `navigator.language`
  - UI 语言（`chrome.i18n.getUILanguage()`）
- Body 明确提示：不要填写任何网页内容或复制内容；
- Body 不包含任何用户复制内容/页面内容/URL/标题。

## 用例 3：默认关闭不记录 pro_* 事件
步骤：
1. 确认 Options 页“匿名使用数据”开关为关闭。
2. 执行清理脚本，确保日志为空。
3. 执行下列动作（任选其一或多项）：
   - Popup 点击“升级 Pro”
   - Popup 点击“加入候补名单”
   - Options 切换到 Pro Tab
   - Options 点击“加入候补名单”
   - Options 点击“复制候补文案”
4. 执行读取日志脚本。

期望：
- `copilot_telemetry_events` 不存在或为空数组。

## 用例 4：开启后记录 pro_* 事件 + props 白名单
步骤：
1. 打开 Options 页，将“匿名使用数据”开关打开。
2. 执行清理脚本，确保从空开始。
3. 执行以下动作：
   - Popup 点击“升级 Pro”（记录 `pro_entry_opened`，`source='popup'`）
   - Options 打开/切换到 Pro Tab（记录 `pro_entry_opened`，`source='options'`）
   - Popup 或 Options 点击“加入候补名单”（记录 `pro_waitlist_opened`，`source` 对应入口）
   - Options 点击“复制候补文案”（记录 `pro_waitlist_copied`，`source='options'`；仅写入剪贴板成功时记录）
4. 执行读取日志脚本，检查事件对象结构与 props。

期望：
- 日志存在且新增 `pro_*` 事件；
- 每条事件仅包含：`name`（字符串）、`ts`（正数时间戳）、可选 `props`；
- `props` 字段严格白名单过滤，且值类型仅 `string|number|boolean`：
  - `pro_entry_opened.props.source` 为 `'popup'|'options'`
  - `pro_waitlist_opened.props.source` 为 `'popup'|'options'`
  - `pro_waitlist_copied.props.source` 为 `'options'`

## 用例 5：最多 100 条 + FIFO（丢弃最旧）
步骤：
1. 打开 Options 页，将“匿名使用数据”开关打开。
2. 在 Options DevTools Console 执行以下脚本，预置 100 条合法事件：
```js
const key = 'copilot_telemetry_events';
const base = Date.now();
const events = Array.from({ length: 100 }, (_, i) => ({ name: 'popup_opened', ts: base + i }));
await chrome.storage.local.set({ [key]: events });
```
3. 触发一次新的可记录事件（例如：Options 切换到 Pro Tab 或 Popup 点击“升级 Pro”）。
4. 执行读取日志脚本，检查数组长度与首条事件。

期望：
- 数组长度仍为 `100`；
- 最旧的一条被丢弃（首条 `ts` 应大于 `base`，即 `base + 1`）。

## 用例 6：关闭后立刻停止追加并清空
步骤：
1. 在“匿名使用数据”开启状态下，先触发至少 1 条 `pro_*` 事件。
2. 读取日志脚本确认存在事件数组。
3. 关闭 Options 页“匿名使用数据”开关。
4. 读取日志脚本确认已清空（key 不存在或为空）。
5. 再次触发任意 `pro_*` 行为（切换 Pro Tab / 点击入口等）。
6. 再次读取日志脚本。

期望：
- 第 4 步后 `copilot_telemetry_events` 被移除或为空；
- 第 6 步仍保持为空（不发送、不缓存、不补发）。
