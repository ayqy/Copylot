1. 执行：`npm run publish:cws -- --dry-run`
2. 检查输出包含以下稳定标记（可复制、无敏感信息）：
   - `-----BEGIN CWS PROXY DIAGNOSTIC BLOCK-----`
   - `-----END CWS PROXY DIAGNOSTIC BLOCK-----`

期望：

## 用例 F（需网络可达）：真实发布（upload/publish）与商店端取证

## 用例 G（需网络可达）：从商店安装回归 + 导出摘要/证据包并落盘

步骤：
1. 从 Chrome Web Store 安装当前版本（禁止用 zip/unpacked 替代）。
2. 打开 Options -> 隐私与可观测性：
   - 开启匿名使用数据开关（若需要采集 WOM/Pro 事件；默认 OFF）
3. 触发最短路径事件（确保有可导出的计数）：
   - Pro：打开 Popup/Options 的 Pro 候补入口（entry/opened/copied）
   - WOM：打开分享/去评价/反馈入口；若评分引导出现，完成一次操作（rate/later/never）
4. 导出并落盘（固定命名，便于对比）：
   - `docs/evidence/v1-45/pro-funnel-summary.json`
   - `docs/evidence/v1-45/pro-funnel-evidence-pack.json`
   - `docs/evidence/v1-45/wom-summary.json`
   - `docs/evidence/v1-45/wom-evidence-pack.json`
5. 更新 `docs/evidence/v1-45/index.md` 的“证据文件清单”条目。

期望：
- 导出文件结构与基线一致，可直接与 `docs/evidence/v1-42/`、`docs/evidence/v1-44/` 对比。
