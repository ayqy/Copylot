## 具体任务
1、Options -> Pro Tab 增加“渠道（campaign）”字段，并写入候补/问卷留资文本（让留资天然可归因）
  - UI：在 `src/options/options.html` 的 Pro 候补行动区新增可选输入项「渠道（campaign）」与提示文案（明确“不要填写敏感信息”）
    - 建议稳定 DOM：`#pro-intent-campaign`
    - 默认值为空；仅允许短字符串（需做长度/字符集校验，避免写入敏感信息或污染 telemetry）
  - 写入范围（只要用户走“留资动作”，就必须携带 campaign）：
    - Options -> Pro：`#pro-waitlist-copy` 复制候补文案：复制内容中新增 `campaign: <value>`（campaign 为空则不写该行）
    - Options -> Pro：`#pro-waitlist-button` 加入候补名单：打开的 GitHub new issue body 中新增 `campaign: <value>`（为空则不写）
    - Options -> Pro：`#pro-waitlist-survey-copy`/`#pro-waitlist-survey-copy-open` 复制问卷：复制的问卷 Markdown 中新增 `campaign: <value>`（为空则不写）
  - 约束：不新增任何网络请求；不改变既有“候补登记不预填网页内容/复制内容”的隐私边界

2、可量化取证：campaign 进入本地匿名事件 props + 周摘要/证据包导出可复核（让商业化证据可执行）
  - Telemetry props（仅匿名开关 ON 时记录；默认 OFF 不记录）：
    - 为以下 Pro 意向事件补齐 `campaign` 字段：`pro_entry_opened/pro_waitlist_opened/pro_waitlist_copied/pro_waitlist_survey_copied`
    - 规则：campaign 为空则不写入 props；非空则写入并通过白名单与合法性校验（避免污染 telemetry）
  - 导出与复核：
    - Options -> 隐私与可观测性 -> Pro 意向漏斗证据包：events 明细中可看到 `props.campaign`
    - “复制本周 Pro 意向证据摘要（7d window）”（`#copy-pro-intent-weekly-digest`）：在 Markdown 末尾追加 campaign 拆分统计（至少覆盖 `pro_waitlist_copied/pro_waitlist_survey_copied` 的计数），用于对比不同渠道的留资效率
  - 文档同步：更新 `docs/growth/telemetry-events.md`，补齐 campaign 的白名单 key、取值约束与隐私说明
  - 兼容性红线：不得修改 v1-51 的 7d 明细 CSV 固定字段与既有统计口径；campaign 只作为新增“证据维度”出现在 events props 与 weekly digest 附加段

3、补齐测试/用例与证据落盘（确保 1~3 天可交付、可回归、可审计）
  - 新增测试用例文档（固定路径）：`docs/test-cases/v1-53.md`
    - 覆盖：campaign 为空/非空；复制候补/打开候补/复制问卷三条链路；匿名开关 OFF 不记录、ON 记录并可导出；weekly digest campaign 拆分口径可复核
  - 自动化测试加固（优先纯函数单测）：覆盖 campaign 合法性校验、telemetry sanitize 白名单、weekly digest 按 campaign 聚合逻辑；并保证 `bash scripts/test.sh` 全量通过
  - 新增证据目录与索引（固定路径）：`docs/evidence/v1-53/` + `docs/evidence/v1-53/index.md`
