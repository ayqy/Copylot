# 问题修复和功能优化（V1-20 反馈模板补齐漏斗摘要：Issue 模板附带 GrowthFunnelSummary）

## 目标
在不新增权限、不新增任何联网发送、且不包含任何网页/复制内容的前提下，扩展 Popup/Options 的“反馈与建议”Issue 模板：在现有 `copilot_settings` 快照 + `copilot_growth_stats` +（last N）`copilot_telemetry_events` 的基础上，追加一段**可审计**的“本地漏斗摘要”（`GrowthFunnelSummary`，含“首次打开 Popup 后 3 分钟内首次成功复制”判断），并补齐单测与测试用例文档，使之达到验收标准、可发布。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、扩展反馈模板生成能力：支持注入并渲染 `GrowthFunnelSummary`（严格隐私合规、仅读取不落盘）
  - 修改 `src/shared/word-of-mouth.ts`
    - 类型扩展：
      - 从 `src/shared/growth-stats.ts` 引入 `GrowthFunnelSummary`
      - 为 `BuildFeedbackIssueParams` 新增可选字段：`growthFunnelSummarySnapshot?: GrowthFunnelSummary`
    - 模板渲染：
      - 在 `buildFeedbackIssueUrl(...)` 中将 `growthFunnelSummarySnapshot` 序列化为 pretty JSON（缺失/失败兜底为 `{}`）
      - 将该 JSON 作为新增占位符注入 `feedbackIssueBodyTemplate`
    - 约束：
      - `GrowthFunnelSummary` 仅允许 `number/boolean/undefined` 字段（不得拼装任何网页内容、URL、标题、复制内容）
      - 不新增任何 storage 写入（仅用于生成 URL query）
      - URL 长度约束保持保守：仍只允许在超长时截断 telemetry events（漏斗摘要/设置/增长统计不得截断）

2、Popup 与 Options 反馈入口接入漏斗摘要（不新增事件名、不改动现有入口行为）
  - Popup：`src/popup/popup.ts` 的 `#feedback-link`
    - 在读取到 `growthStatsSnapshot` 后，使用 `buildGrowthFunnelSummary(growthStatsSnapshot, Date.now())` 构造 `growthFunnelSummarySnapshot`
    - 调用 `buildFeedbackIssueUrl(...)` 时注入 `growthFunnelSummarySnapshot`
    - 失败兜底：任意读取/构造失败不得阻断打开反馈页（仅 `console.warn` + 兜底 `{}`）
    - 埋点约束：仍只复用既有 `wom_feedback_opened`（仅在“匿名使用数据”开关开启时本地记录）
  - Options：`src/options/options.ts` 的 WOM 反馈入口（`#wom-feedback-open` -> `buildWomFeedbackUrl()`）
    - 同 Popup：注入 `growthFunnelSummarySnapshot`
    - 保持与 Popup 行为一致（同样的兜底/隐私约束）

3、更新 i18n 模板（中英文一致、可通过 check-i18n）
  - 更新 `_locales/zh/messages.json` 与 `_locales/en/messages.json`：
    - 在 `feedbackIssueBodyTemplate` 中新增段落（顺序建议放在 `copilot_growth_stats` 之后）：
      - 标题建议：`## 本地漏斗摘要 (copilot_growth_funnel_summary)`
      - 内容：JSON code block
    - 新增占位符 `GROWTH_FUNNEL_JSON`（占位符序号需与 `buildFeedbackIssueUrl(...)` substitutions 顺序一致；不得遗漏/错位）
  - 确保 `npm run check-i18n` 通过

4、补齐单测与测试用例文档（可执行、可回归、可发布）
  - 单测（最小但必须）：
    - 位置：`scripts/unit-tests.ts`
    - 更新 `getMessage('feedbackIssueBodyTemplate', substitutions)` 的测试 stub，使其覆盖新增占位符（包含 `copilot_growth_funnel_summary=$GROWTH_FUNNEL_JSON$` 片段）
    - 覆盖断言（至少）：
      - `body` 中包含 `copilot_growth_funnel_summary`
      - `body` 中的漏斗摘要 JSON 片段包含关键字段名（例如 `activatedWithin3MinutesFromFirstPopup` / `timeFromFirstPopupToFirstCopyMs`）
      - 继续保持既有断言：settings 快照不包含 `userPrompts/template/title` 等用户输入文本；telemetry events 仍按 `ts` 倒序且在开关关闭时为 `[]`
  - 测试用例文档：
    - 新增并维护 `docs/test-cases/v1-20.md`，覆盖（至少）：
      - 用例 A（Popup 入口）：从 Popup `#feedback-link` 打开 GitHub `issues/new`，body 预填包含 5 块：环境信息、`copilot_settings`、`copilot_growth_stats`、`copilot_growth_funnel_summary`、`copilot_telemetry_events`
      - 用例 B（Options 入口）：从 Options `#wom-feedback-open` 打开行为与 Popup 一致
      - 用例 C（隐私校验）：`copilot_growth_funnel_summary` 仅包含时间戳/计数/布尔/耗时；不包含网页内容、URL、标题、复制内容
      - 用例 D（匿名使用数据关闭）：`copilot_telemetry_events` 为 `[]`；其余 4 块仍存在；打开反馈页不报错
      - 自动化回归：记录一次 `bash scripts/test.sh` 结论（命令 + 通过/失败）
  - 验收前完成回归：确保不影响既有“分享/评价/Pro 候补/隐私面板/新手引导/Prompt 管理”等功能，达到可发布状态
