# V1-22 匿名本地埋点枚举值收敛：source/action 严格过滤 + 单测/用例补齐 测试用例

## 目标
验证 V1-22 达到可发布：
- 本地匿名事件 `copilot_telemetry_events` 中枚举型 props（`source/action`）仅允许固定取值；非法值在 `sanitizeTelemetryEvent` 中丢弃（事件保留，但 `props` 可能变为 `undefined`）
- Popup/Options 的现有埋点调用仅产出合法枚举值（不拼接动态字符串）
- “匿名使用数据”开关关闭时：不记录、不缓存、不补发；且关闭即清空本地事件
- `bash scripts/test.sh` 全量通过

## 预置条件
- 已在 Chrome/Chromium 加载本地扩展（开发包或生产包均可）
- 能打开 Popup 与 Options 页面：
  - Popup：点击扩展图标打开 Popup（`src/popup/popup.html`）
  - Options：右键扩展 -> 选项，或 `chrome.runtime.openOptionsPage()`（`src/options/options.html`）
- 能打开 `chrome://extensions` -> 本扩展 -> “Service worker” -> Inspect（用于检查 `chrome.storage.local` / `chrome.storage.sync`）

## 用例 A（开关开启记录：Pro/WOM 的 source 值域）
步骤：
1. 打开 Options -> 隐私与可观测性，开启“匿名使用数据”（`#anonymous-usage-data-switch` 置为 on）。
2. 触发 Popup 的 Pro 入口（任选其一）：
   - 点击 `#upgrade-pro-entry`（`pro_entry_opened`，`source='popup'`）
   - 点击 `#popup-pro-waitlist`（`pro_waitlist_opened`，`source='popup'`）
3. 触发 Popup 的 WOM 入口（任选其一）：
   - 点击 `#feedback-link`（`wom_feedback_opened`，`source='popup'`）
   - 点击 `#share-link`（`wom_share_opened`，`source='popup'`）
   - 点击 `#rate-link`（`wom_rate_opened`，`source='popup'`）
   - 点击“复制分享文案”（`wom_share_copied`，`source='popup'`，需写入剪贴板成功）
4. 触发 Options 的 Pro 入口（任选其一）：
   - 激活 Pro Tab（Options URL `#pro`，或点击 `data-tab="pro"` 的 Tab 按钮）（`pro_entry_opened`，`source='options'`）
   - 点击 `#pro-waitlist-button`（`pro_waitlist_opened`，`source='options'`）
   - 点击 `#pro-waitlist-copy`（`pro_waitlist_copied`，`source='options'`，需写入剪贴板成功）
5. 触发 Options 的 WOM 入口（任选其一）：
   - 点击 `#wom-feedback-open`（`wom_feedback_opened`，`source='options'`）
   - 点击 `#wom-share-open`（`wom_share_opened`，`source='options'`）
   - 点击 `#wom-rate-open`（`wom_rate_opened`，`source='options'`）
   - 点击 `#wom-share-copy`（`wom_share_copied`，`source='options'`，需写入剪贴板成功）
6. 在 Service worker console 执行：
   - `await chrome.storage.local.get('copilot_telemetry_events')`
7. 在结果里找到最近的 `pro_*` / `wom_*` 事件，检查 `props.source`。

期望：
- `pro_entry_opened` / `pro_waitlist_opened`：`props.source` 仅可能为 `'popup' | 'options'`
- `pro_waitlist_copied`：若存在 `props`，则 `props.source === 'options'`
- `wom_*`：若存在 `props`，则 `props.source` 仅可能为 `'popup' | 'options'`
- 不应出现任何其它 `source` 值（非法值会在 sanitize 阶段被丢弃）

## 用例 B（引导埋点：onboarding_* 的 source/action 值域）
步骤：
1. 确保“匿名使用数据”开关开启（`#anonymous-usage-data-switch` 置为 on）。
2. 重置引导完成状态（Service worker console 执行）：
   - `const key='copilot_settings'; const s=(await chrome.storage.sync.get(key))[key]||{}; await chrome.storage.sync.set({[key]: {...s, popupOnboardingCompletedVersion: 0, popupOnboardingCompletedAt: undefined}});`
3. 点击扩展图标打开 Popup（应自动弹出引导）。
4. 完成或跳过引导（任选其一）：
   - 点击完成按钮（`action='finish'`）
   - 点击跳过/关闭（`action='skip'`）
5. 在 Service worker console 执行：
   - `await chrome.storage.local.get('copilot_telemetry_events')`
6. 在结果里找到最近的 `onboarding_shown` / `onboarding_completed`，检查 `props.source/props.action`。
7. 触发一次“手动重开”引导（Service worker console 执行）：
   - `chrome.tabs.create({ url: chrome.runtime.getURL('src/popup/popup.html') + '#onboarding' });`
8. 关闭该 tab 后，重复步骤 5-6，检查新产生的 `onboarding_*` 事件。

期望：
- `onboarding_shown`：若存在 `props`，则 `props.source` 仅可能为 `'auto' | 'manual'`
- `onboarding_completed`：
  - 若存在 `props.source`，则 `props.source` 仅可能为 `'auto' | 'manual'`
  - 若存在 `props.action`，则 `props.action` 仅可能为 `'finish' | 'skip'`
- 不应出现任何其它 `source/action` 值（非法值会在 sanitize 阶段被丢弃）

## 用例 C（评价埋点：rating_prompt_action.action 值域）
步骤：
1. 确保“匿名使用数据”开关开启（`#anonymous-usage-data-switch` 置为 on）。
2. 构造满足触发条件的 Growth Stats（Service worker console 执行）：
   - `await chrome.storage.local.set({ copilot_growth_stats: { installedAt: Date.now() - 72*60*60*1000, successfulCopyCount: 20 } });`
3. 关闭并重新打开 Popup（刷新一次 Popup 即可），等待“评价引导”出现。
4. 点击任一按钮（任选其一）：
   - 立即评价（`action='rate'`）
   - 稍后提醒（`action='later'`）
   - 不再提醒（`action='never'`）
5. 在 Service worker console 执行：
   - `await chrome.storage.local.get('copilot_telemetry_events')`
6. 找到最近的 `rating_prompt_action`，检查 `props.action`。

期望：
- `rating_prompt_action`：若存在 `props`，则 `props.action` 仅可能为 `'rate' | 'later' | 'never'`
- 不应出现任何其它 `action` 值（非法值会在 sanitize 阶段被丢弃）

## 用例 D（开关关闭：不记录 + 关闭即清空 + 不补发）
步骤：
1. 确保已存在至少 1 条上述事件（可先按用例 A/B/C 触发一次）。
2. 在 Options 将“匿名使用数据”开关关闭（`#anonymous-usage-data-switch` 置为 off）。
3. 在 Service worker console 执行并记录结果：
   - `await chrome.storage.local.get('copilot_telemetry_events')`
4. 在开关关闭状态下，再次触发任一 Pro/WOM/引导/评价入口（Popup 或 Options 均可）。
5. 再次执行：
   - `await chrome.storage.local.get('copilot_telemetry_events')`

期望：
- 步骤 3：`copilot_telemetry_events` 被清空（不存在该 key 或值为空数组）。
- 步骤 5：仍不应新增任何事件（不记录、不缓存、不补发）。
