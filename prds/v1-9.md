# V1-9 可见性克隆空白文本节点保留修复（避免文本粘连/丢空格）

## 目标
修复 `src/shared/dom-preprocessor.ts` 的 `createVisibleClone()` 在非代码块场景下丢弃纯空白文本节点导致“词与词粘连（例如 `Hello world` 变为 `Helloworld`）”的问题；在不引入明显噪音空白、不破坏现有复制体验与隐私原则的前提下，确保复制/转换结果的可读性与可用性，并通过可执行的自动化回归与用例文档验证，使之达到验收标准、可发布。

## 具体任务
1、修复可见性克隆对纯空白文本节点的处理（聚焦“避免丢空格”）
  - 背景：`src/shared/dom-preprocessor.ts` 的 `recursiveCopy()` 在非 code 场景对 TEXT_NODE 使用 `text.trim() !== ""` 过滤，会丢弃用于分隔词语/短语的空白，从而造成文本粘连（见 `review-issues.md` 的 ISSUE-005）。
  - 改造要求（必须可回归、避免引入脏空白）：
    - 仅处理 `text.trim() === ""` 的纯空白 TEXT_NODE（且不在 `pre/code` 上下文）
    - 当该空白节点位于“可见文本需要分隔”的上下文时，保留为单个空格 `' '`（不要保留原始缩进/换行）
      - 上下文判定：该空白节点同时存在前后兄弟节点（忽略 Comment），且前后兄弟节点都不是块级分隔标签（见下一条黑名单）
      - 块级分隔标签黑名单（至少包含）：`div/p/section/article/header/footer/nav/main/aside/ul/ol/li/table/thead/tbody/tr/td/th/pre/blockquote/h1~h6/hr/br`
    - 其它场景（例如：块级元素之间/行首行尾/仅一侧有兄弟）继续丢弃该纯空白节点，保持输出干净
    - 保持代码块逻辑不变：`pre/code` 内继续保留所有空白与换行（现有行为）
  - 代码组织要求（便于 code review 与未来单测）：
    - 在 `src/shared/dom-preprocessor.ts` 内新增 helper（例如 `isBlockSeparatorTag(tagName)`、`shouldPreserveWhitespaceTextNode(prev, next)`）
    - 不引入新依赖；避免对每个节点调用 `getComputedStyle` 造成性能回退

2、补齐回归测试（复用现有 DOM 测试框架，不引入新依赖）
  - 新增最小测试用例 HTML：`test/cases/whitespace-between-inline.html`
    - 必须包含至少两类场景：
      - 内联元素之间的换行缩进：`<span>Hello</span>\\n<span>world</span>` 输出应为 `Hello world`（必须有空格）
      - 块级元素之间的换行缩进不应变成行内空格：例如 `<div>Hello</div>\\n<div>world</div>` 输出仍保持块级分隔（不应变成 `Hello world` 同行）
  - 新增/更新期望快照：`test/snapshots/whitespace-between-inline.expected.md`
  - 更新测试清单：执行 `npm run build:tests` 生成最新 `test/test-manifest.json`（确保新用例可在 `public/test/index.html` 一键运行）
  - 验收前必须保证 `bash scripts/test.sh` 全量通过（lint/type-check/check-i18n/unit-tests/build:prod）

3、补齐测试用例文档（可执行、可回归）
  - 新增并维护测试/用例文档：`docs/test-cases/v1-9.md`
  - 文档必须覆盖（至少）：
    - 复现用例：给出最小 HTML 片段与期望输出（`Hello world` 不粘连）
    - 手工回归：在真实网页中复制包含“内联元素 + 换行缩进空白”的段落，确认输出无粘连
    - 负例校验：块级元素之间不引入多余空格/不出现奇怪的行首空格
    - 自动化回归：记录一次 `bash scripts/test.sh` 执行结果（命令 + 结论）
    - 可选：记录一次 `npm run build:tests` + 打开 `public/test/index.html` 运行新用例的结果（PASS/FAIL + 结论）
