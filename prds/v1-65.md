## 具体任务
1、固化“周度跑数归档”目录与命名规范（可持续、可审计）
  - 证据目录固定为：`docs/evidence/v1-65/`
  - 归档文件规范：
    - 周度证据包文件：`copylot-pro-weekly-channel-ops-evidence-pack-YYYY-MM-DD.on.json`（来自 `#download-pro-weekly-channel-ops-evidence-pack`）
    - 截图目录：`docs/evidence/v1-65/screenshots/`（截图文件名必须包含 `YYYY-MM-DD`，避免多周冲突）
  - 新增/刷新 `docs/evidence/v1-65/index.md`（趋势索引），每周新增条目必须包含：
    - 生成时间、扩展版本号、安装方式口径、导出窗口（7d）
    - 本周 campaign 列表（仅标识，禁止敏感信息）
    - 从 `.on.json` 提取的周度 baseline 表（按 campaign 的 `distCopies/leads/leadsPerDistCopy`）
    - 与上一周（最近一次归档）对比的 delta（`distCopies/leads/leadsPerDistCopy`；`distCopies=0 -> N/A` 口径保持一致）
    - 互证结论：PASS/FAIL（引用 `scripts/verify-weekly-channel-ops-evidence-pack.ts` 的断言口径）

2、新增“趋势索引生成”脚本（把手工写 index 升级为一键生成，降低每周跑数摩擦）
  - 新增脚本：`scripts/build-weekly-channel-ops-trend.ts`
    - 输入：证据目录（默认 `docs/evidence/v1-65`）
    - 扫描：目录内所有 `copylot-pro-weekly-channel-ops-evidence-pack-*.on.json`
    - 输出（稳定且可 git 审计）：
      - 覆盖写入 `docs/evidence/v1-65/index.md`（按 `env.windowTo` 或 `env.exportedAt` 排序，生成趋势表 + 每周条目）
      - 额外生成 `docs/evidence/v1-65/trend.csv`（字段：`weekEndDate,distCopies,leads,leadsPerDistCopy,campaignsCount,nonEmptyCampaignsCount`）
    - 约束：不得读取/推断 URL/标题/网页内容/复制内容；只使用 pack 内聚合 `rows` 与 `env`
  - 将脚本接入门禁：
    - `bash scripts/test.sh` 增加一条：对 `docs/evidence/v1-65/` 运行 `scripts/build-weekly-channel-ops-trend.ts` 并确保输出可重复生成（无随机字段/顺序稳定）
    - `scripts/unit-tests.ts` 增加单测：使用 `docs/evidence/v1-64/copylot-pro-weekly-channel-ops-evidence-pack-2026-03-21.on.json` 作为 fixture 生成最小 `v1-65` trend 输出并断言关键口径（`leadsPerDistCopy` 4 位小数、`distCopies=0 -> N/A`）

3、用例/回归与汇报材料闭环（让“持续化”可执行、可复盘、可审计）
  - 新增测试用例文档（固定路径，必须在 PRD 中明确）：`docs/test-cases/v1-65.md`
    - 覆盖：跑数前置设置、最小动作集、下载 `.on.json`、执行趋势索引脚本、互证复核、证据目录检查
    - 必须包含一次回归记录（日期 + PASS），并要求 `bash scripts/test.sh` 全量通过
  - 新增简报（固定路径建议）：`docs/reports/v1-65-report.md`
    - 用于口头汇报：本周趋势表 + 与 v1-64 基线差异 + 风险/偏差 + 下周要跑的 campaign 列表
  - 更新 `docs/roadmap_status.md`：记录 v1-65 规划与交付状态，并保持 Top1/Top2 阻塞信息不断档
