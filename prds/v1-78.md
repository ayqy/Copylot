# 问题修复和功能优化（V1-78 Pro 意向留资闭环最小增强：问卷信号结构化 + 意向分布可导出 + 可审计证据包）

## 目标
基于 `docs/roadmap_status.md` 的收入优先 Top3，本轮因 Top1「（v1-71）Listing 同步落地 + 商店端取证」与 Top2「（v1-70）发布后回归与商店端取证补齐」均依赖商店端账号/权限/网络取证而 BLOCKED，顺延选择 Top3「（S4）升级入口 -> 意向留资 -> 复盘闭环强化（先验证意向，不引入支付）」作为 1~3 天“最小可交付增量”主题，并固化为 v1-78。

收入第一：把“Pro 候补/问卷”从只能做定性反馈，升级为**可量化、可导出、可复盘**的付费意向信号（不引入支付、不新增权限、不上传用户内容），让我们能够用证据回答：
1) 有多少用户愿意付费（Yes/Maybe/No）？  
2) 可接受价位区间如何分布？  
3) 最想要的 Pro 能力是什么？  
并把这些信号纳入现有 Pro 漏斗导出资产，形成可审计的商业化推进证据。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、Pro 问卷意向“信号结构化”埋点（隐私优先，不记录 PII）
  - 将 Options -> Pro ->「复制问卷（含环境信息）/复制并打开候补」视为“意向提交”动作，在现有 `pro_waitlist_survey_copied` 事件基础上补齐**可量化信号**（仅枚举/布尔/计数）：
    - 付费意向：`yes/maybe/no`
    - 价格区间：monthly/annual 的 bucket（使用下拉框 value，而不是展示文案）
    - 能力偏好：4 个能力 checkbox 的选择情况 + other 是否填写（仅 boolean，不记录自由文本）
    - 联系方式：仅记录 `has_contact=true/false`（严禁写入联系方式原文）
  - 同步更新 telemetry 白名单与严格校验（唯一可信口径）：`src/shared/telemetry.ts`
  - 明确隐私红线（必须可被测试断言）：
    - 不得把「使用场景 useCase」自由文本写入 telemetry props
    - 不得把「联系方式 contact」写入 telemetry props
    - 不得写入网页 URL/标题/复制内容

2、Pro 问卷意向分布导出（可复盘的“收入证据”）
  - 在现有 Options ->「Pro 意向漏斗摘要」面板中新增“问卷意向分布”导出能力（默认遵循匿名开关：OFF 时不导出）：
    - 导出过去 7 天问卷意向明细（CSV，不含任何自由文本/联系方式）
    - 导出过去 7 天问卷意向分布（CSV 或 JSON，按 payWilling/monthly/annual/capabilities 聚合）
  - 指标口径（必须写进导出与证据包，避免口径漂移）：
    - `survey_intent = count(pro_waitlist_survey_copied)`
    - `pay_willing_yes/maybe/no = 按 props.pay_willing 聚合计数`
    - capability 分布 = 按 capability 勾选聚合计数（可多选）

3、证据、用例与回归门禁（可审计/可执行）
  - 新增测试/用例文档（固定路径，必须可逐条执行）：`docs/test-cases/v1-78.md`
    - 覆盖：开启/关闭匿名开关 -> 填写问卷（含联系方式）-> 触发复制 -> 导出明细/聚合 -> 断言导出不含联系方式原文与自由文本
    - 必须包含一次回归记录（日期 + PASS/BLOCKED + 导出文件名/截图索引）
  - 新增 v1-78 证据目录（固定结构，便于审计与复盘）：`docs/evidence/v1-78/`
    - `index.md`：记录口径、导出文件清单、sha256、以及“无 PII”断言结论
    - 至少落盘 1 份样例导出（CSV/JSON 均可，但必须可复算 sha256）
  - 自动化回归要求：
    - `bash scripts/test.sh` 全量 PASS
    - 为“无 PII”与导出列头新增/补齐单元测试（建议落在 `scripts/unit-tests.ts`）
