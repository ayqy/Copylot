# V1-2 新手引导 MVP（Popup 首次打开三步引导 + 可回看）

## 目标
在不破坏现有“复制体验/零打扰/隐私优先”的前提下，为新装用户提供可跳过、可回看的 Popup 三步新手引导，并提供“一键应用推荐配置”的最小闭环，使新用户在 3 分钟内完成首次成功复制与基础配置理解；所有能力达到可验收、可测试、可发布状态。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、Popup 首次打开三步引导（可跳过、可回看、状态持久化）
  - 触发条件（必须同时满足“零打扰”与“可验证”）：
    - 当 `copilot_settings` 中“引导完成状态”为未完成时：首次打开 Popup 自动展示引导
    - 已完成后：后续不再自动弹出（除非用户手动打开）
    - 清空配置/重装扩展（`chrome.storage.sync` 被清空）后：再次视为未完成
  - 引导形态（最小可用，不引入新权限、不影响现有按钮可用性）：
    - Popup 内 Modal/Overlay（不新开 Tab）
    - 3 步内容与交互：上一页/下一页、跳过、完成；显示步骤进度（1/3）
    - 文案 i18n（至少中英文），复用现有 `_locales/en/messages.json` 与 `_locales/zh/messages.json`
    - 实现位置（用于落地与 code review 对齐）：
      - UI：`src/popup/popup.html`（新增引导 Modal DOM）+ `src/popup/popup.css`（新增引导样式）
      - 逻辑：`src/popup/popup.ts`（引导展示/跳过/完成/手动打开）
  - 三步内容范围（按主 PRD 的 3 步拆分，但保持 MVP）
    - Step1：如何触发复制（单击/双击/悬停差异）+ 推荐默认配置入口
    - Step2：追加模式（Shift 追加/合并粘贴）+ 对应开关位置说明（指向 Popup 内“Append Mode”开关）
    - Step3：Prompt 与自动跳转 Chat（说明“管理 Prompts”入口；提供按钮跳转 `chrome.runtime.openOptionsPage()`）
  - 状态持久化（可审计、可回归）：
    - 在 `src/shared/settings-manager.ts` 的 `Settings` 中新增引导状态字段（需支持未来升级）：
      - `popupOnboardingVersion: number`（常量，建议初始为 `1`）
      - `popupOnboardingCompletedVersion: number`（默认 `0`；完成后写为当前 `popupOnboardingVersion`）
      - 可选：`popupOnboardingCompletedAt?: number`（仅用于调试/回归，不参与触发逻辑也可）
    - 在 `DEFAULT_SETTINGS` 中给出默认值；确保老用户 settings merge 后不报错
  - “重新查看引导”入口（必须可发现、可测试）：
    - 在 Popup 增加一个稳定入口（按钮/链接均可），点击后强制打开引导 Modal
    - 入口必须 i18n，且具备稳定 DOM 标识（用于手工用例描述与自动化脚本扩展）

2、“一键应用推荐配置”（仅改动基础开关；不影响 Prompt/Chat 列表等存量数据）
  - 在引导 Step1 提供“应用推荐设置”按钮（点击即写入 `copilot_settings`）：
    - 推荐值必须与现有产品原则一致：不打开任何打扰性功能、不新增权限、不改变核心复制逻辑
    - 推荐值建议固定为：`interactionMode=click`、`isHoverMagicCopyEnabled=false`、`outputFormat=markdown`、`tableOutputFormat=markdown`、`attachTitle=false`、`attachURL=false`（其余字段不改）
  - 应用后可验证：
    - Popup 对应开关 UI 立即反映新值
    - 二次打开 Popup 推荐按钮保持可见但不重复弹出引导（由“引导完成状态”控制）

3、测试与用例文档（可执行）
  - 新增并维护测试/用例文档：`docs/test-cases/v1-2.md`
  - 文档必须覆盖（至少）：
    - 首次打开 Popup 自动展示引导；“跳过/完成”后不再自动弹出
    - “重新查看引导”入口可用（从 Popup 内可手动打开引导）
    - “应用推荐设置”对关键开关生效且可回归验证
    - 中英文 i18n 展示
    - 零打扰校验：不影响“一键转换 / 管理 Prompts / 反馈分享评价”等现有入口可用性
  - 在验收前完成并记录：`./scripts/test.sh`、`npm run lint`、`npm run type-check`、`npm run build`（或 `npm run build:prod`）通过

## 验收标准
* 修复上述所有问题，并测试通过，达到可发布状态
* 实现以上所有需求，并测试通过，达到可发布状态
* 新装/清空配置后首次打开 Popup 会自动展示 3 步新手引导；用户可“跳过/完成”；完成后不再自动弹出
* Popup 内提供“重新查看引导”入口（手动打开）；不影响现有设置项与按钮正常使用
* 引导 Step1 的“应用推荐设置”可写入并立即生效：Popup 中相关开关 UI 状态与 `copilot_settings` 一致；且不修改 Prompt/Chat 等存量数据
* Step3 的“打开管理 Prompts”可跳转到 Options 页（`chrome.runtime.openOptionsPage()`），用于继续配置 Prompt/默认 Chat/自动打开等能力
* `docs/test-cases/v1-2.md` 已创建并覆盖所有关键路径；按文档执行手工测试通过；Lint/TypeCheck/Build 通过，可打包发布
