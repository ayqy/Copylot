# 问题修复和功能优化（V1-14 隐私页本地增长统计面板：可查看/导出/重置 copilot_growth_stats）

## 目标
在不新增权限、不新增任何联网埋点、且不影响现有复制体验的前提下，为 Options -> “隐私与可观测性”补齐 **本地增长统计**（`chrome.storage.local` -> `copilot_growth_stats`）的可视化与一键导出/重置能力，支持研发/测试/用户自检“安装时间、成功复制次数、评价引导状态”等关键字段；并补齐可执行的测试用例文档 `docs/test-cases/v1-14.md`，确保 `bash scripts/test.sh` 通过，达到可发布状态。

## 具体任务
1、Options 隐私 Tab 新增“本地增长统计”面板（稳定 DOM、低打扰、可 i18n）
  - 位置：`src/options/options.html` 的 `#privacy-tab`
  - 形态：与现有“本地匿名事件日志”一致的 `details/summary` 折叠面板，避免打扰普通用户，但必须可发现、可测试
  - 稳定 DOM（用于用例与未来自动化）：
    - 容器：`#growth-stats-panel`
    - 只读展示区（textarea 或 pre）：`#growth-stats-view`
    - 操作按钮：刷新 `#growth-stats-refresh`、复制 `#growth-stats-copy`、重置 `#growth-stats-reset`
  - 展示规则（必须可审计、隐私合规）：
    - 展示数据来源仅允许：`chrome.storage.local` 的 `copilot_growth_stats`
    - 展示内容仅允许时间戳/计数/枚举字段（`installedAt/successfulCopyCount/ratingPromptShownAt/ratingPromptAction/ratingPromptActionAt`）；严禁展示/拼装任何网页内容、URL、标题、复制内容
  - i18n：
    - 新增/补齐面板标题、说明、按钮文案等 key
    - 同步更新 `_locales/en/messages.json` 与 `_locales/zh/messages.json`
    - 确保 `npm run check-i18n` 通过

2、实现读取/渲染/导出/重置逻辑（复用 growth-stats 规范化，保证容错）
  - 逻辑落地：`src/options/options.ts`
  - 读取与规范化：
    - 优先复用 `src/shared/growth-stats.ts` 的常量与逻辑（`GROWTH_STATS_KEY`、`getGrowthStats()` 或等价能力），避免 Options 侧直接信任存储内容
    - 容错：key 不存在、数据被污染（非对象/字段类型不对）时，面板应展示“规范化后的结果”，且不得抛错
  - 刷新：
    - 点击 `#growth-stats-refresh` 后重新读取并渲染
    - 建议渲染为 pretty JSON（稳定缩进），以便复制后直接用于排查
  - 复制（用户点击触发）：
    - 点击 `#growth-stats-copy` 后将“规范化后的 stats JSON”写入剪贴板；失败需兜底（如 `document.execCommand('copy')`）但不得影响 Options 其它功能
    - 复制内容必须为可解析 JSON（建议末尾追加换行便于粘贴阅读）
  - 重置（面向测试/自检，必须明确且可回归）：
    - 点击 `#growth-stats-reset` 后重置 `copilot_growth_stats` 为“默认结构”（`installedAt=now`、`successfulCopyCount=0`，并清空 ratingPrompt* 字段）
    - 重置后必须立刻刷新展示区
  - 约束：
    - 不新增任何权限
    - 不引入第三方依赖
    - 面板相关操作不得写入新的 telemetry 事件（避免“自举噪音”）

3、补齐测试用例文档（可执行、可回归、可发布）
  - 新增并维护测试/用例文档：`docs/test-cases/v1-14.md`
  - 文档必须覆盖（至少）：
    - 用例 A（面板可见）：打开 Options -> 隐私 Tab，`#growth-stats-panel` 可见且按钮可点击
    - 用例 B（刷新与展示字段）：点击刷新后，展示为可解析 JSON，且只包含允许字段；不包含任何网页内容/URL/标题/复制内容
    - 用例 C（成功复制计数可验证）：执行一次“成功复制”（任意路径），再刷新，`successfulCopyCount` 递增（对比刷新前后）
    - 用例 D（复制导出）：点击复制后粘贴到文本编辑器，JSON 可解析且与面板一致
    - 用例 E（重置）：点击重置后刷新，`successfulCopyCount=0`，ratingPrompt* 字段为空/不存在，`installedAt` 更新为新的时间戳
    - 自动化回归：记录一次 `bash scripts/test.sh` 执行结果（命令 + 结论）
  - 验收前完成回归：确保不影响既有 Prompt 管理、Chat 服务、隐私开关、Pro/WOM 入口等功能，达到可发布状态
