## 用例 A：Options -> Pro Tab campaign 输入项存在且 DOM 稳定

前置：
- 使用仓库内最新 `plugin-*.zip` 解压后以 unpacked 方式加载（禁止用 `src/` 开发态代替验收）。

步骤：
1. 打开 Options -> Pro Tab。
2. 定位“渠道（campaign，可选）”输入框（稳定 DOM：`#pro-intent-campaign`）。

期望：
- 输入项存在、默认值为空。
- 页面提示明确：不要填写敏感信息；仅允许短字符串（`1-32`，字符集 `[A-Za-z0-9._-]`）。

---

## 用例 B：campaign 为空 -> 文案不写该行（复制候补/打开候补/复制问卷）

步骤：
1. 确保 `#pro-intent-campaign` 为空。
2. 点击 `复制候补文案`（`#pro-waitlist-copy`），粘贴剪贴板内容。
3. 点击 `加入候补名单`（`#pro-waitlist-button`），在打开的 GitHub new issue 页面查看 body（或复制 URL 解码 body）。
4. 点击 `复制问卷（含环境信息）`（`#pro-waitlist-survey-copy`），粘贴剪贴板内容。

期望（必须全部满足）：
- 三处产出的文本均不包含 `campaign:` 行（避免噪音与误导）。
- 仍不包含网页内容/复制内容/URL/标题。

---

## 用例 C：campaign 非空 -> 文案包含 campaign 行（复制候补/打开候补/复制问卷）

步骤：
1. 在 `#pro-intent-campaign` 输入 `twitter`（示例；应满足合法字符集与长度）。
2. 重复用例 B 的 2/3/4 步。

期望：
- 三处产出的文本均包含一行 `campaign: twitter`（可带 `- ` 前缀，但必须可复核）。
- 清空 campaign 后再次执行，`campaign:` 行消失。

---

## 用例 D：匿名使用数据 OFF -> 不记录、不导出（含 campaign 也不写入 telemetry）

步骤：
1. Options -> 隐私与可观测性：确保“匿名使用数据”开关 OFF（默认）。
2. Options -> Pro Tab：填写任意合法 campaign（如 `twitter`），并触发：
   - `加入候补名单`
   - `复制候补文案`
   - `复制问卷（含环境信息）`
3. 回到隐私页：
   - 点击 `复制证据包`（Pro 意向漏斗证据包）
   - 点击 `复制本周 Pro 意向证据摘要`

期望：
- 证据包 `events` 为空数组。
- weekly digest 明确提示“匿名使用数据关闭（无可用事件）”，且不包含任何事件统计明细（更不会出现 campaign 拆分统计）。

---

## 用例 E：匿名使用数据 ON -> campaign 写入 props，可导出可复核（证据包 + weekly digest）

步骤：
1. 隐私页：开启“匿名使用数据”开关，并清空本地匿名事件日志。
2. Options -> Pro Tab：
   - campaign 输入 `twitter`，点击 `复制候补文案` 1 次
   - campaign 输入 `ph`，点击 `复制问卷（含环境信息）` 1 次
3. 回到隐私页：
   - 点击 `复制证据包`，检查 `events[]` 中对应事件的 `props.campaign`
   - 点击 `复制本周 Pro 意向证据摘要`，检查末尾的 campaign 拆分表格

期望：
- 证据包 events 明细中，可看到 `props.campaign` 且值仅来源于用户输入（不得出现 URL/标题/网页内容/复制内容）。
- weekly digest 末尾包含 campaign 拆分统计，至少覆盖：
  - `pro_waitlist_copied` 的 `twitter` 计数
  - `pro_waitlist_survey_copied` 的 `ph` 计数
- 表格可手算复核（计数与操作次数一致）。
