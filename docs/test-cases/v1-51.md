## 用例 C：匿名使用数据 ON 且 7d window 内无事件（导出空集，字段稳定）

步骤：
1. Options -> 隐私与可观测性：先点击“清空”本地匿名事件日志（Telemetry Events 面板）。
2. 开启“匿名使用数据”开关。
3. 展开「Pro 意向漏斗摘要」面板，点击 `导出过去 7 天 Pro 意向明细（CSV）`。
4. 打开下载的 CSV 文件。

期望：
- 文件名符合规范：`copylot-pro-intent-events-7d-YYYY-MM-DD.csv`。
- CSV 为 UTF-8（无 BOM），至少包含表头字段：
  - `exportedAt,extensionVersion,windowFrom,windowTo,lookbackDays,eventTs,eventLocalTime,eventName,source`
- 明细行数为 0（仅表头），或所有行均为白名单事件且不包含网页内容/复制内容/URL/标题。
- `lookbackDays` 固定为 `7`；`windowFrom/windowTo` 可与当前时间手工核对（<=7d 范围）。

## 用例 D：匿名使用数据 ON 且 7d window 内有事件（行级明细可复核）

步骤：
1. Options -> 隐私与可观测性：开启“匿名使用数据”开关（若已开启可跳过）。
2. Options -> Pro Tab（触发 `pro_entry_opened`，source=options）。
3. 在 Pro Tab 依次触发：
   - 点击 `加入候补名单`（触发 `pro_waitlist_opened`）
   - 点击 `复制候补文案`（触发 `pro_waitlist_copied`）
   - 点击 `复制问卷（含环境信息）`（触发 `pro_waitlist_survey_copied`，若该入口存在）
4. 回到 Options -> 隐私与可观测性 ->「Pro 意向漏斗摘要」面板，点击 `导出过去 7 天 Pro 意向明细（CSV）`。
5. 打开下载的 CSV 文件，检查内容。

期望（必须全部满足）：
- CSV 每一行均包含字段：
  - `exportedAt`（ms）/ `extensionVersion`
  - `windowFrom`（ms）/ `windowTo`（ms）/ `lookbackDays`（7）
  - `eventTs`（ms）/ `eventLocalTime`（yyyy-mm-dd hh:mm:ss）/ `eventName` / `source`
- `eventName` 仅包含以下 4 种之一：
  - `pro_entry_opened` / `pro_waitlist_opened` / `pro_waitlist_copied` / `pro_waitlist_survey_copied`
- `source` 仅为 `popup` 或 `options`。
- 不包含网页内容/复制内容/URL/标题等可识别信息（仅事件名/时间戳/来源 + 环境信息）。

## 用例 E：7d window 边界（事件过滤：<from 排除，=from/=to 包含）

说明：本用例通过“手动写入本地 telemetry（仅本地）”验证时间窗边界；适用于回归与审计。

步骤：
1. Options -> 隐私与可观测性：开启“匿名使用数据”开关。
2. 在扩展的 Service Worker（或任意能执行 `chrome.storage.local` 的扩展上下文）运行以下脚本写入本地事件：
   - 写入 1 条 `ts = Date.now() - 8d` 的事件（应被排除）
   - 写入 1 条 `ts = Date.now() - 7d` 的事件（=from，应被包含）
   - 写入 1 条 `ts = Date.now()` 的事件（=to，应被包含）

参考脚本（示意）：
```js
const now = Date.now();
await chrome.storage.local.set({
  copilot_telemetry_events: [
    { name: 'pro_entry_opened', ts: now - 8*24*60*60*1000, props: { source: 'options' } },
    { name: 'pro_entry_opened', ts: now - 7*24*60*60*1000, props: { source: 'options' } },
    { name: 'pro_entry_opened', ts: now, props: { source: 'options' } }
  ]
});
```

3. 回到 Options -> 隐私与可观测性 ->「Pro 意向漏斗摘要」面板，点击 `导出过去 7 天 Pro 意向明细（CSV）`。
4. 打开 CSV，统计 `eventName=pro_entry_opened` 的行数。

期望：
- `pro_entry_opened` 的明细行数为 `2`（8d 的被排除，7d 边界与 now 边界包含）。

## 用例 F：CSV 与 v1-50 摘要口径一致性（可互相印证）

步骤：
1. 匿名开关保持 ON，确保 7d window 内至少有若干 Pro 意向事件（可复用用例 D 的触发步骤）。
2. 导出 CSV（v1-51）：点击 `#export-pro-intent-events-7d-csv`，打开 CSV。
3. 复制摘要（v1-50）：点击 `#copy-pro-intent-weekly-digest`，将 Markdown 粘贴到文本处。
4. 在表格软件中对 CSV 做聚合复核：
   - overall：按 `eventName` 计数（例如透视表 / COUNTIF）
   - bySource：按 `source,eventName` 计数
5. 将 CSV 的聚合计数与 weekly digest 的 overall / bySource 计数逐项对比。

期望：
- CSV 聚合计数与 weekly digest 的计数一致（同一窗口 from/to、同一白名单、同一 source 过滤）。
- 允许的审计说明：由于本地匿名事件日志为 `TELEMETRY_MAX_EVENTS=100` FIFO 窗口，若 7d 内事件量 >100，则导出与摘要均只覆盖“最近 100 条”范围（需在证据索引中说明）。

## 用例 G：关键转化率可由 CSV 复算得到一致结论

步骤：
1. 使用用例 F 的 CSV 聚合结果（overall）。
2. 在表格中按以下公式复算转化率：
   - `waitlist_opened_per_entry_opened = pro_waitlist_opened / pro_entry_opened`
   - `waitlist_copied_per_waitlist_opened = pro_waitlist_copied / pro_waitlist_opened`
   - `survey_copied_per_entry_opened = pro_waitlist_survey_copied / pro_entry_opened`
3. 与 weekly digest 中的转化率字段对比（overall）。

期望：
- 复算结果一致（分母为 0 时按口径输出 `N/A` 或空值均可，但需要一致解释）。
