# 扩展仓安装承接基线统一与无表单增长证据补齐

## 目标
以扩展仓可直接落地的外链、E2E、测试用例与增长资产为交付面，把当前对外口径统一为 Chrome Web Store 安装、上手、隐私与分享，清掉显性表单、问卷、候补语义，并补齐无表单投放包、截图基线和漏斗证据导出路径，使本轮交付可以用可审计证据证明真实获客与商业化主线被推进。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、同步安装承接基线与对外链接
  - 更新 `src/shared/external-links.ts` 及其下游引用，只保留官网首页、`/pricing` 路线说明、Chrome Web Store 安装页与必要隐私承接入口；对外语义统一为安装、上手、隐私与分享。
  - 对齐 `e2e/options-pro-flow.spec.ts` 与 `e2e/popup-growth-flow.spec.ts` 的断言，确保 Options Pro 与 Popup 增长入口不再暴露表单、问卷、候补语义，且打开目标与复制文案都指向无表单闭环。

2、清理扩展仓剩余显性表单、问卷、候补入口与旧命名
  - 只处理会出现在用户界面、分享文案、导出产物和公开文档中的显性命名；将 Pro 入口统一为路线说明 + 分享 / 评价承接，避免把历史 `waitlist` 语义继续暴露到对外路径。
  - 同步更新 `docs/roadmap.md`、`docs/roadmap_status.md` 与相关增长说明，保证公开话术与仓库当前可交付面一致。

3、补齐无表单投放包、截图基线与漏斗证据路径
  - 新增或更新测试用例文档 `docs/test-cases/v1-110.md`，明确覆盖 `src/shared/external-links.ts`、Options Pro 分享工具、Popup 增长入口、无表单投放包与证据导出路径，并以 `bash scripts/test.sh` 作为统一回归入口。
  - 维护 `docs/growth/` 与 `docs/evidence/v1-110/` 下的无表单投放包、截图基线索引、安装/分享漏斗证据导出说明，至少包含 `docs/evidence/v1-110/index.md`、`docs/evidence/v1-110/official-links.json`、`docs/evidence/v1-110/distribution-pack.sample.md`、`docs/evidence/v1-110/screenshot-baseline.md`、`docs/evidence/v1-110/funnel-evidence-paths.md`。
