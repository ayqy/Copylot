# V1-8 代码块清理误删修复（保守移除 Copy 按钮文本）测试用例

## 目标
验证 V1-8 修复“代码块清理误删 Copy/Clone 等合法正文”的问题，在不影响现有复制体验的前提下，确保复制出来的代码可直接使用，并且 `bash scripts/test.sh` 全量通过、可发布。

## 用例 1：手工回归（正文包含 Copy/Clone 不应被误删）
步骤：
1. 打开任意技术文章页面，找到包含代码块的片段，要求代码正文中出现 `Copy` / `Clone`（例如变量名、字符串或注释）。
2. 使用插件的一键转换能力复制该段内容（全文/选区均可）。
3. 粘贴到本地编辑器，核对代码块内容。

期望：
- 代码块正文中的 `Copy` / `Clone` 等字面量/标识符/注释内容保持原样，不缺字、不误删；
- 复制出来的代码可直接运行/可直接作为示例使用（不出现语义被破坏的情况）。

## 用例 2：负例验证（首/末行按钮文案应被移除且不影响正文）
步骤：
1. 构造或找到一个代码块场景，使复制按钮文案被包含进代码文本（常见表现为代码块首行或末行出现 `Copy` / `Copy to clipboard` / `复制` / `复制代码` / `Clone` 等独立一行的文本）。
2. 使用插件复制该代码块输出。
3. 粘贴到本地编辑器核对。

期望：
- 若按钮文案出现在代码块首行或末行，并且该行 `trim()` 后完全等于候选文案，则该整行被移除；
- 代码正文其它行保持不变（包括行内出现的 `Copy/Clone` 字符串、标识符、注释均保留）。

## 用例 3：位置受限验证（按钮文案在中间行不应移除）
步骤：
1. 构造一个代码块文本，确保中间某一行恰好为 `Copy`（或其它候选文案），但该行不是首行/末行。
2. 使用插件复制输出并粘贴查看。

期望：
- 中间行的 `Copy`（或其它候选文案）不被移除，证明“仅清理首/末端连续段”生效。

## 回归验证：不影响现有关键路径
步骤（抽查即可）：
1. 整页一键转换：复制任意长文页面，确认输出结构与历史一致。
2. 选区转换：框选包含代码块的片段，确认只转换选区内容。
3. 智能区块复制/魔法复制：在包含代码块区域触发，确认代码块内容不缺失。
4. 表格转换：对包含表格页面/选区转换，确认表格输出不退化。
5. 追加模式：连续复制两次，确认追加行为不受影响。
6. Prompt 复制：使用 Prompt 相关入口复制，确认输出不受影响。

期望：
- 以上路径均可正常使用，不出现“转换结果丢失内容/结构异常/复制失败”。
