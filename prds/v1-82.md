# 问题修复和功能优化（V1-82 Pro 意向转化提升：Popup 快捷问卷入口 + 可归因跑数取证）

## 目标
在不引入支付、不新增权限、不上传用户内容的前提下，把 Pro 意向问卷的触达路径从“只在 Options 内可见”升级为“Popup 可发现、低摩擦直达”，并把 `pro_waitlist_survey_copied` 的来源可归因到 `popup/options`，形成可导出的漏斗与证据包，用于后续更快跑到 `survey_intent >= 30` 并支持 S4「收款/订阅链路」go/no-go 决策。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、Popup 增加“Pro 问卷（1 分钟）”快捷入口（收入证据增长）
  - 修改：`src/popup/popup.html`、`src/popup/popup.ts`、`src/popup/popup.css`
  - 在 Popup 的 Pro 区新增 1 个可发现入口按钮（i18n 文案），用于引导用户进入 Pro 问卷（不新增任何自由文本采集）
  - 点击后打开 Options 并定位到问卷区：`#pro-waitlist-survey`
  - 同时写入一次性归因参数（例如：`pro_survey_source=popup`），用于后续把 `pro_waitlist_survey_copied` 归因到 `source=popup`

2、Options 支持深链接定位 + 归因透传到 `pro_waitlist_survey_copied`（口径可审计）
  - 修改：`src/options/options.ts`（必要时补充 `src/options/options.html` 的锚点/提示文案）
  - 页面加载时：若 `location.hash === '#pro-waitlist-survey'`，自动滚动到问卷区，并用低打扰方式提示“完成后点击复制”（必须可关闭/不影响复制主流程）
  - 复制问卷时：`copyWaitlistSurveyToClipboard()` 读取归因参数：
    - 若为 `popup`：则 `recordTelemetryEvent('pro_waitlist_survey_copied', { source: 'popup', ... })`
    - 否则：保持现状 `source: 'options'`
  - 归因参数必须在首次读取后清除（例如 `history.replaceState`），避免后续误归因
  - 保持 v1-80 的结构化 props 与隐私门禁不变（本轮只改变 `source` 归因，不新增字段）

3、可导出跑数证据 + 测试/用例（可执行）
  - 自动化门禁：
    - 更新/新增单测：`scripts/unit-tests.ts`，覆盖 `pro_waitlist_survey_copied.source` 仅允许 `popup|options`，且归因参数清除后不再误归因
    - `bash scripts/test.sh` 全量 PASS
  - 新增测试用例文档（固定路径）：`docs/test-cases/v1-82.md`
    - 必须覆盖：Popup 点击 -> Options 定位 -> 填写并复制问卷 -> 在 Options「Pro 意向漏斗摘要」确认 `source=popup` 计数增长 -> 导出/复制证据
    - 必须包含一次回归记录（日期 + PASS/BLOCKED + 导出文件名/证据文件名）
  - 新增证据目录（固定结构）：`docs/evidence/v1-82/`
    - `index.md`：记录口径、文件清单、sha256、“无 PII”断言
    - 至少落盘 1 份可复盘导出（例如 Pro funnel evidence pack / 问卷分布导出 / 截图索引），用于证明“入口可跑数 + 可归因”
