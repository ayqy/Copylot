# V1-107 Options Pro Quick Intent 测试用例

## 目标

验证 `Copylot` 在 `Options -> Pro` 中新增的快捷诉求入口可以把“直接进入 Pro 页”的用户从空白问卷引导到结构化预填问卷，并把这类样本按独立漏斗口径写入本地可审计事件。

## 前置条件

- 扩展版本：`1.2.3`
- 相关模块：
  - `src/options/options.html`
  - `src/options/options.ts`
  - `src/options/options.css`
  - `src/shared/pro-intent-attribution.ts`
  - `src/shared/telemetry.ts`

## 用例 A：Options Pro 显示快捷诉求入口

步骤：
1. 打开 `src/options/options.html#pro-waitlist-survey`。
2. 切换到 `Pro` tab。
3. 检查问卷标题下是否出现一组快捷诉求按钮。

预期：
- 出现 4 个快捷诉求入口：
  - `更干净的正文 / Cleaner article copy`
  - `批量收集整理 / Batch collect and organize`
  - `高级 Prompt 工作流 / Advanced Prompt workflow`
  - `笔记工具导出 / Note-tool export`
- 默认状态文案提示可以直接手填下方问卷。

## 用例 B：快捷诉求可预填问卷

步骤：
1. 在 `Pro` 问卷中点击 `批量收集整理 / Batch collect and organize`。
2. 观察：
  - `#pro-waitlist-survey-use-case`
  - `#pro-waitlist-survey-capability-batch-collection`
  - `#pro-waitlist-survey-quick-intent-status`

预期：
- `use case` 被填入对应场景文案。
- `batch_collection` 复选框被勾选。
- 状态文案显示当前已选择的快捷诉求。

## 用例 C：快捷诉求事件按独立漏斗口径记录

步骤：
1. 在 `Pro` 页点击任一快捷诉求按钮。
2. 读取本地 `copilot_telemetry_events`。

预期：
- 存在 `pro_entry_opened` 事件，且：
  - `props.source === 'options'`
  - `props.content === 'options_quick_intent_cta'`
- 存在 `pro_intent_form_start` 事件，且：
  - `props.source === 'options'`
  - `props.content === 'options_quick_intent_cta'`

## 用例 D：复制问卷后保留预填样本审计字段

步骤：
1. 通过快捷诉求预填问卷。
2. 点击 `复制反馈内容`。
3. 读取 `pro_waitlist_survey_copied` 事件。

预期：
- `props.prefill_used === true`
- `props.prefill_capability_count === 1`
- `props.content === 'options_survey_cta'`
- 不包含用户网页内容、URL、标题或复制内容。

## 用例 E：统一门禁

执行命令：
- `npm run type-check`
- `npm run check-i18n`
- `npm run build:e2e`
- `COPYLOT_E2E_SKIP_BUILD=1 npx playwright test --config=playwright.config.ts --project=main e2e/options-pro-flow.spec.ts`

预期：
- 全部通过。
- 新增快捷诉求相关 E2E 用例通过。
