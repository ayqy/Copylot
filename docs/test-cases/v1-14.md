# V1-14 隐私页本地增长统计面板（查看/导出/重置 copilot_growth_stats）测试用例

## 目标
验证 V1-14 达到可发布：
- Options -> 隐私与可观测性（`#privacy-tab`）新增“本地增长统计”面板 `#growth-stats-panel`（折叠面板，稳定 DOM、可 i18n）
- 面板仅允许读取 `chrome.storage.local` 的 `copilot_growth_stats`，且仅展示允许字段：
  - `installedAt`
  - `successfulCopyCount`
  - `ratingPromptShownAt`
  - `ratingPromptAction`
  - `ratingPromptActionAt`
- 支持刷新/复制导出/重置（均为用户点击触发）
- `bash scripts/test.sh` 全量通过

## 预置条件
- 已在 Chrome/Chromium 加载本地扩展（开发包或生产包均可）
- 可打开 Options 页面（扩展详情页 -> 扩展选项）

## 用例 A（面板可见：稳定 DOM + 可操作）
步骤：
1. 打开 Options 页面。
2. 点击顶部 Tab：隐私（`#privacy-tab`）。
3. 检查“本地增长统计”折叠面板容器 `#growth-stats-panel` 是否可见。
4. 展开面板，检查以下元素存在且按钮可点击：
   - 只读展示区：`#growth-stats-view`
   - 刷新按钮：`#growth-stats-refresh`
   - 复制按钮：`#growth-stats-copy`
   - 重置按钮：`#growth-stats-reset`

期望：
- 面板默认折叠（低打扰），展开后可正常操作。

## 用例 B（刷新与展示字段：可解析 JSON + 字段白名单）
步骤：
1. 在隐私 Tab 展开 `#growth-stats-panel`。
2. 点击 `#growth-stats-refresh`。
3. 复制 `#growth-stats-view` 的文本到任意编辑器，执行 JSON 解析（人工或 Console 均可）。
4. 人工检查 JSON 顶层 key。

期望：
- 文本为可解析 JSON。
- 顶层字段仅包含（顺序不限）：`installedAt/successfulCopyCount/ratingPromptShownAt/ratingPromptAction/ratingPromptActionAt`。
- 不包含任何网页内容/URL/标题/复制内容（可在文本中搜索 `http`/`https`/`title`/`url`/`content` 等关键字应不存在）。

## 用例 C（成功复制计数可验证：successfulCopyCount 递增）
步骤：
1. 在隐私 Tab 点击 `#growth-stats-refresh`，记录当前 `successfulCopyCount`（刷新前值）。
2. 在任意可注入 content script 的网页上执行一次“成功复制”（任意路径均可，确保复制成功）。
3. 回到 Options -> 隐私 Tab，再次点击 `#growth-stats-refresh`，记录 `successfulCopyCount`（刷新后值）。

期望：
- 刷新后值相对刷新前值递增（通常为 +1）。

## 用例 D（复制导出：剪贴板 JSON 可解析且与面板一致）
步骤：
1. 在隐私 Tab 展开 `#growth-stats-panel`，先点击 `#growth-stats-refresh`。
2. 点击 `#growth-stats-copy`。
3. 打开任意文本编辑器，粘贴内容。
4. 对比粘贴内容与 `#growth-stats-view` 内容是否一致（忽略末尾换行差异）。

期望：
- 粘贴内容为可解析 JSON。
- 粘贴内容与面板展示一致。

## 用例 E（重置：恢复默认结构并立即刷新）
步骤：
1. 在隐私 Tab 展开 `#growth-stats-panel`。
2. 点击 `#growth-stats-reset`。
3. 检查 `#growth-stats-view` 中的 JSON。

期望：
- `successfulCopyCount === 0`
- `ratingPromptShownAt/ratingPromptAction/ratingPromptActionAt` 为空或不存在
- `installedAt` 更新为新的时间戳（相对重置前变化）
- 重置后展示区已立即刷新（无需再次手动刷新）
