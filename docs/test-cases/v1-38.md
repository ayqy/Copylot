## 用例 A（匿名开关关闭：不记录/清空/不导出）
步骤：
1. 打开 Options -> 隐私与可观测性。
2. 确认“匿名使用数据”开关为关闭（默认关闭）。
3. 打开「本地匿名事件日志」面板：
   - 点击“刷新”，确认事件数为 0 且文本框为 `[]`。
4. 打开「Pro 意向漏斗摘要」面板：
   - 确认面板内提示“未开启，不记录/不导出”可见。
   - 点击“刷新”，确认摘要 JSON 中 `enabled=false` 且各来源计数均为 0。
5. 在 Popup/Options 触发 Pro 入口与候补动作（升级 Pro / 加入候补 / 复制候补文案）。
6. 回到隐私页再次刷新「本地匿名事件日志」与「Pro 意向漏斗摘要」。
7. 点击「复制证据包」，检查导出的 JSON：
   - `settings.isAnonymousUsageDataEnabled=false`
   - `events=[]`

期望：
- 开关关闭时：不新增事件；事件日志为空；摘要与证据包导出为空/不可用（不补发、不回填）。

## 用例 B（匿名开关开启：事件记录 + 分来源统计 + 转化率派生）
步骤：
1. Options -> 隐私与可观测性，开启“匿名使用数据”开关。
2. 触发 Popup/Options 的 Pro 漏斗事件（需覆盖来源）：
   - Popup 点击升级 Pro（记录 `pro_entry_opened`，`props.source=popup`）
   - Options 打开 Pro Tab（记录 `pro_entry_opened`，`props.source=options`）
   - Popup/Options 打开候补（记录 `pro_waitlist_opened`，`props.source=popup|options`）
   - Popup/Options 复制候补文案成功（记录 `pro_waitlist_copied`，`props.source=popup|options`）
3. 回到 Options -> 隐私与可观测性：
   - 刷新「本地匿名事件日志」，检查包含以上 `pro_*` 事件，且 `props` 仅包含白名单字段（`source`），`source` 枚举仅为 `popup|options`。

## 用例 C（证据包导出：字段白名单 + 无敏感信息）
步骤：
1. 在开启匿名使用数据前提下，点击「Pro 意向漏斗摘要」面板内“复制证据包”。
2. 将剪贴板 JSON 粘贴到任意文本编辑器做自检：
   - 顶层字段严格包含：`meta/settings/proFunnel/events`
   - `meta` 包含：`exportedAt`（ms）、`extensionVersion`（`manifest.json`）、`source=options`
   - `settings` 仅包含白名单字段（至少包含 `isAnonymousUsageDataEnabled`）
   - `proFunnel` 为摘要对象（分来源计数/最近时间戳/转化率）
   - `events` 仅包含 `pro_*` 事件，且事件字段白名单为 `name/ts/props`，`props` 仅包含 `source`
   - 不包含任何 URL/标题/网页内容/用户复制内容

期望：
- 证据包为合法 JSON，可复核、可留档，隐私合规。
