## 具体任务
1、补齐 `wom_*` 本地匿名事件的 `source` props（白名单过滤、隐私合规、向后兼容）
  - 修改：`src/shared/telemetry.ts`
  - 事件范围（不新增事件名，仅扩展 props allowlist）：
    - `wom_feedback_opened`
    - `wom_share_opened`
    - `wom_share_copied`
    - `wom_rate_opened`
  - props 规则：
    - allowlist 增加 `source`
    - 值类型：仅 `string`，取值固定为 `'popup' | 'options'`（其它值应被 `sanitizeTelemetryEvent` 丢弃）
  - 兼容性与约束：
    - 旧数据（无 props）仍应通过 sanitize 并正常展示
    - “匿名使用数据”开关关闭时：仍不记录（不发送、不缓存、不补发）
    - 不新增任何网页/复制内容字段；仅允许标量 props

2、Popup/Options 口碑入口接入 `source` props + 统一商店 UTM campaign（可审计、可回归）
  - Popup：`src/popup/popup.ts`
    - `#feedback-link` / `#share-link` / `#rate-link`：
      - `recordTelemetryEvent('wom_*', { source: 'popup' })`
    - 打开商店详情页链接：
      - `buildChromeWebStoreDetailUrl(...)` 的 UTM 必须包含：
        - `utm_source=copylot-ext`
        - `utm_medium=popup-entry`
        - `utm_campaign=v1-21`
      - “复制分享文案”必须使用同一条 store URL（保证可审计一致）
  - Options：`src/options/options.ts`
    - `#wom-feedback-open` / `#wom-share-open` / `#wom-share-copy` / `#wom-rate-open`：
      - `recordTelemetryEvent('wom_*', { source: 'options' })`
    - Options 打开商店详情页链接：
      - `utm_source=copylot-ext`
      - `utm_medium=options-entry`
      - `utm_campaign=v1-21`
  - 约束：
    - 不新增权限、不新增依赖、不新增任何联网发送
    - 不改变现有 UI 结构与 DOM id（仅更新埋点 props 与 UTM 参数）

3、文档与测试闭环（单测 + 用例文档可执行）
  - 更新埋点清单：`docs/growth/telemetry-events.md`
    - 为上述 `wom_*` 事件补充 props：`{ source: 'popup' | 'options' }`（隐私约束不变）
  - 单测：`scripts/unit-tests.ts`
    - 新增/更新断言覆盖（至少）：
      - `sanitizeTelemetryEvent({ name:'wom_share_opened', ts, props:{ source:'popup' } })` 保留 props
      - 非白名单 props（例如 `{ source:'popup', foo:'bar' }`）必须被过滤
      - 非法 source 值（例如 `'unknown'`）必须被过滤（props 变 `undefined`）
      - `buildChromeWebStoreDetailUrl(...)` 的 UTM 参数符合本需求（含 `utm_campaign=v1-21`）
  - 测试用例文档：新增 `docs/test-cases/v1-21.md`
    - 必须覆盖（至少）：
      - 用例 A（Popup 分享 UTM）：点击 `#share-link` 打开商店页，校验 query 含 `utm_source/utm_medium/utm_campaign`
      - 用例 B（Options 分享 UTM）：点击 `#wom-share-open` 打开商店页，校验 query
      - 用例 C（WOM 埋点 source）：开启“匿名使用数据”后分别从 Popup/Options 触发 WOM 入口，检查 `copilot_telemetry_events` 中对应事件 `props.source` 正确
      - 用例 D（开关关闭）：关闭“匿名使用数据”后触发 WOM 入口，不应新增事件；且关闭时清空仍符合“不发送、不缓存、不补发”
      - 自动化回归：记录一次 `bash scripts/test.sh` 结论（命令 + 通过/失败）
