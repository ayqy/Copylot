# 结构化导出与下游工作流 Pro 路线验证

## 目标
只完成 `docs/roadmap.md` 中 `S3 Pro 路线验证` 的第 3 项：围绕结构化导出与下游工作流补齐路线说明、验证素材、匿名归因与回归证据，让这条路线也具备独立的验证闭环。

## 具体任务
1、在 `Options -> Pro` 的验证素材实验区中新增“结构化导出与下游工作流验证”卡片，提供“打开路线页 / 复制路线链接 / 复制验证简报 / 复制验证清单”4 个动作，并复用现有 campaign 门禁。

2、扩展 `src/shared/pro-route-validation.ts`、`src/shared/pro-intent-attribution.ts`、`src/shared/telemetry.ts`、`src/options/options.ts` 与对应本地化文案，使结构化导出验证动作能以 `options_structured_export_cta` 进入现有匿名 intent / distribution 导出链路。

3、同步更新 `scripts/unit-tests.ts`、`scripts/ui-integration-tests.ts`、`e2e/options-pro-flow.spec.ts` 与 `bash scripts/test.sh` 覆盖范围，保证结构化导出路线的打开、复制、归因、离线 zip 检查和浏览器级回归都能通过。

4、创建并维护 `docs/evidence/v4-6/*`、`docs/test-cases/v4-6.md`、`docs/reports/v4-6-report.md`，同步更新 `docs/roadmap.md`、`docs/roadmap_status.md`，将 S3 第 3 项推进为已完成。

5、在本轮最后新增一份三轮闭环汇总报告，归档 `v4-4`、`v4-5`、`v4-6` 的进展、结果与观察反思，并明确下一阶段最重要的 3 件事。

## 验收标准
* `Options -> Pro` 中出现结构化导出与下游工作流验证卡片，且 campaign 门禁与前两条路线保持一致。
* 本地 intent 导出中可归因到 `pro_waitlist_opened + content=options_structured_export_cta`。
* 路线链接、验证简报、验证清单的复制动作均可进入本地匿名分发导出，并通过 `content` 区分为结构化导出路线。
* `docs/evidence/v4-6/*` 与三轮汇总报告可复核路线边界、固定链接、匿名信号、隐私边界和下一阶段判断。
* `bash scripts/test.sh` 退出码为 `0`，达到可发布状态。

1、当前只做 S3 第 3 项，不提前接入支付或收款逻辑。
