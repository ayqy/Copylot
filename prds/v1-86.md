# 问题修复和功能优化（V1-86 Pro 意向转化提升：Popup 问卷入口前置 + 来源归因 + 可导出复盘证据）

## 目标
在不引入支付、不新增权限、不上传用户复制内容的前提下，把 Pro 意向问卷的触达路径从“仅 Options 内可见”升级为“Popup 可发现、低摩擦直达”，并将 `pro_waitlist_survey_copied` 的来源严格归因到 `popup|options`，产出可导出、可审计的漏斗证据，直接推动 `survey_intent` 样本量增长，为后续 S4「收款/订阅链路打通」提供可复盘的 go/no-go 决策输入。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、Popup 增加“Pro 问卷（1 分钟）”快捷入口（收入证据增长）
  - 修改：`src/popup/popup.html`、`src/popup/popup.ts`、`src/popup/popup.css`、`_locales/*/messages.json`
  - 在 Popup 的 Pro 区新增 1 个清晰可发现的入口按钮（i18n 文案），用于引导用户进入 Options 的 Pro 意向问卷区（不新增任何自由文本采集）
  - 点击后打开 Options 并定位到问卷区：`src/options/options.html#pro-waitlist-survey`
  - 透传一次性归因参数（querystring）：`?pro_survey_source=popup`，用于后续把 `pro_waitlist_survey_copied` 归因到 `source=popup`

2、Options 支持深链接定位 + 一次性归因透传到 `pro_waitlist_survey_copied`（口径可审计）
  - 修改：`src/options/options.ts`（必要时补充 `src/options/options.html` 的提示文案/锚点）
  - 页面加载时：
    - 若 `location.hash === '#pro-waitlist-survey'`，自动滚动到问卷区（`section#pro-waitlist-survey`）
    - 若 `pro_survey_source=popup`，则写入一次性归因（仅用于下一次“复制问卷”事件），并在读取后清除（例如 `history.replaceState` 移除 querystring），避免后续误归因
  - 复制问卷时（`copyWaitlistSurveyToClipboard()`）：
    - 若一次性归因为 `popup`：`recordTelemetryEvent('pro_waitlist_survey_copied', { source: 'popup', ... })`
    - 否则保持现状：`source: 'options'`
  - 严格约束：本轮只改变 `source` 归因；不新增任何遥测字段；不改变 v1-80 的结构化 props 与隐私门禁（枚举/布尔 allowlist 仍以 `src/shared/telemetry.ts` 为唯一可信口径）

3、可导出跑数证据 + 测试/用例/证据落盘（可执行、可审计）
  - 自动化门禁：
    - 更新/新增单测：`scripts/unit-tests.ts`，覆盖 `pro_waitlist_survey_copied.source` 仅允许 `popup|options`，且未知来源被丢弃（防口径漂移）
    - `bash scripts/test.sh` 全量 PASS
  - 新增测试用例文档（固定路径）：`docs/test-cases/v1-86.md`
    - 必须覆盖：Popup 点击入口 -> Options 定位到 `#pro-waitlist-survey` -> 填写并复制问卷 -> 在 Options「Pro 意向漏斗摘要」确认 `bySource.popup.counts.pro_waitlist_survey_copied >= 1` -> 复制/导出证据（如「复制证据包」「导出过去 7 天 Pro 意向明细（CSV）」）
    - 必须覆盖：匿名开关 ON/OFF（OFF 时不记录/不导出，并提示）
    - 必须包含一次回归记录（日期 + PASS/BLOCKED + 导出文件名/证据文件名/截图索引）
  - 新增证据目录（固定结构）：`docs/evidence/v1-86/`
    - `index.md`：记录口径、文件清单、sha256（可复算）、“无 PII”断言结论
    - 至少落盘 1 份可复盘导出（例如 Pro funnel evidence pack / Pro intent events 7d CSV / 问卷分布导出 / 截图索引），用于证明“入口可跑数 + 可归因 + 可导出复盘”
