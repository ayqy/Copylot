# V1-11 含表格混合内容转换一致性修复（复用 convertToMarkdown/PlainText 链路）测试用例

## 目标
验证 V1-11 修复并达到可发布：
- 含表格 mixed content 路径不再绕过 `convertToMarkdown()/convertToPlainText()`：
  - `outputFormat=markdown` 时：链接正文不丢失、相对链接可规范化、无 `[](#)` 噪音
  - `outputFormat=plaintext` 时：整体为纯文本（不出现意外 Markdown 语法串），表格按 `tableOutputFormat` 插入且与上下文有稳定分隔
- `test/cases/mixed-content-with-table-and-links.html` 在 `public/test/index.html` 一键跑可通过
- `bash scripts/test.sh` 全量通过

## 本地复现准备（避免 file:// 权限差异）
1. 在项目根目录启动静态服务器：
   - `python3 -m http.server 8000`
2. 打开用例页面：
   - `http://localhost:8000/test/cases/mixed-content-with-table-and-links.html`

说明：
- Popup 的 **AI Convert** 按钮与右键菜单 **Convert Page** 均走同一条 `processContent()` 转换链路。

## 用例 A（Markdown 输出 + 含表格 mixed content）
前置：
- Popup 设置：
  - `Output format = Markdown`
  - `Table output format = Markdown`（默认）

步骤：
1. 在页面正文区域点击任意位置（确保转换目标包含表格与段落混合内容）。
2. 触发一键转换：
   - Popup：点击 **AI Convert**
   - 或右键菜单：点击 **Convert Page**

期望：
- 表格输出为 Markdown 表格（与上下文之间有稳定空行分隔，不粘连）。
- 相对链接被规范化为绝对 URL（例如 `getting-started` → `https://example.com/docs/getting-started`）。
- “含装饰性 img/svg 的链接”仍保留可见正文（例如 `Download now` 不丢失），且不输出空 `![](...)` 或大段 SVG 噪音。
- 不出现 `[](#)`（空锚点链接噪音应被清理或不产生）。

## 用例 B（PlainText 输出 + 含表格 mixed content）
前置：
- Popup 设置：
  - `Output format = Plain text`
  - `Table output format = Markdown` 或 `CSV`（两者都应回归一次）

步骤：
1. 触发一键转换（同用例 A）。

期望：
- 整体输出为纯文本（不出现 `#` 标题、`[text](url)` 等 Markdown 语法串）。
- 表格按 `tableOutputFormat` 以“文本形式”插入：
  - `CSV`：多行逗号分隔文本
  - `Markdown`：管道表格文本
- 表格与前后段落有稳定换行分隔（不粘连）。

## 回归检查（不影响既有路径）
覆盖点（建议快速冒烟）：
- 无表格页面转换：输出与既有快照保持一致（可在 `public/test/index.html` 跑全量快照用例验证）。
- 纯表格元素点击复制：仍按既有规则输出 Markdown 表格/CSV（不应被 mixed content 逻辑改变）。
