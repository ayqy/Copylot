## 用例 B：dry-run + 证据包落盘（离线可推进）

步骤：
1. 执行：
   - `npm run publish:cws -- --dry-run --evidence-dir docs/evidence/v1-69/preflight/`
2. 断言（从生成的 `*.dry-run.json` 中复核）：
   - `packVersion === "v1-62"`
   - `dryRun === true`
   - `extensionVersion` 为当前 `manifest.json` 版本
   - `zip.sha256` 为非空 64 位 hex
   - `proxyDiagnostic` 存在且脱敏（不得包含 token/secret）
   - `preflightReport.reportVersion === "v1-47"`
   - Preflight FAIL 时：`preflightFixHints.length > 0` 且建议可执行
   - `publishAttempt.errorCode === "skipped"`

## 用例 C：真实发布（非 dry-run）+ `.publish.json` 证据包落盘

前置条件：
- 代理/VPN 可用，且可访问：
  - `https://www.googleapis.com/`
  - `https://chromewebstore.googleapis.com/`
- CWS 凭据齐全（`.env` 内 `CWS_EXTENSION_ID/CWS_CLIENT_ID/CWS_CLIENT_SECRET/CWS_REFRESH_TOKEN`）

步骤：
1. 执行：
   - `npm run publish:cws -- --evidence-dir docs/evidence/v1-69/preflight/`
2. 断言（成功态）：
   - 退出码 0
   - 生成 1 个 `*.publish.json`
   - `dryRun === false`
   - `publishAttempt.published === true`
   - `publishAttempt.channel === "default"`
3. 断言（阻断态：Preflight FAIL）：
   - 退出码 1
   - 仍生成 1 个 `*.publish.json`
   - `publishAttempt.published === false`
   - `publishAttempt.errorCode === "preflight_failed"`（或同义阻断原因）

## 用例 E：从商店安装回归 + Pro 漏斗导出取证（真实发布后执行）

步骤：
1. 必须从 CWS 安装当前版本（禁止用 `plugin-*.zip`/unpacked 替代）。
2. 按 `docs/test-cases/v1-45.md` 用例 G 的最短路径回归触发 Pro 候补链路事件（至少覆盖）：
   - `pro_entry_opened`
   - `pro_waitlist_opened`
   - `pro_waitlist_copied`
3. 在 Options -> 隐私与可观测性 -> Pro 面板导出并落盘（文件名固定）：
   - `docs/evidence/v1-69/pro-funnel-summary.json`
   - `docs/evidence/v1-69/pro-funnel-evidence-pack.json`
4. 断言：
   - 事件计数非 0
   - `meta.extensionVersion` 与商店版本一致
   - 导出文件不包含 token/secret/网页内容/URL/用户复制内容
5. 在 `docs/evidence/v1-69/index.md` 记录导出文件名与关键字段断言结论。
