# V1-39-1 拆分：publish:cws 代理链路确定性修复（undici dispatcher + env 优先级 + NO_PROXY）

## 范围（MVP）
在不新增扩展权限、不改动扩展运行时逻辑（禁止改 `src/`）的前提下，仅通过发布脚本把 `npm run publish:cws` 的代理链路做成：

- 可确定生效（全局 `fetch` 走代理）
- 可诊断、可复现（稳定诊断块 + 强校验）

## 子需求清单（按 MVP）
- [x] 代理环境变量优先级（确定性）
  - [x] 支持：`CWS_PROXY` > `HTTPS_PROXY/https_proxy` > `HTTP_PROXY/http_proxy` > `ALL_PROXY/all_proxy`
  - [x] 记录命中项（envKey）并在诊断块中输出

- [x] 代理 URL 强校验（禁止静默失败）
  - [x] 必须包含 scheme（如 `http://127.0.0.1:7890`）
  - [x] 不支持的 protocol（如 `socks5:`）明确报错并给出替代方案

- [x] `NO_PROXY/no_proxy` 支持（边界明确）
  - [x] 未设置时默认：`localhost,127.0.0.1,::1`
  - [x] 显式设置时尊重原值，并在诊断块中记录

- [x] undici 全局 dispatcher 安装（保证 global fetch 走代理）
  - [x] 使用 `undici.EnvHttpProxyAgent` + `setGlobalDispatcher(...)`
  - [x] 不引入任何真实外网依赖（仅配置链路）
