## 目标
验证子 PRD `prds/v1-90.md` 的验收点：在不引入支付/不新增权限/不上传用户复制内容的前提下，把「Pro 意向跑数取证」做成可执行闭环：

本轮验收对象（固定路径）：

## 用例 A：匿名开关 ON -> 一键下载 Pro 意向跑数证据包（JSON）

步骤：
1. 在真实 Chrome 环境安装当前插件版本（以 `manifest.json` 为准）。
2. 打开 Options -> 隐私与可观测性，开启「匿名使用数据」开关。
3. 触发至少 1 次 `pro_waitlist_survey_copied`（例如：进入 Pro 问卷区并点击「复制问卷」）。
4. 展开「Pro 意向漏斗摘要」面板，点击「下载 Pro 意向跑数证据包（JSON）」并下载文件。
5. 打开导出的 JSON 文件，复核字段与隐私红线。

期望：
- 导出文件名符合：`copylot-pro-intent-run-evidence-pack-v1-90-YYYY-MM-DD.on.json`。
- 导出 JSON 为单文件，至少包含以下字段：
  - `env.exportedAt` / `env.extensionVersion` / `env.isAnonymousUsageDataEnabled`
  - `proFunnelSummary`（含 `bySource.popup|options` 与 `overall`）
  - `proWaitlistSurveyIntentDistribution`（含 `survey_intent` 等关键分布字段）
  - `proIntentEvents7dCsv`（CSV 字符串内嵌，便于离线对账）
  - `proIntentWeeklyDigestMarkdown`（Markdown 字符串内嵌，便于复盘）
- 隐私红线：JSON 不包含 URL/标题/网页内容/复制内容/联系方式明文（可用 `rg \"https?://|@\" <file>` 辅助复核）。

## 用例 B：离线落盘与 sha256 固化（可复核）

步骤：
1. 准备 1 份用例 A 导出的证据包 JSON（文件路径作为脚本入参）。
2. 执行离线落盘脚本：
   - `node --no-warnings=ExperimentalWarning --loader=ts-node/esm scripts/build-pro-intent-run-evidence-pack.ts <downloaded-pack.json>`
3. 打开 `docs/evidence/v1-90/index.md` 与 `docs/evidence/v1-90/sha256.json`，复核清单与哈希。
4. 执行 sha256 复算：
   - `shasum -a 256 docs/evidence/v1-90/*`

期望：
- `docs/evidence/v1-90/` 至少包含：
  - `index.md`：口径说明、文件清单、sha256（可复算）、“无 PII”断言结论
  - `pro-intent-run-evidence-pack.json`：证据包落盘文件
  - `sha256.json`：固化 `index.md` 与 `pro-intent-run-evidence-pack.json` 的 sha256
- `sha256.json` 可用于后续复盘：对比任意两次落盘的 `index.md`/证据包是否发生变化。

## 用例 C：匿名开关 OFF -> 下载禁用证据包（不得包含明细/CSV）

步骤：
1. 打开 Options -> 隐私与可观测性，关闭「匿名使用数据」开关（会清空本地匿名事件日志）。
2. 展开「Pro 意向漏斗摘要」面板，点击「下载 Pro 意向跑数证据包（JSON）」并下载文件。
3. 打开导出的 JSON 文件，检查门禁与明细字段。

期望：
- 顶层门禁字段：
  - `enabled=false`
  - `disabledReason=anonymous_usage_data_disabled`
- 明细/CSV 不得导出：
  - `proIntentEvents7dCsv` 必须为空串（不得包含 CSV 头/行）
