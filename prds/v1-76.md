## 具体任务
1、定稿“敏感词/夸大口径”门禁口径（单一事实来源，规则可执行）
  - 新增口径文档：`docs/publish/cws-listing-redlines-policy.md`（本轮唯一口径）
  - 明确三类规则并写成“可执行断言”：
    - Keywords：零容忍（命中即 BLOCKED）
    - 长描述（EN/ZH）：拦截“误导性宣称/诱导付费”，允许“否定语境免责声明”出现，但必须可被规则识别并在证据中解释归因
    - 证据化要求：扫描必须输出命中清单（文件/行号/命中词/归因规则/allowed|blocked），并落盘到固定证据目录

2、落地自动化扫描脚本 + 证据落盘 + 门禁接入（收入优先，降低审核与返工风险）
  - 新增脚本：`scripts/scan-cws-listing-redlines.ts`
    - 输入：`docs/ChromeWebStore-Description-EN.md`、`docs/ChromeWebStore-Description-ZH.md`、`docs/aso/keywords.md`
    - 输出目录（固定）：`docs/evidence/v1-76/`
    - 输出文件（文件名固定，便于审计与复用）：
      - `docs/evidence/v1-76/cws-listing-redlines-scan.json`（命中明细：file/line/term/rule/result）
      - `docs/evidence/v1-76/index.md`（扫描摘要：PASS/BLOCKED + 修复建议 + 引用源 sha256）
    - 断言：
      - 存在 blocked 命中：脚本退出码非 0（阻断 `bash scripts/test.sh`）
      - 仅有 allowed 命中：脚本退出码 0（允许免责声明存在，但必须在 index.md 中解释归因）
  - 接入门禁：在 `scripts/test.sh` 增加扫描脚本执行步骤，确保 CI 与本地回归一致
  - 单元测试：在 `scripts/unit-tests.ts` 增加用例，覆盖：
    - “否定语境免责声明命中 payment/subscription/付费/订阅”应为 allowed
    - “误导性宣称命中（Pro 已上线/subscribe now/upgrade to pro/立即订阅/马上付费）”应为 blocked
    - keywords 命中任意敏感词应为 blocked

3、断点续孵化收尾：解除口径阻塞 + 用例/简报闭环（可复现、可追责）
  - 更新 `docs/evidence/v1-71/index.md`：将“口径阻塞（敏感词/夸大口径争议）”替换为“已由 v1-76 门禁与证据覆盖”，仅保留网络/权限相关阻塞（若仍不可达）
  - 新增测试用例文档（固定路径，必须可逐条执行）：`docs/test-cases/v1-76.md`
    - 覆盖：口径文档 -> 扫描脚本 -> 证据落盘 -> `bash scripts/test.sh` 门禁 -> v1-71 阻塞更新
    - 必须包含一次回归记录（日期 + PASS/BLOCKED + 证据文件名）
  - 新增简报（固定路径建议）：`docs/reports/v1-76-report.md`
