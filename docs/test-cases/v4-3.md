## 用例 A：Shift 追加复制能形成多段合并文本

步骤：

1. 打开长文 fixture 页面并启用 `isClipboardAccumulatorEnabled=true`。
2. 选中第一段内容，按住 `Shift` 触发复制。
3. 选中第二段内容，继续按住 `Shift` 触发复制。
4. 检查剪贴板文本与扩展 badge。

期望：

- 剪贴板内出现两段正文，并使用 `---` 分隔。
- badge 文本显示 `2`。
- 本地 `copilot_append_session` 只记录 `clipCount`、计数、时间戳和状态枚举，不记录正文、URL、标题。

## 用例 B：Popup 明确承接“继续追加 / 清空后重来”

步骤：

1. 准备 `copilot_append_session.clipCount=2` 的本地状态。
2. 打开 Popup。
3. 检查追加状态卡片文案并点击“清空收集”。

期望：

- `#append-session-card` 可见，并展示已收集段数。
- 清空后卡片隐藏、badge 归零。
- 新一轮 Shift 追加可以从 `clipCount=1` 重新开始，不继承脏状态。

## 用例 C：Options 面板可查看匿名审计摘要并清理当前收集

步骤：

1. 准备 `copilot_append_session` 为进行中的两段收集状态。
2. 打开 `src/options/options.html#privacy`。
3. 点击“刷新数据”“复制数据”“清空当前收集”。

期望：

- 面板 JSON 包含 `workflowState`、`clipCount`、`sessionsCompleted`、`sessionsCleared`、`lastCompletedClipCount`。
- 复制结果不包含网页正文、URL 或复制文本。
- 清空后面板中的 `clipCount` 变为 `0`，`sessionsCleared` 累加。

## 用例 D：统一测试入口与证据文件保持可复核

步骤：

1. 执行 `bash scripts/test.sh`。
2. 检查 `docs/evidence/v4-3/append-mode-audit.md`、`append-mode-funnel-sample.json` 与 `append-mode-commercialization-proof.md`。

期望：

- `bash scripts/test.sh` 退出码为 `0`。
