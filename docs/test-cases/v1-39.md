## 用例 A（dry-run：无代理 + 无凭据，稳定通过前置门禁且无网络调用）
步骤：
1. 在当前 shell 清空代理与 CWS 凭据环境变量（避免本地 `.env` 污染演练口径）：
   - `CWS_EXTENSION_ID= CWS_CLIENT_ID= CWS_CLIENT_SECRET= CWS_REFRESH_TOKEN= CWS_PROXY= HTTPS_PROXY= HTTP_PROXY= ALL_PROXY= NO_PROXY=`
2. 执行：`npm run publish:cws -- --dry-run`
3. 观察脚本输出开头包含：
   - `-----BEGIN CWS PROXY DIAGNOSTIC BLOCK-----`

## 用例 B（代理正确：命中任一代理环境变量，网络请求走代理）
前置：
- 人类提供可访问 Google API 的网络环境（可用代理/VPN），并提供代理地址（包含 scheme，例如 `http://127.0.0.1:7890`）
- `.env` 或环境变量已配置 CWS 发布凭据：`CWS_EXTENSION_ID/CWS_CLIENT_ID/CWS_CLIENT_SECRET/CWS_REFRESH_TOKEN`

步骤：
1. 设置代理（任选其一）：
   - 方案 1：`export HTTPS_PROXY=http://127.0.0.1:7890`
   - 方案 2：`export ALL_PROXY=http://127.0.0.1:7890`
   - （可选更高优先级）`export CWS_PROXY=http://127.0.0.1:7890`
2. （可选）设置不走代理：`export NO_PROXY=localhost,127.0.0.1,::1`
3. 执行：`npm run publish:cws`
4. 观察诊断块：
   - `proxy.enabled=true`
   - `proxy.envKey` 命中实际使用的环境变量名
   - `fetch.dispatcher` 显示 `undici.EnvHttpProxyAgent (setGlobalDispatcher)`
5. 若发布成功：按“商业化取证动作”完成截图/证据导出，并将索引写入 `docs/reports/v1-39-report.md`
6. 若失败：日志必须给出可执行排障指引（含可复制环境变量示例，且不打印 token/secret）

期望：
- 上传/发布阶段请求走代理；不再出现 `ENOTFOUND www.googleapis.com` 这类“直连 DNS 失败”阻塞（若代理可达）
- 若代理不可用/配置错误，错误日志包含下一步动作指引（可复制 env 示例 + 最小必需信息）
