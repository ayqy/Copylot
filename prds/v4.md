# Copylot V4 复用与口碑增长闭环

## 目标
在已有“安装后完成第一次干净复制”能力的基础上，按照当前 `docs/roadmap.md` 与 `docs/roadmap_status.md` 的阶段判断，开启一条新的主迭代路线：把 Copylot 从“首次成功”推进到“再次复用、稳定复用、价值验证后再触发口碑”，形成更短、更可审计的增长闭环。

## 当前判断
- 当前 roadmap 阶段为 `S2 / G2 第二次复用与口碑循环`。
- 旧 `v1-xxx` 路线不再作为后续子 PRD 的续线参考。
- 本主 PRD 只服务当前 roadmap 主线，不回到表单、问卷、候补、支付或其他偏航承接。

## 核心任务
1. 第二次打开理由
  - 在 Popup、Onboarding 或其他现有扩展内入口中强化明确的复用动作，例如快捷 Prompt 槽位、追加模式、连续复制工作流等。
  - 每个子 PRD 必须把“为什么用户会再打开一次”落成真实功能、真实文案或真实操作路径，而不是抽象说明。

2. 高频场景稳定性
  - 围绕长文、表格、代码块三类高频场景持续减少返工。
  - 每轮优先聚焦最可能阻碍复用的一类场景，给出回归与证据，而不是平均铺开。

3. 口碑触发门禁
  - 分享与评价动作必须绑定到“至少 2 次成功复制”之后。
  - 门禁、事件、导出证据与回归口径必须可审计，不记录复制内容、网页内容或用户原文。

4. 文档与验证
  - 每轮同步更新 `docs/roadmap_status.md`、`docs/roadmap.md`、对应子 PRD、测试用例、证据目录与简报。
  - 统一通过 `bash scripts/test.sh`，必要时补充 Popup / Options / Playwright 专项回归。

## 验收标准
- 至少一个“第二次打开理由”被做成真实可发布的产品路径。
- 本轮重点稳定性场景获得实质性改进，并有测试或证据支持。
- 分享/评价提示与“至少 2 次成功复制”建立清晰门禁，并能导出复核证据。
- 子 PRD、roadmap 状态、测试文档、证据目录、简报和代码实现保持一致。
- `bash scripts/test.sh` 通过，达到可发布状态。

## 注意
1. 后续子 PRD 从 `v4-1.md` 开始按顺序推进。
2. 若任务需要修改官网源码、支付链路或当前仓库外资源，应在文档中单列阻塞，不把扩展内复用主线冲散。
3. 继续坚持隐私优先与零打扰，不新增联网采集，不记录任何用户原文内容。
