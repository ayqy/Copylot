## 用例 A（重点，离线可执行）：dry-run + evidence-dir 仍可落盘 FAIL 证据（可审计）

步骤：
1. 清理代理环境变量（示例）：
   - `unset CWS_PROXY HTTPS_PROXY HTTP_PROXY ALL_PROXY https_proxy http_proxy all_proxy`
2. 执行（必须可复制）：
   - `npm run publish:cws -- --dry-run --evidence-dir docs/evidence/v1-62/preflight/`
3. 断言：目录中新增 1 个 JSON 文件，且命名符合规范：
   - `copylot-cws-publish-diagnostic-pack-<version>-YYYY-MM-DD-HHMMSS.dry-run.json`
4. `node` 验证 JSON 可解析（示例）：
   - `node -e 'const fs=require(\"fs\");const p=process.argv[1];const j=JSON.parse(fs.readFileSync(p,\"utf8\"));console.log(\"ok\",j.packVersion,j.dryRun);' <file>`
5. 字段断言（手动或脚本均可）：
   - `packVersion === "v1-62"`
   - `dryRun === true`
   - `exportedAt` 为 ISO 字符串
   - `extensionVersion` 为 manifest 版本号
   - `zip.sha256` 为非空 64 位 hex
   - `proxyDiagnostic` 存在（脱敏）
   - `preflightReport.reportVersion === "v1-47"`
   - `preflightFixHints` 为非空数组（FAIL 时必须可执行）
   - `credentials` 仅包含 4 个 boolean（不得出现任何具体值）
   - `publishAttempt.errorCode === "skipped"`（dry-run 必须明确 skipped）
6. 安全红线（grep/rg 断言，示例）：
   - `rg -n \"CWS_CLIENT_SECRET|CWS_REFRESH_TOKEN\" docs/evidence/v1-62/preflight/` 期望无匹配

期望：
- 即使 Preflight FAIL（网络不可达），也必须生成证据包文件（FAIL 也可审计/可复核）。
- dry-run 退出码为 0（仍严格不进行 upload/publish 网络调用）。

## 用例 B（可选：网络可达时执行）：Preflight PASS 也必须落盘证据（dry-run）

步骤：
1. 配置可用代理/VPN（示例）：
   - `export HTTPS_PROXY=http://127.0.0.1:7890`
   - `export NO_PROXY=localhost,127.0.0.1,::1`
2. 执行：
   - `npm run publish:cws -- --dry-run --evidence-dir docs/evidence/v1-62/preflight/`
3. 断言：
   - 生成 1 个 `.dry-run.json` 证据包文件
   - `preflightReport.summary.fail === 0`
   - `preflightFixHints` 允许为空数组（PASS 无需修复建议）

期望：
- Preflight PASS/FAIL 两态均可落盘，证据包 schema 一致、字段顺序稳定，便于长期归档与 diff。

## 用例 C（重点，需网络可达 + 凭据齐全）：非 dry-run 的 Preflight 阻断语义 + 证据落盘

步骤：
1. （可选）故意让 Preflight FAIL（例如关闭代理），执行：
   - `npm run publish:cws -- --evidence-dir docs/evidence/v1-62/preflight/`
2. 断言：
   - 退出码为 1（阻断真实发布）
   - 目录中仍产生 1 个 `.publish.json` 证据包文件
   - `dryRun === false`
   - `publishAttempt.errorCode` 为 `preflight_failed` 或等价阻断原因（不得包含鉴权 header/response 原文）

期望：
- 非 dry-run 下 Preflight FAIL 必须阻断真实发布（退出码为 1），且证据包仍必须落盘（FAIL 可审计）。
