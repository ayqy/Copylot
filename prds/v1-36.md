# 问题修复和功能优化（V1-36 Pro 候补转化闭环：商店页 CTA + Popup 复制候补文案 + 可导出意向证据）

## 目标
在不新增任何扩展权限、不新增任何联网发送、且不破坏现有免费核心体验的前提下，补齐“Pro 候补（付费意向）”的最小可交付转化闭环：让用户在商店页与产品内都能发现一致的候补入口，并能以低摩擦方式复制/提交候补文案，同时通过本地可导出的匿名事件日志形成可验证的商业化推进证据，使之达到验收标准。

## 具体任务
1、商店页（CWS）商业化入口补齐：在长描述中加入“Pro 候补”CTA（不夸大、可审计）
  - 修改：
    - `docs/ChromeWebStore-Description-ZH.md`
    - `docs/ChromeWebStore-Description-EN.md`
  - 要求：
    - 新增一个小节（例如“Copylot Pro（候补/规划中）/ Copylot Pro (Planned)”），明确：
      - Pro 未上线、不可用（Not shipped yet）
      - 候补加入方式：引导用户在扩展内 Options -> Pro Tab 点击“加入候补名单”（避免商店页直接承诺支付/订阅）
      - 提供可对外复用且稳定的“范围/边界”说明链接：`docs/monetization/pro-scope.md` 的 GitHub 链接
    - 文案必须与仓库现状一致：不得暗示 Pro 已提供/可购买；不得承诺具体上线时间/价格

2、Popup 低摩擦候补：补齐“复制候补文案”入口 + 事件埋点（可导出、可审计）
  - 修改：
    - `src/popup/popup.html`：在现有 Pro 入口区域补齐一个“复制候补文案”按钮（建议 id：`popup-pro-waitlist-copy`）
    - `src/popup/popup.ts`：实现复制逻辑（与 `src/options/options.ts` 的 `#pro-waitlist-copy` 保持一致：复制 waitlist issue body 文本），并在复制成功时记录本地匿名事件
    - `src/popup/popup.css`：补齐按钮样式与布局（不破坏现有 Popup 主流程）
    - `_locales/en/messages.json`、`_locales/zh/messages.json`：补齐按钮文案 i18n key（以及“Copied/已复制”复用既有 key 或新增 key）
    - `src/shared/telemetry.ts`：放开 `pro_waitlist_copied` 的 `props.source` 白名单以允许 `popup`（目前仅允许 `options`），保证 Popup 事件可被审计与导出
    - `docs/growth/telemetry-events.md`：同步更新埋点清单（`pro_waitlist_copied` 支持 `source: popup|options`）
  - 行为约束：
    - 不新增任何权限（例如不引入 `downloads` 等）
    - 不新增任何联网发送；仅本地写入匿名事件（且受开关控制，默认关闭）
    - 复制内容不得包含网页内容/URL/标题/剪贴板历史等；仅为候补模板文案（含扩展版本、扩展 ID、语言信息与用户可填写占位符）

3、测试/用例闭环（确保“商业化证据”可验证、可复现）
  - 修改：
    - `scripts/unit-tests.ts`：更新 telemetry 的纯函数用例断言（`pro_waitlist_copied` 允许 `source: popup` 后，sanitize 行为应随之调整）
  - 新增：
    - `docs/test-cases/v1-36.md`：手工用例 + 回归记录（必须包含“商业化证据验证”步骤）
  - 回归要求：
