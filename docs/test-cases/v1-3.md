# V1-3 测试用例（低打扰评价引导 MVP：触发条件计数 + 仅提示 1 次）

## 范围
- 本地可审计使用计数：`chrome.storage.local` -> `copilot_growth_stats`
- 成功复制次数累计：主复制 / Prompt 复制 / Convert Page / Append Mode（成功才 +1；失败不 +1；分享文案复制不计入）
- Popup 一次性评价引导：满足条件时展示且仅展示 1 次（关闭 Popup 也不再重复提示）
- 三个按钮行为：去评价（reviews 跳转）/ 稍后 / 不再提示；并校验落盘字段
- i18n：至少中英文
- 隐私校验：本地存储仅包含计数/枚举/时间戳，不包含任何复制内容/页面内容/URL

## 运行自动化检查
```bash
./scripts/test.sh
```

## 手工用例

### TC-01 清空本地存储后的初始化（installedAt / successfulCopyCount）
前置：
- 打开扩展 Service Worker Console（`chrome://extensions` -> Copylot -> Inspect views）
步骤：
1. 执行 `await chrome.storage.local.clear()`
2. 打开 Popup
3. 执行 `await chrome.storage.local.get('copilot_growth_stats')`
期望：
- `copilot_growth_stats.installedAt` 为合法时间戳（number）
- `copilot_growth_stats.successfulCopyCount` 为 `0`
- 对象内不包含任何复制内容/页面内容/URL

### TC-02 成功复制次数累计：主复制（成功才 +1）
前置：
- 在任意可注入 content script 的网页（https 页面）上使用 Magic Copy
- Service Worker Console 中可查看 `copilot_growth_stats`
步骤：
1. 记录当前 `successfulCopyCount`
2. 使用 Magic Copy 主按钮完成一次复制（确保复制成功）
3. 再次查看 `successfulCopyCount`
期望：
- `successfulCopyCount` 恰好 +1

### TC-03 成功复制次数累计：Prompt 复制（成功才 +1）
步骤：
1. 记录当前 `successfulCopyCount`
2. 使用 Magic Copy 的 Prompt 菜单执行一次复制（确保复制成功）
3. 再次查看 `successfulCopyCount`
期望：
- `successfulCopyCount` 恰好 +1

### TC-04 成功复制次数累计：Convert Page（Popup / Context Menu 均可）
步骤：
1. 记录当前 `successfulCopyCount`
2. 使用 Popup 内 “AI Convert” 或右键菜单 “Convert to AI-Friendly Format” 完成一次复制（确保复制成功）
3. 再次查看 `successfulCopyCount`
期望：
- `successfulCopyCount` 恰好 +1

### TC-05 成功复制次数累计：Append Mode（成功才 +1）
前置：
- 在 Popup 中开启 “Enable Append Mode”
步骤：
1. 记录当前 `successfulCopyCount`
2. 按住 Shift，使用 Magic Copy 主按钮复制一次（追加复制）
3. 再次查看 `successfulCopyCount`
期望：
- `successfulCopyCount` 恰好 +1

### TC-06 失败不累计（content script 不可用/复制未成功）
步骤：
1. 记录当前 `successfulCopyCount`
2. 打开 `chrome://settings`（或其他 content script 不可注入页面）
3. 打开 Popup，点击 “AI Convert”（此时应无法完成复制）
4. 再次查看 `successfulCopyCount`
期望：
- `successfulCopyCount` 不变

### TC-07 分享文案复制不计入成功复制次数
步骤：
1. 记录当前 `successfulCopyCount`
2. 在 Popup 底部点击 “复制文案 / Copy Text”
3. 再次查看 `successfulCopyCount`
期望：
- `successfulCopyCount` 不变

### TC-08 满足触发条件后打开 Popup 展示评价引导（且仅提示 1 次）
前置：
- 在 Service Worker Console 执行（构造满足条件的本地统计）：
  - `installedAt = Date.now() - 72*60*60*1000`
  - `successfulCopyCount = 20`
  - 删除 `ratingPromptShownAt/ratingPromptAction/ratingPromptActionAt`
步骤：
1. 执行：
   - `await chrome.storage.local.set({ copilot_growth_stats: { installedAt: Date.now() - 72*60*60*1000, successfulCopyCount: 20 } })`
2. 打开 Popup
3. 验证评价引导容器存在：`#rating-prompt`
4. 立即查看 `copilot_growth_stats.ratingPromptShownAt`
5. 关闭 Popup，再次打开 Popup
期望：
- 第一次打开 Popup：展示评价引导卡片（`#rating-prompt` 可见）
- 展示时立即落盘：`ratingPromptShownAt` 为合法时间戳
- 第二次打开 Popup：不再展示评价引导（“仅提示 1 次”）

### TC-09 三个按钮行为与落盘字段校验
前置：
- 先按 TC-08 重新构造满足条件且未展示过的状态
步骤（去评价）：
1. 打开 Popup，点击 `#rating-prompt-rate`
期望：
- 打开 Chrome Web Store reviews 页面
- `copilot_growth_stats.ratingPromptAction === 'rate'`
- `copilot_growth_stats.ratingPromptActionAt` 为合法时间戳

步骤（稍后）：
1. 重新构造状态（同 TC-08）
2. 打开 Popup，点击 `#rating-prompt-later`
期望：
- 评价引导卡片在 Popup 内关闭（隐藏）
- `ratingPromptAction === 'later'` 且 `ratingPromptActionAt` 有值
- 关闭并再次打开 Popup：不再出现评价引导

步骤（不再提示）：
1. 重新构造状态（同 TC-08）
2. 打开 Popup，点击 `#rating-prompt-never`
期望：
- 评价引导卡片在 Popup 内关闭（隐藏）
- `ratingPromptAction === 'never'` 且 `ratingPromptActionAt` 有值
- 关闭并再次打开 Popup：不再出现评价引导

### TC-10 i18n 展示（至少中英文）
步骤：
1. 将浏览器 UI 语言切换为中文，打开 Popup
2. 检查评价引导卡片标题/描述/按钮文案为中文
3. 将浏览器 UI 语言切换为英文，打开 Popup
4. 检查上述文案为英文
期望：
- 评价引导相关文案随 UI 语言切换

### TC-11 隐私校验：本地统计不包含任何复制内容/页面内容/URL
步骤：
1. 执行 `await chrome.storage.local.get('copilot_growth_stats')`
2. 人工检查对象字段
期望：
- 仅包含时间戳/计数/枚举字段（`installedAt/successfulCopyCount/ratingPromptShownAt/ratingPromptAction/ratingPromptActionAt`）
- 不包含任何用户复制内容、页面内容、URL
