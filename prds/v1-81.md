# 问题修复和功能优化（V1-81 Pro 意向强度验证：真实样本导出取证 + 收款/订阅推进决策阈值固化）

## 目标
在不引入支付、不新增权限、不上传用户内容的前提下，把 v1-80 的“问卷意向分布（7d）可导出”推进为**可决策、可复盘、可审计**的商业化证据：用真实样本跑出一份可落盘的分布导出与决策摘要，并固化是否推进 S4「收款/订阅链路」的量化决策阈值。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、真实样本跑数 + 证据包落盘（收入证据）
  - 前置条件：
    - 真实 Chrome 环境可用（用于 Options 操作与文件下载）
    - 「匿名使用数据」开关为 ON（否则不记录/不导出）
  - 收集真实样本（以 v1-80 口径为准）：
    - 让真实用户完成 Pro 意向问卷并点击「复制问卷（含环境信息）」产生 `pro_waitlist_survey_copied`
    - 最小可交付：先跑通流程并落盘（建议至少产生 `survey_intent >= 5`）；样本量不足时，决策结论必须落在“继续收集”
    - 建议最小样本量门槛：过去 7 天 `survey_intent >= 30`（不足则先继续收集，不进入“收款/订阅”实施）
  - 导出并落盘：
    - 导出入口：Options -> 隐私页 ->「Pro 意向漏斗摘要」->「导出问卷意向分布（过去 7 天，JSON）」
    - 落盘目录：`docs/evidence/v1-81/`
    - `index.md`：记录口径、导出文件清单、sha256、以及“无 PII”断言结论
    - 至少落盘 1 份“真实样本”导出 JSON（文件名保持与 v1-80 一致的命名风格）
  - 简报同步：新增 `docs/reports/v1-81-report.md`，写清楚本轮样本量、关键分布结论、以及对 S4 的决策建议（推进/暂缓/继续收集）

2、固化 S4「收款/订阅链路」推进的决策阈值（可复盘口径）
  - 新增或更新阈值口径文档（固定路径）：`docs/monetization/pro-subscription-decision-thresholds.md`
  - 阈值必须完全基于 v1-80 导出的可审计字段（禁止引入新采集项）：
    - 样本量门槛：`survey_intent`（7d）
    - 高意向占比：`(pay_willing_yes + pay_willing_maybe) / survey_intent`
    - 价格区间集中度：`price_monthly_* / price_annual_*` 的主峰 bucket 与占比
    - 能力偏好：`capability_*` Top2 与覆盖率
  - 输出必须给出明确的三段式结论（可被验收判定）：
    - A：继续收集（样本不足/分布不稳定）
    - B：先迭代价值表达与能力包（意向不足但有明确偏好）
    - C：可推进收款/订阅链路最小实现（样本与意向达到门槛）

3、自动生成“意向决策摘要”+ 测试/用例（可执行）
  - 新增一个离线可运行的决策摘要生成脚本（固定入口）：`scripts/build-pro-intent-decision-pack.ts`
    - 输入：v1-80 导出的分布 JSON（路径参数）
    - 输出：写入 `docs/evidence/v1-81/` 的 Markdown 摘要 + JSON 摘要（含阈值版本与口径字段，便于审计复盘）
  - 单元测试门禁（避免口径漂移）：
    - 在 `scripts/unit-tests.ts` 增加用例：给定分布 JSON 样例，断言阈值判定结果稳定、字段齐全、且不包含任何自由文本/联系方式原文/URL/标题/复制内容
    - `bash scripts/test.sh` 全量 PASS
  - 新增测试用例文档（固定路径）：`docs/test-cases/v1-81.md`
    - 必须覆盖：真实样本收集/导出、脚本生成摘要、sha256 复算、以及“无 PII”复核
    - 必须包含一次回归记录（日期 + PASS/BLOCKED + 导出文件名/摘要文件名）
