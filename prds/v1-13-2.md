# V1-13-2 Options WOM 行为落地（UTM 可审计 + 模板化反馈 + 本地匿名事件）

## 目标
复用既有 WOM 链路实现 Options 页四个入口的实际行为：分享打开/复制分享文案/去评价/反馈与建议；确保 UTM 可审计、反馈模板隐私合规，并复用既有本地匿名事件（开关默认关闭）。

## 具体任务
1. 逻辑落地：`src/options/options.ts`
2. 分享（可审计 UTM）：
   - 复用 `src/shared/word-of-mouth.ts` 的 `buildChromeWebStoreDetailUrl(...)`
   - 固定 UTM：
     - `utm_source=copylot-ext`
     - `utm_medium=options-entry`
     - `utm_campaign=v1-13`
   - 打开方式：`chrome.tabs.create({ url })`
3. 复制分享文案：
   - 复用 `buildShareCopyText(getMessage, storeUrl)`
   - 仅用户点击后写入剪贴板；失败提示且不影响 Options 其它功能
4. 去评价：
   - 复用 `buildChromeWebStoreReviewsUrl(chrome.runtime.id)`，`chrome.tabs.create` 打开
5. 反馈与建议（模板化 + 隐私约束）：
   - 复用 `buildFeedbackIssueUrl(...)` + `buildFeedbackSettingsSnapshot(currentSettings)`
   - env 字段仅允许：扩展版本/扩展 ID/userAgent/navigator.language/UI language（`chrome.i18n.getUILanguage()`）
   - 严禁拼装任何网页内容、URL、标题、复制内容
6. 本地匿名事件（复用既有事件名，不新增事件）：
   - 点击入口后调用 `recordTelemetryEvent`：
     - `wom_share_opened` / `wom_share_copied` / `wom_rate_opened` / `wom_feedback_opened`
   - 开关关闭时不记录、不缓存、不补发（由 `recordTelemetryEvent` 既有逻辑保证）

## 验收标准
- 四个入口均按要求打开对应页面/复制文案
- 分享详情页 URL 携带固定 UTM（可在用例中审计）
- 反馈 issue body 仅含环境信息 + settings 快照 + 复现模板，不含任何网页/复制/URL/标题内容
- “匿名使用数据”开关关闭不记录，开启后记录 `wom_*`
