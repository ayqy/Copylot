# 问题修复和功能优化（V1-34 Prompt 作用域一致性修复：悬浮按钮 Prompt 优先选区）

## 目标
在不新增任何扩展权限、不改变既有复制/Prompt/引导/口碑/商业化功能主逻辑的前提下，修复“通过悬浮按钮的 Prompt 菜单使用 Prompt 时会忽略用户选区、始终作用于当前块”的 UX 不一致问题：当用户在当前块内存在有效选区时，Prompt 必须优先作用于选区（含表格整表优先与精确选区 fragment），使行为与右键 Prompt 入口对齐，并通过可执行测试用例验证达到可发布状态。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、修复悬浮按钮 Prompt 的作用范围（选区优先，避免忽略划词内容）
  - 修改：`src/content/content.ts`
  - 调整 `handlePromptClick(promptId)` 的处理根节点选择逻辑：
    - 当存在“有效选区”且选区位于当前 `currentTarget` 内：使用 `getSelectionRootElementForProcessing()` 作为 `processContent()` 的输入根节点
      - 表格内选区：优先整表（table ancestor）
      - 非表格选区：优先精确选区 fragment（保留 `<a>/<b>/<code>` 等内联结构）
    - 当无有效选区，或选区不在 `currentTarget` 内：保持现有行为，仍对 `currentTarget` 做 `processContent()`
  - 兼容性约束：
    - 不改变无选区时的行为（仍作用于当前块）
    - 不改变右键菜单 Prompt（background message）路径的行为
    - 不引入新的权限、不新增依赖

2、回归隐私与统计链路（不因作用域修复引入新数据暴露或重复计数）
  - 确认一次 Prompt 操作只产生一次：
    - 剪贴板写入（`finalText`）
    - telemetry 事件：`copy_success` + `prompt_used`
    - growth stats：成功复制计数 + 首次 Prompt 使用时间等字段更新逻辑保持正确
  - “匿名使用数据”关闭时仍不记录/不缓存/不补发；开启时本地事件日志可审计（仍不联网发送）

3、补齐测试/用例闭环（让验收可执行、可回归）
  - 新增测试用例文档：`docs/test-cases/v1-34.md`
  - 文档必须覆盖（至少）：
    - 用例 A（选区优先）：在某内容块内划词，使用悬浮按钮 Prompt，结果仅包含选区内容（非整块）
    - 用例 B（选区不在当前块）：页面其它位置保留选区，但在另一块上使用悬浮按钮 Prompt，结果仍作用于该块（不被其它选区“劫持”）
    - 用例 C（无选区）：无选区时使用悬浮按钮 Prompt，行为与当前版本一致（作用于当前块）
    - 用例 D（表格选区）：在表格单元格内划词后使用悬浮按钮 Prompt，结果作用于整表（与右键选区路径一致）
    - 用例 E（统计可审计）：开启“匿名使用数据”，执行一次用例 A，Options -> 隐私 -> “本地匿名事件日志”中可看到 `prompt_used` 与 `copy_success`；关闭开关后事件日志被清空
    - 自动化回归记录：记录一次 `bash scripts/test.sh` 的执行结论与日期（作为可发布审计证据）

## 验收标准
* 修复上述所有问题，并测试通过，达到可发布状态
* 实现以上所有需求，并测试通过，达到可发布状态
* 悬浮按钮 Prompt 作用域与用户预期一致：
  - 当前块内存在有效选区时：Prompt 仅作用于选区（表格内选区优先整表）
  - 无选区时：Prompt 仍作用于当前块（行为不变）
  - 选区不在当前块时：Prompt 不被其它选区影响，仍作用于当前块
* 右键菜单 Prompt 行为不变：有选区作用于选区，无选区作用于整页（按既有实现）
* 统计与隐私无回归：
  - 一次 Prompt 操作仅记录一次 `copy_success` 与一次 `prompt_used`
  - “匿名使用数据”关闭时不记录/不缓存/不补发；开启时仅本地可审计，不产生任何联网发送
* 已新增 `docs/test-cases/v1-34.md` 且可执行：按用例 A-E 完成回归并记录结论；`bash scripts/test.sh` 全量通过
