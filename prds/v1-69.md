# 问题修复和功能优化（V1-69 真实 CWS 发布 + 商店端取证闭环：上架获客入口打通 + 可审计商业化证据落盘）

## 目标
基于 `docs/roadmap_status.md` 的收入优先 Top3，本轮选择 Top1「（v1-69）完成一次真实 CWS 发布 + 商店端取证」作为 1~3 天最小可交付增量主题。

收入第一：把“上架=获客入口 + Pro 候补 CTA=商业化验证入口”的链路真正跑通并可取证/可复核——在网络可达（代理/VPN 可用）后完成一次发布到 CWS `default` channel，并落盘发布诊断证据包（`.publish.json`）+ 商店端截图索引；随后从商店安装回归并导出 Pro 意向漏斗证据（摘要/证据包）作为可验证的商业化推进证据。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、发布前门禁与“可审计发布证据”预演（离线可执行，先确保流程与证据口径稳定）
  - 按 `docs/publish/cws-preflight-checklist.md` 完成离线门禁（版本/产物一致性 + `bash scripts/test.sh` 全量回归）。
  - 执行 dry-run 并落盘诊断证据包（必须可复核、可审计）：
    - 命令：`npm run publish:cws -- --dry-run --evidence-dir docs/evidence/v1-69/preflight/`
    - 断言：目录内产生 1 个 `*.dry-run.json`（`packVersion="v1-62"`，且包含 Proxy Diagnostic + Preflight Report + zip sha256）。

2、真实发布到 CWS（需网络可达 + 代理/VPN 可用；必须落盘 `.publish.json` 证据包）
  - 在代理/VPN 可达后执行真实发布并落盘证据：
    - 命令：`npm run publish:cws -- --evidence-dir docs/evidence/v1-69/preflight/`
    - 断言：目录内产生 1 个 `*.publish.json`，且 `publishAttempt.published=true`、`channel="default"`、`zip.sha256` 可复核。
  - 将本轮真实发布的关键可审计信息写入证据索引：`docs/evidence/v1-69/index.md`（包含：发布版本号、`*.publish.json` 文件名、`zip.sha256`、执行命令、代理环境变量“脱敏口径”）。

3、商店端取证（“上架获客入口”与“商业化 CTA”必须可见且可审计）
  - 打开 Chrome Web Store 商店页完成截图取证（截图断言参考 `docs/evidence/v1-62/index.md`）并落盘到：`docs/evidence/v1-69/screenshots/`
    - `01-cws-store-version-and-publish-time.png`：版本号与发布时间可见且与发布版本一致
    - `02-cws-store-pro-cta-visible.png`：Pro 候补 CTA 可见（不暗示 Pro 已上线/可订阅）
    - `03-cws-store-detail-top.png`：顶部关键物料（名称/图标/简介）可见
  - 在 `docs/evidence/v1-69/index.md` 中补齐截图索引（文件名 + 每张图的断言结论 + 与 `docs/evidence/v1-66/` / `docs/evidence/v1-68/` 的 `inputs.sha256` 复核口径）。

4、从商店安装回归 + 转化入口取证（证明“能获客、能转化、能导出数据”）
  - 必须从 CWS 安装当前版本（禁止用 `plugin-*.zip`/unpacked 替代）并完成最短路径回归（动作参考 `docs/test-cases/v1-45.md` 的用例 G）：
    - 触发 Pro 候补链路事件（至少覆盖：entry/opened/waitlist_opened/waitlist_copied）
    - 导出并落盘（文件名固定，便于审计与对比基线 `docs/evidence/v1-42/` / `docs/evidence/v1-44/`）：
      - `docs/evidence/v1-69/pro-funnel-summary.json`
      - `docs/evidence/v1-69/pro-funnel-evidence-pack.json`
  - 在 `docs/evidence/v1-69/index.md` 记录导出文件名与关键字段断言（例如：事件计数非 0、导出窗口、extensionVersion 与商店版本一致）。

5、测试/用例与汇报材料闭环（确保“做了就能复盘，且能复现”）
  - 新增测试用例文档（固定路径，必须可逐条执行）：`docs/test-cases/v1-69.md`
    - 覆盖：离线门禁 -> dry-run 证据 -> 真实发布 -> 商店截图 -> 商店安装回归 -> 导出 Pro 漏斗证据 -> 更新证据索引
    - 必须包含一次回归记录（日期 + PASS 或 BLOCKED + 阻塞原因/证据文件名）
  - 新增简报（固定路径建议）：`docs/reports/v1-69-report.md`
    - 只写结论：是否真实发布成功、是否取证齐全、商业化推进证据是什么、阻塞是什么（若有）
  - 更新 `docs/roadmap_status.md`：勾选/更新 v1-69 状态、刷新当前进度与 Top3（保持阻塞不断档）
