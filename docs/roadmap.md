# Copylot 产品路线图与商业化/增长方向（截至 2026-03-18）

## 0. 现状摘要（以仓库为准）
- 当前版本：`manifest.json` 显示为 `1.1.18`
- 现有核心能力（用户可感知）
  - 智能区块复制（单击/双击触发）与代码块悬停复制
  - 追加模式（Shift 追加合并）
  - 表格一键转 Markdown / CSV
  - Prompt 管理器（`chrome.storage.sync` 同步、内置 Prompt、使用次数统计）
  - Prompt 可配置自动打开 Chat 服务（ChatGPT/Claude/Gemini/Kimi 等内置入口）
- 已知体验缺口（来自 `docs/Future-Features.md` 的方向性记录）
  - “真正的页面去噪/阅读模式级清洗”仍有提升空间
  - Popup 语言切换 UI 缺失（当前跟随系统语言）
  - Prompt 数据（usageCount/lastUsedAt）未充分转化为排序/效率功能
  - 不同入口下 Prompt 应用范围可能不一致，需要统一预期

## 1. 北极星指标与关键指标（Growth Metrics）
- 北极星指标：Activated WAU（每周活跃用户中“成功复制 >= 1 次”的人数）
- 核心漏斗：安装 -> 首次打开 Popup -> 首次成功复制 -> 首次使用 Prompt -> 7 日内复用
- 关键指标（按优先级）
  - Activation：新装 3 分钟内首次成功复制率
  - Retention：D1/D7 复用率、每活跃用户周复制次数
  - Adoption：Prompt 使用率、表格转换使用率、追加模式使用率
  - Word of Mouth：商店评分、评价率、分享点击率、反馈转化率
  - Revenue（后续）：升级入口点击率、付费转化率、付费留存、ARPPU

## 2. 产品路线图（2026）
> 节奏建议：4~6 周一个小版本；每个版本都有“可验证的指标提升假设”。

### 2.1 2026-03-18 ~ 2026-04-30：V1（增长与商业化基础能力）
- 目标：把“好用”变成“用得起来、会推荐”，建立可观测性与口碑闭环
- 交付：
  - 事件与漏斗体系 + 隐私合规的“匿名使用数据”开关（默认关闭）
  - 新手引导（首次打开 Popup 三步激活）
  - 评价/分享/反馈闭环（低打扰、可永久关闭）
  - ASO 素材规范与内容资产最小库（截图脚本/关键词/教程模版）
- 验证：Activation +10~20%，商店评分稳定提升（以 30 天滚动为观察窗）

### 2.2 2026-05-01 ~ 2026-06-30：V1.2（核心体验“确定性”提升）
- 目标：降低“复制出来还要再整理”的概率，减少预期落差
- 方向：
  - 真正的页面级去噪（参考阅读模式/文本密度/黑名单）与“转换整页”体验升级
  - Prompt：按最常用/最近使用排序；统一不同入口下的应用范围与文案提示
  - 语言：补回 Popup 语言切换（或明确“跟随系统”并在 UI 中解释）
  - 富媒体：Canvas/视频复制期望对齐（至少提供更明确的结果说明与选项）
- 验证：负面反馈率下降，复制后返工率（主观）下降，评价文本出现“干净/省时间”占比提升

### 2.3 2026-07-01 ~ 2026-08-31：V1.3（工作流增强与模板增长）
- 目标：让用户形成“Copylot = 我的 AI 输入工作台”的心智
- 方向：
  - Prompt Pack：内置更多高复用模板（总结/翻译/改写/代码审阅/提炼表格结论）
  - 一键工作流：复制 + 自动打开指定 Chat + 可选附带来源信息（更可控）
  - 面向笔记工具的导出能力（优先：Obsidian/Notion 的可粘贴格式与结构化模板）
- 验证：Prompt 使用率提升、每活跃用户复制次数提升、分享点击率提升

### 2.4 2026-09-01 ~ 2026-10-31：商业化 MVP（Pro 试水）
- 原则：不破坏免费核心体验；Pro 必须是“更强工作流/更省时间”，而非“基础功能收费”
- 推荐 Pro 功能包（可按实现成本分层）
  - 高级页面清洗（阅读模式 + 去广告/评论区/推荐区规则）
  - 批量采集与整理（跨页面收集、命名、导出、历史检索）
  - Prompt Pack 高级库/团队共享（需要后端则后置）
- 价格与套餐（建议先做验证，再决定最终价格）
  - Pro：月付/年付；优先验证“升级入口点击 -> 付费意向”而非立刻追求规模
  - Team（可选）：面向小团队的共享 Prompt 库与统一规范
- 验证：升级入口点击率、意向留资率/购买转化率、付费用户留存与满意度

### 2.5 2026-11-01 ~ 2026-12-31：增长规模化（渠道 + 口碑复利）
- Chrome Web Store ASO 持续迭代：关键词/本地化/截图顺序/视频/GIF/更新日志
- 内容增长：围绕“表格转 CSV/MD”“代码块清理”“AI 输入工作流”做 SEO 与分发
- 社区与合作：Obsidian/Notion/开发者社区；与 AI 工具博主/教程作者合作
- 机制化复盘：每月一轮增长实验（假设 -> 实施 -> 指标 -> 复盘 -> 迭代）

## 3. 获客与增长抓手（可立即执行）
- ASO：围绕“copy to markdown / table to csv / prompt template / ai friendly”做标题与关键词矩阵；中英文双语一致化
- 口碑：低打扰评价引导 + 反馈闭环（问题模板化、快速响应、更新日志可感知）
- 内容：每个核心功能至少 1 篇教程 + 1 个 30 秒短视频脚本 + 1 组可复用截图
- 裂变：分享入口（带 UTM）+ 可选“复制结果附带 Copylot 来源标记”（默认关闭，避免打扰）

## 4. 商业化策略（方向与边界）
- 定位：Copylot = “AI 时代的复制/采集输入层”；Pro = “更强清洗 + 更强整理 + 更强导出”
- 约束：
  - 默认不收集内容；任何数据采集必须匿名、可关闭、可审计
  - Pro 不应依赖读取/上传用户复制内容；优先做本地增强与可选导出
- 路径：
  - 先验证意向（升级入口、候补名单、问卷）-> 再做最小可交付 Pro -> 再扩展到团队与高级能力
