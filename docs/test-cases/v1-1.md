# V1-1 测试用例（口碑闭环 MVP）

## 范围
- Popup：反馈入口（带模板）、分享入口（带 UTM + 复制文案）、去评价入口
- i18n：中英文展示
- 隐私：反馈模板不包含任何用户复制内容

## 运行自动化检查
```bash
./scripts/test.sh
```

## 手工用例

### TC-01 反馈入口：跳转 URL 预填 title/body
步骤：
1. 打开插件 Popup
2. 点击“反馈与建议 / Feedback & Suggestions”
期望：
- 打开 GitHub `issues/new`
- URL 中包含 `title` 与 `body` 参数，页面标题/正文已被预填

### TC-02 反馈模板字段校验（环境信息 + 开关快照）
步骤：
1. 打开插件 Popup
2. 点击“反馈与建议 / Feedback & Suggestions”
3. 在新建 Issue 页面检查 body 内容
期望（必须包含）：
- 插件版本、扩展 ID
- `navigator.userAgent`、`navigator.language`、`chrome.i18n.getUILanguage()`
- `copilot_settings` 关键开关快照（至少：启用复制/悬停/追加模式/输出格式/表格格式/是否附带标题与 URL/交互模式）
- 复现步骤 / 期望结果 / 实际结果占位区块

### TC-03 隐私校验：不包含任何用户复制内容
步骤：
1. 在任意网页进行一次复制（确保剪贴板存在内容）
2. 打开插件 Popup，点击“反馈与建议 / Feedback & Suggestions”
3. 检查新建 Issue 的 body
期望：
- body 中不出现任何用户复制的正文内容（仅包含环境信息与开关配置快照）

### TC-04 分享入口：打开商店页（UTM 可审计）
步骤：
1. 打开插件 Popup
2. 点击“分享给朋友 / Share”
期望：
- 打开 Chrome Web Store 商店页
- URL 携带固定 UTM 参数：`utm_source`、`utm_medium`、`utm_campaign`

### TC-05 分享入口：一键复制分享文案
步骤：
1. 打开插件 Popup
2. 点击“复制文案 / Copy Text”
3. 粘贴到任意输入框
期望：
- 粘贴内容包含商店链接（带 UTM）
- 文案包含简短卖点描述（中英文随 UI 语言切换）

### TC-06 去评价入口：跳转 reviews
步骤：
1. 打开插件 Popup
2. 点击“去评价 / Rate”
期望：
- 打开 Chrome Web Store reviews 页面
- 本版本不出现任何自动评价弹窗/强引导（零打扰）

### TC-07 i18n 展示（至少中英文）
步骤：
1. 将浏览器 UI 语言切换为中文，打开 Popup
2. 确认“反馈 / 分享 / 复制文案 / 去评价”等文案为中文
3. 将浏览器 UI 语言切换为英文，打开 Popup
4. 确认上述文案为英文
期望：
- 文案与反馈模板内容随 UI 语言切换
