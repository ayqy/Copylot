## 具体任务
1、统一产品话术（Slogan/卖点/差异化）并落地到“短描述”（可审计、可回归）
  - 新增并维护话术基准文档：`docs/aso/value-prop.md`
    - 必须包含：1 句主 Slogan + 3 个核心卖点 + 3 个差异化卖点（中英文对照）
    - 所有表述必须与仓库真实能力一致（以 `README.md`、`docs/roadmap.md`、`manifest.json` 为准），禁止“夸大/未实现能力”（避免预期落差导致差评）
    - 必须包含隐私声明口径（中英文对照）：默认本地处理；不收集/不上传复制内容；“匿名使用数据”默认关闭，开启后仅本地记录事件名/时间戳/少量枚举字段（不含内容），不联网发送
  - 更新扩展短描述（Chrome Web Store 入口处可见，需控制长度与一致性）：
    - 更新 `_locales/en/messages.json` 的 `appDescription`
    - 更新 `_locales/zh/messages.json` 的 `appDescription`
    - 约束：短描述需要可直接用于商店展示（简洁、聚焦价值、避免堆砌）；避免出现“收集数据/联网分析”等歧义表述
  - 变更必须保证 `npm run check-i18n` 通过

2、重写 Chrome Web Store 长描述（中英文）+ 关键词矩阵（可直接粘贴、可持续迭代）
  - 更新并固化商店长描述（要求“非 Markdown”可直接粘贴到 CWS）：
    - `docs/ChromeWebStore-Description-EN.md`
    - `docs/ChromeWebStore-Description-ZH.md`
    - 结构要求（两份保持一致）：
      - 开头 3-5 行：问题 -> 价值（对齐 Slogan）
      - 核心功能：只写仓库已实现能力，按“最强 3 个卖点优先”排序（例如：智能区块复制/表格转 MD&CSV/代码块清理/追加模式/Prompt 工作流）
      - 使用场景：面向知识工作者/开发者/数据工作者各 1 段（短而具体，避免泛化）
      - 隐私与权限：明确“本地处理 + 最小权限 + 匿名使用数据开关（默认关闭，仅本地记录）”
      - CTA：反馈入口（GitHub Issues）与分享/评价（不强引导）
    - 禁止项：Markdown 标题语法（`#`/`##`）、emoji 密集排版、以及任何与实际不符的“悬停复制图片/视频”等描述
  - 新增关键词矩阵与标题建议（便于 ASO 迭代与本地化一致性）：
    - 新增文档：`docs/aso/keywords.md`
    - 至少包含：
      - EN：关键词组（copy to markdown/table to csv/prompt template/ai ready 等）+ 3 套可选标题/副标题组合
      - ZH：关键词组（智能复制/网页转 Markdown/表格转 CSV/Prompt 模板 等）+ 3 套可选标题/副标题组合

3、隐私政策更新 + 发布前检查用例（可执行、可验证）
  - 更新隐私政策文档：`docs/privacy-policy.md`
    - 将历史命名（如 MagicCopy）统一为 Copylot
    - 更新“Last Updated”日期
    - 明确说明本地存储的内容范围（settings/prompts/growth_stats/telemetry_events 等均为本地），并写清“匿名使用数据”开关语义：默认关闭；开启后仅本地记录匿名事件；关闭后清空；不联网发送
    - 文案必须可支撑 Chrome Web Store 的“数据使用披露”填写（避免与实际实现冲突）
  - 新增并维护测试/用例文档：`docs/test-cases/v1-7.md`
    - 必须覆盖（至少）：
      - 短描述校验：`_locales/*/messages.json` 的 `appDescription` 与 `docs/aso/value-prop.md` 口径一致；中英文均为可上架文案
      - 长描述校验：`docs/ChromeWebStore-Description-*.md` 不包含 Markdown 标题语法；描述能力与 `README.md` 一致；隐私段落包含“匿名使用数据默认关闭且仅本地记录”
      - 隐私政策校验：`docs/privacy-policy.md` 命名/日期/开关语义正确；不出现“上传内容/联网分析”等冲突表述
      - 回归：执行 `bash scripts/test.sh` 全量通过（lint/type-check/check-i18n/unit-tests/build:prod）
    - 验收前必须按文档执行手工检查，并在文档中记录检查结论与 `bash scripts/test.sh` 结果
