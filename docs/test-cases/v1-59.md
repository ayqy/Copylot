## 用例 C：匿名开关 ON（可复制含表格/洞察的获客效率周报 Markdown）

目标：覆盖至少 2 个 campaign + 空 campaign；覆盖 `distCopies=0 -> N/A`；排序稳定；小数格式固定 4 位。

步骤：
1. Options -> 隐私与可观测性：打开“匿名使用数据”开关为 ON。
2. Options -> Pro Tab：设置 `#pro-intent-campaign=twitter`（合法 campaign）。
3. 生成 distCopies（v1-57 口径）：
   - 点击渠道分发工具包任意复制按钮至少 2 次（例如 `#pro-store-url-copy` / `#pro-distribution-pack-copy`），确保记录 `pro_distribution_asset_copied`。
4. 生成 leads（v1-54 口径）：
   - 点击加入候补/问卷相关复制按钮，至少触发 1 次 `pro_waitlist_copied` 或 `pro_waitlist_survey_copied`。
5. 生成第二个 campaign（例如 `xhs`）：
   - 将 `#pro-intent-campaign` 改为 `xhs`，重复步骤 3~4。
6. 生成空 campaign 桶：
   - 清空 `#pro-intent-campaign`（留空），再触发 1 次分发复制或留资复制，使事件 `props.campaign` 缺失/空字符串。
7. 生成 `distCopies=0 -> N/A` 场景：
   - 选择一个 campaign（例如 `ads`），仅触发 leads（步骤 4），不触发分发复制（步骤 3）。
8. 回到 Options -> 隐私与可观测性 -> Pro 面板，点击 `#copy-pro-acquisition-efficiency-by-campaign-weekly-report`。
9. 将剪贴板 Markdown 粘贴到周报/复盘文档中查看。

期望：
- Markdown 标题/Env/表格/Insights/Privacy 区块顺序稳定。
- 表格列固定：`campaign/leads/distCopies/leadsPerDistCopy`。
- 排序稳定：`leadsPerDistCopy desc -> leads desc -> distCopies desc -> campaign asc`（与 v1-58 rows 一致）。
- `distCopies=0` 的行必须输出 `leadsPerDistCopy=N/A`。
- 小数格式固定 4 位（例如 `0.5000`），便于审计与对比。
- Insights 至少 3 条且可复核：
  - Top1 campaign（按 `leadsPerDistCopy`）
  - Top1 campaign 的 `leads/distCopies/leadsPerDistCopy`
  - 空 campaign leads 占比

截图索引建议：
- `03-privacy-pro-acq-eff-weekly-report-on-copied.png`：匿名 ON 时复制成功通知 + 粘贴内容可见表格与 Insights

## 用例 D：互证复算（v1-59 Markdown 表格必须与 v1-58 合并 CSV 口径一致）

目标：证明 v1-59 周报 Markdown 可被审计复核（同一 window 下与 v1-58 合并 CSV 对齐复算）。

步骤：
1. 匿名使用数据 ON。
2. 导出 v1-58：点击 `#export-pro-acquisition-efficiency-by-campaign-7d-csv` 下载合并 CSV。
3. 复制 v1-59：点击 `#copy-pro-acquisition-efficiency-by-campaign-weekly-report` 得到 Markdown。
4. 用同一 `windowFrom/windowTo` 的口径按 `campaign` 对齐 CSV 与 Markdown 表格，逐行核对：
   - `leads` 必须一致
   - `distCopies` 必须一致
   - `leadsPerDistCopy` 必须一致（分母为 0 时均为 `N/A`）

期望：
- 任意 campaign 行可与 v1-58 CSV 互证一致；可复算、可审计。
