## 用例 A：Pro Tab 出现融合判断区块

步骤：

1. 打开 `src/options/options.html#pro`。
2. 检查 `#pro-route-verdict-panel`、`#copy-pro-route-validation-verdict-summary`、`#download-pro-route-validation-verdict-json`。

期望：

- 新区块可见。
- 两个动作按钮存在且默认可点击。
- 隐私提示明确说明不包含页面正文、复制内容、URL、标题或联系方式。

## 用例 B：复制出的摘要同时包含三项检查与最终 verdict

步骤：

1. 在 `Options -> Pro` 点击“复制 Pro 路线融合判断摘要”。
2. 读取最近一次剪贴板文本。

期望：

- 文本包含 `V4-11 Pro 路线融合判断摘要`。
- 文本包含 `route_leader_consistent=`、`route_stability_ready=`、`gate_allows_payment_evaluation=`。
- 文本包含 `stay_validation` 或“继续验证”结论。
- 文本不包含页面正文、复制内容、URL、标题或联系方式。

## 用例 C：下载出的 JSON 可复核统一判断

步骤：

1. 点击“下载 Pro 路线融合判断 JSON”。
2. 打开生成文件并检查关键字段。

期望：

- 文件名符合 `copylot-pro-route-validation-verdict-v4-11.json`。
- 文件内包含 `routeLeaderTrackId`、`routeLeaderConsistent`、`routeStabilityReady`、`gateAllowsPaymentEvaluation`、`verdictCode`、`reasons`。
- 当前样例数据下，结果仍回到 `verdictCode = stay_validation`。

## 用例 D：状态文档与边界一致

步骤：

1. 复核 `docs/roadmap.md`、`docs/roadmap_status.md`、`docs/reports/v4-11-report.md`。

期望：

- Roadmap 明确最后一项已进入“统一融合判断已完成，后续只剩收费评估审计”的状态。
- 状态文档明确当前 verdict 仍停在 `stay_validation`，还没有进入支付/收款实现。
