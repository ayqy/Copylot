# Copylot 增长与商业化基础能力 V1（可观测性 + 新手引导 + 评价/分享闭环）

## 目标
在不破坏“本地处理、隐私优先、零打扰”的产品原则下，补齐增长与商业化的基础设施：让更多新安装用户在 3 分钟内完成首次成功复制并形成复用习惯，提升 Chrome Web Store 评分与自然流量，并为后续 Pro 版本定价/转化提供数据与入口。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、增长数据与指标体系（隐私合规）
  - 明确北极星指标：WAU 中“发生过至少 1 次成功复制”的激活用户数（Activated WAU）
  - 定义关键漏斗：安装 -> 首次打开 Popup -> 首次成功复制 -> 首次使用 Prompt -> 7 日内复用
  - 设计并落地事件规范（事件名、字段、采样、去标识化规则、保留周期），并提供可审计的埋点清单
  - 产品内新增“匿名使用数据”开关：默认关闭；开启后才允许发送匿名事件；关闭时不发送、不缓存、不补发

2、首次安装新手引导（Activation）
  - 新安装/清空配置后首次打开 Popup：展示 3 步引导（可跳过、可回看）
    - Step1：如何触发复制（单击/双击/悬停差异）+ 建议默认配置
    - Step2：追加模式（Shift 追加/合并粘贴）+ 开关位置
    - Step3：Prompt 与自动跳转 Chat 服务（选择默认 Chat、是否自动打开）
  - 引导完成后写入“已完成引导”状态，后续不再自动弹出；在设置页保留“重新查看引导”入口

3、口碑与获客闭环（Rating / Share / Feedback）
  - 评价引导：满足触发条件后（例如：安装满 3 天且成功复制 >= 20 次）仅提示 1 次，可“稍后/不再提示”
  - 分享入口：Popup 与设置页新增“分享给朋友”入口（打开带 UTM 的商店页/落地页），支持一键复制分享文案
  - 反馈入口：将“反馈与建议”升级为可用问题模板（包含版本号、浏览器、系统语言、功能开关状态等信息的自动拼装文本）

4、商业化准备（不影响现有免费功能）
  - 明确 Pro 功能包与边界：哪些永久免费、哪些进入 Pro（优先选择“高价值但不破坏隐私”的能力）
  - 新增“升级 Pro / 加入候补名单”入口（先跳转到公开页面/文档即可），并纳入埋点以评估付费意向
  - 设计转化链路的最小闭环：入口 -> 价值说明 -> 行动（购买/登记/联系）-> 回流

5、增长运营资产与商店转化优化（ASO）
  - 统一产品话术：明确 1 句主 Slogan + 3 个核心卖点 + 3 个差异化卖点（面向知识工作者/开发者/数据工作者）
  - 更新并固化商店素材规范：标题/副标题、关键词、截图顺序、GIF/视频脚本、更新日志模板
  - 以“表格转 CSV/MD”“代码块清理”“Prompt 工作流”为主线补齐示例与教程，形成可复用的增长内容库

6、测试用例与发布准备
  - 补齐自动化测试：覆盖引导展示/埋点开关/评价触发/分享链接/反馈模板生成等关键路径
  - 补齐手工测试清单：Chrome/Edge 最新版、不同语言环境、隐私开关开启/关闭、无障碍与性能基本检查
  - 发布前完成回归：确保不增加不必要权限、不引入明显性能退化、不破坏现有复制体验

## 验收标准
* 修复上述所有问题，并测试通过，达到可发布状态
* 实现以上所有需求，并测试通过，达到可发布状态
* 新装用户首次打开 Popup 时可看到引导，完成后不再自动出现；设置页可手动重新打开引导
* “匿名使用数据”默认关闭；开启/关闭行为可通过抓包或日志验证（关闭时不发送、不缓存、不补发）
* 评价引导满足条件最多提示 1 次，且不会在用户明确选择“不再提示”后再次出现
* 分享与反馈入口可用：链接可打开、文案可复制、反馈模板包含版本号与关键环境信息
* 新增功能覆盖对应自动化测试与手工测试用例，CI/Lint/TypeCheck 通过，可打包发布
