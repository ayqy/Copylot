## 用例 A：Pro Tab 出现结构化导出验证卡片

步骤：

1. 打开 `src/options/options.html#pro`。
2. 检查 `#pro-validation-structured-export`、`#pro-validation-structured-open`。
3. 未填写 campaign 时检查 3 个复制按钮。
4. 填写 `twitter` 后再次检查按钮状态。

期望：

- 结构化导出与下游工作流验证卡片可见。
- 未填写 campaign 时，路线链接/验证简报/验证清单复制按钮为 disabled。
- 填写 campaign 后，3 个复制按钮恢复可用。

## 用例 B：结构化导出验证素材带上独立 content

步骤：

1. 在 `Options -> Pro` 填写 `campaign=twitter`。
2. 点击“复制路线链接”“复制验证简报”“复制验证清单”。
3. 读取最近一次剪贴板文本。

期望：

- 路线链接包含 `utm_content=options_structured_export_cta`。
- 验证简报包含路线/商店/隐私 3 类固定链接。
- 验证清单包含 campaign 和路线链接。
- 素材不包含复制正文、网页正文、页面 URL 或标题。

## 用例 C：匿名导出可以区分结构化导出路线

步骤：

1. 填写 `campaign=twitter`。
2. 点击“打开结构化导出路线”。
3. 依次复制路线链接、验证简报、验证清单。
4. 导出本地 Pro intent / distribution 摘要。

期望：

- intent 导出中出现 `pro_waitlist_opened`，且 `content=options_structured_export_cta`。
- distribution 行为仍落在 `validation_route / validation_brief / validation_checklist` 动作中，但 `content` 可区分为 `options_structured_export_cta`。
- 浏览器级回归可同时覆盖高级页面清洗、批量采集与结构化导出三条路线。

## 用例 D：三轮汇总报告与 roadmap 口径一致

步骤：

1. 执行 `bash scripts/test.sh`。
2. 复核 `docs/reports/v4-6-report.md`、`docs/reports/v4-4-to-v4-6-summary.md` 与 `docs/roadmap_status.md`。

期望：

- `bash scripts/test.sh` 退出码为 `0`。
- `docs/roadmap.md` 中 S3 第 3 项被标记为完成。
- 三轮汇总报告中的里程碑、观察和下一步 Top3 与 roadmap 状态保持一致。
