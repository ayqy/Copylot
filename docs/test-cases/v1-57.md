## 用例 D：campaign 合法（复制内容可复核：商店 URL UTM + 候补 URL campaign + 投放包包含四要素）

步骤：
1. 在 `#pro-intent-campaign` 输入 `twitter`，并失焦（触发 change，输入会被规范化）。
2. 观察：4 个按钮均变为可点击（enabled）。
3. 点击 `#pro-store-url-copy`，从剪贴板粘贴得到商店安装链接。
4. 解码复核：URL query 参数必须包含：
   - `utm_source=copylot-ext`
   - `utm_medium=distribution_toolkit`
   - `utm_campaign=twitter`
5. 点击 `#pro-waitlist-url-copy`，从剪贴板粘贴得到候补 URL；解码复核 `body` 中包含 `campaign: twitter` 行（可用 `node -e` 解码或直接打开 URL 查看）。
6. 点击 `#pro-waitlist-recruit-copy`，从剪贴板粘贴得到招募文案：必须包含候补链接与 campaign 信息。
7. 点击 `#pro-distribution-pack-copy`，从剪贴板粘贴得到 Markdown 投放包，必须包含：
   - 商店安装链接
   - 候补链接
   - 招募文案
   - 可选问卷引导

期望：
- 商店安装链接可解码复核，且 `utm_campaign` 与输入 campaign 一致。
- 候补链接为 GitHub New Issue URL（title/body 预填），且 body 解码后包含 `campaign: twitter` 行。
- 招募文案与投放包均不拼入任何网页内容/复制内容/当前网页 URL/标题等（仅固定模板 + 环境信息（如有）+ campaign + 商店/候补链接）。

截图索引建议：
- `03-pro-distribution-toolkit-enabled.png`：campaign 合法时 4 个按钮可用
- `04-pro-distribution-toolkit-copied.png`：任一按钮复制成功后短暂变为“已复制”

## 用例 E：匿名开关与分发动作取证导出（7d 按 campaign 聚合 CSV）

步骤：
1. Options -> 隐私与可观测性：确保“匿名使用数据”开关为 OFF。
2. 观察导出入口：`#export-pro-distribution-by-campaign-7d-csv` 应为 disabled（置灰）。
3. 打开开关为 ON。
4. 回到 Options -> Pro Tab，确保 `campaign=twitter`，依次点击 4 个复制按钮各 1 次（触发写入剪贴板成功）。
5. Options -> 隐私与可观测性 -> 本地匿名事件日志：点击“刷新”，确认存在事件：
   - `name=pro_distribution_asset_copied`
   - `props.source='options'`
   - `props.campaign='twitter'`
   - `props.action` 为 4 个枚举之一（与按钮对应）
6. 点击导出按钮 `#export-pro-distribution-by-campaign-7d-csv`，下载 CSV。

期望：
- 匿名 OFF：导出入口置灰且不读取/不推断本地 events。
- 匿名 ON：复制成功会记录 `pro_distribution_asset_copied`；且可导出过去 7 天按 campaign 聚合 CSV。
- CSV 字段顺序稳定且无 BOM；空 campaign（缺失/空字符串）归入 `空 campaign` 桶；`distCopies` 为 4 列复制计数之和。

截图索引建议：
- `05-privacy-pro-distribution-export-disabled.png`：匿名 OFF 时导出入口置灰
- `06-privacy-pro-distribution-export-success.png`：匿名 ON 时导出成功（可见下载文件/提示）

## 用例 F：与 v1-54 leads 对齐复算（leads_per_dist_copy）

目标：把「分发动作 -> 留资 leads」效率变成可复算、可审计的证据。

步骤：
1. 匿名使用数据 ON。
2. 导出 v1-54：`#export-pro-intent-by-campaign-7d-csv`（得到每个 campaign 的 `leads`）。
3. 导出 v1-57：`#export-pro-distribution-by-campaign-7d-csv`（得到每个 campaign 的 `distCopies`）。
4. 在表格中按 campaign 对齐两份 CSV，复算：
   - `leads_per_dist_copy = leads / distCopies`
   - 若 `distCopies=0`，输出 `N/A`

期望：
- 可在同一时间窗内按 campaign 对齐复算出“每次分发带来留资”的效率指标，且可被审计复核。
