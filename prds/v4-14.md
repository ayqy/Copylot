# v4-14 stay_validation 外部话术守门复核包

## 背景

`v4-13` 已把跨 `campaign` 的领先路线复核做成了产品内可导出的审计包，但 roadmap 里仍缺一层关键约束：即便当前结论仍是 `stay_validation`，官网、商店和扩展内的话术也必须有单独的守门检查，避免任何“已验证收费 / 已上线支付 / 可直接升级订阅”的误导表达继续流出。

## 目标

1. 新增 `stay_validation` 外部话术守门复核包，覆盖路线页标题、商店短描述、价值 bullet 与摘要判断。
2. 对每个对外 surface 输出 `aligned / needs_review / blocked` 状态、命中的验证信号和被禁止收费话术。
3. 明确这轮完成的是“话术守门能力”，不是“收费实现开始”。

## 交付范围

1. 新增共享逻辑 `src/shared/pro-stay-validation-messaging-guard.ts`。
2. 新增脚本 `scripts/build-pro-stay-validation-messaging-guard-pack.ts`，基于 `v4-9` writeback 与 `v4-13` campaign review 生成 `docs/evidence/v4-14/messaging-guard-pack/*`。
3. 更新 `src/options/options.ts`、`src/options/options.html`、`_locales/en/messages.json`、`_locales/zh/messages.json`，让 `Options -> Pro` 支持复制 Markdown 与下载 JSON。
4. 更新 `scripts/check-i18n.ts`、`scripts/unit-tests.ts`、`scripts/ui-integration-tests.ts`、`e2e/options-pro-flow.spec.ts`、`scripts/test.ts`，覆盖规则常量、区块可见性和导出产物。
5. 补齐 `docs/test-cases/v4-14.md`、`docs/evidence/v4-14/*`、`docs/reports/v4-14-report.md`、`docs/roadmap.md`、`docs/roadmap_status.md`。

## 边界

1. 只检查 `stay_validation` 边界内的对外文案，不新增支付、收款、留资、问卷或候补链路。
2. 守门规则只使用本地匿名聚合结论与当前 writeback 文案，不写入任何复制内容、网页正文、URL、标题或联系方式。
3. 即使守门状态为 `aligned`，也只代表对外话术没有越界，不代表已经满足收费门槛。
