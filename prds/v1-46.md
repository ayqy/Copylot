# 问题修复和功能优化（V1-46 Pro 候补提示（低打扰）+ 漏斗取证补齐：补齐“曝光 -> 行动”的可量化商业化证据）

## 目标
基于 `docs/roadmap_status.md` 的收入优先 Top3：由于 Top2「真实 CWS 发布 + 商店端取证」仍被“网络可达性（需可用代理/VPN）”阻塞，本轮选择 Top1「Pro 候补提示（低打扰）+ 漏斗取证补齐」作为 1~3 天最小可交付增量主题。收入第一：在不新增权限、不引入支付/订阅、不采集用户复制内容的前提下，把 Pro 候补 CTA 从“仅有入口”升级为“可量化漏斗（曝光分母 -> 行动）”，并能通过现有本地导出/证据包形成可审计材料，为真实发布后的转化对比与 Pro MVP 决策提供证据。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、Popup 内新增“Pro 候补提示”（一次性、低打扰、可永久关闭）
  - 设计触发门槛（收入优先 + 精准性）：仅对高意图用户展示（示例：安装满 48h 且成功复制次数达到阈值，且已使用过 Prompt 或达到重度复制阈值），避免打扰新手与低活跃用户
  - UI 交互要求（不破坏复制主流程）：
    - 展示位置：Popup 内可见但不遮挡主操作（可折叠/可关闭）
    - 操作：`加入候补名单`（跳转现有候补入口/Pro Tab）、`稍后`（延迟再提示）、`不再提示`（永久不再提示）
  - 状态持久化：记录已展示/已选择的动作，保证“最多提示 1 次或按延迟策略再提示”，并可审计（本地可导出）

2、补齐“Pro 候补提示 -> 行动”的匿名事件与漏斗摘要（用于商业化证据）
  - 新增匿名事件（仅在“匿名使用数据”开关开启后写入本地 telemetry；默认关闭不记录）：
    - `pro_prompt_shown`：提示曝光（props：`source=popup`）
    - `pro_prompt_action`：用户动作（props：`source=popup`，`action=join|later|never`）
  - 更新 Pro 意向漏斗摘要/证据包口径：把 `pro_prompt_shown/pro_prompt_action` 纳入 Options -> 隐私页的导出摘要与证据包，形成可计算的漏斗转化率（例如：`entry_opened_per_prompt_shown`、`waitlist_opened_per_prompt_shown`）
  - 兼容与约束：延续事件白名单（仅允许枚举值 props）、不新增任何联网上报、导出文件可被 v1-45 的取证/复盘模板直接引用

3、补齐测试用例与证据落盘路径（可执行、可回归、可审计）
  - 新增测试用例文档（固定路径）：`docs/test-cases/v1-46.md`
    - 覆盖：触发条件、展示次数上限/延迟策略、“稍后/不再提示”行为、埋点开关 OFF 时不记录、开关 ON 时可导出事件与摘要
  - 新增证据目录（固定路径）：`docs/evidence/v1-46/` + `docs/evidence/v1-46/index.md`
    - 覆盖：Popup 提示截图索引、导出的「Pro 意向漏斗摘要/证据包」文件清单与命名规范（用于后续真实发布对比）
  - 新增交付简报（固定路径建议）：`docs/reports/v1-46-report.md`

  - 在开启“匿名使用数据”开关后，可通过 Options -> 隐私页导出包含 `pro_prompt_shown/pro_prompt_action` 的「Pro 意向漏斗摘要/证据包」，并能计算至少 2 个转化率口径（曝光 -> 进入 Pro、曝光 -> 打开候补）
