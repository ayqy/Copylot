
专项覆盖：

## 用例 A：Pro Tab 出现批量采集验证卡片

步骤：

1. 打开 `src/options/options.html#pro`。
2. 检查 `#pro-validation-bulk-collection`、`#pro-validation-bulk-open`。
3. 未填写 campaign 时检查 3 个复制按钮。
4. 填写 `twitter` 后再次检查按钮状态。

期望：

- 批量采集与整理验证卡片可见。
- 未填写 campaign 时，路线链接/验证简报/验证清单复制按钮为 disabled。
- 填写 campaign 后，3 个复制按钮恢复可用。

## 用例 B：批量采集验证素材带上独立 content

步骤：

1. 在 `Options -> Pro` 填写 `campaign=twitter`。
2. 点击“复制路线链接”“复制验证简报”“复制验证清单”。
3. 读取最近一次剪贴板文本。

期望：

- 路线链接包含 `utm_content=options_bulk_collection_cta`。
- 验证简报包含路线/商店/隐私 3 类固定链接。
- 验证清单包含 campaign 和路线链接。
- 素材不包含复制正文、网页正文、页面 URL 或标题。

## 用例 C：匿名导出可以区分批量采集路线

步骤：

1. 填写 `campaign=twitter`。
2. 点击“打开批量采集路线”。
3. 依次复制路线链接、验证简报、验证清单。
4. 导出本地 Pro intent / distribution 摘要。

期望：

- intent 导出中出现 `pro_waitlist_opened`，且 `content=options_bulk_collection_cta`。
- distribution 行为仍落在 `validation_route / validation_brief / validation_checklist` 动作中，但 `content` 可区分为 `options_bulk_collection_cta`。
- 浏览器级回归可同时覆盖高级页面清洗与批量采集两条路线。

## 用例 D：统一测试入口保持通过

步骤：

1. 执行 `bash scripts/test.sh`。
2. 复核 `docs/reports/v4-5-report.md` 与 `docs/roadmap_status.md`。

期望：

- `bash scripts/test.sh` 退出码为 `0`。
- `docs/roadmap.md` 中 S3 第 2 项被标记为完成。
- 简报、roadmap 与测试结果口径一致。
