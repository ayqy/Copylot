# V2 E2E 自动化与发布回归（测试用例）

## 目标
- 建立 `npm run e2e` 真实浏览器回归入口
- 覆盖 content / popup / options 三条关键用户路径
- 将 E2E 接入现有总门禁 `bash scripts/test.sh`

## 覆盖范围
- Content 页面：
  - 打开真实扩展注入内容脚本的网页 fixture
  - 选中文本并点击页面内容
  - 校验悬浮 Magic Copy 按钮可见
- Popup 页面：
  - 打开真实扩展 popup 页面
  - 校验 `#convert-button`、`#add-prompt-button`、`#popup-pro-waitlist-copy` 可见
- Options 页面：
  - 打开真实扩展 options 页面
  - 校验内置 Prompt 卡片存在：`data-id="builtin-summary-article"`
  - 新建 `E2E Prompt`
  - 校验新 Prompt 卡片渲染成功

## 执行命令
- 首次安装浏览器：
  - `npx playwright install chromium`
- 单独跑 E2E：
  - `npm run e2e`
- 不重复构建直接复测：
  - `COPYLOT_E2E_SKIP_BUILD=1 npm run e2e`
- 总门禁：
  - `bash scripts/test.sh`

## 备注
- 原生浏览器右键菜单 UI 不适合 Playwright 直接点击，本轮通过共享 `getActivePrompts()` 与右键菜单纯函数模型覆盖同一业务分支。
- DevTools 页面纳入 `scripts/ui-integration-tests.ts` smoke test，不走主 E2E。
