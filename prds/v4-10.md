# v4-10 领先路线稳定性摘要

## 背景

`v4-8` 已经能比较三条路线样本，`v4-9` 已经能把领先路线回写成对外文案，但当前仍缺少一个专门回答“这条领先路线是不是稳定领先”的中间层，容易把一次领先误判成长期需求。

## 本轮目标

1. 新增共享纯逻辑，把最近 `7d / 14d` 的领先路线、campaign 支撑和冲突情况整理成稳定性摘要。
2. 在 `Options -> Pro` 新增稳定性摘要入口，支持复制 Markdown 与下载 JSON。
3. 保持当前边界：只做路线稳定性复核，不做支付、候补、问卷或收款实现。

## 验收口径

1. `Options -> Pro` 可见“领先路线稳定性摘要”区块、复制按钮和下载按钮。
2. 摘要输出至少包含 `7d / 14d` 窗口 leader、`supporting_campaigns`、`conflicting_campaigns` 和 verdict。
3. `bash scripts/test.sh` 通过，覆盖 shared 逻辑、CLI 产物、Options 入口、UI 集成、Playwright 和 determinism 门禁。
