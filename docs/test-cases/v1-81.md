# V1-81 Pro 意向强度验证：真实样本导出取证 + 收款/订阅推进决策阈值固化（测试用例）

## 目标
验证子 PRD `prds/v1-81.md` 的验收点：基于 v1-80 的 7d 分布导出，用真实样本落盘证据包，并用离线脚本生成可审计的“意向决策摘要”，最终产出可用于判断是否推进 S4「收款/订阅链路」的商业化证据。

本轮验收对象（固定路径）：
- 阈值口径：`docs/monetization/pro-subscription-decision-thresholds.md`
- 样本导出（v1-80 入口）：Options -> 隐私页 ->「Pro 意向漏斗摘要」->「导出问卷意向分布（过去 7 天，JSON）」
- 决策摘要脚本：`scripts/build-pro-intent-decision-pack.ts`
- 用例文档（本文件）：`docs/test-cases/v1-81.md`
- 证据目录：`docs/evidence/v1-81/`
- 简报：`docs/reports/v1-81-report.md`
- 自动化回归入口：`bash scripts/test.sh`

## 用例 A：真实样本收集与 7d 分布导出（v1-80 入口）

步骤：
1. 在真实 Chrome 环境安装当前插件版本（以 `manifest.json` 为准）。
2. 打开 Options -> 隐私页，开启「匿名使用数据」开关。
3. 引导真实用户完成 Pro 意向问卷并点击「复制问卷（含环境信息）」；先确保流程可跑通（建议至少产生 `survey_intent >= 5`），样本量不足时结论应落在“继续收集”。
4. 展开「Pro 意向漏斗摘要」面板，点击「导出问卷意向分布（过去 7 天，JSON）」并下载文件。
5. 将导出文件移动到 `docs/evidence/v1-81/` 并记录 sha256（`shasum -a 256 docs/evidence/v1-81/*.json`）。

期望：
- 导出文件名符合：`copylot-pro-waitlist-survey-intent-distribution-7d-YYYY-MM-DD.json`。
- JSON 自带口径字段：`lookbackDays/windowFrom/windowTo/definitions`。
- JSON 不包含任何自由文本/联系方式原文/网页信息/复制内容（可用 `rg` 复核）。

---

## 用例 B：生成“意向决策摘要”（离线脚本）

步骤：
1. 准备 1 份用例 A 导出的分布 JSON（位于 `docs/evidence/v1-81/`）。
2. 执行脚本生成决策摘要（命令以 PRD 约定为准）：
   - `node scripts/build-pro-intent-decision-pack.ts --input docs/evidence/v1-81/<export>.json --out docs/evidence/v1-81/`
3. 打开输出的 Markdown/JSON 摘要文件，复核字段与结论。

期望：
- `docs/evidence/v1-81/` 生成：
  - 1 份 Markdown 摘要（包含样本量、意向占比、价格区间主峰、能力偏好 Top2、阈值判定结果）
  - 1 份 JSON 摘要（包含阈值版本与口径字段，便于审计复盘）
- 摘要结论必须为三段式之一：A（继续收集）/B（先迭代价值表达与能力包）/C（可推进收款/订阅链路最小实现）。
- 摘要不包含任何自由文本/联系方式原文/URL/标题/复制内容（可用 `rg` 复核）。
