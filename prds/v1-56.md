## 具体任务
1、新增“按 campaign 的本周复盘摘要（Markdown）”一键复制入口（收入证据资产化）
  - 入口位置：Options -> 隐私与可观测性 -> Pro 意向漏斗摘要面板（与 `#copy-pro-intent-weekly-digest` / `#export-pro-intent-by-campaign-7d-csv` 同区域）
  - 新增稳定 DOM（建议）：`#copy-pro-intent-by-campaign-weekly-report`
  - 输出格式：Markdown（可直接粘贴到 Notion/飞书/文档），至少包含：
    - 环境信息：`exportedAt`、`extensionVersion`、`lookbackDays=7`、`windowFrom/windowTo`
    - 按 campaign 的汇总表（每行一个 campaign，必须包含 `空 campaign` 行）：`pro_entry_opened` / `pro_waitlist_opened` / `pro_waitlist_copied` / `pro_waitlist_survey_copied` / `leads` / `leads_per_entry_opened`
    - 关键结论区（仅模板文字 + 数字）：例如 Top1/Top3 campaign（按 leads 排序）、空 campaign 占比提示（用于发现漏填与归因失真）
  - 隐私与开关行为（必须与既有口径一致）：
    - 匿名使用数据 OFF：不得读取/推断历史事件；摘要必须明确提示“匿名使用数据关闭（无可用事件）”，仅输出环境信息 + 隐私声明（不包含任何事件计数）
    - 匿名使用数据 ON：仅基于本地匿名事件日志 `copilot_telemetry_events`（白名单 + source 过滤 + FIFO window）派生统计；不得包含 URL/标题/网页内容/复制内容

2、证据落盘与复盘口径固化（可审计、可复核、可持续）
  - 证据目录（固定路径）：`docs/evidence/v1-56/` + `docs/evidence/v1-56/index.md`
  - 必须落盘（脱敏、可审计）：
    - `pro-intent-by-campaign-weekly-report.off.md`：匿名 OFF 的周报样例（不读取/不推断事件）
    - `pro-intent-by-campaign-weekly-report.on.md`：匿名 ON 的周报样例（7d window 按 campaign 汇总 + leads/转化率）
    - 截图索引（按钮入口/telemetry off/on 输出态），截图放在 `docs/evidence/v1-56/screenshots/`
  - 复盘口径必须与 v1-50/v1-51/v1-54 互证：
    - overall 互证：周报中所有 campaign 行按事件列求和，应与 v1-51 的 7d 明细 CSV（按 `eventName` 计数）一致
    - campaign 互证：周报中各 campaign 行应与 v1-54 的按 campaign 聚合 7d CSV 对应列一致（同一 window）

3、测试与回归（确保可发布、可执行）
  - 自动化测试加固（优先纯函数 + `scripts/unit-tests.ts`）：
    - Markdown 生成：字段齐全、表格列顺序稳定、数值计算正确（`leads = pro_waitlist_copied + pro_waitlist_survey_copied`；分母为 0 的比率输出 `N/A`）
    - 空 campaign 桶：当 `props.campaign` 缺失/空字符串时，必须归入 `空 campaign`
    - 开关行为：匿名 OFF 时输出不包含事件计数（可通过 snapshot/包含断言验证）
  - 新增测试用例文档（固定路径）：`docs/test-cases/v1-56.md`
    - 覆盖：匿名 OFF/ON；至少 2 个 campaign + 空 campaign 的数据样例；与 v1-51/v1-54 的三方互证步骤；并包含一次回归记录（日期 + PASS）
    - 必须要求 `bash scripts/test.sh` 全量通过
