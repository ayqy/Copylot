## 用例 C：匿名使用数据 ON（按 campaign 汇总表 + leads/效率指标）

前置：
- 匿名使用数据开关为 ON。
- 7d window 内至少覆盖：2 个非空 campaign + 1 个空 campaign。

步骤（示例数据构造，确保可复核）：
1. Options -> 隐私与可观测性：先点击“清空”本地匿名事件日志（Telemetry Events 面板）。
2. 开启“匿名使用数据”开关。
3. 构造至少 2 个 campaign：
   - Options -> Pro Tab：在 `#pro-intent-campaign` 填写 `twitter`，触发若干次 Pro 意向动作（打开/加入候补/复制等）。
   - 再将 `#pro-intent-campaign` 改为 `xhs`，重复触发若干次 Pro 意向动作。
4. 构造空 campaign：
   - 清空 `#pro-intent-campaign`（保持为空），再触发至少 1 次 `pro_entry_opened` 或留资动作。
5. 回到 Options -> 隐私与可观测性，点击 `#copy-pro-intent-by-campaign-weekly-report`，粘贴输出。

期望（Markdown 输出至少包含）：
- 环境信息：`exportedAt`、`extensionVersion`、`lookbackDays=7`、`windowFrom/windowTo`。
- 按 campaign 的汇总表（必须包含 `空 campaign` 行）：
  - `pro_entry_opened` / `pro_waitlist_opened` / `pro_waitlist_copied` / `pro_waitlist_survey_copied` / `leads` / `leads_per_entry_opened`
  - `leads = pro_waitlist_copied + pro_waitlist_survey_copied`
  - 分母为 0 的比率输出 `N/A`
- 关键结论区（模板文字 + 数字）：
  - Top1/Top3 campaign（按 leads 排序）
  - 空 campaign leads 占比提示

## 用例 D：三方互证（v1-51 明细 CSV / v1-54 按 campaign 聚合 CSV / 本次 Markdown 周报）

目标：保证“按 campaign 的计数/效率”可复核、可审计、可对齐（同一 window、同一白名单、同一 source 过滤）。

步骤：
1. 匿名开关保持 ON，确保 7d window 内有若干 Pro 意向事件，且覆盖：
   - 至少 2 个非空 campaign
   - 至少 1 次空 campaign
2. 导出 v1-51 7d 明细 CSV：
   - Options -> 隐私与可观测性 ->「Pro 意向漏斗摘要」面板，点击 `#export-pro-intent-events-7d-csv`。
3. 导出 v1-54 按 campaign 聚合 7d CSV：
   - 点击 `#export-pro-intent-by-campaign-7d-csv`。
4. 导出本次 v1-56 Markdown 周报：
   - 点击 `#copy-pro-intent-by-campaign-weekly-report` 并粘贴保存。
5. 复核（允许行排序差异，不允许计数差异）：
   - overall 互证：将 v1-54 CSV 中所有 campaign 行按事件列求和，应与 v1-51 明细 CSV（按 `eventName` 计数）一致。
   - campaign 互证：将 v1-56 Markdown 周报表格中的每个 campaign 行，与 v1-54 CSV 对应行的 4 个事件计数一致；并验证 `leads` 与 `leads_per_entry_opened` 可复算。

期望：
- 三方口径一致：同一 7d window、同一事件白名单与 source 过滤，得到的 campaign 行计数一致。
