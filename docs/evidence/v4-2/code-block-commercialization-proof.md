# v4-2 代码块商业化证明

## 直接证据

- 代码块悬停复制样例现在可直接输出：
  - `function buildPrompt() { ... }`
  - `const steps = [ ... ];`
- 整页转换样例会保留说明段落与 fenced code，可直接作为“网页内容 + Prompt -> 粘贴到 AI”的输入。
- 自动化回归覆盖了噪声清理、缩进保持、空行保持与 fenced code 输出，降低了第二次复制前的人工整理成本。

## 复用价值判断

- 返工步骤由“删首尾按钮文案 + 删行号 + 补缩进”收敛为 `0` 个显式人工步骤。
- 代码块场景天然高频，若第一次代码复制结果已经可直接粘贴，用户更容易继续触发第二次 `copy_success`。
- 代码块是最接近“复制后立即投入编辑器 / AI”的场景，因此比长文整理更适合作为第二次复用入口。

## 可验证的证明路径

1. 在 `e2e/fixtures/code.html` 对 `#code-block` 执行悬停复制。
2. 直接粘贴到编辑器或 AI，确认无需清理噪声即可使用。
3. 再对 `#table-code-block` 或整页转换执行第二次复制，验证同一会话内仍能得到可直接复用结果。

## 量化口径

- 样例通过率口径：本轮要求 2/2 代码块样例通过“无需返工”判定。
- 返工成本口径：输出中若仍出现行号、按钮噪声、缩进缺失任一项，则判定返工成本 > 0。
- 第二次复用判断口径：首次复制后无需整理即可继续发起第二次复制，记为更有利于第二次 `copy_success`。
