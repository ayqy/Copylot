# 跨 campaign 路线复核审计包

## 背景

`S3 Pro 路线验证` 已经通过 `v4-8 / v4-9 / v4-10 / v4-11 / v4-12` 串起路线比较、领先路线回写、稳定性摘要、统一 verdict 与收费评估审计包，但当前仍缺少一份把“逐个 campaign 是否支持同一领先路线”直接导出的复核包。既然 Chrome Web Store 已公开可安装、并行增长循环已经启动，本轮收入优先的最小增量不应再扩支付入口，而应先把跨 campaign 样本复核做成产品内能力，避免把 acquisition 偏差误判成可收费需求。

## 目标

实现一轮 1-3 天内可交付的收入优先增量：把 `advanced_cleaning` 的跨 campaign 样本复核能力接入现有 `Options -> Pro` 验证链路，支持直接复制和下载审计摘要，让并行增长循环具备可导出的 campaign 证据，并继续把对外结论约束在 `stay_validation`，为后续是否继续扩量或进入收费评估提供可执行、可测试、可复核的依据。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 结合项目代码的需求澄清

- `src/options/options.html` 与 `src/options/options.ts` 已经承载路线比较、领先路线回写、稳定性摘要、统一 verdict、收费评估审计包等导出面板；本轮应沿用同一 `Options -> Pro` 导出模式新增 cross-campaign review 区块，而不是新开一条入口。
- `src/shared/pro-route-validation-stability.ts` 已经输出 `campaigns`、`supportingCampaigns`、`conflictingCampaigns` 等跨 campaign 摘要，`src/shared/pro-route-validation-verdict.ts` 已经把 `stay_validation` 与收费前门槛收敛成统一 verdict；新增能力必须复用这两层现有口径，不得再发明一套并行 telemetry 统计。
- 当前工作树已经存在与目标直接相关的草稿文件 `src/shared/pro-route-validation-campaign-review.ts` 与 `scripts/build-pro-route-validation-campaign-review-pack.ts`；本轮应优先把它们收敛到可导出、可测试、可回归状态，并同步接入 `_locales/en/messages.json`、`_locales/zh/messages.json`、`scripts/unit-tests.ts`、`scripts/ui-integration-tests.ts` 与 `e2e/options-pro-flow.spec.ts`。
- 历史 `docs/test-cases/v4-3.md`、`docs/reports/v4-3-report.md` 已对应 Shift 追加模式，不能再复用为本轮验收路径；本轮测试、证据与简报应新增到 `docs/test-cases/v4-13.md`、`docs/evidence/v4-13/*`、`docs/reports/v4-13-report.md`，而 `prds/v4-3.md` 仅作为当前子 PRD 入口文件。

## 具体任务

1、收敛 `src/shared/pro-route-validation-campaign-review.ts` 与 `scripts/build-pro-route-validation-campaign-review-pack.ts`，基于现有 stability / verdict 摘要生成跨 campaign 路线复核审计包；至少显式输出每个 campaign 的 leader、`signalGap`、`reviewStatus`、`reviewAction`、`prioritizedCampaigns`、`conclusion`、`nextStep` 与来源证据，并确保全程只使用本地匿名聚合字段，不包含页面正文、复制内容、URL、标题或联系方式。
2、更新 `src/options/options.html`、`src/options/options.ts`、`_locales/en/messages.json`、`_locales/zh/messages.json`，在 `Options -> Pro` 新增“跨 campaign 路线复核审计包”区块，提供复制 Markdown 摘要与下载 JSON 的入口；文案必须明确当前仍处于“当前优先验证方向 / stay_validation”，服务于并行增长循环和样本复核，不新增支付、问卷、候补、留资或其他偏航承接。
3、补齐 `scripts/test.ts`、`scripts/unit-tests.ts`、`scripts/ui-integration-tests.ts`、`e2e/options-pro-flow.spec.ts` 与文档产物 `docs/test-cases/v4-13.md`、`docs/evidence/v4-13/*`、`docs/reports/v4-13-report.md`，让冲突 campaign、薄样本 campaign、支持性 campaign、无信号 campaign、复制下载、话术边界与商业化证据导出都有可执行回归，并把整体任务规模控制在 1-3 天内完成。
