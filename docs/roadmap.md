# Copylot 商业计划 Roadmap（验证日期：2026-07-03）

一句话结论：建议做“免费浏览器插件增长闭环优先”的路线，因为 Copylot 当前仍处于 Chrome Web Store 冷启动阶段，最缺的是安装后的首次成功、二次复用、分享与评分，而不是表单、问卷或候补名单。

高置信判断：
- Chrome 官方在 Discovery 文档中明确把商店发现质量与评分、安装/卸载、listing 质量、真实需求匹配、onboarding 清晰度绑定在一起；这意味着增长主线必须围绕激活与留存，而不是问卷样本。
- Copylot 在 2026-07-03 的 Chrome Web Store 公开信号只有 `51 users`、`No ratings`、版本 `1.2.2`、更新时间 `2026-06-27`，仍属于极早期插件冷启动阶段。
- 竞品的增长路径也证明“先形成稳定工作流，再做付费增强”更合理：Glasp 先靠高亮、同步、社区与 AI 工作流建立使用习惯，再以 `US$12.50/月` 承接 Pro；Just Read 先把阅读和保存体验做强，再承接付费增强。
- 当前阶段完全不做表单、问卷、候补收集，更符合浏览器插件的增长规律，也更符合用户的真实心理路径。

你的关注点：
- 现有 roadmap 把问卷、候补、调查样本放在了过高优先级。
- 最近新增的“你最想先解决什么”功能把 Popup 主路径从“快速复制”带偏到了“引导填表”。
- 官网和 `/pricing` 也没有围绕浏览器插件的增长获客循环来承接。

抽象后的研究问题：
- 浏览器插件在冷启动阶段，应该优先优化哪条链路，才能直接改善商店发现与自然增长？
- Copylot 当前是否已经具备做支付承接或强 Pro 导流的信任基础？
- 哪些功能最可能形成“安装后会回来继续用”的复用理由？
- Pro 何时才应该进入主链路，而不是过早挤占安装与激活资源？

目标用户的顾虑：
- 我装完以后，能不能在 30 秒内完成第一次干净复制？
- 我复制的内容会不会被上传、记录、滥用？
- 这个插件和原生复制相比，到底帮我减少了多少返工？
- 它值不值得我推荐给同事，而不是只用一次就忘？

## 核心结论

Copylot 现在必须进入的是浏览器插件增长循环：

`官网 / Chrome Web Store -> 安装 -> 首次成功复制 -> 第一次复用 -> 第二次回访 -> 分享 / 评价 -> 再回到安装`

Pro 只保留“路线说明”和“增强方向承接”，不做表单、不做问卷、不做候补、不做支付承接。

## 一级结论 1：当前增长瓶颈是激活与评分，不是需求收集

官方直接证据：
- Chrome 官方 Discovery 文档把评分、下载/卸载、listing 质量、真实需求匹配、onboarding 清晰度列为商店发现关键因素。
- Copylot 在 2026-07-03 的 CWS 公开信号只有 `51 users`、`No ratings`。

基于公开数据的判断：
- 当前任何把用户导向表单、问卷、候补的动作，都会稀释安装后的首次成功复制与二次复用。
- 插件自然增长的第一杠杆不是“更多意向文本”，而是“更多满意用户完成真实任务并愿意留评”。

仍存在不确定性：
- 目前没有公开的完整安装到激活后台数据，因此本判断主要基于商店公开信号、官网承接状态和本地事件模型。

## 一级结论 2：官网应该承担安装承接，而不是支付或留资

官方直接证据：
- 官网 `https://copy.useai.online/` 当前需要更明确地解释价值、上手路径与隐私边界。
- `/pricing` 之前仍是通用支付页模板，本轮已改为 Pro 路线页目标。

基于仓库事实的判断：
- 在没有稳定评分、没有成熟复用漏斗之前，过早导向支付或留资只会增加不信任感。
- 官网主任务应该是降低理解成本、降低信任门槛、给出明确的安装和上手路径。

## 一级结论 3：Pro 方向必须围绕“高频增量工作流”

官方直接证据：
- Glasp 把高亮、同步、AI 总结、导出和社区作为主路径，并在此基础上承接 Pro。
- Just Read 把阅读体验、保存和个性化增强作为长期留存抓手。

基于公开数据的判断：
- Copylot 的 Pro 只值得围绕三类“高频返工消除器”推进：
  - 更强页面清洗
  - 批量采集与整理
  - 面向下游工具的结构化导出
- 这些方向只有在用户已经完成多次成功复制后，才有资格进入强化验证。

## 90 天路线图

### Phase 0：立即修正（1 周内）
- 回退“你最想先解决什么”相关功能。
- 扩展内完全去掉表单、问卷、候补显性入口。
- 官网改为安装闭环站：首页强调价值、上手、隐私、安装；`/pricing` 改为 Pro 路线页；补齐 `/privacy`。

### Phase 1：安装与首次成功（第 1-3 周）
- 北极星：新安装用户首次 `copy_success` 转化率。
- 优先打磨：
  - Popup 首屏操作速度
  - 首次上手说明
  - 官网 / 商店截图与文案一致性
  - 隐私解释与本地处理信任感

### Phase 2：第二次复用与口碑（第 3-6 周）
- 北极星：`7 日内成功复制 >= 2 次` 的活跃用户数。
- 优先打磨：
  - Prompt 槽位与模板复用
  - 表格、代码块、长文清洗稳定性
  - Shift 追加模式
  - 合适时机的分享与评价提示

### Phase 3：Pro 路线验证（第 6-12 周）
- 只在以下前提成立后，才进入更强 Pro 设计：
  - 商店评分开始形成正向积累
  - 首次成功复制与二次复用稳定改善
  - 至少一类增强方向表现出高频真实需求
- 这一阶段仍然不做表单、问卷、候补收集；只做路线页说明与真实行为数据观察。

## 当前执行优先级

1. 安装承接：官网、商店、Popup 统一围绕“30 秒完成第一次干净复制”。
2. 复用抓手：Prompt、表格、代码块、追加模式。
3. 口碑循环：分享、评价、反馈都要低摩擦、晚触发、可追踪。
4. Pro 承接：只保留路线说明，不做留资和支付。

## 最终判断

不建议继续把 Copylot 当前版本的增长主线建立在表单、问卷、候补之上。现在的正确路径是把所有产品、官网、商店和分发动作统一回浏览器插件增长循环：`安装 -> 成功复制 -> 复用 -> 分享 / 评价 -> 再带来安装`。
