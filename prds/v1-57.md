# 问题修复和功能优化（V1-57 渠道分发工具包升级：商店安装链接/完整投放包一键复制 + 分发动作取证导出）

## 目标
基于 `docs/roadmap_status.md` 的收入优先 Top3：由于 Top1/Top2 仍受“需要可用代理/VPN + 商店可达”的网络可达性阻塞，本轮顺延选择 Top3「渠道分发/投放跑数与复盘固化 v3」作为 1~3 天最小可交付增量主题。

收入第一：在不引入支付/订阅、不新增权限、不联网发送数据、不采集用户复制内容的前提下，把渠道分发从“能复制候补链接/招募文案”升级为“可直接拿去获客 + 可审计取证 + 可量化复盘”的最小闭环——补齐商店安装链接（带 UTM + campaign）与完整投放包一键复制，并新增分发动作的匿名本地事件与按 campaign 聚合导出（CSV），用于对齐「分发动作 -> 留资 leads」效率证据。

## 具体任务
1、升级 Options -> Pro 的“渠道分发工具包”（让对外分发可直接用于获客）
  - 入口位置：`src/options/options.html` 的 `#pro-waitlist-distribution-toolkit`
  - 在既有按钮（`#pro-waitlist-url-copy` / `#pro-waitlist-recruit-copy`）基础上新增 2 个一键复制入口（建议稳定 DOM）：
    - `#pro-store-url-copy`：复制商店安装链接（必须带 UTM + campaign）
    - `#pro-distribution-pack-copy`：复制完整投放包（Markdown，必须包含“商店安装链接 + 候补链接 + 招募文案 + 可选问卷引导”）
  - 强制 campaign 规则（与 v1-55 一致，仅对“渠道分发工具包”生效）：
    - `campaign` 为空/非法：上述 4 个按钮全部置灰 + 显示 `#pro-waitlist-distribution-campaign-required` 提示
    - `campaign` 合法：允许复制；复制内容不得包含网页内容/复制内容/URL/标题（仅允许固定模板 + 扩展环境信息 + campaign + 商店/候补链接）
  - 商店安装链接口径（用于可转化获客增长）：
    - URL：Chrome Web Store detail URL（使用扩展 ID）
    - UTM：`utm_source=copylot-ext`、`utm_medium=distribution_toolkit`、`utm_campaign=<campaign>`
    - 要求：链接可解码复核（query 参数齐全、campaign 与输入一致）

2、分发动作取证：新增匿名本地事件 + 按 campaign 聚合 7d CSV 导出（让“分发动作 -> 留资 leads”可量化）
  - 新增匿名本地事件（仅匿名开关 ON 时记录；默认 OFF 不记录）：
    - 事件名：`pro_distribution_asset_copied`
    - 触发：仅在“写入剪贴板成功”后记录（避免虚假计数）
    - props（白名单 + 枚举校验，需写入 `docs/growth/telemetry-events.md`）：
      - `source: 'options'`
      - `campaign: <string>`（必填，来自分发工具包的强制 campaign）
      - `action: 'waitlist_url' | 'recruit_copy' | 'store_url' | 'distribution_pack'`
  - 新增导出入口（固定路径/稳定 DOM，确保可审计可复核）：
    - 入口位置：Options -> 隐私与可观测性 -> Pro 面板（与 v1-51/v1-54 同区域）
    - 稳定 DOM（建议）：`#export-pro-distribution-by-campaign-7d-csv`
    - 开关约束：匿名使用数据 OFF 时入口置灰且不读取/不推断本地 events；ON 时允许导出
    - 时间窗：过去 7 天（window 口径与 v1-50/v1-51/v1-54 保持一致，便于互证）
    - 聚合维度：`campaign`（缺失/空字符串归入 `空 campaign` 桶）
    - CSV 固定字段建议：
      - `exportedAt/extensionVersion/windowFrom/windowTo/lookbackDays/campaign/waitlistUrlCopied/recruitCopyCopied/storeUrlCopied/distributionPackCopied/distCopies`
    - 复盘用法（必须写入用例与证据索引）：与 v1-54 的 `leads` 按 campaign 对齐，可在表格中复算 `leads_per_dist_copy = leads / distCopies`（分母为 0 输出 N/A）

3、证据落盘 + 测试回归闭环（确保可发布、可执行）
  - 更新事件清单文档：`docs/growth/telemetry-events.md` 补齐 `pro_distribution_asset_copied` 的触发与 props 白名单/枚举/隐私说明
