# V1-19 本地漏斗摘要面板：3 分钟激活与漏斗进度自检 测试用例

## 目标
验证 V1-19 达到可发布：
- Options -> 隐私与可观测性 新增“本地漏斗摘要”面板（`#growth-funnel-panel`），可刷新/复制导出，且导出内容为可解析 JSON
- 漏斗摘要严格隐私合规：仅包含时间戳/计数/布尔（以及由其派生的毫秒耗时），不包含任何网页内容、URL、标题、复制内容
- “3 分钟激活”规则固定且可审计：
  - 当 `firstPopupOpenedAt` 与 `firstSuccessfulCopyAt` 均存在时，计算 `timeFromFirstPopupToFirstCopyMs` 并给出 `activatedWithin3MinutesFromFirstPopup`
  - 缺失任一时间戳时，上述字段均为 `undefined` 且不报错
- `bash scripts/test.sh` 全量通过

## 预置条件
- 已在 Chrome/Chromium 加载本地扩展（开发包或生产包均可）
- 能打开 Options 页面（`src/options/options.html`）
- 可在 Options -> 隐私页找到“本地漏斗摘要”面板：
  - 面板：`#growth-funnel-panel`
  - 刷新按钮：`#growth-funnel-refresh`
  - 复制按钮：`#growth-funnel-copy`
  - 展示区域：`#growth-funnel-view`
- 成功复制链路可用（右键菜单“Convert Page to AI-Friendly Format”可成功复制到剪贴板）

## 用例 A（面板可见：DOM 稳定且按钮可点击）
步骤：
1. 打开 Options -> 隐私与可观测性。
2. 找到并展开 `#growth-funnel-panel`。
3. 点击 `#growth-funnel-refresh`。
4. 点击 `#growth-funnel-copy`。

期望：
- `#growth-funnel-panel` 可见且可展开/折叠。
- `#growth-funnel-refresh` 与 `#growth-funnel-copy` 可点击，不影响隐私页其它面板。
- `#growth-funnel-view` 中内容为可解析 JSON（pretty 格式，末尾带换行）。

## 用例 B（真实链路激活：3 分钟内首次成功复制）
步骤：
1. 打开 Options -> 隐私页，点击 `#growth-stats-reset` 重置增长统计（确保漏斗从干净状态开始）。
2. 点击 `#growth-funnel-refresh`，记录当前摘要（此时通常 `isActivated=false`，且 `timeFromFirstPopupToFirstCopyMs/activatedWithin3MinutesFromFirstPopup` 为 `undefined`）。
3. 点击扩展图标打开 Popup，然后关闭 Popup（写入 `firstPopupOpenedAt`）。
4. 在 3 分钟内，在任意普通网页（非 `chrome://`）右键选择“Convert Page to AI-Friendly Format”完成一次成功复制。
5. 回到 Options -> 隐私页，点击 `#growth-funnel-refresh`，查看 `#growth-funnel-view`。

期望：
- `isPopupOpened = true`
- `isActivated = true`
- `timeFromFirstPopupToFirstCopyMs` 为 `number` 且 `<= 180000`
- `activatedWithin3MinutesFromFirstPopup = true`

## 用例 C（超时负例：人工调整 firstPopupOpenedAt 为 3 分钟之前）
说明：仅修改时间戳，不写入任何内容字段，用于验证阈值判断。

步骤：
1. 打开 `chrome://extensions`，找到该扩展的 Service Worker（或 Background）并点击“Inspect”打开控制台。
2. 在控制台执行（将 `firstPopupOpenedAt` 调整到 3 分钟之前；保留其它字段）：

```js
const key = 'copilot_growth_stats';
const existing = (await chrome.storage.local.get(key))[key] || {};
await chrome.storage.local.set({
  [key]: {
    ...existing,
    firstPopupOpenedAt: Date.now() - 3 * 60 * 1000 - 1
  }
});
```

3. 回到 Options -> 隐私页，点击 `#growth-funnel-refresh`，查看 `#growth-funnel-view`。

期望：
- 当 `firstSuccessfulCopyAt` 已存在时：`activatedWithin3MinutesFromFirstPopup = false`
- 当 `firstSuccessfulCopyAt` 不存在时：`activatedWithin3MinutesFromFirstPopup` 仍为 `undefined`（不应强行输出 false）

## 用例 D（隐私校验：摘要仅时间戳/计数/布尔/耗时）
步骤：
1. 在 Options -> 隐私页点击 `#growth-funnel-refresh`，查看 `#growth-funnel-view` 的 JSON 内容。
2. 检查所有字段：
   - 允许：时间戳（number）、计数（number）、布尔（boolean）、毫秒耗时（number）
   - 禁止：任何网页内容、URL、标题、复制内容、用户输入文本等（例如出现 `http`、长文本段落、HTML 片段）

期望：
- 摘要 JSON 不包含任何网页/复制内容、URL、标题或用户输入文本字段。
