# V1-42 转化入口跑数基线：zip 安装回归 + Pro 意向漏斗证据落盘 测试用例

## 目标
验证子 PRD `prds/v1-42.md` 的“可量化/可审计/可对比”商业化基线（不依赖外网）：

- 使用仓库内最新 `plugin-*.zip` 解压后以 unpacked 方式加载（禁止用 `src/` 开发态代替验收）
- Popup/Options 转化入口可用，且候补文案口径一致
- 隐私与可观测性：匿名使用数据开关默认关闭；关闭状态下不记录、不导出（不缓存、不补发）
- 开启匿名使用数据后触发 `pro_entry_opened/pro_waitlist_opened/pro_waitlist_copied`（含 `props.source=popup|options`），并从隐私页导出两份 JSON 落盘到 `docs/evidence/v1-42/`
- 自动化回归：`bash scripts/test.sh` 通过，并记录日期/结论

## 前置
1) 解压最新 zip 产物（本次基线版本）：
- `plugin-1.1.20.zip`（仓库根目录）

2) Chrome 加载 unpacked（强制验收方式）：
- 打开 `chrome://extensions`
- 开启 Developer mode
- 点击 `Load unpacked`
- 选择 zip 解压后的目录（目录内应包含 `manifest.json`、`src/popup/popup.html`、`src/options/options.html` 等）

3) 建议使用全新 Chrome Profile（避免历史匿名事件影响），或在隐私页先点击“清空”。

## 用例 A（默认关闭：不记录/不导出/不补发）
步骤：
1. 打开 Options -> 隐私与可观测性。
2. 确认“匿名使用数据”开关为关闭（默认关闭）。
3. 打开「本地匿名事件日志」面板：
   - 点击“刷新”，确认事件数为 0 且文本框为 `[]`。
4. 打开「Pro 意向漏斗摘要」面板：
   - 确认面板提示“未开启，不记录/不导出”可见。
   - 点击“刷新”，确认摘要 JSON 中 `enabled=false` 且各来源计数均为 0。
5. 在 Popup/Options 触发 Pro 入口与候补动作（升级 Pro / 加入候补 / 复制候补文案）。
6. 回到隐私页再次刷新「本地匿名事件日志」与「Pro 意向漏斗摘要」。
7. 点击「复制证据包」，检查导出的 JSON：
   - `settings.isAnonymousUsageDataEnabled=false`
   - `events=[]`

期望：
- 开关关闭时：不新增事件；事件日志为空；摘要与证据包导出为空/不可用（不缓存、不补发）。

## 用例 B（开启：最短路径触发 pro_* + 导出并落盘证据）
步骤：
1. Options -> 隐私与可观测性，开启“匿名使用数据”开关。
2. 触发 Popup 来源事件（source=popup）：
   - Popup 点击“升级 Pro/Upgrade Pro”（记录 `pro_entry_opened`）
   - Popup 点击“加入候补名单”（记录 `pro_waitlist_opened`）
   - Popup 点击“复制候补文案”成功（记录 `pro_waitlist_copied`）
3. 触发 Options 来源事件（source=options）：
   - Options 打开 Pro Tab（记录 `pro_entry_opened`）
   - 点击“加入候补名单”（记录 `pro_waitlist_opened`）
   - 点击“复制候补文案”成功（记录 `pro_waitlist_copied`）
4. 回到 Options -> 隐私与可观测性：
   - 刷新「本地匿名事件日志」，检查包含以上 `pro_*` 事件，且 `props` 仅包含白名单字段（`source`），`source` 枚举仅为 `popup|options`。
   - 刷新「Pro 意向漏斗摘要」，检查按 `popup|options` 分别统计计数与转化率。
5. 导出并落盘（固定路径，供审计/对比）：
   - 点击「复制摘要」并将 JSON 保存到：`docs/evidence/v1-42/pro-funnel-summary.json`
   - 点击「复制证据包」并将 JSON 保存到：`docs/evidence/v1-42/pro-funnel-evidence-pack.json`
6. 关键路径截图（固定路径，供商业化审计）：
   - 按“截图命名规范”保存截图到：`docs/evidence/v1-42/screenshots/`
   - 并确保 `docs/evidence/v1-42/index.md` 中截图索引与断言可复查

期望：
- 开关开启后：可记录并导出 `pro_*` 事件；摘要按来源统计正确，转化率派生正确；证据包字段白名单且无敏感信息。

## 用例 C（关闭后导出为空：不补发）
步骤：
1. 在已产生 `pro_*` 事件的前提下，关闭“匿名使用数据”开关。
2. 回到隐私页：
   - 刷新「本地匿名事件日志」应为空
   - 刷新「Pro 意向漏斗摘要」应为 `enabled=false` 且计数为 0
   - 点击「复制证据包」导出应满足：`events=[]`
3. 将关闭状态下的导出落盘：
   - `docs/evidence/v1-42/pro-funnel-summary-telemetry-off.json`
   - `docs/evidence/v1-42/pro-funnel-evidence-pack-telemetry-off.json`

期望：
- 关闭状态下：不缓存、不补发；导出为空；不会出现“关闭后补回旧事件”的情况。
