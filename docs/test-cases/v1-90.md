# V1-90 Pro 意向跑数取证执行闭环：证据包一键导出 + 离线落盘复盘材料（测试用例）

## 目标
验证子 PRD `prds/v1-90.md` 的验收点：Options 内可一键导出“Pro 意向跑数证据包（JSON）”，且该证据包可被离线脚本稳定落盘到 `docs/evidence/v1-90/` 并复算 sha256；同时满足隐私门禁（匿名开关 OFF 时不得包含任何事件明细/CSV），用于后续提升 `survey_intent` 样本量与订阅 MVP go/no-go 决策复盘。

本轮验收对象（固定路径）：
- 子 PRD：`prds/v1-90.md`
- Options 页面：`src/options/options.html` / `src/options/options.ts`
- Pro 意向漏斗摘要：Options -> 隐私与可观测性 ->「Pro 意向漏斗摘要」
- 证据包导出按钮：Options ->「下载 Pro 意向跑数证据包（JSON）」
- 离线落盘脚本：`scripts/build-pro-intent-run-evidence-pack.ts`
- 自动化回归入口：`bash scripts/test.sh`
- 用例文档（本文件）：`docs/test-cases/v1-90.md`
- 证据目录（固定结构）：`docs/evidence/v1-90/`

## 用例 A：匿名开关 ON 时可导出包含关键证据的单文件

前置：
- 在 Options -> 隐私与可观测性，开启「匿名使用数据」开关（ON）。
- 至少产生 1 条 `pro_waitlist_survey_copied`（建议从 Popup 入口进入问卷并复制一次）。

步骤：
1. 打开 Options -> 隐私与可观测性 ->「Pro 意向漏斗摘要」，点击刷新（如有）。
2. 点击「下载 Pro 意向跑数证据包（JSON）」并保存文件（记录文件名）。
3. 打开导出的 JSON 文件，检查关键字段。

期望：
- JSON 内包含 `env.exportedAt`、`env.extensionVersion`、`env.isAnonymousUsageDataEnabled=true`。
- JSON 内包含：
  - `proFunnelSummary`（含 `bySource.popup|options` 与 overall 统计）
  - `proWaitlistSurveyIntentDistribution`（含 `survey_intent` 等关键字段）
  - `proIntentEvents7dCsv`（非空字符串，且可离线打开）
- 导出文件不包含：网页 URL/标题/网页内容/复制内容/联系方式明文/问卷自由文本明文。

---

## 用例 B：匿名开关 OFF 时证据包必须被门禁（不包含明细/CSV）

前置：
- 在 Options -> 隐私与可观测性，关闭「匿名使用数据」开关（OFF）。

步骤：
1. 点击「下载 Pro 意向跑数证据包（JSON）」并保存文件（记录文件名）。
2. 打开导出的 JSON 文件，检查关键字段。

期望：
- JSON 内 `enabled=false`，且存在 `disabledReason=anonymous_usage_data_disabled`。
- JSON 内不得包含任何事件明细/导出 CSV 内容（例如 `proIntentEvents7dCsv` 为空串或被省略；以 PRD 为准）。
- 导出行为必须给出明确提示（toast/notice，文案以实现为准）。

---

## 用例 C：离线落盘与 sha256 复算（证据可复盘）

前置：
- 已完成用例 A，拿到证据包文件（JSON）。

步骤：
1. 执行离线落盘脚本（示例命令，参数以实现为准）：
   - `ts-node scripts/build-pro-intent-run-evidence-pack.ts --input /path/to/pro-intent-run-evidence-pack.json`
2. 检查 `docs/evidence/v1-90/` 是否生成/更新：
   - `index.md`
   - `pro-intent-run-evidence-pack.json`
   - `sha256.json`
3. 复算 sha256 并与 `sha256.json` 或 `index.md` 中记录一致：
   - `shasum -a 256 docs/evidence/v1-90/*`

期望：
- `docs/evidence/v1-90/index.md` 存在，且包含：口径说明、文件清单、sha256（可复算）、“无 PII”断言结论。
- `pro-intent-run-evidence-pack.json` 与导出文件内容等价（允许文件名规范化，但内容必须可复盘）。
