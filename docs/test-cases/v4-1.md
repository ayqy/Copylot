# v4-1 快捷 Prompt 槽位第二次复用路径测试用例

## 范围

- `src/popup/popup.html`
- `src/popup/popup.ts`
- `src/popup/popup.css`
- `src/options/options.ts`
- `src/shared/growth-stats.ts`
- `src/shared/telemetry.ts`
- `_locales/zh/messages.json`
- `_locales/en/messages.json`
- `docs/evidence/v4-1/second-open-reuse-audit.md`
- `docs/evidence/v4-1/reuse-funnel-sample.json`
- `docs/reports/v4-1-report.md`
- 相关 `e2e/*.spec.ts`

## 自动化入口

```bash
bash scripts/test.sh
```

通过标准：

- 统一测试入口退出码为 `0`
- 首次 `copy_success` 之后，Popup 或 Onboarding 会把下一步主路径切换为快捷 Prompt 槽位复用
- 至少 1 个快捷 Prompt 槽位能直接触发一次真实复制，并形成第二次 `copy_success`
- 复用证据可导出、可复核，且不包含复制内容或网页原文

## 用例 A：首次成功前后主路径切换正确

步骤：

1. 清空本地 `copilot_growth_stats` 与相关匿名事件。
2. 打开 Popup，记录首次成功前的 Hero、主按钮、快捷入口与 Onboarding 文案。
3. 在任意支持页面完成 1 次真实复制，再次打开 Popup 或 Onboarding。

期望：

- 首次成功前的主路径仍围绕“第一次干净复制”。
- 首次成功后，主路径切换为“再次复用”，且明确指向快捷 Prompt 槽位或等价入口。
- 文案明确表达“复制网页内容 + 套用 Prompt + 粘贴到 AI”的第二次打开理由。

## 用例 B：快捷 Prompt 槽位可以直接完成第二次复制

步骤：

1. 保持至少存在 1 个可用 Prompt，并完成第 1 次成功复制。
2. 打开 Popup，点击第 1 个展示中的快捷 Prompt 槽位。
3. 在目标 AI 聊天框、文档或纯文本编辑器中粘贴结果。

期望：

- 快捷 Prompt 槽位可见、可点击，且标题、快捷键信息与 Prompt 绑定关系正确。
- 点击后能够生成“Prompt + 页面内容”的复制结果，而不是只打开设置页。
- 第 2 次复制成功后，本地增长状态与复用摘要被正确更新。

## 用例 C：Onboarding 与 Popup 的复用途径保持一致

步骤：

1. 在首次成功后分别打开 Popup 首屏和 Onboarding。
2. 对比两处的主标题、副标题、主行动与辅助说明。
3. 用同一套 Prompt 槽位分别从 Popup 与 Onboarding 触发一次操作。

期望：

- Popup 与 Onboarding 都围绕同一条“第二次打开 -> Prompt 复用 -> 粘贴验证”路径。
- 两处不会出现互相冲突的主行动，也不会退回表单、候补或支付承接。
- 任一入口都能完成相同的复用动作，并记录为同一套可审计漏斗。

## 用例 D：复用漏斗证据可导出且可复核

步骤：

1. 完成 1 次首次成功复制与 1 次快捷 Prompt 复用复制。
2. 导出 `docs/evidence/v4-1/second-open-reuse-audit.md` 与 `docs/evidence/v4-1/reuse-funnel-sample.json`。
3. 复核字段来源、关键时间顺序与事件白名单。

期望：

- 证据文档明确说明 `firstSuccessfulCopyAt`、Prompt 槽位点击或使用、第二次 `copy_success` 的字段来源与判定口径。
- 样例中的关键顺序至少满足“首次成功 <= Prompt 复用 <= 第二次成功复制”。
- 导出内容不包含网页原文、复制文本、Prompt 原文以外的敏感数据。

## 用例 E：低打扰与商业化证据同时成立

步骤：

1. 完成首次成功后复用路径全流程。
2. 检查 Popup、Onboarding、Options 与证据文档。
3. 确认分享、评价、Pro、表单、问卷、支付链路未被提前插入当前主路径。

期望：

- 当前最短路径只服务第二次复用，不额外插入高打扰弹窗或偏航入口。
- 可导出的商业化证据至少覆盖快捷 Prompt 槽位曝光、点击、成功复制或复用转化摘要中的一种。
- 证据足以判断本轮增量是否推动了真实复用，而不是只增加了文案或静态入口。
