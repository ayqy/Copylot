# 问题修复和功能优化（V1-38 Pro 转化漏斗取证包：本地摘要 + 一键导出 + 用例落盘）

## 目标
以“真实收入回报路径可证明”为优先的最小可交付增量（1-3 天）：在**不新增任何扩展权限**、**不做任何联网发送**、**不收集用户复制内容/页面内容**的前提下，把现有 Pro 候补转化入口（Popup/Options）升级为“可量化、可审计、可复盘”的证据链——在 Options 的「隐私与可观测性」中提供 **Pro 意向漏斗本地摘要**与**一键导出证据包**，使得后续一旦完成 CWS 真实发布，即可快速从商店安装回归并拿到可验证的商业化推进证据，达到验收标准。

## 具体任务
1、在 Options -> 隐私与可观测性中新增「Pro 意向漏斗摘要」面板（仅本地、可审计）
  - 数据来源：仅使用本地匿名事件日志（`copilot_telemetry_events`，已存在），不得新增任何采集维度与任何联网发送。
  - 摘要口径（最小但可复盘）：
    - 按 `props.source`（`popup|options`）分别统计：`pro_entry_opened`、`pro_waitlist_opened`、`pro_waitlist_copied` 的次数与最近一次时间戳
    - 派生转化率（仅展示/导出，不落盘）：`waitlist_opened / entry_opened`、`waitlist_copied / waitlist_opened`
    - 明确口径边界：事件日志最多 100 条、FIFO；统计仅基于当前日志窗口（避免误解为全量历史）
  - 交互要求：
    - 支持刷新与复制（复制内容为 JSON，便于贴入简报/Issue/审计记录）
    - 当匿名使用数据开关关闭时：面板应明确显示“未开启，不记录/不导出”（避免误判）

2、一键导出「商业化证据包（Pro Funnel Evidence Pack）」并固化可审计格式
  - 导出内容（JSON 文本，严禁包含用户内容/URL/标题）：
    - `meta`：导出时间（ms）、扩展版本号（`manifest.json`）、导出来源（`options`）
    - `settings`：仅包含与转化相关且无敏感信息的设置快照（例如：匿名使用数据开关状态；其余字段必须白名单化）
    - `proFunnel`：任务 1 的摘要对象（含分来源计数/转化率/最近时间戳）
    - `events`：仅筛选 `pro_*` 事件（必要时可额外包含 `popup_opened` 作为上下文），并再次经过 `sanitizeTelemetryEvents`（保证 props 白名单与枚举收敛）
  - 交互要求：
    - 提供“复制证据包”按钮（写入剪贴板成功后给出明确反馈）；可复用现有剪贴板工具链
    - 证据包应可直接粘贴到 `docs/reports/v1-38-report.md` 与后续“从商店安装回归”审计记录中（证明商业化路径推进）

3、补齐可执行测试用例与自动化回归断言（保证证据链可复现）
  - 新增并填写手工用例文档（路径固定）：`docs/test-cases/v1-38.md`
    - 必须覆盖：匿名开关开/关行为、Pro Tab 入口点击、候补打开/复制、摘要统计正确性、证据包导出内容自检（不含敏感信息）、以及一次 `bash scripts/test.sh` 回归记录（日期 + PASS）
  - 补齐/更新自动化断言（按仓库既有模式）：
    - 至少覆盖：证据包导出 JSON 的字段白名单、`pro_*` 事件的 `props.source` 枚举校验、匿名开关关闭时不产出事件/导出为空的行为
  - 输出交付简报（用于口头汇报，路径固定）：`docs/reports/v1-38-report.md`（记录状态/效果/修改范围/测试结论/示例证据片段-脱敏）
