# 问题修复和功能优化（V1-91 Pro 意向样本量提升：Popup 问卷引导 + 可导出漏斗证据）

## 目标
在不引入支付/订阅、不新增权限、不上传用户复制内容的前提下，把“Pro 候补问卷”从「有入口但转化弱」升级为「Popup 低打扰引导 -> 1 分钟问卷 -> 可观测/可导出证据」的最小闭环，优先提升 `survey_intent = count(pro_waitlist_survey_copied)` 样本量，为 S4「收款/订阅链路打通」提供更可信的决策输入。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、Popup “Pro 候补提示”改为“问卷优先引导”（提升 `survey_intent` 分母）
  - 修改：`src/popup/popup.html`、`src/popup/popup.ts`、`src/shared/growth-stats.ts`、`_locales/*/messages.json`
  - 行为要求（低打扰、可关闭）：
    - 仍复用现有触发门槛（`shouldShowProPrompt`），不新增强打断弹窗
    - 提示文案改为“1 分钟问卷”价值解释（帮助决定是否做订阅/做哪些 Pro 能力）
    - 主按钮点击后：打开 Options 并定位到问卷区（必须带来源归因）：
      - `src/options/options.html?pro_survey_source=popup#pro-waitlist-survey`
    - 「稍后/不再提示」逻辑保持不变（仍可 snooze/永久关闭）
  - 口径要求（可复盘）：
    - Popup 出现提示 -> 记录 `pro_prompt_shown`
    - 点击主按钮 -> 记录 `pro_prompt_action`（复用既有 action 枚举，不新增隐私字段）

2、问卷区“计入样本”提示与漏斗口径补齐（让商业化证据可验证）
  - 修改：`src/options/options.html`、`src/options/options.ts`、`src/shared/pro-funnel.ts`、`_locales/*/messages.json`
  - 问卷区提示（不强迫，信息透明）：
    - 当「匿名使用数据」开关为 OFF 时，在 `#pro-waitlist-survey` 显示一条提示：
      - “未开启匿名使用数据：本次填写/复制不会计入 `survey_intent` 统计（仍可继续填写）”
    - 当开关为 ON 时隐藏该提示（避免噪音）
  - 漏斗口径补齐（可量化）：
    - 在 `ProFunnelSummary.rates` 新增 1 个稳定字段：
      - `survey_copied_per_prompt_shown = pro_waitlist_survey_copied / pro_prompt_shown`
    - Options -> 隐私与可观测性 ->「Pro 意向漏斗摘要」与其证据包导出中必须可观察该字段（避免“做了引导但无法量化”）

3、测试/用例/门禁（必须可执行）
  - 自动化门禁：
    - 更新/新增单测：`scripts/unit-tests.ts`
      - 断言 `survey_copied_per_prompt_shown` 字段存在、类型稳定、且在分母为 0 时为 `null`
    - `bash scripts/test.sh` 全量 PASS
  - 新增测试用例文档（固定路径）：`docs/test-cases/v1-91.md`

      - Popup 点击主按钮 -> Options 自动定位到 `#pro-waitlist-survey`（来源归因 `popup`）
      - 匿名开关 OFF/ON 两种情况下的提示文案与统计口径差异
      - 产生至少 1 条 `pro_waitlist_survey_copied` 后导出 v1-90 证据包，并验证 `survey_intent >= 1`
    - 必须包含一次回归记录（PASS/BLOCKED + 导出文件名/证据文件名/截图索引）
