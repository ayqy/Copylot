## 用例 A：Pro Tab 出现 tracker 区块

步骤：

1. 打开 `src/options/options.html#pro`。
2. 检查 `#pro-human-approval-window-tracker-panel`、复制按钮和下载 JSON 按钮。

期望：

- tracker 区块可见。
- 文案说明需要三项检查在同一窗口内一起通过。
- 当前状态默认显示 `hold_validation`。

## 用例 B：Markdown 摘要能列出三项检查

步骤：

1. 在 `Options -> Pro` 点击复制 tracker 摘要。
2. 检查剪贴板内容。

期望：

- 文本包含 `tracker_status=`、`ready_for_human_approval=`、`audit_status=`、`guard_status=`。
- `payment_audit_ready`、`campaign_review_clear`、`messaging_guard_aligned` 三项检查都存在。
- 未通过项会进入 `blockers` 列表。

## 用例 C：JSON 产物正确保持 hold_validation

步骤：

1. 下载 `v4-15` JSON。
2. 读取关键字段。

期望：

- 文件名为 `copylot-pro-human-approval-window-tracker-v4-15.json`。
- 文件内包含 `checks`、`blockers`、`prioritizedCampaigns`、`nextStep`、`sources`。
- 本轮样例 `readyForHumanApproval=false`，`trackerStatus=hold_validation`。

## 用例 D：三项检查同窗逻辑回归

步骤：

1. 运行 `scripts/unit-tests.ts` 与 `e2e/options-pro-flow.spec.ts`。
2. 检查三项检查同时通过与部分失败的断言。

期望：

- 三项检查都通过时才会进入 `enter_human_approval_review`。
- 任意一项未通过时继续 `hold_validation`。
- 当前样例因 `payment_audit_ready=false`、`campaign_review_clear=false` 而被拦截。

## 用例 E：证据与路线图收口

步骤：

1. 复核 `docs/evidence/v4-15/*`、`docs/roadmap.md`、`docs/roadmap_status.md`、`docs/reports/v4-15-report.md`。

期望：

- 证据索引说明 tracker 如何把三项检查固化到同一窗口。
- roadmap 把人类批准门槛 tracker 标记为完成，但不把收费实现规划标记完成。
- 报告明确说明当前仍未进入人类批准。
