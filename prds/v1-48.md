## 具体任务
1、Options -> Pro Tab 增加“付费意向问卷（可选）”与一键留资入口（降摩擦、可审计）
  - 新增一个“可选问卷”区块（仅用户主动填写，不弹窗、不打扰主流程），建议放在 Pro（规划中）列表与行动按钮附近，降低“看完不知道下一步做什么”的流失。
  - 问卷字段（最小集，全部可为空）：
    - 使用场景（短文本）
    - 期望的 Pro 能力（勾选或短文本，沿用 `docs/monetization/pro-scope.md` 的候选能力口径）
    - 付费意向（结构化选项，便于后续统计与决策）：是否愿意付费（Yes/Maybe/No）+ 可接受价位区间（按月/按年）
    - 联系方式（可选）：仅提示“用户自愿填写”，不做任何自动采集/自动填充
  - 行动按钮（至少一个主按钮）：
    - `复制问卷（含环境信息）`：把“环境信息 + 问卷内容”组装成 Markdown 文本写入剪贴板（用于粘贴到 GitHub Issue/邮件/任何渠道）
    - 可选增强：`复制并打开候补`（先复制再打开现有候补入口），用于减少步骤；若不做该按钮，则保持现有“加入候补名单”按钮即可
  - 文案与隐私要求：
    - 明确提示：不包含网页内容/复制内容/URL/标题；所有内容均来自“扩展环境信息 + 用户手动填写”
    - 与既有候补模板的隐私提示一致（参考 `proWaitlistIssueBodyTemplate`）

2、漏斗取证增强：把“问卷留资”纳入本地匿名事件与 Pro 证据包（可量化商业化证据）
  - 新增匿名事件（仅在“匿名使用数据”开关开启后写入本地 telemetry；默认关闭不记录）：
    - `pro_waitlist_survey_copied`：用户复制问卷（props：`source=options`）
    -（打开候补页继续沿用既有事件）`pro_waitlist_opened` / `pro_waitlist_copied`
  - 更新 Pro 意向漏斗摘要/证据包口径（Options -> 隐私页）：
    - 证据包需包含新事件 `pro_waitlist_survey_copied`（events 白名单与 props 枚举过滤保持一致）
    - 摘要至少能展示该事件的 count/lastTs（可不新增复杂转化率，但必须可从导出结果中验证“问卷留资动作发生过”）
  - 文档同步：
    - 更新埋点清单：`docs/growth/telemetry-events.md` 增补新事件与字段白名单

3、补齐测试/用例与证据落盘（确保可执行、可回归、可用于商业化取证）
  - 新增测试用例文档（固定路径）：`docs/test-cases/v1-48.md`

  - 自动化测试加固（优先纯函数/可离线跑的单测）：覆盖新事件白名单过滤、Pro 漏斗摘要/证据包字段稳定性；并保证 `bash scripts/test.sh` 全量通过
