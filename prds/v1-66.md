# 问题修复和功能优化（V1-66 CWS Listing 物料证据包一键生成：上架=获客入口的口径可审计/可复核）

## 目标
基于 `docs/roadmap_status.md` 的收入优先 Top3，本轮选择 Top1「CWS Listing 物料证据包一键生成 + 门禁（离线可推进）」作为 1~3 天最小可交付增量主题。

收入第一：把“上架=获客入口”的关键资产（Chrome Web Store 中英文长描述、关键词矩阵、截图顺序/拍摄脚本、更新日志模板）从“散落文档/容易跑偏”升级为**可一键生成、可审计、可复核、可复用的结构化证据包**，让后续真实发布/ASO 迭代具备可追溯基线；同时避免“发了但物料不一致/夸大/缺 CTA 导致转化断档”的商业化风险。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、新增「CWS Listing 物料证据包」一键生成脚本（把上架资产变成结构化、可归档的证据）
  - 新增脚本（建议固定路径）：`scripts/build-cws-listing-evidence-pack.ts`
  - 输入（必须来自仓库可审计文件，禁止手填/复制粘贴到代码）：
    - `docs/ChromeWebStore-Description-EN.md`
    - `docs/ChromeWebStore-Description-ZH.md`
    - `docs/aso/keywords.md`
    - `docs/aso/store-assets.md`
    - `docs/aso/cws-release-notes-template.md`
    - （可选）`docs/aso/value-prop.md`（用于一致性校验）
  - 输出（落盘目录固定，便于审计与归档）：`docs/evidence/v1-66/`
    - 证据包文件（JSON，命名稳定，便于按时间归档）：`cws-listing-evidence-pack-<version>-YYYY-MM-DD-HHMMSS.json`
    - `docs/evidence/v1-66/index.md`（索引：生成时间/版本/输入文件哈希清单/关键断言/使用说明）
  - 证据包最小字段（字段命名/顺序稳定，便于长期 diff）：
    - `packVersion: "v1-66"`
    - `exportedAt`（ISO 时间）
    - `extensionVersion`（来自 `manifest.json`，并与 `dist/manifest.json` 一致性校验）
    - `inputs`（每个输入文件：`path, bytes, sha256`）
    - `listing`（结构化摘要，用于“可验证的商业化证据”）：
      - `descriptions.en/zh`（建议存放原文或 sha256 + 关键片段索引；不得引入任何 secret）
      - `keywords`（按 EN/ZH 分组的数组）
      - `screenshotPlan`（从 `docs/aso/store-assets.md` 提取 1~7 张截图标题与断言要点）
      - `releaseNotesTemplateSha256`
    - `assertions`（用于自动验收，至少包含）：
      - `hasProWaitlistCta`（长描述是否包含 Pro waitlist 加入方式与 `docs/monetization/pro-scope.md` 稳定链接）
      - `hasTutorialLinks`（长描述是否包含 tutorials 链接）
      - `hasPrivacyClaims`（长描述是否包含“本地处理/不上传复制内容/匿名开关默认 OFF”的口径）
      - `noOverclaimKeywords`（关键词/描述不得出现“已上线 Pro/订阅/付费”等误导性词汇，词表需可维护）

2、补齐自动化测试 + 门禁（确保“物料证据包”长期稳定且可重复生成）

    - 输出文件命名规则（version/时间戳后缀/目录固定）
    - Schema 稳定（必含字段齐全；`packVersion` 固定；字段顺序稳定）
    - 安全红线：证据包不得包含任何凭据/Token/环境变量明文
    - 关键断言：`assertions.*` 在当前仓库素材下为 PASS（否则必须给出可执行修复提示）
    - 可重复性：同一份输入重跑输出 deterministic（除 `exportedAt` 外其余字段稳定；或提供可选 `--stable-exported-at` 用于门禁）
  - 将脚本接入 `bash scripts/test.sh` 门禁：
    - 生成/校验 `docs/evidence/v1-66/` 的样例证据包（或对 fixtures 目录生成），并断言可重复生成

3、用例/证据/汇报材料闭环（让“上架物料资产化”可执行、可审计、可复盘）

    - 必须包含一次回归记录（日期 + PASS）与执行命令（可复制）
  - 落盘证据目录与索引（固定路径）：`docs/evidence/v1-66/`
