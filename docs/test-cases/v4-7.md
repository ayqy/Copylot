## 用例 A：Pro Tab 出现收费前门槛判断区块

步骤：

1. 打开 `src/options/options.html#pro`。
2. 检查 `#pro-decision-gate-panel`、`#copy-pro-intent-decision-summary`、`#download-pro-intent-decision-summary-json`。

期望：

- 新区块可见。
- 两个动作按钮存在且默认可点击。
- 隐私提示明确说明不包含页面正文、复制内容、URL、标题或联系方式。

## 用例 B：复制出的摘要符合 A/B/C 口径

步骤：

1. 在 `Options -> Pro` 点击“复制收费前门槛摘要”。
2. 读取最近一次剪贴板文本。

期望：

- 文本包含 `V1-81 Pro 意向决策摘要（可审计）`。
- 文本包含 `survey_intent`、`price_monthly_peak`、`price_annual_peak`、`capability_top2`。
- 文本包含 `code：\`A\``、`code：\`B\`` 或 `code：\`C\`` 之一。
- 文本不包含页面正文、复制内容、URL、标题或联系方式。

## 用例 C：下载出的 JSON 可复核

步骤：

1. 点击“下载收费前门槛 JSON”。
2. 打开生成文件并检查关键字段。

期望：

- 文件名符合 `copylot-pro-intent-decision-summary-v1-81.json`。
- 文件内包含 `thresholds`、`metrics`、`decision` 三大区块。
- 若匿名数据关闭，结果仍可导出，但必须落在 `A` 且原因包含 `anonymous_usage_data_disabled`。

## 用例 D：状态文档与当前边界一致

步骤：

1. 复核 `docs/roadmap.md`、`docs/roadmap_status.md`、`docs/reports/v4-7-report.md`。

期望：

- Roadmap 明确最后一项已进入“门槛摘要导出已完成，后续只剩路线样本比较与回写”的状态。
- 状态文档明确当前仍未进入支付/收款实现，只是把收费前判断变成了可导出的产品能力。
