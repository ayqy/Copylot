# V1-5 可观测性 MVP（匿名事件本地打点）测试用例

## 目标
验证“匿名使用数据”开关与本地匿名事件日志真实打通：
- 默认关闭：不记录任何本地匿名事件
- 开启后：关键增长动作写入 `chrome.storage.local`（`copilot_telemetry_events`）
- 关闭后：立刻停止追加并清空 `copilot_telemetry_events`
- 事件结构/props 白名单/隐私约束满足要求
- 自动化测试入口 `bash scripts/test.sh` 全量通过，可发布

## 测试前置
1. 确保扩展已加载（建议加载 `dist/` 打包产物或开发构建产物）。
2. 打开 Options 页，找到“匿名使用数据”开关（默认应为关闭）。
3. 打开 Options 页 DevTools Console，用于执行下列校验脚本。

校验脚本（读取日志）：
```js
await chrome.storage.local.get('copilot_telemetry_events')
```

清理脚本（清空日志）：
```js
await chrome.storage.local.remove('copilot_telemetry_events')
```

## 用例 1：默认关闭不记录
步骤：
1. 确认 Options 页“匿名使用数据”开关为关闭。
2. 执行清理脚本，确保日志为空。
3. 执行以下动作（任选其一或多项）：
   - 打开 Popup（触发 `popup_opened`）
   - 页面内执行一次复制成功（触发 `copy_success`）
   - 触发新手引导（自动弹出或手动重开），并结束（触发 `onboarding_*`）
   - 满足条件时展示评价引导并点击按钮（触发 `rating_prompt_*`）
   - Popup 内点击反馈/分享/去评价、复制分享文案（触发 `wom_*`）
4. 执行读取日志脚本。

期望：
- `copilot_telemetry_events` 不存在或为空数组。

## 用例 2：开启后记录事件 + 字段白名单
步骤：
1. 打开 Options 页，将“匿名使用数据”开关打开。
2. 执行清理脚本，确保从空开始。
3. 依次执行以下动作（尽量覆盖多事件）：
   - 打开 Popup（`popup_opened`）
   - 手动打开新手引导并“跳过/完成”（`onboarding_shown`/`onboarding_completed`）
   - 点击反馈/分享/去评价（`wom_feedback_opened`/`wom_share_opened`/`wom_rate_opened`）
   - 点击“复制分享文案”，确保提示“已复制”（`wom_share_copied`）
   - 页面内复制成功一次（`copy_success`）
   - 用 Prompt 复制成功一次（应出现 `copy_success` + `prompt_used`）
4. 执行读取日志脚本，检查数组内事件对象结构。

期望：
- 日志存在且新增事件；
- 每条事件仅包含：`name`（字符串）、`ts`（正数时间戳）、可选 `props`；
- `props` 仅包含白名单字段：
  - `onboarding_shown.props.source` 为 `'auto'|'manual'`
  - `onboarding_completed.props.source` 与 `onboarding_completed.props.action` 为 `'finish'|'skip'`
  - `rating_prompt_action.props.action` 为 `'rate'|'later'|'never'`
- `copy_success` / `prompt_used` 无 `props`。

## 用例 3：关闭后立刻停止追加并清空
步骤：
1. 在“匿名使用数据”开启状态下，先触发至少 1 条事件（例如打开 Popup）。
2. 读取日志脚本确认存在事件数组。
3. 关闭 Options 页“匿名使用数据”开关。
4. 读取日志脚本确认已清空（key 不存在或为空）。
5. 再次执行用例 2 中任意动作（打开 Popup/复制成功等）。
6. 再次读取日志脚本。

期望：
- 第 4 步后 `copilot_telemetry_events` 被移除或为空；
- 第 6 步仍保持为空（不发送、不缓存、不补发）。

## 用例 4：最多 100 条 + FIFO（丢弃最旧）
步骤：
1. 打开 Options 页，将“匿名使用数据”开关打开。
2. 在 Options DevTools Console 执行以下脚本，预置 100 条合法事件：
```js
const key = 'copilot_telemetry_events';
const base = Date.now();
const events = Array.from({ length: 100 }, (_, i) => ({ name: 'popup_opened', ts: base + i }));
await chrome.storage.local.set({ [key]: events });
```
3. 触发一次新的可记录事件（例如打开一次 Popup）。
4. 执行读取日志脚本，检查数组长度与首条事件。

期望：
- 数组长度仍为 `100`；
- 最旧的一条被丢弃（首条 `ts` 应大于 `base`，即 `base + 1`）。

## 用例 5：隐私校验（无用户内容/页面内容/URL/标题）
步骤：
1. 在“匿名使用数据”开启状态下，触发多条事件（尤其是复制成功与 Prompt 复制成功）。
2. 读取日志脚本，人工检查每条事件对象：
   - 不出现复制内容正文
   - 不出现页面 URL、标题、DOM 文本
   - 不出现任何可识别用户的字段

期望：
- 满足隐私约束；事件只表达“动作发生”，不携带用户内容。

## 回归校验
步骤：
- 验证现有功能不受影响：复制、Prompt 复制、引导、评价引导、分享/反馈入口均可正常使用。

期望：
- 交互体验与原逻辑一致；埋点失败不影响主流程。
