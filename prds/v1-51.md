## 具体任务
1、Options -> 隐私与可观测性：新增「导出过去 7 天 Pro 意向明细（CSV）」入口（可审计、可复核、稳定 DOM）
  - 在 Options -> 隐私与可观测性 -> Pro 区域新增按钮，建议稳定 DOM：`#export-pro-intent-events-7d-csv`
  - 导出内容为 CSV（UTF-8，无 BOM），行级明细至少包含字段：
    - `exportedAt`（ms）/ `extensionVersion`
    - `windowFrom`（ms）/ `windowTo`（ms）/ `lookbackDays`（固定 7）
    - `eventTs`（ms）/ `eventLocalTime`（yyyy-mm-dd hh:mm:ss）/ `eventName` / `source`
  - 事件来源与过滤规则：
    - 仅导出 `pro_entry_opened/pro_waitlist_opened/pro_waitlist_copied/pro_waitlist_survey_copied`
    - 仅导出 `source in ('popup','options')` 的事件
    - 时间窗与 v1-50 一致：`from = now - 7d`，过滤规则为 `<from` 排除，`=from`/`=to` 包含
  - 隐私与开关行为必须一致：
    - 匿名开关 OFF：不读取/不推断历史，不导出事件（按钮置灰或导出空集并明确提示均可，但必须可在用例中验证“未读取本地 events”）
    - 匿名开关 ON：仅基于本地 telemetry（白名单 + FIFO window）过滤生成 CSV；不新增任何网络请求
  - 文件命名规范（可用于证据落盘与复盘对齐）：`copylot-pro-intent-events-7d-YYYY-MM-DD.csv`

2、数据口径与复核闭环：让“摘要（v1-50）”与“明细（v1-51）”可互相印证（收入取证可执行）
  - 明细 CSV 的 eventName 聚合计数，必须可与「复制本周 Pro 意向证据摘要（7d window）」中的 overall 计数一致（允许因 TELEMETRY_MAX_EVENTS=100 截断导致的上限差异，但必须在 CSV/文档中明确可审计说明）
  - 在 `docs/evidence/v1-51/index.md` 固化最小复盘口径（可直接用 CSV 计算并出图/出表）：
    - `pro_entry_opened/pro_waitlist_opened/pro_waitlist_copied/pro_waitlist_survey_copied` 的 7d 总量与按 source 拆分
    - 关键转化率（用 CSV 可复算）：`waitlist_opened_per_entry_opened`、`waitlist_copied_per_waitlist_opened`、`survey_copied_per_entry_opened`
  - 明确非目标：不引入“支付/订阅/价格页”，不做与收入证据无关的 UI 美化

3、补齐测试/用例与证据落盘（确保 1~3 天内可交付、可回归、可审计）
  - 新增测试用例文档（固定路径）：`docs/test-cases/v1-51.md`
    - 覆盖：匿名开关 OFF/ON；7d window 边界；CSV 字段与行内容断言；CSV 与 v1-50 摘要计数一致性复核；导出文件命名规范
  - 自动化测试加固（优先纯函数单测）：为“7d 过滤 -> CSV 序列化”新增/补齐单测，并保证 `bash scripts/test.sh` 全量通过
