# V2 E2E 自动化与发布回归（测试用例）

## 目标
- 建立 `npm run test` 的统一 QA 验收级自动化回归入口
- 优先覆盖用户真实入口：页面选择、内容脚本按钮、popup、options、background context menu
- 将所有自动化测试收口到单一命令

## 执行入口
- 首次安装浏览器：
  - `npx playwright install chromium`
- 统一回归：
  - `npm run test`
- 跳过扩展重建直接复测：
  - `COPYLOT_TEST_SKIP_BUILD=1 npm run test`
- 仅执行原生 UI 真实入口回归：
  - `COPYLOT_TEST_ONLY_NATIVE_UI=1 npm run test`

## 覆盖矩阵
- Content 页面：
  - 打开真实网页 fixture：`article.html`
  - 真实选中文本并点击页面
  - 校验悬浮 Magic Copy 按钮可见
  - 点击主按钮，校验 growth stats 的 `successfulCopyCount` 增长
  - 打开编辑器 fixture：`editor.html`
  - 校验编辑器排除逻辑生效，按钮不可见
  - 打开代码块页面 fixture：`code.html`
  - 校验 hover 代码块复制会移除 copy-label 与可识别行号
  - 校验整页转换保留上下文段落与 fenced code
  - 打开表格 fixture：`table.html` / `table-complex.html`
  - 校验表格复制支持 RFC4180 风格 CSV 与复杂 markdown 表格
- Context Menu：
  - 打开真实网页 fixture 并创建真实选区
  - 通过 background 复用生产 `handleContextMenuClick()` 触发 prompt 菜单路径
  - 校验自动打开 chat 页
  - 校验 prompt `usageCount` 持久化
  - 校验 growth stats 的 prompt copy 成功计数
  - 触发整页 `Convert Page` 入口
  - 校验页面级抽取会过滤噪音区块
- Popup 页面：
  - 基于当前活动 tab 打开真实 popup 页面
  - 自动完成 onboarding
  - 点击 `Convert`，校验 growth stats 成功计数增长
  - 点击 `Manage Prompts`，校验真实 options 页打开
  - 点击 `复制候补文案`，校验按钮状态变化与 telemetry 写入
  - 切换 interaction / output / table / extras / append mode 配置并校验 sync 持久化
  - 校验 onboarding 推荐配置、完成状态持久化、重新打开与打开 options deeplink
  - 校验 popup 增长链路：
    - `Pro 问卷`
    - `Share with friends`
    - `Feedback`
    - `Rate`
    - 校验真实点击后的外链目标 URL 审计与 telemetry
- Options 页面：
  - 打开真实 options 页面
  - 校验内置 Prompt：`data-id="builtin-summary-article"`
  - 新建 `E2E Prompt`
  - 校验新 Prompt 卡片渲染成功
  - 校验自定义 chat service 增删改、默认服务与 fallback
  - 校验 prompts 编辑、搜索、过滤、排序、预览、批量删除、导入导出
  - 校验 telemetry / growth funnel / WOM / Pro funnel 面板复制、导出、清空与 evidence pack
  - 校验 Pro survey / distribution toolkit / waitlist open/copy attribution
- Native UI：
  - 真实点击浏览器工具栏 `扩展程序 / Extensions`
  - 真实点击扩展面板中的 `Copylot`
  - 校验 popup 真实打开，并能完成一次 `Convert`
  - 真实系统级右键选区
  - 真实打开 `智能复制+自定义提示 / Magic Copy with Prompt`
  - 真实点击 Prompt 子菜单项 `QA Native Summary`
  - 校验 prompt `usageCount` 与 growth stats 成功计数
  - 真实系统级右键页面
  - 真实打开 `Copylot > 转换为AI友好格式`
  - 校验整页转换真实执行成功

## 备注
- 主 E2E 使用 `.tmp_e2e/extension` 产物，不污染 `dist/`
- Popup 主套件使用带 `?tab=` 的真实 popup 页面，确保可稳定绑定当前 QA tab
- 原生浏览器 icon 点击与原生右键菜单点击继续单独归入 `native-ui` project；统一命令默认跳过该组，需显式设置 `COPYLOT_TEST_NATIVE_UI_SKIP=0` 或 `COPYLOT_TEST_ONLY_NATIVE_UI=1`
- `native-ui` 依赖 macOS 辅助功能权限、桌面会话和 `Google Chrome for Testing` 可见窗口
