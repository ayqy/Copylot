# v4-12 收费评估审计包

## 背景

`v4-7 / v4-8 / v4-9 / v4-10 / v4-11` 已经把收费前门槛、三路线样本比较、领先路线回写包、领先路线稳定性摘要和统一 verdict 串成一条完整判断链，但最后一项 roadmap 还停在“已经有判断，却还没有把 go/no-go、阻塞项、证据链和边界整理成同一份审计包”的状态。要把 `S3 Pro 路线验证` 从 `3/4` 正式收口到 `4/4`，需要补上这份审计包，同时继续明确本轮不进入支付实现。

## 目标

1. 在 `Options -> Pro` 新增“收费评估审计包”区块，支持复制 Markdown 审计摘要与下载 JSON。
2. 把统一 verdict 进一步整理成可复核的审计结构，至少显式输出 `audit_status`、三项检查、阻塞项、边界、下一步与证据链。
3. 补齐脚本、测试与文档，把 roadmap 最后一项正式收口，但继续保持“不做支付实现”的边界。

## 交付范围

1. 新增共享纯逻辑 `src/shared/pro-payment-evaluation-audit.ts`，负责从统一 verdict 构建收费评估审计包。
2. 新增脚本 `scripts/build-pro-payment-evaluation-audit-pack.ts`，使用 `docs/evidence/v4-11/verdict-pack/*` 生成 `docs/evidence/v4-12/payment-evaluation-audit-pack/*`。
3. 更新 `src/options/options.ts`、`src/options/options.html`、`_locales/en/messages.json`、`_locales/zh/messages.json`，让 `Options -> Pro` 可直接导出审计包。
4. 更新 `scripts/test.ts`、`scripts/unit-tests.ts`、`scripts/ui-integration-tests.ts`、`e2e/options-pro-flow.spec.ts`，覆盖共享逻辑、按钮可见性、复制摘要与脚本确定性。
5. 新增 `docs/test-cases/v4-12.md`、`docs/evidence/v4-12/*`、`docs/reports/v4-12-report.md`，并同步刷新 `docs/roadmap.md`、`docs/roadmap_status.md`。

## 边界

1. 本轮只做收费评估审计，不实现支付、收款、订阅、问卷、候补或其他偏航承接。
2. 审计包只使用本地匿名聚合字段和本地判断结果，不包含网页正文、复制内容、URL、标题或联系方式。
3. 即便审计包给出“可进入收费评估复核”，本轮也只停留在判断与文档层，不直接进入支付实现。
