## 用例 D：三方互证（v1-54 leads + v1-57 distCopies -> v1-58 合并结果可复算一致）

目标：证明 v1-58 合并导出结果可被审计复核（同一 campaign 下可由 v1-54/v1-57 复算）。

步骤：
1. 匿名使用数据 ON。
2. 导出 v1-54：点击 `#export-pro-intent-by-campaign-7d-csv`，拿到每个 campaign 的 `leads`。
3. 导出 v1-57：点击 `#export-pro-distribution-by-campaign-7d-csv`，拿到每个 campaign 的 `distCopies`。
4. 导出 v1-58：点击 `#export-pro-acquisition-efficiency-by-campaign-7d-csv`，拿到合并表。
5. 在表格中按 `campaign` 对齐三份 CSV，并逐行复算校验：
   - `leads`（v1-58）应等于 v1-54 导出中的 `leads`
   - `distCopies`（v1-58）应等于 v1-57 导出中的 `distCopies`
   - `leadsPerDistCopy`（v1-58）应等于 `leads / distCopies`
   - 当 `distCopies=0` 时，v1-58 必须输出 `N/A`

期望：
- 任意 campaign 行可被 v1-54/v1-57 复算一致；分母为 0 时不除零、不误读（输出 `N/A`）。
