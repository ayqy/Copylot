# 问题修复和功能优化（V1-67 CWS Listing ASO 迭代证据化 v2：证据包 diff + 变更摘要一键生成）

## 目标
基于 `docs/roadmap_status.md` 的收入优先 Top3：Top1「真实 CWS 发布 + 商店端取证」与 Top2「发布后回归与 24h/7d 复盘取证」当前均受“网络可达性/商店可达”阻塞，本轮顺延选择 Top3「CWS Listing ASO 迭代节奏固化（离线可推进）」作为 1~3 天最小可交付增量主题。

收入第一：把“上架=获客入口”的 ASO 迭代从“手工对比、不可审计、容易丢转化入口/合规口径”升级为**可一键生成 diff 证据 + 变更摘要 + 门禁**的可复盘资产，确保每次 Listing 调整都能提供可验证的商业化推进证据（变更点、基线、断言、红线），并为网络恢复后的真实发布与商店端转化复盘提供可追溯素材链路。

## 具体任务
1、新增「CWS Listing ASO diff 证据包」一键生成脚本（让每次物料改动都有可审计 diff）
  - 新增脚本（固定路径）：`scripts/build-cws-listing-diff-evidence-pack.ts`
  - 输入（必须来自可审计文件；禁止手填/复制粘贴到代码）：
    - baseline：1 份 v1-66 的 `cws-listing-evidence-pack-*.json`（默认取 `docs/evidence/v1-66/index.md` 中最新/当前引用的 pack；也允许通过参数显式指定）
    - current：基于仓库当前 `docs/` 输入，用 v1-66 的 `buildCwsListingEvidencePackFromRepo` 即时生成（要求 `requireDistManifest: true`，避免版本漂移）
  - 输出（落盘目录固定，便于按“每次 ASO 迭代”归档）：`docs/evidence/v1-67/`
    - diff 证据包（JSON，命名稳定）：`cws-listing-diff-evidence-pack-<extensionVersion>-<utcCompact>.json`
    - `docs/evidence/v1-67/index.md`：索引 baseline/current pack 的 sha256、关键变更摘要、红线断言结果、执行命令
  - diff 证据包最小字段（字段命名/顺序稳定，便于长期 diff 与审计）：
    - `packVersion: "v1-67"`
    - `exportedAt`（ISO）
    - `extensionVersion`（来自 current pack）
    - `baseline`：`{ packPath, sha256, extensionVersion, exportedAt, assertions }`
    - `current`：`{ sha256, extensionVersion, exportedAt, assertions }`（如落盘 current pack，可额外记录 `packPath`）
    - `diff`（至少包含）：
      - `keywords.enAdded/enRemoved/zhAdded/zhRemoved`（集合 diff）
      - `descriptions.enSha256From/enSha256To/zhSha256From/zhSha256To`（内容级变更的可复核指纹；不强制实现全文 diff 算法）
      - `screenshotPlan.changedIds`（截图顺序/标题/断言要点的变化标记）
      - `assertions.changedKeys`（从 true->false 或 false->true 的键）
    - `redlines`（数组，门禁用）：至少包含 `lostProWaitlistCta/lostPrivacyClaims/overclaimDetected` 等可执行枚举
  - CLI 约束（为门禁与可重复性服务）：
    - 支持 `--stable-exported-at`（用于 `bash scripts/test.sh` 的 deterministic 校验）
    - 支持 `--baseline-pack <path>`（可选；不传则按 `docs/evidence/v1-66/index.md` 自动解析）
    - 支持 `--evidence-dir <dir>`（可选；默认 `docs/evidence/v1-67`）
  - 安全红线：diff 证据包/索引不得包含任何发布凭据、Token、环境变量明文

2、补齐自动化测试 + 门禁（保证“ASO 迭代证据化”长期可用且不跑偏）
  - 更新 `scripts/unit-tests.ts`（至少覆盖）：
    - diff pack Schema 稳定（必含字段齐全；`packVersion` 固定；字段顺序稳定）
    - redlines 触发逻辑：当 `current.assertions.hasProWaitlistCta=false` / `hasPrivacyClaims=false` / `noOverclaimKeywords=false` 时必须产出对应 redline
    - baseline pack 自动解析：从 `docs/evidence/v1-66/index.md` 能稳定解析出 pack 文件名（与 v1-66 口径一致）
    - 可重复性：`--stable-exported-at` 下输出文件名与 JSON 内容 deterministic（便于长期门禁）
  - 接入 `bash scripts/test.sh` 门禁（建议放在 v1-66 listing pack 门禁之后）：
    - 连续执行两次 `scripts/build-cws-listing-diff-evidence-pack.ts --stable-exported-at`，对 `docs/evidence/v1-67/index.md` 与 diff JSON 做 sha256 对比，确保输出可重复

3、用例/证据/汇报材料闭环（让“ASO 迭代”变成可执行、可审计的商业化动作）
  - 新增测试用例文档（固定路径，必须在本 PRD 中明确）：`docs/test-cases/v1-67.md`
    - 覆盖：生成 baseline pack（v1-66）-> 生成 diff pack（v1-67）-> 校验 `JSON.parse` -> 校验 `redlines=[]` -> sha256 复核 -> 敏感信息搜索（`rg`/`grep`）
    - 必须包含一次回归记录（日期 + PASS）与可复制命令
  - 落盘证据目录（固定路径）：`docs/evidence/v1-67/`
    - 至少包含 1 份 `cws-listing-diff-evidence-pack-*.json` 与 `index.md`（可审计/可复核/可复用）
  - 新增简报（固定路径建议）：`docs/reports/v1-67-report.md`
    - 用于口头汇报：本轮 ASO 变更点（关键词/描述/截图计划）、红线是否触发、对“获客入口/Pro 候补转化入口”风险的影响
  - 更新 `docs/roadmap_status.md`：勾选 v1-67 里程碑并刷新“当前进度/Top3/阻塞”一句话结论（保持网络阻塞信息不断档）
