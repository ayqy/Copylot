# V1-16 反馈入口增强：Issue 模板自动附带本地诊断信息（Settings + GrowthStats + TelemetryEvents）测试用例

## 目标
验证 V1-16 达到可发布：
- Popup（`src/popup/popup.html`）的反馈入口 `#feedback-link` 可用
- Options（`src/options/options.html`）的反馈入口 `#wom-feedback-open` 可用
- 打开 GitHub `issues/new` 时 body 预填包含 4 块（顺序不限，但必须存在）：
  - 环境信息（版本/扩展 ID/UA/language 等）
  - 关键开关快照 `copilot_settings`（仅标量字段；不含任何网页/复制内容/用户输入文本）
  - 增长统计 `copilot_growth_stats`
  - 匿名事件日志 `copilot_telemetry_events`（最近最多 N 条，按 `ts` 倒序；仅 `name/ts/props(白名单)`）
- “匿名使用数据”开关关闭时：`copilot_telemetry_events` 为空数组 `[]`，且读取失败不影响打开反馈页
- `bash scripts/test.sh` 全量通过

## 预置条件
- 已在 Chrome/Chromium 加载本地扩展（开发包或生产包均可）
- 能打开 Popup 与 Options 页面

## 用例 A（Popup 入口可用：打开 issues/new 且包含四块诊断信息）
步骤：
1. 打开 Popup（点击扩展图标）。
2. 点击“反馈与建议”链接（DOM：`#feedback-link`）。
3. 检查是否打开新 Tab，URL 是否为 GitHub `.../issues/new`。
4. 在打开的页面地址栏 query 中复制 `body` 参数（或直接在 issue 编辑器中查看 body 预填内容）。

期望：
- body 中包含：
  - `## Environment`（或中文对应段落）
  - `copilot_settings` 的 JSON code block
  - `copilot_growth_stats` 的 JSON code block
  - `copilot_telemetry_events` 的 JSON code block（包含 “last / 最近最多 N 条” 提示）

## 用例 B（Options 入口可用：行为与 Popup 一致）
步骤：
1. 打开 Options 页面，找到 WOM 区域反馈按钮（DOM：`#wom-feedback-open`）。
2. 点击按钮。
3. 检查是否打开新 Tab，URL 是否为 GitHub `.../issues/new`。
4. 检查 body 预填是否同样包含四块诊断信息。

期望：
- 与用例 A 一致，且 Options 的入口不会依赖 Popup 状态。

## 用例 C（匿名使用数据关闭：telemetry_events 为空数组，且不影响打开）
步骤：
1. 打开 Options -> 隐私/可观测性相关区域，将“匿名使用数据”开关关闭（`isAnonymousUsageDataEnabled = false`）。
2. 分别从 Popup 与 Options 点击反馈入口打开 GitHub `issues/new`。
3. 检查 body 中 `copilot_telemetry_events` 的 JSON code block。

期望：
- `copilot_telemetry_events` 为 `[]`。
- 打开反馈页不报错、不阻塞（读取失败/不存在也必须兜底为 `[]`）。
- 关闭后不会继续记录事件；重新开启后不会“补发”关闭期间历史事件（应为空或仅包含重新开启后的新事件）。

## 用例 D（匿名使用数据开启并产生事件：telemetry_events 倒序且不含网页/复制内容）
步骤：
1. 打开 Options，将“匿名使用数据”开关开启（`isAnonymousUsageDataEnabled = true`）。
2. 触发若干本地匿名事件（示例）：
   - 打开 Popup（`popup_opened`）
   - 在任意页面执行一次复制成功链路（`copy_success`）
   - 点击反馈入口（`wom_feedback_opened`）
3. 再次打开反馈入口并查看 body 中 `copilot_telemetry_events`。

期望：
- `copilot_telemetry_events` 至少包含上述事件名中的一部分（以实际触发为准），且按 `ts` 倒序（最新在最前）。
- 每条事件仅包含 `name/ts/props(白名单)`；不得包含任何网页内容/URL/标题/复制内容。
