# 问题修复和功能优化（V1-16 反馈入口增强：Issue 模板自动附带本地诊断信息（Settings + GrowthStats + TelemetryEvents））

## 目标
在不新增权限、不新增任何联网发送、且不包含任何网页/复制内容的前提下，升级 Popup/Options 的“反馈与建议”入口：GitHub `issues/new` 预填内容除现有环境信息与关键开关快照外，自动附带可审计的 `copilot_growth_stats` 与（截断后的）`copilot_telemetry_events`，降低问题定位与来回沟通成本，使之达到验收标准并可发布。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、扩展反馈模板生成能力（Shared 模块可复用；严格隐私合规）
  - 修改 `src/shared/word-of-mouth.ts`
    - 扩展 `FeedbackSettingsSnapshot` 字段：补齐仅标量字段（boolean/number/string 枚举），至少包含：
      - `isAnonymousUsageDataEnabled`
      - `popupOnboardingVersion` / `popupOnboardingCompletedVersion`
      - `defaultChatServiceId` / `defaultAutoOpenChat`
      - `language`
    - 严禁包含任何用户输入文本（例如 `userPrompts[].title/template`、自定义 Chat 服务 URL 等）
    - 扩展 `buildFeedbackIssueUrl(...)` 入参，使其支持额外注入两段“本地诊断快照”：
      - `growthStatsSnapshot`：`src/shared/growth-stats.ts` 的 `GrowthStats`（即 `copilot_growth_stats` 规范化结果）
      - `telemetryEventsSnapshot`：`src/shared/telemetry.ts` 的 `sanitizeTelemetryEvents(...)` 过滤结果（即 `copilot_telemetry_events`），并**仅保留最近 N 条**（建议 `N=20`），按 `ts` 倒序输出
    - 失败兜底：读取失败/不存在时必须用 `{}` / `[]` 兜底；不得影响“打开反馈页面”主路径
    - URL 长度约束：为避免 query 过长导致打开失败，必须做截断（只截断 telemetry events；growth stats/settings 不截断）

2、Popup 与 Options 的反馈入口接入诊断信息（不新增事件名、不改动现有链路）
  - Popup：`src/popup/popup.ts` 的 `#feedback-link`
    - 在构造 `buildFeedbackIssueUrl(...)` 之前读取：
      - `await getGrowthStats()`（本地增长统计）
      - `await chrome.storage.local.get(TELEMETRY_EVENTS_KEY)` + `sanitizeTelemetryEvents(...)`（本地匿名事件日志）
    - 将诊断快照注入到 `buildFeedbackIssueUrl(...)`
    - 保持既有本地匿名事件记录逻辑不变：仍只复用 `wom_feedback_opened`（仅在开关开启时写入本地）
  - Options：`src/options/options.ts` 的 `#wom-feedback-open`
    - 同样注入 growth stats + telemetry events（与 Popup 行为一致）
  - 约束：
    - 不新增权限、不引入第三方依赖、不新增联网请求
    - 严禁拼装任何网页内容/URL/标题/复制内容
    - telemetry events 只允许 `name/ts/props(白名单)`；props 仍按 `src/shared/telemetry.ts` allowlist 过滤

3、更新 i18n 模板 + 单测/用例闭环（可执行、可回归、可发布）
  - i18n：
    - 更新 `_locales/en/messages.json` 与 `_locales/zh/messages.json` 的 `feedbackIssueBodyTemplate`，在原有“环境信息 + settings 快照”后追加两个段落：
      - `copilot_growth_stats`（JSON code block）
      - `copilot_telemetry_events`（JSON code block，注明“last N”）
    - 确保 `npm run check-i18n` 通过
  - 单测（最小但必须）：
    - 更新 `scripts/unit-tests.ts`，至少覆盖：
      - `buildFeedbackSettingsSnapshot(...)` 输出不包含任何用户输入文本字段（例如不存在 `userPrompts`/`template`/`title` 等 key）
      - `buildFeedbackIssueUrl(...)` 生成 URL 的 query 中包含 `copilot_growth_stats` 与 `copilot_telemetry_events` 对应的 JSON 片段（至少校验关键字段/片段存在）
  - 测试用例文档：
    - 新增并维护 `docs/test-cases/v1-16.md`，覆盖：
      - Popup/Options 两处反馈入口均可用；打开 GitHub `issues/new` 时 body 预填包含四块：环境信息、settings 快照、growth_stats、telemetry_events
      - “匿名使用数据”开关关闭：telemetry_events 为空数组；且不会因读取失败报错（不影响打开）
      - 开关开启并产生若干事件后：telemetry_events 至少包含 `popup_opened/copy_success/wom_feedback_opened` 等（按 `ts` 倒序），且不包含任何网页/复制内容
      - 执行 `bash scripts/test.sh` 全量通过，并在文档记录结论
