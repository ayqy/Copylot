## 用例 A（离线）：触发门槛（高意图、低打扰）

目的：
- 仅对高意图用户展示 Popup 内「Pro 候补提示」
- 避免打扰新手/低活跃用户

前置（建议用 zip/unpacked 验收口径）：
- 使用仓库内最新 `plugin-*.zip` 解压后以 unpacked 方式加载（禁止用 `src/` 开发态代替验收）。

准备（强制模拟 growth stats，避免“安装满 48h + 累积复制次数”等待）：
1. 打开 Options -> 隐私与可观测性 -> Growth Stats（或任意可打开 devtools 的扩展页面）。
2. 在 Console 执行（设置为“满足条件”的高意图用户）：
   - `await chrome.storage.local.set({ copilot_growth_stats: { installedAt: Date.now() - 48*60*60*1000, successfulCopyCount: 20, firstPromptUsedAt: Date.now() - 1000 } })`
3. 打开 Popup。

期望：
- Popup 内出现「Pro 候补（可选）」提示，且不遮挡主操作（非弹窗遮罩，只在页面流内展示）。
- 提示可折叠（`details/summary`），默认展开展示 CTA。

反例（低活跃用户不应提示）：
1. Console 执行：
   - `await chrome.storage.local.set({ copilot_growth_stats: { installedAt: Date.now() - 48*60*60*1000, successfulCopyCount: 19 } })`
2. 打开 Popup。

期望：
- 不展示 Pro 候补提示。

## 用例 B（离线）：展示次数上限 + “稍后”延迟策略

准备：
1. 先按用例 A 设置为可触发状态。
2. 打开 Popup，确认提示出现（会写入 `proPromptShownCount`）。

步骤与期望：
1) 不点击任何按钮，直接关闭 Popup，再次打开 Popup
- 期望：不再出现（最多提示 1 次；除非用户显式选择“稍后”）。

2) 重新设置为“可触发状态且未决策”的干净数据：
- `await chrome.storage.local.set({ copilot_growth_stats: { installedAt: Date.now() - 48*60*60*1000, successfulCopyCount: 20, firstPromptUsedAt: Date.now() - 1000, proPromptShownCount: 0 } })`

3) 打开 Popup，点击「稍后」
- 期望：提示立即隐藏；growth stats 写入：
  - `proPromptAction='later'`
  - `proPromptSnoozedUntil` 为未来时间（延迟策略）

4) 立即再次打开 Popup
- 期望：不出现（snooze 未到期）。

5) 手动将 snooze 设置为已过期（用于回归验证“按策略延迟后才可再出现”）：
- `const s = (await chrome.storage.local.get('copilot_growth_stats')).copilot_growth_stats; await chrome.storage.local.set({ copilot_growth_stats: { ...s, proPromptSnoozedUntil: Date.now() - 1 } })`

6) 再次打开 Popup
- 期望：提示可再次出现（最多再提示 1 次；总展示次数上限）。

7) 再次关闭 Popup 并反复打开
- 期望：达到展示次数上限后不再出现（可通过检查 `proPromptShownCount` 验证）。

## 用例 C（离线）：按钮行为（加入候补 / 不再提示）

准备：按用例 A 设置为可触发状态。

1) 点击「加入候补名单」
期望：
- Popup 关闭，并在新标签打开 Options -> Pro Tab（hash `#pro`）。
- growth stats 写入 `proPromptAction='join'`（用于永久不再提示）。

2) 重置为可触发状态后，点击「不再提示」
期望：
- 提示立即隐藏。
- growth stats 写入 `proPromptAction='never'`。
- 之后无论如何打开 Popup，都不再出现该提示（永久关闭）。

## 用例 D（离线）：匿名使用数据开关 OFF（不记录、不缓存、不补发）

步骤：
1. Options -> 隐私与可观测性：确保“匿名使用数据”开关为 OFF（默认）。
2. 通过用例 A 的方式让 Pro 候补提示在 Popup 出现，并点击任意动作（join/later/never）。
3. 回到 Options -> 隐私与可观测性：
   - Telemetry Events 面板应为空（或不新增本次事件）。
   - Pro 意向漏斗摘要/证据包应提示不可用（或导出为空，不包含 `pro_prompt_shown/pro_prompt_action`）。

期望：
- 开关 OFF 时：不记录、不缓存、不补发；之后再打开开关也不应补发此前事件。

## 用例 E（离线）：匿名使用数据开关 ON（可导出摘要/证据包 + 至少 2 个转化率）

步骤：
1. Options -> 隐私与可观测性：开启“匿名使用数据”开关。
2. 按用例 A 让 Popup 展示 Pro 候补提示（触发 `pro_prompt_shown`）。
3. 点击任意动作按钮（触发 `pro_prompt_action`）。
4. 可选：再点击 Popup 内既有 Pro 入口按钮（用于形成可计算转化率）：
   - 「升级 Pro」（`pro_entry_opened`）
   - 「加入候补名单」（`pro_waitlist_opened`）
5. 回到 Options -> 隐私与可观测性：
   - 刷新 Pro 意向漏斗摘要
   - 复制「摘要」与「证据包」，并按 `docs/evidence/v1-46/index.md` 命名规范落盘到 `docs/evidence/v1-46/`

期望：
- 导出内容包含 `pro_prompt_shown/pro_prompt_action`（仅白名单字段，且 `props.source='popup'`）。
- Pro 意向漏斗摘要中可计算至少 2 个转化率口径（示例）：
  - `entry_opened_per_prompt_shown`
  - `waitlist_opened_per_prompt_shown`
