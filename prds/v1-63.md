# 问题修复和功能优化（V1-63 渠道跑数/复盘证据包再降摩擦：一键下载「周度渠道复盘证据包（JSON）」）

## 目标
基于 `docs/roadmap_status.md` 的收入优先 Top3：Top1「完成一次真实 CWS 发布 + 商店端取证」仍受“需要可用代理/VPN + 商店可达”的网络可达性阻塞，本轮顺延选择 Top2「渠道分发/投放跑数与复盘固化 v6」作为 1~3 天最小可交付增量主题。

收入第一：在不引入支付/订阅、不新增权限、不联网发送数据、不采集用户复制内容的前提下，把“渠道跑数 -> 复盘 -> 证据归档”的链路进一步从“多入口、多文件、易丢证据”升级为**单文件、可落盘归档、可审计可复核的周度证据包**——新增「下载周度渠道复盘证据包（JSON）」入口，将获客效率（leads/distCopies/leadsPerDistCopy）与其关键互证资产（分发动作 7d CSV、Pro 意向明细 7d CSV、周报 Markdown、Env）一次性打包，显著降低渠道复盘与商业化审计摩擦，持续推进真实收入回报路径（可转化获客增长 -> 留资 -> 后续 Pro 转化验证）。

1、新增「下载周度渠道复盘证据包（JSON）」入口（把每周跑数复盘从“多次导出”升级为“单文件证据包落盘”）
  - 入口位置：Options -> 隐私与可观测性 -> Pro 面板（与 v1-51/v1-57/v1-61 同区域）

    - 匿名使用数据 ON：读取本地匿名事件日志并生成完整证据包（含互证资产）
  - 证据包内容（字段命名/顺序稳定，便于长期归档与 diff；不得包含 URL/标题/网页内容/复制内容）：

    - `assets.acquisitionEfficiencyEvidencePack`：复用 v1-61 的获客效率 evidence pack（CSV + 周报 Markdown + rows + Env），确保同口径可互证
    - `assets.proDistributionByCampaign7dCsv`：复用 v1-57 的“分发动作按 campaign 聚合（7d CSV）”导出结果（同口径、同 header）
    - `assets.proIntentEvents7dCsv`：复用 v1-51 的“Pro 意向明细（7d CSV）”导出结果（同口径、同 header；至少覆盖 lead 事件）
    - `assets.verifyMarkdown`：最短可执行互证说明（如何从 CSV/rows 复算并核对 leadsPerDistCopy；以及如何核对 leads/distCopies 与两份 CSV 的计数一致）
  - 文件名规范（必须稳定、可单测、便于按周归档；不得包含 campaign 等用户输入）：
    - 建议：`copylot-pro-weekly-channel-ops-evidence-pack-YYYY-MM-DD.off.json`
    - 建议：`copylot-pro-weekly-channel-ops-evidence-pack-YYYY-MM-DD.on.json`
  - 实现建议（保持最小改动、不新增依赖/权限）：

    - 新增 `src/shared/pro-weekly-channel-ops-evidence-pack-filename.ts`：文件名生成纯函数（OFF/ON 后缀 + 本地日期），并在 `scripts/unit-tests.ts` 覆盖
    - 在 `src/options/options.html` 增加按钮；在 `src/options/options.ts` 复用既有下载逻辑（`downloadTextFile(...)` / `downloadCsvFile(...)`）实现下载
