Markdown is a lightweight markup language that lets you format text with plain syntax. This guide covers the essential syntax, practical steps, common pitfalls, and when to consider alternatives.

## What Is Markdown and Why Use It?

Markdown is a plain-text formatting syntax designed to be readable without rendering. It's used everywhere: GitHub READMEs, Notion notes, documentation sites, forums, and even emails. You don't need a WYSIWYG editor — just a simple text file.

**This guide is for:**
- Developers writing documentation or README files
- Content creators who prefer plain-text editing
- Anyone who wants to format text quickly without HTML

## Step-by-Step: How to Write Markdown

### 1. Headings
Use `#` for headings. One `#` is the largest (H1), six `######` is the smallest (H6).

```markdown
# Heading 1
## Heading 2
### Heading 3
```

### 2. Bold and Italic
- `**bold**` or `__bold__` → **bold**
- `*italic*` or `_italic_` → *italic*
- `***both***` → ***both***

### 3. Lists
**Unordered:** use `-`, `*`, or `+`
```markdown
- Item 1
- Item 2
  - Subitem
```

**Ordered:** use numbers
```markdown
1. First
2. Second
```

### 4. Links and Images
- Link: `[text](url)` → [Copylot](https://copy.useai.online)
- Image: `![alt text](image-url)`

### 5. Code
- Inline: use backticks: `code`
- Block: use triple backticks with optional language:
```javascript
console.log("Hello, Markdown!");
```

### 6. Blockquotes
Use `>` for quotes.
```markdown
> This is a blockquote.
```

### 7. Horizontal Rules
Use three or more dashes, asterisks, or underscores: `---`, `***`, `___`.

### 8. Tables
```markdown
| Syntax | Description |
|--------|-------------|
| Header | Title |
| Paragraph | Text |
```

## Common Mistakes and How to Avoid Them

- **Forgetting a space after `#`**: `#Heading` won't render as a heading. Always put a space: `# Heading`.
- **Mixing list markers inconsistently**: Stick to one style per list.
- **Not escaping special characters**: Use `\` before characters like `*` or `_` if you want them literal.
- **Broken image links**: Always double-check the URL and include alt text for accessibility.
- **Assuming all Markdown flavors are identical**: Some platforms (GitHub, Reddit, Discord) have slight differences. Test your output.

## When Markdown Might Not Be Enough

Markdown is great for simple formatting, but it has limits:
- No support for complex layouts (columns, advanced tables).
- Limited styling (no font sizes, colors, or margins).
- No dynamic content or interactive elements.

**Alternatives:**
- For advanced documents: use HTML/CSS, LaTeX, or a word processor.
- For rich content on the web: consider a CMS with a visual editor or lightweight frameworks like AsciiDoc.

## FAQ

### How to markdown 适合谁？

适合任何需要快速格式化文本的人，尤其是开发者写 README、文档，以及内容创作者写博客或笔记。如果你需要复杂排版或高级样式，Markdown 可能不够用。

### How to markdown 最容易踩的坑是什么？

最常见的是忘记在标题符号后加空格，导致格式失效。其次是混合使用不同的列表标记，或者没有转义特殊字符，导致渲染错误。

### How to markdown 失败时的备用方案是什么？

如果 Markdown 无法满足需求（如需要复杂表格、自定义样式或动态内容），可以改用 HTML/CSS、LaTeX 或 AsciiDoc。对于非技术用户，使用富文本编辑器也是可行的替代方案。
