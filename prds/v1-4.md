# V1-4 可观测性 MVP（匿名使用数据开关 + 本地匿名事件日志）

## 目标
在不引入任何联网埋点、且不影响现有复制体验/隐私优先/零打扰的前提下，新增“匿名使用数据”开关（默认关闭），并落地一套可审计的本地匿名事件日志（仅事件名/时间戳/少量枚举字段），用于验证关键漏斗与增长动作是否被正确记录；同时补齐可执行的测试用例文档与最小单测，达到可发布状态。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、匿名使用数据开关（默认关闭、可 i18n、可回归）
  - 在 Options 页新增“匿名使用数据 / Anonymous Usage Data”开关，落盘到 `copilot_settings.isAnonymousUsageDataEnabled`（新增字段，默认 `false`）
  - 开关打开后才允许写入本地匿名事件日志；关闭后必须立刻停止记录，并清空本地日志（确保“不发送、不缓存、不补发”的可验证行为）
  - 开关控件需有稳定 DOM 标识（用于用例描述与未来自动化），例如：`#anonymous-usage-data-switch`

2、本地匿名事件日志（可审计、无用户内容、固定保留策略）
  - 新增 `chrome.storage.local` Key：`copilot_telemetry_events`
  - 事件结构（仅允许白名单字段；不得包含任何用户复制内容/页面内容/URL/标题）：
    - `name: string`（事件名）
    - `ts: number`（毫秒时间戳）
    - `props?: Record<string, string | number | boolean>`（可选；仅允许少量枚举/布尔/计数类字段）
  - 保留策略：仅保留最近 `100` 条；超出时丢弃最旧；任何写入失败不得影响主流程
  - 最小事件清单（覆盖主 PRD 漏斗与现有增长动作；本阶段仅本地记录，不做联网发送）：
    - `popup_opened`（打开 Popup）
    - `copy_success`（成功写入剪贴板后）
    - `prompt_used`（通过 Prompt 链路触发的一次成功复制）
    - `onboarding_shown` / `onboarding_completed`（自动/手动；skip/finish 需可区分）
    - `rating_prompt_shown` / `rating_prompt_action`（`rate|later|never`）
    - `wom_feedback_opened` / `wom_share_opened` / `wom_share_copied` / `wom_rate_opened`
  - 新增并维护“埋点清单”文档：`docs/growth/telemetry-events.md`
    - 必须逐项写清：触发位置（文件/函数/交互）、事件名、props 字段、隐私约束、保留策略

3、测试与用例文档（可执行）
  - 新增并维护测试/用例文档：`docs/test-cases/v1-4.md`
  - 文档必须覆盖（至少）：
    - 默认关闭：不产生 `copilot_telemetry_events`；执行复制/引导/分享等动作后本地仍为空
    - 开关开启：上述动作会追加事件；事件字段符合白名单；日志最多 100 条且先进先出
    - 开关关闭：立即停止追加，并清空 `copilot_telemetry_events`
    - 隐私校验：事件与 props 中不出现任何复制内容、页面内容、URL、标题等
    - 回归校验：现有功能（复制、引导、分享/反馈/评价、评价引导计数）不受影响
  - 自动化/单测最小补齐：
    - 在 `scripts/unit-tests.ts` 增加对“事件白名单过滤 + 保留 100 条裁剪”的纯函数单测覆盖
  - 在验收前完成并记录：`./scripts/test.sh` 通过
