# 快捷 Prompt 槽位第二次复用路径

## 目标
实现 Popup / Onboarding 中快捷 Prompt 槽位的第二次复用路径，让首次成功用户拥有明确、可验证、可审计的再次打开理由，并使其达到验收标准

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、更新 `src/popup/popup.html`、`src/popup/popup.ts`、`src/popup/popup.css`、`src/options/options.ts` 与 `_locales/zh/messages.json`、`_locales/en/messages.json`，在首次 `copy_success` 之后把 Popup / Onboarding 的主行动切到快捷 Prompt 槽位，至少提供 1 个可直接触发的真实复用入口，并明确“复制网页内容 + 套用 Prompt + 粘贴到 AI”的第二次打开理由。
2、更新 `src/shared/growth-stats.ts`、`src/shared/telemetry.ts`、Popup 相关状态渲染与本地可导出摘要，补齐“首次成功 -> 快捷 Prompt 槽位点击或使用 -> 第二次 `copy_success`”的审计字段、事件口径与证据导出，输出到 `docs/evidence/v4-1/second-open-reuse-audit.md` 与 `docs/evidence/v4-1/reuse-funnel-sample.json`，且全程不记录任何复制内容或网页原文。
3、创建并维护 `docs/test-cases/v4-1.md`，同步更新相关 `e2e/*.spec.ts`、`scripts/test.sh` 覆盖范围与 `docs/reports/v4-1-report.md`，确保第二次打开路径、快捷 Prompt 槽位复用、商业化证据导出与回归结果都可执行、可复核、可汇报。

## 验收标准
* 修复上述所有问题，并测试通过，达到可发布状态
* 实现以上所有需求，并测试通过，达到可发布状态
* `docs/test-cases/v4-1.md` 中定义的自动化与手工用例均可执行，且 `bash scripts/test.sh` 退出码为 `0`
* 首次成功后的 Popup 或 Onboarding 中至少存在 1 个快捷 Prompt 槽位可直接触发，用户能够在当前主路径内完成第二次 `copy_success`
* `docs/evidence/v4-1/second-open-reuse-audit.md` 能复核“首次成功 -> Prompt 复用 -> 第二次成功复制”的字段来源、判定口径与样例，`docs/evidence/v4-1/reuse-funnel-sample.json` 可导出且不包含用户原文内容
* 本轮必须产出至少 1 项可验证的商业化证据：快捷 Prompt 槽位曝光、点击、成功复制或复用转化摘要中的任一组合，能够支持判断第二次打开理由是否推动真实复用与后续分享前置质量
