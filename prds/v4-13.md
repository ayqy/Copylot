# v4-13 跨 campaign 领先路线复核包

## 背景

`v4-12` 已把统一 verdict 固化成收费评估审计包，但当前 roadmap 已经明确：问题不再是“缺少更多判断面板”，而是“还没有把不同 campaign 对领先路线的支持、冲突和样本薄弱点整理成一份可导出的复核包”。如果这一层继续靠人工拼读，就仍然容易把单一获客来源的领先误判成长期需求。

## 目标

1. 在 `Options -> Pro` 新增“跨 campaign 领先路线复核包”区块，支持复制 Markdown 与下载 JSON。
2. 基于 `v4-10` 稳定性摘要和 `v4-11` 统一 verdict，输出每个 campaign 的 `supporting / conflicting / thin / no_signals` 状态、动作建议、优先补样列表、结论与下一步。
3. 把 roadmap 从已收口的 `S3` 正式推进到新的 `S4`，但继续停留在 `stay_validation` 边界，不进入收费或支付实现。

## 交付范围

1. 新增共享纯逻辑 `src/shared/pro-route-validation-campaign-review.ts`。
2. 新增脚本 `scripts/build-pro-route-validation-campaign-review-pack.ts`，使用 `docs/evidence/v4-10/stability-pack/*` 与 `docs/evidence/v4-11/verdict-pack/*` 生成 `docs/evidence/v4-13/campaign-review-pack/*`。
3. 更新 `src/options/options.ts`、`src/options/options.html`、`_locales/en/messages.json`、`_locales/zh/messages.json`，让 `Options -> Pro` 可直接导出跨 campaign 复核包。
4. 更新 `scripts/test.ts`、`scripts/unit-tests.ts`、`scripts/ui-integration-tests.ts`、`e2e/options-pro-flow.spec.ts`，覆盖共享逻辑、按钮可见性、剪贴板输出和脚本确定性。
5. 更新 `docs/test-cases/v4-13.md`、`docs/evidence/v4-13/*`、`docs/reports/v4-13-report.md`、`docs/roadmap.md`、`docs/roadmap_status.md`。

## 边界

1. 本轮只做跨 campaign 的领先路线复核，不新增支付、收款、问卷、留资或其他偏航承接。
2. 复核包只使用本地匿名聚合 campaign 字段和本地 verdict 结果，不包含网页正文、复制内容、URL、标题或联系方式。
3. 复核包的目标是帮助继续收集样本和约束对外话术，而不是证明已经可以进入收费实现。
