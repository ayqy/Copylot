## 自动化入口

```bash
bash scripts/test.sh
```

通过标准：

- 统一测试入口退出码为 `0`
- Popup 首屏、商店说明、官网承接规范、截图顺序和安装指引都围绕“第一次干净复制”
- 不新增表单、问卷、候补或支付承接

## 用例 A：Popup 首屏对齐第一次干净复制

步骤：

1. 打开任意包含长文、表格或代码块的页面。
2. 打开 Popup。
3. 检查首屏主标题、前三个卖点、三步指引和状态文案。

期望：

- 主标题为“30 秒完成第一次干净复制”或英文等价文案。
- 前三个卖点顺序固定为：长文、表格、代码块。
- 三步指引明确说明“先复制什么、怎么触发、复制后贴到哪里”。
- 第一次成功前状态为“待完成”，完成一次复制后再次打开 Popup 状态切换为“已完成”。

## 用例 B：商店说明与截图顺序一致

步骤：

1. 检查 `docs/ChromeWebStore-Description-ZH.md`。
2. 检查 `docs/ChromeWebStore-Description-EN.md`。
3. 检查 `docs/evidence/v1-111/screenshot-sequence.md`。

期望：

- 中英文主标题语义一致，第一目标都是“第一次干净复制”。
- 前三个卖点顺序一致：长文、表格、代码块。
- 截图顺序一致：Popup 首屏、长文、表格、代码块、粘贴结果。
- 快速上手步骤一致：安装、复制、粘贴验证。

## 用例 C：官网承接规范与安装指引可直接交付

步骤：

1. 检查 `docs/evidence/v1-111/homepage-alignment-spec.md`。
2. 检查 `docs/evidence/v1-111/first-copy-install-guide.md`。
3. 检查 `docs/ChromeWebStoreSetup.md`。

期望：

- 官网首页规范包含主标题、副标题、固定 CTA、截图顺序与页面禁区。
- 安装指引只要求三步即可完成第一次干净复制。
- 站点仓无需依赖当前仓库额外源码，也能按规范实现首页承接。

## 用例 D：官方链接资产只保留四类固定入口

步骤：

1. 检查 `docs/evidence/v1-111/official-links.json`。
2. 检查 `docs/growth/install-first-launch-pack-v1-111.md`。
3. 运行 Popup / Options 相关 E2E，确认对外分享口径不包含表单、问卷或候补。

期望：

- 只保留官网首页、商店安装页、隐私说明页、Pro 路线页。
- 不出现表单、问卷、候补或支付承接。
- UTM 样例在 `popup`、`options`、`distribution_toolkit` 三类 surface 下都成立。

## 用例 E：安装 -> Popup -> 首次成功链路可审计

步骤：

1. 执行 `bash scripts/test.sh`。
2. 检查 `docs/evidence/v1-111/funnel-audit-spec.md`。
3. 复核 `copilot_growth_stats` 的关键字段说明。

期望：

- 文档明确安装入口、首次 Popup 打开和首次 `copy_success` 的字段来源。
- 对账规则至少覆盖 `installedAt <= firstPopupOpenedAt <= firstSuccessfulCopyAt`。
- 即使匿名使用数据关闭，仍能用 `copilot_growth_stats` 证明首次成功已发生。
