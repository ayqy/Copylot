## 用例 A：Pro Tab 出现话术守门区块

步骤：

1. 打开 `src/options/options.html#pro`。
2. 检查 `#pro-stay-validation-messaging-guard-panel`、复制按钮和下载 JSON 按钮。

期望：

- 新区块可见。
- 文案明确当前外部口径仍是 `stay_validation`。
- 说明中清楚标出这是守门检查，不是收费放行判断。

## 用例 B：Markdown 摘要能逐项列出 surface 状态

步骤：

1. 在 `Options -> Pro` 点击复制守门摘要。
2. 读取最近一次剪贴板文本。

期望：

- 文本包含 `guard_status=`、`messaging_boundary=stay_validation`、`prioritized_campaigns=`。
- `route_headline`、`store_short_description`、`store_value_bullet`、`summary_judgement` 四个 surface 都存在。
- 每项都包含 `status=`、`validation_signals=`、`blocked_claim_hits=` 和 `reason=`。

## 用例 C：JSON 产物锁定验证边界

步骤：

1. 下载 `v4-14` JSON。
2. 打开文件并检查关键字段。

期望：

- 文件名为 `copylot-pro-stay-validation-messaging-guard-v4-14.json`。
- 文件内包含 `requiredValidationSignals`、`blockedClaims`、`surfaces`、`guardStatus`、`conclusion`、`nextStep`、`sources`。
- 不包含复制内容、网页正文、URL、标题或联系方式。

## 用例 D：守门规则回归

步骤：

1. 运行 `scripts/unit-tests.ts`。
2. 检查 `aligned / needs_review / blocked` 三类分支断言。

期望：

- 缺少验证词时为 `needs_review`。
- 命中收费 / 订阅等禁词时为 `blocked`。
- 本轮现有样例产物保持 `aligned`，并继续落在 `stay_validation`。

## 用例 E：证据与路线图收口

步骤：

1. 复核 `docs/evidence/v4-14/*`、`docs/roadmap.md`、`docs/roadmap_status.md`、`docs/reports/v4-14-report.md`。

期望：

- 证据索引明确守门区块、样例输出和当前对外边界。
- roadmap 把 `stay_validation` 外部话术守门标记为完成。
- 报告明确说明这轮完成的是“守门能力”，不是“收费实现”。
