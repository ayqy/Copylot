# v4-11 Pro 路线融合判断摘要

## 背景

`v4-8` 已经能比较三条路线样本，`v4-9` 已经能把领先路线回写成对外文案，`v4-10` 已经能复核领先路线的 `7d / 14d + campaign` 稳定性，`v1-81` 已经能导出收费前 A/B/C 门槛摘要，但现在仍需要人工拼读四份摘要，缺少一个统一 verdict 来明确当前是否还应停留在验证阶段。

## 本轮目标

1. 新增共享纯逻辑，把路线比较、回写、稳定性和收费前门槛融合成统一 verdict。
2. 在 `Options -> Pro` 新增融合判断入口，支持复制 Markdown 与下载 JSON。
3. 保持当前边界：只做统一判断与证据闭环，不做支付、候补、问卷或收款实现。

## 验收口径

1. `Options -> Pro` 可见“Pro 路线融合判断摘要”区块、复制按钮和下载按钮。
2. 摘要输出至少包含 `route_leader_consistent`、`route_stability_ready`、`gate_allows_payment_evaluation` 和最终 `verdictCode`。
3. `bash scripts/test.sh` 通过，覆盖 shared 逻辑、CLI 产物、Options 入口、UI 集成、Playwright 和 determinism 门禁。
