## 用例 D：campaign 合法（两种复制均成功，且 URL 解码可复核包含 campaign 行）

步骤：
1. 在 `#pro-intent-campaign` 输入 `twitter`，并失焦（触发 change，输入会被规范化）。
2. 观察：`#pro-waitlist-url-copy` 与 `#pro-waitlist-recruit-copy` 变为可点击（enabled）。
3. 点击 `#pro-waitlist-url-copy`，从剪贴板粘贴得到候补 URL。
4. 解码复核（推荐二选一）：
   - 方法 1：将 URL 粘贴到浏览器地址栏，观察 query 参数 `body`（解码后应包含 `campaign: twitter`）。
   - 方法 2：用 Node 解码校验：
     - `node -e 'const u=new URL(process.argv[1]); console.log(u.searchParams.get(\"body\"));' \"<PASTE_URL>\"`
5. 点击 `#pro-waitlist-recruit-copy`，从剪贴板粘贴得到招募文案。

期望：
- 候补 URL 为 GitHub New Issue URL（title/body 预填）。
- 解码后的 issue body 中**必须包含** `campaign: twitter` 行（用于渠道归因与后续 ROI 对齐）。
- 招募文案包含候补链接（同一 URL），且包含 campaign 信息；文案不拼入任何网页内容/复制内容/当前网页 URL/标题等。
