# V1-9 可见性克隆空白文本节点保留修复（避免文本粘连/丢空格）测试用例

## 目标
验证 V1-9 修复“非 `pre/code` 场景丢弃纯空白文本节点导致词语粘连”的问题；在不引入明显噪音空白的前提下，确保相邻文本可读性，并保证 `bash scripts/test.sh` 全量通过、可发布。

## 用例 1：复现用例（最小 HTML，不应粘连）
最小 HTML（示例）：
```html
<p>
  <span>Hello</span>
  <span>world</span>
</p>
```

步骤：
1. 将以上 HTML 放到任意页面/沙盒中渲染（确保 `<span>` 之间存在换行/缩进产生的纯空白文本节点）。
2. 使用插件一键转换（全文/选区均可）。

期望：
- 输出为 `Hello world`（中间必须有单个空格），不应变成 `Helloworld`。

## 用例 2：手工回归（真实网页：内联元素 + 换行缩进空白）
步骤：
1. 打开任意内容站点（博客/文档/新闻页均可），找到段落内存在多个内联元素（如 `<span>/<a>/<strong>/<em>`）且源码中通过换行/缩进排版的片段。
2. 使用插件复制该段内容（全文/选区均可）。
3. 粘贴到纯文本/Markdown 编辑器核对。

期望：
- 相邻词语/短语之间保留合理的单空格分隔；不出现“词与词粘连”。

## 用例 3：负例校验（块级元素之间不引入行内空格）
示例 HTML：
```html
<div>Hello</div>
<div>world</div>
```

步骤：
1. 构造或寻找与上述等价的块级相邻场景（`div/p/section/...` 等）。
2. 使用插件一键转换并粘贴查看。

期望：
- 输出仍保持块级分隔（不应变成同行 `Hello world`）。
- 不出现奇怪的行首空格、块级之间多余的“空格行”。
