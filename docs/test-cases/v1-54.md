## 用例 C：匿名使用数据 ON 且 7d window 内无事件（导出空集，字段稳定）

步骤：
1. Options -> 隐私与可观测性：先点击“清空”本地匿名事件日志（Telemetry Events 面板）。
2. 开启“匿名使用数据”开关。
3. 展开「Pro 意向漏斗摘要」面板，点击 `导出过去 7 天 Pro 意向按 campaign 聚合（CSV）`。
4. 打开下载的 CSV 文件。

期望：
- 文件名符合规范：`copylot-pro-intent-by-campaign-7d-YYYY-MM-DD.csv`。
- CSV 为 UTF-8（无 BOM），至少包含表头字段（固定顺序）：
  - `exportedAt,extensionVersion,windowFrom,windowTo,lookbackDays,campaign,proEntryOpened,proWaitlistOpened,proWaitlistCopied,proWaitlistSurveyCopied,leads`
- 数据行数为 0（仅表头）；或所有行均为白名单事件的聚合结果且不包含网页内容/复制内容/URL/标题。
- `lookbackDays` 固定为 `7`；`windowFrom/windowTo` 可与当前时间手工核对（<=7d 范围）。

## 用例 D：campaign 非空/为空两种情况（空桶聚合 + leads 可复算）

### D1：campaign 非空（例如 `twitter`）

步骤：
1. Options -> Pro Tab：在渠道输入项 `#pro-intent-campaign` 填写 `twitter`。
2. 回到 Options -> Pro Tab 触发一次打开（应记录 `pro_entry_opened`，source=options，campaign=twitter）。
3. 在 Pro Tab 依次触发：
   - 点击 `加入候补名单`（`pro_waitlist_opened`）
   - 点击 `复制候补文案`（`pro_waitlist_copied`）
   - 点击 `复制问卷（含环境信息）`（`pro_waitlist_survey_copied`，若该入口存在）
4. Options -> 隐私与可观测性 ->「Pro 意向漏斗摘要」面板，点击 `#export-pro-intent-by-campaign-7d-csv`。
5. 打开下载的 CSV 文件，定位 `campaign=twitter` 行。

期望：
- `campaign` 列存在 `twitter` 行，且 4 个事件计数为整数：
  - `proEntryOpened/proWaitlistOpened/proWaitlistCopied/proWaitlistSurveyCopied`
- `leads = proWaitlistCopied + proWaitlistSurveyCopied` 可在表格中复算一致。

### D2：campaign 为空（空桶：`空 campaign`）

步骤：
1. Options -> Pro Tab：清空 `#pro-intent-campaign`（保持为空）。
2. 重复 D1 的触发步骤（至少触发 `pro_entry_opened/pro_waitlist_opened/pro_waitlist_copied`）。
3. 导出 `#export-pro-intent-by-campaign-7d-csv`。
4. 打开 CSV，定位 `campaign=空 campaign` 行。

期望：
- `campaign=空 campaign` 行存在（用于发现渠道分发漏填）。
- 对应 4 个事件计数为整数；`leads` 可复算。

## 用例 E：三方互证（证据包 events / v1-51 明细 CSV / 本次聚合 CSV）

目标：保证“按 campaign 的计数”可复核、可审计、可对齐。

步骤：
1. 匿名开关保持 ON，确保 7d window 内至少有若干 Pro 意向事件，且同时覆盖：
   - 至少 1 个非空 campaign（例如 twitter）
   - 至少 1 次空 campaign（留空输入后触发）
2. 导出证据包（用于获得 campaign 维度的事件明细）：
   - Options -> 隐私与可观测性 ->「Pro 意向漏斗摘要」面板，点击 `复制证据包`；
   - 将剪贴板内容保存为本地文件（JSON）。
3. 导出 v1-51 7d 明细 CSV：
   - 点击 `#export-pro-intent-events-7d-csv`，保存 CSV。
4. 导出 v1-54 按 campaign 聚合 CSV：
   - 点击 `#export-pro-intent-by-campaign-7d-csv`，保存 CSV。
5. 在表格中做复核：
   - 从证据包 JSON 的 `events[]`：按 `props.campaign`（为空归为“空 campaign”）与 `name` 聚合计数，得到每个 campaign 的 4 个事件计数。
   - 从 v1-54 聚合 CSV：直接读取每个 campaign 行的 4 个事件计数与 `leads`。
   - 从 v1-51 明细 CSV：按 `eventName` 聚合计数得到 overall；并将 v1-54 的所有 campaign 行对同一事件的计数求和，应与 v1-51 overall 一致（同一 7d window，同一白名单，同一 source 过滤）。

期望：
- campaign 维度一致性：证据包 events 聚合出的每个 campaign 的 4 个计数，与 v1-54 聚合 CSV 对应行一致（允许行排序差异，不允许计数差异）。
- overall 一致性：v1-54 所有 campaign 行按事件求和，与 v1-51 明细 CSV 的 overall 计数一致。
- `leads` 字段可复算：`leads = proWaitlistCopied + proWaitlistSurveyCopied`。
