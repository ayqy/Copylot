# V1-43-1 WOM 入口 UTM 统一（Popup / Options / 评分引导）

## 目标
为所有打开 Chrome Web Store 的 WOM 入口统一 UTM 口径，形成可核验的商店端来源拆分证据，且不新增权限、不影响复制体验。

统一规则：
- `utm_source=copylot-ext`
- `utm_campaign=v1-43`
- `utm_medium` 至少区分三类入口：`popup` / `options` / `rating_prompt`

## 具体任务
1. 统一构造 UTM 的纯函数
   - 修改：`src/shared/word-of-mouth.ts`
   - 新增：`buildWomUtmParams(medium: 'popup' | 'options' | 'rating_prompt')`
   - `buildChromeWebStoreDetailUrl(...)` / `buildChromeWebStoreReviewsUrl(...)` 统一追加 query（不新增权限）
2. Popup / Options / 评分引导接入
   - Popup：`src/popup/popup.ts`
     - 分享入口与复制分享文案复用同一条 store URL（`utm_medium=popup`）
     - 去评价入口打开 reviews URL（`utm_medium=popup`）
     - 评分引导点击 “Rate” 打开 reviews URL（`utm_medium=rating_prompt`）
   - Options：`src/options/options.ts`
     - 分享入口与复制分享文案使用 `utm_medium=options`
     - 去评价入口打开 reviews URL（`utm_medium=options`）
3. 自动化断言（可回归）
   - 更新：`scripts/unit-tests.ts`
   - 覆盖：
     - `buildChromeWebStoreDetailUrl(...)` / `buildChromeWebStoreReviewsUrl(...)` 的 UTM 参数（含 `utm_campaign=v1-43`）

## 验收标准
- Popup/Options/评分引导三处打开 CWS 链接均满足 UTM 统一口径，且可在新标签正常打开
- “复制分享文案”中的商店链接与对应入口打开的链接一致（同 UTM）
- 不新增权限
- `bash scripts/test.sh` 全量回归通过
