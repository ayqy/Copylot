# V1-8 代码块清理误删修复（保守移除 Copy 按钮文本）

## 目标
修复当前代码块文本清理逻辑对正常代码内容的误删问题（例如误删 `Copy/Clone` 等合法标识符或字符串），在不影响现有复制体验与隐私原则的前提下，确保“复制出来的代码可直接使用”，并通过可执行的自动化测试与手工用例验证，使之达到验收标准、可发布。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、修复代码块文本清理的误删（聚焦 P0：避免丢失代码正文）
  - 问题背景：`src/shared/content-processor.ts` 内 `cleanCodeBlockTextConservatively()` 通过全局正则删除 `Copy/Clone/复制...`，会误删代码中的正常文本（见 `review-issues.md` 的 ISSUE-003）。
  - 改造策略：将“复制按钮文本”清理规则改为**行级、位置受限、强约束**的保守清理：
    - 仅当某一行 `trim()` 后**完全等于**以下候选之一时才移除该行：`Copy` / `COPY` / `Copy to clipboard` / `复制` / `复制代码` / `Clone` / `克隆` / `拷贝`
    - 默认仅允许移除代码块的**首行或末行**命中的按钮文案（避免误伤正文中间行）；若存在连续多行按钮文案，允许仅清理首/末端连续段
    - 严禁对“行内子串”做替换（例如 `const Copy = ...`、`"Copy"`、注释中的 Copy 等必须保留）
  - 代码组织要求（便于单测与复用）：
    - 抽出纯函数模块：`src/shared/code-block-cleaner.ts`
    - 对外导出：`cleanCodeBlockText(text: string): string`
    - `src/shared/content-processor.ts` 内部统一改为调用该纯函数（确保所有 code/pre 路径一致）
  - 保持既有能力不退化：继续保留当前“Markdown 字符反转义”逻辑，但仅作用于清理后的最终文本。

2、补齐自动化单测（可在 CI/本地一键执行）
  - 在 `scripts/unit-tests.ts` 增加针对 `src/shared/code-block-cleaner.ts` 的单测用例（不少于以下场景）：
    - “正文包含 Copy/Clone”不应被删除（标识符/字符串/注释三类至少各 1 个用例）
    - 代码块首/末行存在按钮文案时应被移除（中英文各 1 个用例）
    - 按钮文案出现在中间行时**不**移除（证明“位置受限”生效）
  - 验收前必须保证 `bash scripts/test.sh` 全量通过（lint/type-check/check-i18n/unit-tests/build:prod）。

3、补齐测试用例文档（可执行、可回归）
  - 新增并维护测试/用例文档：`docs/test-cases/v1-8.md`
  - 文档必须覆盖（至少）：
    - 手工回归：从任意技术文章复制包含 `Copy/Clone` 关键字的代码块，确认输出不缺字、不误删
    - 负例验证：构造“代码块首/末行带 Copy/复制代码”场景，确认按钮文案被移除且不影响正文
    - 回归验证：不影响现有“整页一键转换/选区转换/智能区块复制/表格转换/追加模式/Prompt 复制”等路径
    - 自动化验证：记录一次 `bash scripts/test.sh` 的执行结果（命令 + 结论）
