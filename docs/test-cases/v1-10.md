# V1-10 表格内选区复制一致性修复（整表优先 + 去除双重可见性裁剪）测试用例

## 目标
验证 V1-10 修复以下问题并达到可发布：
- 表格内选区触发转换时不再只输出局部碎片，输出为整表（Markdown/CSV 按设置）
- 表格外选区仍保持精确选区能力（仅输出选区内容，保留基础内联结构）
- 选区回退链路最多只执行一次 `createVisibleClone()`（避免双重可见性裁剪）
- `bash scripts/test.sh` 全量通过

## 本地复现准备（避免 file:// 权限差异）
1. 在项目根目录启动静态服务器：
   - `python3 -m http.server 8000`
2. 打开用例页面：
   - `http://localhost:8000/test/cases/table-selection-inside-table.html`

说明：
- Popup 的 **AI Convert** 按钮与右键菜单 **Convert Page** 均走 `CONVERT_PAGE_WITH_SELECTION` 入口。

## 用例 A（表格内选区：整表优先）
步骤：
1. 在表格任意单元格内划词（例如选中 `code-link` 或 `link`）。
2. 触发转换：
   - Popup：点击 **AI Convert**
   - 或右键菜单：点击 **Convert Page**

期望：
- 输出为整张表格（而不是仅当前单元格/fragment）。
- 表格输出格式遵循设置：
  - `Table output format = Markdown` 时输出为 Markdown 表格
  - `Table output format = CSV` 时输出为 CSV

## 用例 B（表格外选区：精确选区不退化）
步骤：
1. 在表格外段落（Outside paragraph A/B）中划词（例如选中 `Bold` 或 `link` 的一段）。
2. 触发转换（同用例 A）。

期望：
- 输出仅包含选区内容（不包含整表、也不包含整页其它内容）。
- 内联结构尽量保留（例如链接/加粗/内联 code 等）。

## 负例（无选区：整页转换保持一致）
步骤：
1. 点击空白处取消选区（或确保无任何选中内容）。
2. 触发转换（同用例 A）。

期望：
- 仍为整页转换（与现有行为一致）。

## 回归检查（不影响既有路径）
覆盖点（建议快速冒烟）：
- Magic Copy 主按钮复制：点击/双击触发主按钮复制，结果正常
- 表格转 CSV：将 `Table output format` 切到 CSV 后，整表输出为 CSV
- Prompt 处理整页：无选区时使用 prompt 功能，仍能处理整页并输出
- 追加模式（Shift append）：开启累积模式后，追加复制行为保持正常
