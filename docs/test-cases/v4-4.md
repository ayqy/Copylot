## 用例 A：Pro Tab 出现高级页面清洗验证卡片并复用 campaign 门禁

步骤：

1. 打开 `src/options/options.html#pro`。
2. 确认 `#pro-validation-advanced-cleaning`、`#pro-validation-advanced-open` 可见。
3. 未填写 campaign 时检查 3 个复制按钮。
4. 填写 `twitter` 后再次检查按钮状态。

期望：

- 高级页面清洗验证卡片可见，且不出现旧候补/问卷入口。
- 未填写 campaign 时，路线链接/验证简报/验证清单复制按钮为 disabled，并显示需要 campaign 的提示。
- 填写 campaign 后，3 个复制按钮恢复可用。

## 用例 B：高级页面清洗验证素材可复制并带上统一 UTM

步骤：

1. 在 `Options -> Pro` 填写 `campaign=twitter`。
2. 依次点击“复制路线链接”“复制验证简报”“复制验证清单”。
3. 读取最近一次复制的剪贴板文本。

期望：

- 路线链接包含 `/pricing`、`utm_medium=distribution_toolkit`、`utm_campaign=twitter`、`utm_content=options_advanced_cleaning_cta`。
- 验证简报包含路线/商店/隐私 3 类固定链接。
- 验证清单包含 campaign 和路线链接。
- 素材不包含复制正文、页面 URL、页面标题或其他用户原文。

## 用例 C：匿名导出能归因高级页面清洗验证动作

步骤：

1. 填写 `campaign=twitter`。
2. 点击“打开高级清洗路线”。
3. 依次复制路线链接、验证简报、验证清单。
4. 导出本地 Pro intent / distribution 摘要。

期望：

- intent 导出中出现 `pro_waitlist_opened`，且 `content=options_advanced_cleaning_cta`。
- distribution 摘要中新增 `validationRouteCopied`、`validationBriefCopied`、`validationChecklistCopied` 3 列。
- 以上 3 个动作会计入同一 campaign 的 `distCopies`。

## 用例 D：统一测试入口与离线证据包继续可审计

步骤：

1. 执行 `bash scripts/test.sh`。
2. 检查根目录最新 `plugin-1.2.3.zip`。
3. 检查 `docs/evidence/v1-63/*`、`docs/evidence/v1-64/*`、`docs/evidence/v1-65/*` 的周度渠道证据包。

期望：

- `bash scripts/test.sh` 退出码为 `0`。
- 最新离线安装包中能看到 `#pro-validation-advanced-open`、`#pro-validation-advanced-brief-copy` 与新 JS 关键字。
- 历史周度渠道证据包的 `proDistributionByCampaign7dCsv` 使用包含 3 个新验证列的最新表头，并能通过互证脚本。
