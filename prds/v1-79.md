# 问题修复和功能优化（V1-79 Pro 意向留资闭环最小增强：问卷信号结构化 + 7d 分布导出（无 PII））

## 目标
在不引入支付、不新增权限、不上传用户内容的前提下，把 Options -> Pro「付费意向问卷」从“只能定性反馈”升级为**可量化、可导出、可复盘**的商业化证据：结构化记录付费意向/价位区间/能力偏好信号，并支持导出过去 7 天意向分布（可用于判断是否值得推进收款链路）。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、Pro 问卷意向“结构化信号”写入本地 telemetry（隐私优先，无 PII）
  - 在 `src/options/options.ts` 的 `copyWaitlistSurveyToClipboard()` 记录 `pro_waitlist_survey_copied` 事件时，补齐以下 props（仅枚举/布尔；严禁自由文本）：
    - `pay_willing`: `yes|maybe|no|unknown`（空选择 -> `unknown`）
    - `pay_monthly`: `lt_5|5_10|10_20|20_50|50_plus|unknown`（空选择 -> `unknown`）
    - `pay_annual`: `lt_50|50_100|100_200|200_500|500_plus|unknown`（空选择 -> `unknown`）
    - `cap_advanced_cleaning|cap_batch_collection|cap_prompt_pack|cap_note_export`: boolean（对应 4 个能力 checkbox 是否勾选）
    - `has_other_capability`: boolean（`#pro-waitlist-survey-capabilities-other` 是否非空；不得记录其文本）
    - `has_contact`: boolean（`#pro-waitlist-survey-contact` 是否非空；不得记录其文本）
  - 更新唯一可信白名单与严格校验：`src/shared/telemetry.ts`
    - `pro_waitlist_survey_copied` 的 allowlist 必须包含上述字段，并对枚举字段做严格 value 校验（非法值丢弃）
    - 隐私红线（必须能被单元测试断言）：
      - 不得把 `useCase`/`contact`/`otherCapabilities` 的原文写入 telemetry props
      - 不得写入网页 URL/标题/复制内容

2、问卷意向分布导出（7d window，作为“收入证据”）
  - 在 Options ->「Pro 意向漏斗摘要」面板新增 1 个导出按钮（遵循匿名开关：OFF 时不导出）：
    - 导出过去 7 天「问卷意向分布」（JSON；按 `pay_willing/pay_monthly/pay_annual/cap_*` 聚合计数）
  - 导出文件命名（与现有导出风格一致，便于归档）：`copylot-pro-waitlist-survey-intent-distribution-7d-YYYY-MM-DD.json`
  - 指标口径（必须写入导出 JSON，避免口径漂移）：
    - `survey_intent = count(pro_waitlist_survey_copied)`
    - `pay_willing_* = 按 props.pay_willing 聚合计数`
    - `price_monthly_* / price_annual_* = 按 props.pay_monthly / props.pay_annual 聚合计数`
    - `capability_* = 按 cap_* 为 true 聚合计数（可多选，允许总和 > survey_intent）`

3、测试/用例/证据落盘（可执行、可审计）
  - 新增测试用例文档（固定路径）：`docs/test-cases/v1-79.md`
    - 用例必须覆盖：匿名开关 ON/OFF、填写问卷（含联系方式与“其他能力”自由文本）、点击复制问卷、导出分布 JSON、断言导出不包含联系方式原文/使用场景自由文本/其他能力原文
    - 必须包含一次回归记录（日期 + PASS/BLOCKED + 导出文件名/截图索引）
  - 新增证据目录（固定结构）：`docs/evidence/v1-79/`
    - `index.md`：记录口径、导出文件清单、sha256、以及“无 PII”断言结论
    - 至少落盘 1 份样例导出（JSON），可复算 sha256
  - 自动化回归门禁：
    - `bash scripts/test.sh` 全量 PASS
    - 在 `scripts/unit-tests.ts` 补齐单元测试：枚举值严格校验、以及“导出不含 PII 字段/内容”的断言（至少覆盖 contact/useCase/otherCapabilities 的原文不会出现在导出结果中）
