## 用例 A：匿名使用数据开关 OFF（不记录/不导出）

步骤：
1. 打开 Options -> 隐私页。
2. 关闭「匿名使用数据」开关。
3. 展开「Pro 意向漏斗摘要」面板，观察提示文案与按钮状态。
4. 填写「付费意向问卷」任意内容（包括：联系方式 + 其他能力自由文本），点击「复制问卷（含环境信息）」。
5. 展开「本地匿名事件日志」面板，点击「刷新」。

期望：
- 「Pro 意向漏斗摘要」面板显示提示：`未开启，不记录/不导出`。
- 导出类按钮（含「问卷意向分布（7d）」）不可用。
- 本地事件日志不会新增 `pro_waitlist_survey_copied`（或事件数保持不变）。

## 用例 B：问卷结构化信号写入 telemetry（匿名使用数据 ON）

步骤：
1. 打开 Options -> 隐私页，开启「匿名使用数据」开关。
2. 在「付费意向问卷」中填写：
   - 使用场景：输入任意自由文本（例如：`用于整理竞品资料`）
   - 勾选任意能力 checkbox（至少勾选 2 项）
   - 其他能力：输入任意自由文本（例如：`需要自定义导出模板`）
   - 是否愿意付费：选择 `Yes/Maybe/No` 之一
   - 可接受价位（按月/按年）：各选择 1 档
   - 联系方式：输入任意自由文本（例如：邮箱/Telegram）
3. 点击「复制问卷（含环境信息）」。
4. 展开「本地匿名事件日志」面板，点击「刷新」，在日志中定位 `pro_waitlist_survey_copied` 事件。

期望：
- `pro_waitlist_survey_copied.props` 中包含（仅枚举/布尔）：
  - `pay_willing/pay_monthly/pay_annual`
  - `cap_advanced_cleaning/cap_batch_collection/cap_prompt_pack/cap_note_export`
  - `has_other_capability/has_contact`
- 日志中不出现以下任何原文（隐私红线）：
  - 使用场景原文（useCase）
  - 联系方式原文（contact）
  - “其他能力”原文（otherCapabilities）
  - 网页 URL/标题/复制内容

## 用例 C：导出问卷意向分布（7d，JSON）

步骤：
1. 保持「匿名使用数据」为 ON。
2. 重复执行用例 B 的第 3 步（可改动价位/意向/能力勾选），至少产生 2 次 `pro_waitlist_survey_copied`。
3. 展开「Pro 意向漏斗摘要」面板，点击「导出问卷意向分布（过去 7 天，JSON）」。
4. 打开下载得到的 JSON 文件，复核字段与口径。

期望：
- 文件名符合：`copylot-pro-waitlist-survey-intent-distribution-7d-YYYY-MM-DD.json`。
- JSON 自带口径字段：`lookbackDays/windowFrom/windowTo/definitions`。
- JSON 包含可量化指标：
  - `survey_intent`
  - `pay_willing_*`
  - `price_monthly_* / price_annual_*`
  - `capability_*`
- JSON 不包含任何自由文本/联系方式原文/网页信息/复制内容（可用 `rg` 复核）。
