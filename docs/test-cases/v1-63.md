## 用例 B：匿名开关 OFF（允许下载证据包 JSON，但不读取/不推断事件）

步骤：
1. Options -> 隐私与可观测性：确保“匿名使用数据”开关为 OFF。
2. 点击 `#download-pro-weekly-channel-ops-evidence-pack`。
3. 在浏览器下载目录找到下载文件并打开查看。

期望：
- 下载成功（有成功通知）。
- 文件名稳定且可归档：`copylot-pro-weekly-channel-ops-evidence-pack-YYYY-MM-DD.off.json`
  - `YYYY-MM-DD` 取 `exportedAt` 的本地日期（与既有导出命名风格一致）。
  - 文件名不得包含 campaign/URL/标题/网页内容/复制内容等敏感信息。
- 文件可被 `JSON.parse(...)`。
- JSON 必须包含稳定 schema 字段（命名/顺序稳定，便于长期归档与 diff）：
  - 顶层：`packVersion/enabled/disabledReason/telemetryOffNotice/env/assets`
  - `packVersion` 固定为 `v1-63`
  - `env` 必含：`extensionVersion/exportedAt/lookbackDays/windowFrom/windowTo/isAnonymousUsageDataEnabled`
  - `assets` 必含：`acquisitionEfficiencyEvidencePack/proDistributionByCampaign7dCsv/proIntentEvents7dCsv/verifyMarkdown`
- 必须包含明确的 OFF 提示（例如“匿名使用数据关闭（不读取/不推断事件）”）。
- 不读取/不推断本地 events（OFF 时不应访问本地匿名事件日志；`assets.*` 为空/占位但可审计；`assets.acquisitionEfficiencyEvidencePack.enabled=false`）。

## 用例 C：匿名开关 ON（可下载完整周度证据包 JSON：互证资产齐全）

目标：覆盖至少 2 个 campaign + 空 campaign；覆盖 `distCopies=0 -> N/A`；小数格式固定 4 位；并可按 `assets.verifyMarkdown` 完成互证复核。

步骤：
1. Options -> 隐私与可观测性：打开“匿名使用数据”开关为 ON。
2. Options -> Pro Tab：设置 `#pro-intent-campaign=twitter`（合法 campaign）。
3. 生成 distCopies（v1-57 口径）：
   - 点击渠道分发工具包任意复制按钮至少 2 次（例如 `#pro-store-url-copy` / `#pro-distribution-pack-copy`），确保记录 `pro_distribution_asset_copied`。
4. 生成 leads（v1-51/v1-54 口径）：
   - 点击加入候补/问卷相关复制按钮，至少触发 1 次 `pro_waitlist_copied` 或 `pro_waitlist_survey_copied`。
5. 生成第二个 campaign（例如 `xhs`）：
   - 将 `#pro-intent-campaign` 改为 `xhs`，重复步骤 3~4。
6. 生成空 campaign 桶：
   - 清空 `#pro-intent-campaign`（留空），再触发 1 次分发复制或留资复制，使事件 `props.campaign` 缺失/空字符串。
7. 生成 `distCopies=0 -> N/A` 场景：
   - 选择一个 campaign（例如 `ads`），仅触发 leads（步骤 4），不触发分发复制（步骤 3）。
8. 回到 Options -> 隐私与可观测性 -> Pro 面板，点击 `#download-pro-weekly-channel-ops-evidence-pack`。
9. 打开下载的 JSON 文件查看。

期望：
- 文件名稳定且可归档：`copylot-pro-weekly-channel-ops-evidence-pack-YYYY-MM-DD.on.json`
- JSON 内字段稳定且可审计：
  - `packVersion=v1-63`
  - `env.windowFrom/windowTo` 为 7d window（边界一致，可与互证资产核对）
  - `assets.acquisitionEfficiencyEvidencePack`：复用 v1-61 同口径获客效率 evidence pack（CSV + 周报 Markdown + rows + Env）
  - `assets.proDistributionByCampaign7dCsv`：复用 v1-57 同口径分发动作按 campaign 聚合 7d CSV（header/字段顺序稳定）
  - `assets.proIntentEvents7dCsv`：复用 v1-51 同口径 Pro 意向明细 7d CSV（header/字段顺序稳定，至少覆盖 lead 事件）
  - `assets.verifyMarkdown`：包含可执行的互证步骤（复算 leadsPerDistCopy + 核对 leads/distCopies 与两份 CSV 的计数一致）
- 互证可复核：
  - `distCopies=0` 的行必须输出 `leadsPerDistCopy=N/A`
  - 小数格式固定 4 位（例如 `0.5000`），便于审计与对比

## 用例 D：互证复核（按 `assets.verifyMarkdown` 可完成“分发动作 -> leads -> 获客效率”复核）

步骤：
1. 匿名使用数据 ON。
2. 下载 v1-63：点击 `#download-pro-weekly-channel-ops-evidence-pack` 得到 evidence pack JSON 文件。

   - 对 `assets.proIntentEvents7dCsv` 统计 lead 事件行数，并与 rows 的 `leads` 求和一致

期望：
- 互证步骤可执行；可审计、可复核、可落盘归档。
