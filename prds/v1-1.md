# V1-1 口碑闭环 MVP（反馈模板 + 分享入口 + 评价入口）

## 目标
在不打扰用户正常复制体验的前提下，补齐最小“口碑闭环”基础能力：让用户可以一键分享、快速评价、并提交包含关键环境信息的反馈，从而降低反馈成本并提升商店自然增长效率；所有能力达到可验收、可测试、可发布状态。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、反馈入口模板化（最小可用）
  - 将 Popup 现有“Feedback & Suggestions”升级为“带模板”的反馈入口：点击后跳转到 GitHub `issues/new`，并在 URL 中预填 `title/body`
  - 反馈模板必须自动拼装并包含（仅环境与配置，不包含任何用户复制内容）：
    - 插件版本（`chrome.runtime.getManifest().version`）、扩展 ID（`chrome.runtime.id`）
    - 浏览器与系统信息（`navigator.userAgent`、`navigator.language`、`chrome.i18n.getUILanguage()`）
    - 关键功能开关快照（来自 `copilot_settings`，例如：启用复制/悬停/追加模式/输出格式/是否附带标题与 URL/交互模式等）
    - 复现步骤/期望结果/实际结果占位区块（便于用户填写）
  - 反馈入口文案与模板需可 i18n（至少中英文）

2、分享与评价入口（低打扰、可验证）
  - Popup 新增“分享给朋友”入口：支持
    - 打开带 UTM 的 Chrome Web Store 商店页（使用 `chrome.runtime.id` 构造 URL；UTM 字段固定且可审计）
    - 一键复制分享文案（含商店链接，文案需 i18n）
  - Popup 新增“去评价”入口：跳转到商店评价页（reviews），不做自动弹窗触发（避免打扰）

3、测试与用例文档（可执行）
  - 新增并维护测试/用例文档：`docs/test-cases/v1-1.md`
  - 文档必须覆盖：反馈模板字段校验、分享链接 UTM 校验、分享文案复制、评价跳转、i18n 展示、以及“不会包含用户复制内容”的隐私校验
  - 在验收前完成并记录：`npm run lint`、`npm run type-check`、`npm run build`（或 `npm run build:prod`）通过

## 验收标准
* 修复上述所有问题，并测试通过，达到可发布状态
* 实现以上所有需求，并测试通过，达到可发布状态
* Popup 中可见“反馈 / 分享 / 去评价”入口，且不影响现有复制与设置功能
* 反馈入口跳转的 GitHub 新建 Issue 页面已预填 title/body，且 body 中包含：版本、扩展 ID、浏览器/系统语言、关键开关快照、复现模板；不包含任何用户复制内容
* 分享入口可正常打开商店页，且 URL 携带约定的 UTM 参数；“一键复制分享文案”复制内容包含商店链接与简短卖点文案
* “去评价”入口可打开商店 reviews 页面；本版本不出现任何自动评价弹窗或强引导（零打扰）
* `docs/test-cases/v1-1.md` 已创建并覆盖所有关键路径；按文档执行手工测试通过；Lint/TypeCheck/Build 通过，可打包发布
