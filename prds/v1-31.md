## 具体任务
1、生产构建“干净产物”自检脚本（自动化可执行、失败即阻断发布）
  - 新增脚本：`scripts/verify-prod-build.sh`
  - 脚本约束：
    - 必须 `set -euo pipefail`，任一检查失败必须退出非 0
    - 不新增任何 npm 依赖；允许使用系统 `find/grep` 与 `node -e` 做 JSON 校验
  - 自检项（必须全部覆盖，且输出可读的失败原因）：
    - `dist/` 必须存在（否则提示先执行 `npm run build:prod`）
    - 生产包不得包含测试资源：
      - `find dist -name "*test*" -o -name "*runner*"` 必须为空
      - `dist/manifest.json` 不得包含 `test` 字样引用（例如 `grep -i test dist/manifest.json` 为空）
    - 生产包不得包含开发彩蛋/调试入口（防 CWS 审核风险）：
      - `dist/src/popup/popup.js` 中不得出现 `clickCount|easter|test/index.html`（正则 grep 为空）
    - 权限/隐私底线校验（防“无意新增权限”）：
      - 解析 `dist/manifest.json`，确保 `permissions` 集合严格等于 `["storage","clipboardWrite","contextMenus"]`（顺序不敏感）
  - 接入统一回归入口（让 CI/本地一键回归可阻断不干净产物）：
    - 修改 `scripts/test.sh`：在 `npm run build:prod` 后追加执行 `bash scripts/verify-prod-build.sh`

2、沉淀 V1 全量 Smoke 手工回归用例（覆盖核心链路、可在 30 分钟内跑完）
  - 新增并维护测试/用例文档：`docs/test-cases/v1-31.md`
  - 文档必须覆盖（至少）：
    - 用例 A（权限/隐私回归）：确认无新增权限；“匿名使用数据”默认关闭；关闭时不记录/不缓存/不补发本地匿名事件
    - 用例 B（新手引导回归）：新装/清空配置后首次打开 Popup 自动展示 3 步引导；完成/跳过后不再自动弹出；Options 页入口可重新打开
    - 用例 C（复制主流程）：单击/双击/悬停触发复制均可用；复制成功后剪贴板内容可用且不含明显无关内容
    - 用例 D（追加模式）：启用 Append Mode 后，Shift 追加复制可合并；关闭后回到单次复制
    - 用例 E（表格转换）：表格输出 Markdown/CSV 与设置一致；表格内选区/整表路径不应出现明显结构损坏
    - 用例 F（Prompt 工作流）：右键 Prompt“选区优先”可用；选区 Prompt 的“复制成功后自动打开 Chat”可用；关闭开关后不打开
    - 用例 G（口碑闭环）：分享/复制分享文案/去评价/反馈模板入口均可用；反馈模板仅包含环境信息与开关快照（不包含网页/复制内容/URL/标题）
    - 用例 H（商业化入口回归）：Popup/Options 的升级 Pro/候补入口可用；Pro Tab Free vs Planned 口径可见
    - 自动化回归：记录一次 `bash scripts/test.sh` 结论（命令 + 通过/失败），并注明本次已包含生产包自检脚本

3、发布简报交付（用于口头验收与发版记录）
  - 新增并维护简报：`docs/reports/v1-31-report.md`
  - 简报必须包含（至少）：
    - 本次交付的 2 项核心产物（自检脚本 + Smoke 用例文档）的完成状态
    - `bash scripts/test.sh` 回归结论（PASS/FAIL + 日期）
    - 修改范围（涉及的目录与文件列表）
