## 用例 B（Path A 可复现：代码块悬停复制）
步骤（按 `docs/tutorials/code-block-cleaning.md`）：
1. 选择示例页面（MDN / GitHub Docs 任一）。
2. 确保 Popup：`Enable Magic Copy: ON`；`Enable Hover Trigger: ON`；`Format: Plain Text`；`Extra: OFF`（建议）。
3. 在 `<pre>`/`<code>` 代码块上悬停出现 Copylot 按钮并点击复制。
4. 分别粘贴到：
   - VS Code / 任意纯文本编辑器
   - 任意 Chat 输入框

期望：
- 缩进/内部空行保留；仅裁剪首尾空行。
- 不混入你未开启的额外信息（`Extra` 关闭时不追加页面标题/URL）。
- 若页面存在可识别的行号结构，应被去除；若页面不存在行号或结构不可识别，则不强制要求去除（与教程边界描述一致）。

执行记录：
- 结果：通过
- 备注：代码块提取走 `convertToPlainText(...pre/code...) -> extractCodeBlockText()`，清理与边界可在 `src/shared/content-processor.ts` / `src/shared/code-block-cleaner.ts` 审计。

## 用例 C（Path B 可复现：Magic Copy 块级/页级复制）
步骤（按 `docs/tutorials/code-block-cleaning.md`）：
1. 确保 Popup：`Enable Magic Copy: ON`，选择 `Mode: Single-Click`（或 `Double-Click`）。
2. 选择 `Format: MD`（便于观察 code fence；也可换成 `Plain Text` 验证无 fence）。
3. 在示例页面点击包含代码块的段落/内容块（块级），或通过右键菜单页级入口复制主要内容。
4. 粘贴到任意 Chat 输入框 / Markdown 编辑器。
5. 对照教程中“Path A vs Path B 的范围差异”描述，确认一致。

期望：
- Path B 输出包含上下文（段落/标题/列表 + 代码块）；代码块在 `Format: MD` 时以 code fence 表达。
- 与 Path A 的差异：Path A 更适合只复制“代码本体”；Path B 更适合携带上下文发给 AI（与实现一致）。

执行记录：
- 结果：通过
- 备注：复制内容统一走 `processContent(currentTarget, settings)`；Markdown 时由 `processCodeBlock(...)` 包裹 code fence，代码文本仍由 `extractCodeBlockText()` 提取并清理（见 `src/shared/content-processor.ts` / `src/content/content.ts`）。
