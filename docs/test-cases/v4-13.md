```bash

```

专项覆盖：

- `scripts/unit-tests.ts`：纯逻辑覆盖 `supporting / conflicting / thin / no_signals` 四类分支、blocker 分类、`stay_validation` 边界与证据来源字段。
- `scripts/ui-integration-tests.ts`：`Options -> Pro` 中复核区块的复制 Markdown、下载 JSON 与通知路径。

## 用例 A：Pro Tab 出现跨 campaign 复核区块

步骤：

1. 打开 `src/options/options.html#pro`。
2. 检查 `#pro-route-validation-campaign-review-panel`、`#copy-pro-route-validation-campaign-review-summary`、`#download-pro-route-validation-campaign-review-json`。

期望：

- 新区块可见。
- 复制与下载按钮存在且默认可点击。
- 说明文案明确当前仍是“当前优先验证方向 / stay_validation”。
- 隐私提示明确说明不包含网页正文、复制内容、URL、标题或联系方式。

## 用例 B：复制出的 Markdown 摘要包含边界、分支与优先补样列表

步骤：

1. 在 `Options -> Pro` 点击“复制跨 campaign 复核包”。
2. 读取最近一次剪贴板文本。

期望：

- 文本包含 `V4-13 跨 campaign 领先路线复核包` 或英文标题。
- 文本包含 `messaging_boundary=stay_validation`、`overall_leader_track_id=`、`prioritized_campaigns=`。
- 文本至少包含一个 campaign 级别条目，且条目带有 `status=`、`signal_gap=`、`conclusion=`、`next_step=`。
- 文本不包含页面正文、复制内容、URL、标题或联系方式。

## 用例 C：下载出的 JSON 可区分 acquisition 偏差与样本过薄

步骤：

1. 点击“下载跨 campaign 复核 JSON”。
2. 打开生成文件并检查关键字段。

期望：

- 文件名符合 `copylot-pro-route-validation-campaign-review-v4-13.json`。
- 文件内包含 `overallLeaderTrackId`、`supportingCampaigns`、`conflictingCampaigns`、`thinCampaigns`、`prioritizedCampaigns`、`blockers`、`conclusion`、`nextStep`、`sources`。
- `blockers` 至少能区分 `acquisition_bias_unresolved` 与 `sample_still_thin`；当无信号样例进入纯逻辑测试时，还能区分 `no_campaign_signals`。

## 用例 D：四类复核分支回归

步骤：

1. 运行 `scripts/unit-tests.ts` 与 `e2e/options-pro-flow.spec.ts`。
2. 复核 `supporting / conflicting / thin / no_signals` 四类状态的断言。

期望：

- `supporting`：支持当前 leader，且 `prioritized = false`。
- `conflicting`：leader 与总体 leader 不一致，并落入 `acquisition_bias_unresolved`。
- `thin`：leader 与总体一致，但样本不足，落入 `sample_still_thin`。
- `no_signals`：无有效路线信号，落入 `no_campaign_signals`。

## 用例 E：证据、路线图与交付文档闭环

步骤：

1. 复核 `docs/evidence/v4-13/*`、`docs/roadmap.md`、`docs/roadmap_status.md`、`docs/reports/v4-13-report.md`。

期望：

- `docs/evidence/v4-13/campaign-review-pack/*` 给出按 campaign 的路线支持情况、优先补样列表与证据来源。
- `docs/evidence/v4-13/campaign-review-sample.md` 明确说明这些证据如何支撑“继续扩量 / 继续验证 / 暂缓收费判断”。
- `docs/roadmap.md` 与 `docs/roadmap_status.md` 的 S4 进度数字与 checkbox 统计一致。
- `docs/reports/v4-13-report.md` 记录测试结果、修改范围与 commit / push 状态。
