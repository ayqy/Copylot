# V1-43 WOM 小实验：分享/评价/反馈入口低打扰优化 + WOM 摘要/证据包 测试用例

## 目标
验证子 PRD `prds/v1-43.md` 的“低打扰 + 可量化 + 可审计 + 可对比”口碑（WOM）闭环：

- Popup / Options / 评分引导三处打开 Chrome Web Store 的链接均携带统一可核验 UTM（`utm_source/utm_medium/utm_campaign=v1-43`）
- “复制分享文案”中的商店链接与“分享入口打开的链接”一致（同 UTM）
- 隐私与可观测性：新增「WOM 摘要」面板，可刷新、复制摘要、复制 WOM Evidence Pack（字段白名单、脱敏）
- 匿名使用数据默认关闭；关闭时不记录/不导出（不缓存、不补发），且 UI 明确提示
- 自动化回归：`bash scripts/test.sh` 通过，并记录日期/结论

## 前置
1) 解压最新 zip 产物（本次验收版本）：
- `plugin-1.1.20.zip`（仓库根目录）

2) Chrome 加载 unpacked（强制验收方式）：
- 打开 `chrome://extensions`
- 开启 Developer mode
- 点击 `Load unpacked`
- 选择 zip 解压后的目录（目录内应包含 `manifest.json`、`src/popup/popup.html`、`src/options/options.html` 等）

3) 建议使用全新 Chrome Profile（避免历史匿名事件影响），或在隐私页先点击“清空”。

## 用例 A（默认关闭：不记录/不导出/不补发）
步骤：
1. 打开 Options -> 隐私与可观测性。
2. 确认“匿名使用数据”开关为关闭（默认关闭）。
3. 打开「本地匿名事件日志」面板：
   - 点击“刷新”，确认事件数为 0 且文本框为 `[]`。
4. 打开「WOM 摘要」面板：
   - 确认面板提示“未开启，不记录/不导出”可见。
   - 点击“刷新”，确认摘要 JSON 中 `enabled=false` 且各来源计数均为 0。
   - 点击“复制证据包”，检查导出的 JSON：
     - `settings.isAnonymousUsageDataEnabled=false`
     - `events=[]`
5. 在 Popup/Options 触发 WOM 入口（分享/复制文案/去评价/反馈）与评分引导按钮（如可触发）。
6. 回到隐私页再次刷新「本地匿名事件日志」与「WOM 摘要」。

期望：
- 开关关闭时：不新增事件；事件日志为空；WOM 摘要与证据包导出为空/不可用（不缓存、不补发）。

## 用例 B（UTM 校验：Popup / Options / 评分引导）
步骤：
1. 开启匿名使用数据（便于同时验证事件记录，但 UTM 校验不依赖开关）。
2. Popup 入口：
   - 点击“分享给朋友”，打开的商店详情页 URL query 必须包含：
     - `utm_source=copylot-ext`
     - `utm_medium=popup`
     - `utm_campaign=v1-43`
   - 点击“去评价”，打开的商店评价页 URL query 必须包含同一套 UTM（`utm_medium=popup`）。
   - 点击“复制文案”，粘贴到任意文本框，检查其中的商店链接 query 与“分享给朋友”打开的链接一致（同 `utm_medium=popup`）。
3. Options 入口（Pro Tab -> 口碑与支持）：
   - 点击“分享给朋友”，打开的商店详情页 URL query 必须包含同一套 UTM（`utm_medium=options`）。
   - 点击“去评价”，打开的商店评价页 URL query 必须包含同一套 UTM（`utm_medium=options`）。
   - 点击“复制文案”，粘贴检查商店链接 query 与 Options “分享给朋友”一致（同 `utm_medium=options`）。
4. 评分引导入口（Popup 内 rating prompt）：
   - 在满足条件时出现评分引导，点击 “Rate/去评价”，打开的商店评价页 URL query 必须包含同一套 UTM：
     - `utm_medium=rating_prompt`

期望：
- Popup / Options / 评分引导三类入口均满足 UTM 统一口径，且可在新标签打开；不引入新增权限。

## 用例 C（开启：触发 WOM / rating_prompt 事件 + 摘要/证据包字段白名单）
步骤：
1. Options -> 隐私与可观测性，开启“匿名使用数据”开关。
2. 触发 Popup 来源 WOM 事件（source=popup）：
   - Popup 点击“分享给朋友”（记录 `wom_share_opened`）
   - Popup 点击“复制文案”成功（记录 `wom_share_copied`）
   - Popup 点击“去评价”（记录 `wom_rate_opened`）
   - Popup 点击“反馈与建议”（记录 `wom_feedback_opened`）
3. 触发 Options 来源 WOM 事件（source=options）：
   - Options Pro Tab 点击“分享给朋友”（记录 `wom_share_opened`）
   - 点击“复制文案”成功（记录 `wom_share_copied`）
   - 点击“去评价”（记录 `wom_rate_opened`）
   - 点击“反馈与建议”（记录 `wom_feedback_opened`）
4. 触发评分引导事件（source=rating_prompt）：
   - 评分引导展示时应记录 `rating_prompt_shown`
   - 点击按钮记录 `rating_prompt_action`（`action=rate|later|never`）
5. 回到 Options -> 隐私与可观测性：
   - 刷新「本地匿名事件日志」，检查包含以上 `wom_*` / `rating_prompt_*` 事件，且 `props` 仅包含白名单字段：
     - `wom_*`：仅 `{ source: 'popup' | 'options' }`
     - `rating_prompt_shown`：仅 `{ source: 'rating_prompt' }`
     - `rating_prompt_action`：仅 `{ source: 'rating_prompt', action: 'rate'|'later'|'never' }`
   - 刷新「WOM 摘要」，检查按 `popup/options/rating_prompt` 分别统计 count/lastTs，并派生转化率：
     - `share_copied_per_share_opened`
     - `rating_prompt_rate_clicked_per_prompt_shown`
6. 导出并落盘（固定路径，供审计/对比）：
   - 点击「复制摘要」并将 JSON 保存到：`docs/evidence/v1-43/wom-summary.json`
   - 点击「复制证据包」并将 JSON 保存到：`docs/evidence/v1-43/wom-evidence-pack.json`
7. 关键路径截图（固定路径，供审计）：
   - 按“截图命名规范”保存截图到：`docs/evidence/v1-43/screenshots/`
   - 并确保 `docs/evidence/v1-43/index.md` 中截图索引与断言可复查

期望：
- 开关开启后：可记录并导出 WOM/rating_prompt 事件；摘要按来源统计正确，转化率派生正确；证据包字段白名单且无敏感信息。

## 用例 D（关闭后导出为空：不补发）

   - `docs/evidence/v1-43/wom-summary-telemetry-off.json`
   - `docs/evidence/v1-43/wom-evidence-pack-telemetry-off.json`

期望：
- 关闭状态下：不缓存、不补发；导出为空；不会出现“关闭后补回旧事件”的情况。
