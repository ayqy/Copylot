# V1-12 隐私页匿名事件日志可视化与一键导出（可审计）

## 目标
在不引入任何联网埋点、不新增权限、且保持“匿名使用数据”默认关闭的前提下，为 Options 页“隐私与可观测性”提供本地匿名事件日志（`copilot_telemetry_events`）的查看/刷新/复制/清空能力，使测试与用户自检无需 DevTools 即可验证事件白名单与“关闭即清空、不发送、不缓存、不补发”；并补齐可执行的测试用例文档与最小单测，使之达到验收标准、可发布。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、Options 隐私 Tab 增加“本地匿名事件日志”面板（只读查看 + 刷新/复制/清空）
  - 位置：`src/options/options.html` 的 `#privacy-tab`
  - 新增 UI 区块（建议默认折叠，避免打扰普通用户；但必须可发现、可测试）：
    - 容器：`#telemetry-events-panel`
    - 事件条数：`#telemetry-events-count`
    - 只读展示区（textarea 或 pre 均可）：`#telemetry-events-view`
    - 操作按钮：刷新 `#telemetry-events-refresh`、复制 `#telemetry-events-copy`、清空 `#telemetry-events-clear`
  - 展示与交互规则（必须可回归、可审计）：
    - 仅展示本地匿名事件：事件结构严格限定为 `name/ts/props(白名单)`；不得展示/拼接任何页面内容、复制内容、URL、标题等
    - 当 `Settings.isAnonymousUsageDataEnabled=false` 时：
      - 允许用户查看当前日志应为空（用于自检），但需在 UI 文案中明确提示“开关关闭时不会记录；关闭会清空”
    - “复制”必须在用户点击按钮后才写入剪贴板，复制内容为可解析 JSON（建议 pretty print）；复制失败不得影响 Options 其它功能
    - “清空”点击后立刻移除本地 key，并同步刷新 UI（清空后展示应为空数组或提示为空）
  - i18n：新增/补齐 `_locales/en/messages.json` 与 `_locales/zh/messages.json` 对应文案 key，并确保 `npm run check-i18n` 通过
  - 样式：`src/options/options.css` 中补齐面板样式，保持与现有 Options 视觉风格一致（不做大改版）

2、实现读取/渲染/导出逻辑（复用 telemetry 白名单过滤，保证容错）
  - 逻辑落地：`src/options/options.ts`
    - 读取：从 `chrome.storage.local` 读取 `copilot_telemetry_events`（建议复用 `src/shared/telemetry.ts` 的白名单过滤能力，避免 Options 侧直接信任存储内容）
    - 渲染：
      - UI 展示应稳定（建议按 `ts` 升序或降序固定一种；并在用例中按该排序验收）
      - 空/异常兜底：key 不存在、数据被污染（非数组/字段缺失）时，面板应显示为空且不抛错
    - 复制：将“过滤后的事件数组”序列化为 JSON 写入剪贴板；推荐追加换行便于粘贴阅读
    - 清空：复用 `src/shared/telemetry.ts` 的 `clearTelemetryEvents()`，并在清空后刷新 UI
  - 与开关联动（必须可验证“不发送、不缓存、不补发”）：
    - 关闭“匿名使用数据”开关后：除现有 `clearTelemetryEvents()` 外，需确保面板立即刷新为空（避免 UI 残留旧数据造成误解）
  - 最小单测补齐（保证可持续回归）：
    - 若新增/导出任何纯函数（例如事件数组序列化/排序/格式化），必须在 `scripts/unit-tests.ts` 增加断言覆盖
  - 约束：
    - 不新增任何权限
    - 不引入第三方依赖
    - 不新增任何联网请求
    - 面板相关操作（刷新/复制/清空）不得写入新的 telemetry 事件，避免“自举噪音”

3、补齐测试用例文档（可执行、可回归）
  - 新增并维护测试/用例文档：`docs/test-cases/v1-12.md`
  - 文档必须覆盖（至少）：
    - 用例 A（开关关闭）：确认日志面板可见；刷新后为空；执行若干动作（打开 Popup/复制成功/分享等）后再次刷新仍为空
    - 用例 B（开关开启）：触发至少 3 类事件（例如 `popup_opened/copy_success/wom_share_copied`），刷新后可见；结构仅包含 `name/ts/props(白名单)`，不包含任何 URL/标题/复制内容
    - 用例 C（复制导出）：点击“复制”后粘贴到文本编辑器，JSON 可解析且内容与面板一致
    - 用例 D（清空）：点击“清空”后刷新为空；并验证 `chrome.storage.local.get('copilot_telemetry_events')` 为空/不存在（用于最终兜底校验）
    - 用例 E（关闭即清空）：开启状态下产生事件后关闭开关，应立刻清空；再次触发动作不再记录（不发送、不缓存、不补发）
    - 自动化回归：记录一次 `bash scripts/test.sh` 执行结果（命令 + 结论）
