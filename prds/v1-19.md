# 问题修复和功能优化（V1-19 本地漏斗摘要面板：3 分钟激活与漏斗进度自检）

## 目标
在不新增权限、不新增任何联网发送、且不记录任何网页/复制内容的前提下，基于本地 `chrome.storage.local` 的 `copilot_growth_stats`（安装/首次打开 Popup/首次成功复制/首次 Prompt 使用/7 日内复用等里程碑时间戳与计数），在 Options -> 隐私与可观测性 中新增“本地漏斗摘要”面板：以**可审计、可复制导出**的方式展示漏斗进度与关键派生指标（尤其“首次打开 Popup 后 3 分钟内首次成功复制”），并补齐单测与测试用例文档，使之达到验收标准、可发布。

## 具体任务
1、补齐“漏斗摘要”纯函数（不改写存储；可单测；隐私合规）
  - 位置：`src/shared/growth-stats.ts`
  - 新增（或等价）纯函数与类型（命名可微调但需在 PR 中固定）：
    - `export interface GrowthFunnelSummary { ... }`（仅 `number/boolean/undefined`，不引入任何文本内容字段）
    - `export function buildGrowthFunnelSummary(stats: GrowthStats, now: number): GrowthFunnelSummary`
  - `GrowthFunnelSummary` 字段要求（最小可交付即可，但必须覆盖主 PRD 漏斗与“3 分钟激活”判断）：
    - 原始里程碑透传（便于审计）：
      - `installedAt: number`
      - `successfulCopyCount: number`
      - `firstPopupOpenedAt?: number`
      - `firstSuccessfulCopyAt?: number`
      - `firstPromptUsedAt?: number`
      - `reusedWithin7DaysAt?: number`
    - 派生布尔（便于一眼判断漏斗进度；不得写回 storage）：
      - `isPopupOpened: boolean`（`Boolean(firstPopupOpenedAt)`）
      - `isActivated: boolean`（以 `firstSuccessfulCopyAt` 存在为准；兜底可用 `successfulCopyCount > 0`）
      - `isPromptUsed: boolean`（`Boolean(firstPromptUsedAt)`）
      - `isReusedWithin7Days: boolean`（`Boolean(reusedWithin7DaysAt)`）
    - “3 分钟激活”指标（对齐主 PRD：新装用户 3 分钟内完成首次成功复制）：
      - `timeFromFirstPopupToFirstCopyMs?: number`（当 `firstPopupOpenedAt && firstSuccessfulCopyAt` 时计算；否则为 `undefined`）
      - `activatedWithin3MinutesFromFirstPopup?: boolean`（当且仅当 `timeFromFirstPopupToFirstCopyMs` 存在时输出；阈值固定为 `3 * 60 * 1000`）
  - 约束：
    - 严禁新增任何会落盘的字段（本需求只做“读取 + 派生 + 展示/导出”）
    - 严禁包含任何网页内容、URL、标题、复制内容；摘要仅允许时间戳/计数/布尔
    - 对异常/缺失值必须容错：任何非法/缺失时间戳时，相应派生字段为 `false/undefined`，不得抛错

2、Options 隐私页新增“本地漏斗摘要”面板（稳定 DOM、可 i18n、可复制导出）
  - UI 位置：`src/options/options.html` 的 `#privacy-tab` 中，建议放在 `#growth-stats-panel` 之前（便于先看摘要再看原始 JSON）
  - 形态：复用现有 `details/summary` 折叠面板样式（与“本地匿名事件日志/本地增长统计”一致），避免打扰普通用户
  - 稳定 DOM（用于用例与未来自动化）：
    - 面板：`#growth-funnel-panel`
    - 展示区（textarea 或 pre，只读）：`#growth-funnel-view`
    - 操作按钮：刷新 `#growth-funnel-refresh`、复制 `#growth-funnel-copy`
  - i18n：
    - 新增/补齐 `_locales/en/messages.json` 与 `_locales/zh/messages.json` 文案 key（至少包含：面板标题/说明、刷新/复制按钮文案、复制成功/失败 toast 文案）
    - 确保 `npm run check-i18n` 通过
  - 逻辑落地：`src/options/options.ts`
    - 新增读取与渲染函数（命名可微调但需在 PR 中固定）：
      - `readGrowthFunnelSummaryForDisplay(): Promise<GrowthFunnelSummary>`（内部 `await getGrowthStats()` + `buildGrowthFunnelSummary(stats, Date.now())`）
      - `refreshGrowthFunnelPanel(): Promise<GrowthFunnelSummary | null>`（渲染为 pretty JSON，末尾追加换行便于粘贴）
      - `copyGrowthFunnelSummaryToClipboard(): Promise<void>`（失败走 `fallbackCopyText` 兜底）
    - 初始化时机：在 `initialize()` 中与 `refreshTelemetryEventsPanel()/refreshGrowthStatsPanel()` 同层级调用一次 `refreshGrowthFunnelPanel()`（保证初次打开 Options 即可看到可审计结果）
    - 交互要求：
      - 点击刷新：重新读取并渲染
      - 点击复制：复制“规范化后的摘要 JSON”（不得复制整段 HTML）
    - 约束：
      - 面板操作不得写入任何 telemetry 事件（避免自举噪音）
      - 任意读取/渲染/复制失败不得影响 Options 其它功能；必须 try/catch 并降级（仅 `console.warn` + toast）
  - 样式：优先复用现有 `.telemetry-events-panel` 与 `.telemetry-events-view`；如需新增 CSS，只允许最小改动且放在 `src/options/options.css`，不得大改版

3、补齐单测与测试用例文档（可执行、可回归、可发布）
  - 单测（最小但必须）：
