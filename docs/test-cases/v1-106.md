# V1-106 Popup Pro Intent Prefill Funnel 测试用例

## 目标

验证本轮 `Copylot` 功能迭代已经把 Popup 的高意向 Pro 承接从“空白问卷”升级为“先选诉求，再带着预填信息进入问卷”，并且新增样本字段仍满足本地可审计、隐私安全的约束。

## 前置条件

- 扩展版本：`1.2.3`
- 相关模块：
  - `src/popup/popup.ts`
  - `src/options/options.ts`
  - `src/shared/pro-waitlist-survey-prefill.ts`
  - `src/shared/telemetry.ts`

## 用例 A：Popup 意向卡片可见且不影响主路径

步骤：
1. 打开任意真实 fixture 页面并打开 Popup。
2. 确认原有 `复制给AI`、`快捷键设置`、`Manage Prompts`、`升级 Pro / Pro 问卷 / 加入候补名单` 仍可见。
3. 确认新增意向卡片可见，且包含 4 个入口：
   - `更干净的正文 / Cleaner article copy`
   - `批量收集整理 / Batch collect and organize`
   - `高级 Prompt 工作流 / Advanced Prompt workflow`
   - `笔记工具导出 / Note-tool export`

预期：
- Popup 主复制路径和增长入口没有被挤掉或隐藏。
- 新增意向卡片只作为增强承接，不改动既有入口语义。

## 用例 B：Popup 意向卡片可深链到预填问卷

步骤：
1. 在 Popup 点击 `更干净的正文 / Cleaner article copy`。
2. 跳转到 `Options -> Pro -> #pro-waitlist-survey`。
3. 检查问卷字段：
   - `#pro-waitlist-survey-use-case`
   - `#pro-waitlist-survey-capability-advanced-cleaning`

预期：
- `use case` 已自动填入预设场景文案。
- `advanced_cleaning` 复选框已自动勾选。
- URL 在解析后会去掉 `pro_survey_prefill`，避免后续误归因。

## 用例 C：预填样本可写入本地可审计字段

步骤：
1. 通过 Popup 意向卡片进入预填问卷。
2. 点击 `复制反馈内容` 或 `复制后前往候补`。
3. 读取本地 `copilot_telemetry_events` 中的 `pro_waitlist_survey_copied`。

预期：
- 事件存在。
- `props.source === 'popup'`。
- `props.prefill_used === true`。
- `props.prefill_capability_count >= 1`。
- 不包含用户原始 use case、contact、网页 URL、标题或复制内容。

## 用例 D：统一门禁

执行命令：
- `npm run type-check`
- `npm run check-i18n`
- `node --no-warnings=ExperimentalWarning --loader=ts-node/esm scripts/unit-tests.ts`
- `npm run build:e2e`
- `COPYLOT_E2E_SKIP_BUILD=1 npx playwright test --config=playwright.config.ts --project=main e2e/popup-flow.spec.ts e2e/options-pro-flow.spec.ts`

预期：
- 全部通过。
- `options-pro-flow` 中的预填样本用例通过。
- `popup-flow` 中的 Popup 意向卡片深链用例通过。
