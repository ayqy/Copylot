# 问题修复和功能优化（V1-23 ASO 素材规范最小闭环：截图顺序/GIF 脚本/更新日志模板）

## 目标
在不新增权限、不引入任何联网分析、且不改动核心复制/Prompt/引导主流程的前提下，补齐 `prds/v1.md` 中“增长运营资产与商店转化优化（ASO）”尚未固化的交付物：沉淀可复用的 Chrome Web Store 截图顺序与脚本、GIF/短视频脚本、以及更新日志模板与发布前检查清单，使后续每次发版都能快速产出一致且真实可验证的商店素材，并按测试用例文档执行回归后达到可发布状态。

## 具体任务
1、固化 Chrome Web Store 截图顺序与素材规范（可直接复用、口径真实可验证）
  - 新增并维护文档：`docs/aso/store-assets.md`
  - 文档必须包含（至少）：
    - 截图顺序（建议 6-8 张，按“价值最强 -> 场景覆盖 -> 隐私与可信”排序），每张截图包含：
      - 截图标题（EN/ZH 各 1 条，避免夸大、避免未实现能力）
      - 要展示的真实功能点（必须可在仓库中验证，参考：`README.md`、`docs/roadmap.md`、`manifest.json`）
      - 拍摄步骤（从“干净浏览器 Profile/安装扩展/打开某个示例页面”到“点击哪个按钮/看到什么结果”）
      - 需要的开关前置（例如：输出格式、是否附加标题/URL、追加模式、默认 Chat、自动打开 Chat 等）
    - 统一口径的隐私提示（EN/ZH）：默认本地处理；不收集/不上传复制内容；“匿名使用数据”默认关闭，开启后仅本地记录匿名事件（不联网发送）
  - 约束：
    - 禁止把“未来能力/设想”写进截图文案（避免预期落差带来差评）
    - 禁止引导用户进行任何高打扰操作（例如强制弹窗、强制评价），只描述“用户点击触发”的真实体验

2、补齐 GIF/短视频脚本 + 更新日志模板（提升可持续发版效率）
  - 新增并维护 GIF/短视频脚本：`docs/aso/gif-script.md`
    - 至少包含 1 个 20-30 秒脚本（分镜/字幕/镜头切换），主线聚焦一个“最强场景”（建议二选一并固定）：
      - 场景 A：网页表格一键转 CSV/Markdown 并可直接粘贴到表格工具/Notion
      - 场景 B：追加模式跨页面收集资料（Shift 追加 + 分隔符 + 一次性粘贴）
    - 字幕/旁白文案必须提供 EN/ZH 两套，且与 `docs/aso/value-prop.md` 口径一致
  - 新增并维护 Chrome Web Store 更新日志模板：`docs/aso/cws-release-notes-template.md`
    - 必须包含固定结构（可复制粘贴到 CWS）：
      - Highlights（1-3 条，面向用户语言）
      - Fixes（避免过度技术化，但可定位）
      - Privacy（如涉及隐私/权限变更必须明确；无变更也要声明“无新增权限/无新增数据收集”）
      - Verification（内部用：必须要求 `bash scripts/test.sh` 通过）
    - 模板内容必须明确“禁止夸大/禁止写未实现能力”

3、测试/用例与发布检查闭环（可执行、可回归）
  - 新增并维护测试/用例文档：`docs/test-cases/v1-23.md`
  - 文档必须覆盖（至少）：
    - 用例 A（素材规范完整性）：`docs/aso/store-assets.md` 包含截图顺序（>=6 张）+ 每张截图的拍摄步骤 + 开关前置说明 + EN/ZH 文案
    - 用例 B（GIF 脚本可执行）：按 `docs/aso/gif-script.md` 的步骤能完成一次录制/截帧，且脚本描述的每个 UI/结果均可在当前版本中复现
    - 用例 C（更新日志模板可复用）：`docs/aso/cws-release-notes-template.md` 可直接复制粘贴，且包含隐私/验证段落；不包含未实现能力
    - 用例 D（口径一致性抽检）：随机抽检 3 条文案，确认与 `README.md`/`docs/aso/value-prop.md` 不冲突，且不出现“联网分析/上传内容”等违背隐私原则的表述
    - 自动化回归：记录一次 `bash scripts/test.sh` 结论（命令 + 通过/失败）
