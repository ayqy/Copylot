# V1-15 Options 补齐“重新查看新手引导”入口：从 Options 一键打开 Popup 三步引导（popup.html#onboarding）测试用例

## 目标
验证 V1-15 达到可发布：
- Options（`src/options/options.html`）在 `#prompts-tab` 顶部新增稳定入口：
  - 容器：`#options-onboarding-panel`
  - 按钮：`#options-onboarding-open`
- 点击按钮后在新 Tab 打开 `src/popup/popup.html#onboarding`，并强制展示 3 步引导（可跳过/可完成/可回看）
- Popup 仅在 deeplink 命中时强制打开；不携带 `#onboarding` 时仍按 `shouldAutoShowOnboarding(currentSettings)` 自动展示（仅未完成状态）
- `bash scripts/test.sh` 全量通过

## 预置条件
- 已在 Chrome/Chromium 加载本地扩展（开发包或生产包均可）
- 可打开 Options 页面（扩展详情页 -> 扩展选项）

## 用例 A（入口可见：稳定 DOM + 可点击）
步骤：
1. 打开 Options 页面（默认进入 Prompt 管理 Tab，即 `#prompts-tab`）。
2. 检查 `#options-onboarding-panel` 是否可见且不遮挡其它操作。
3. 检查 `#options-onboarding-open` 是否可见且可点击。

期望：
- `#options-onboarding-panel` 默认可见（稳定入口、低打扰）。
- 点击按钮不会影响 Options 其它功能（如搜索、新建 Prompt、导入导出等）。

## 用例 B（一键打开：新 Tab + URL + 引导自动展示 1/3）
步骤：
1. 在 Options 的 `#prompts-tab` 点击 `#options-onboarding-open`。
2. 检查是否打开新 Tab。
3. 检查新 Tab 的地址栏 URL 是否为：
   - `chrome-extension://<extension_id>/src/popup/popup.html#onboarding`
4. 检查引导 Modal 是否自动展示：
   - 容器：`#popup-onboarding-modal` 可见
   - 进度：`#popup-onboarding-progress` 文案为 `1/3`

期望：
- 新 Tab 打开成功，URL 命中 `.../src/popup/popup.html#onboarding`。
- 引导自动展示且从第 1 步开始（1/3）。

## 用例 C（可回看：已完成后仍可强制展示）
步骤：
1. 按用例 B 进入 `popup.html#onboarding`，将引导走完（点击“下一步”直到“完成”）或点击“跳过”关闭。
2. 再次回到 Options -> `#prompts-tab`，再次点击 `#options-onboarding-open`。
3. 在新打开的 Tab 中检查引导是否仍会自动展示，并且进度为 `1/3`。

期望：
- 即使已完成/已跳过，引导仍会被 deeplink 强制展示（证明“回看”不受 completed 状态限制）。

## 用例 D（交互回归：引导关闭方式与 Popup 其它功能不受影响）
步骤：
1. 在 `popup.html#onboarding` 的引导 Modal 中依次验证：
   - “上一步/下一步”可用，且步骤切换正确（1/3 -> 2/3 -> 3/3）
   - “完成”可关闭引导
   - “跳过”可关闭引导
   - 按 `ESC` 可关闭引导
   - 点击遮罩（Modal 外部）可关闭引导
2. 关闭引导后，随便切换一个 Popup 设置（如输出格式、附加标题等），再关闭页面并重新打开 Popup（不带 `#onboarding`）。

期望：
- 所有关闭方式均可用，且不影响其它按钮/设置保存。

## 用例 E（状态可审计：completedVersion 与 version 关系正确，且后续不会自动弹出）
步骤：
1. 在引导完成或跳过后，在 `popup.html` 页面打开 DevTools（Console）。
2. 执行以下命令检查设置字段：
   - `chrome.storage.sync.get('copilot_settings', (r) => console.log(r.copilot_settings.popupOnboardingCompletedVersion, r.copilot_settings.popupOnboardingVersion))`
3. 再次打开 Popup（不带 `#onboarding`，例如点击扩展图标打开默认 Popup，或打开 `chrome-extension://<id>/src/popup/popup.html`）。

期望：
- `popupOnboardingCompletedVersion >= popupOnboardingVersion`（通常完成后两者相等）。
- 后续打开 Popup（不带 `#onboarding`）不会自动弹出引导（除非手动清空配置/重置 storage）。
