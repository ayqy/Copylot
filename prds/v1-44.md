## 具体任务
1、评分引导触发优化（更早但更精准，低打扰）
  - 调整 `shouldShowRatingPrompt(stats, now)` 触发条件（仅本地、最多展示 1 次，仍保留 `later/never` 按钮但不二次打扰）：
    - 安装时长门槛：从 72h 下调到 48h（`RATING_PROMPT_MIN_INSTALL_AGE_MS`）
    - 成功复制次数门槛：从 20 次下调到 10 次（`RATING_PROMPT_MIN_SUCCESSFUL_COPY_COUNT`）
    - 精准性补充：满足以下任一条件才触发（避免“刚装就烦”）：
      - 已发生过一次 Prompt 使用（`firstPromptUsedAt` 存在）
      - 或成功复制次数 >= 20（对重度纯复制用户保持覆盖）
  - 优化评分引导文案（i18n，中英文一致口径）：更强调“帮助我们获得更多曝光/自然增长”，避免夸大承诺；不出现任何“Pro 已上线/付费能力已实现”的暗示。

2、商店/评价链接 UTM 更新为 v1-44（可核验归因）
  - 将所有打开 Chrome Web Store 详情页/评价页的入口（Popup/Options/评分引导/复制分享文案内的链接）统一切换为 `utm_campaign=v1-44`（保留 `utm_source=copylot-ext`，并继续使用 `utm_medium=popup|options|rating_prompt` 区分入口）。
  - 确保不引入新增权限；所有链接在新标签页可正常打开。

3、量化证据、用例与回归（可执行、可审计、可对比）
  - 新增测试用例文档（固定路径）：`docs/test-cases/v1-44.md`
    - 覆盖：评分引导触发阈值（含 Prompt 用户与非 Prompt 用户两条路径）、三类 `utm_medium` 的 UTM 参数校验、匿名开关开/关下的 WOM 摘要/证据包导出行为、以及一次 `bash scripts/test.sh` 回归记录（日期 + PASS）。
  - 补齐自动化断言（单元测试/集成回归）：更新 `scripts/unit-tests.ts` 覆盖新阈值与 `utm_campaign=v1-44`，并确保 `bash scripts/test.sh` 全量回归通过。
  - 落盘本轮可验证商业化证据资产目录（固定路径）：`docs/evidence/v1-44/`
    - 至少包含：`wom-summary.json`、`wom-evidence-pack.json`、telemetry off 对照导出、截图索引 `index.md`（含版本号/时间/断言与对比口径说明：与 `docs/evidence/v1-43/` 的可比字段）。
