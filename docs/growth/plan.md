# Copylot 增长执行计划（无表单版）

更新时间：2026-07-03

## 一句话目标

把 Copylot 的主增长链路固定为：

`官网 / 商店曝光 -> 安装 -> 第一次成功复制 -> 第一次复用 -> 第二次回访 -> 分享 / 评价 -> 带来新的安装`

当前阶段完全不做表单、问卷、候补收集。

## 本期北极星

- 北极星：`7 日内成功复制 >= 2 次的活跃用户数`
- 第一优先级：
  - 新安装用户首次 `copy_success` 转化率
  - 首次 `copy_success` 后 7 日内再次 `copy_success` 的比例
- 第二优先级：
  - `prompt_used` 占活跃用户比例
  - `wom_share_copied`
  - `wom_rate_opened`
- 第三优先级：
  - `pro_entry_opened`
  - `pro_waitlist_opened`
  - `pro_distribution_asset_copied`

## 经营原则

1. 安装前讲价值，不讲留资。
2. 安装后先让用户省时间，再请求分享或评价。
3. Pro 当前只做路线说明，不做表单、问卷、候补、支付。
4. 所有新动作都必须直接服务于激活、复用或口碑。

## 30 天动作

### A. 安装承接
- 官网首屏只做三件事：
  - 解释网页复制为什么会返工
  - 说明安装后 30 秒能完成什么
  - 直接给出安装与上手入口
- 商店与官网文案统一，不再使用企业支付模板式话术。
- 补齐隐私页，明确“本地处理、默认不上传、匿名事件默认关闭”。

### B. 首次激活
- Popup 主路径只保留：
  - 复制给 AI
  - 打开 Prompt 管理
  - 打开上手和快捷键设置
- 首次上手内容重点覆盖：
  - 长文复制
  - 代码块清理
  - 表格转 Markdown / CSV
  - Shift 追加模式

### C. 第二次复用
- 重点打磨：
  - Prompt 槽位
  - 表格转换
  - 追加模式
  - 来源附带
- 评估标准只有一个：它是否让用户第二天还会再次打开插件。

### D. 口碑循环
- 分享：默认复制官网链接、商店安装链接和分享文案。
- 评价：在用户完成成功使用后再提示去商店评价。
- 反馈：继续保留 GitHub 反馈入口，但不进入主路径。

### E. Pro 路线
- Pro 当前只验证三类增强方向：
  - 高级页面清洗
  - 批量采集与整理
  - 笔记工具 / Prompt 工作流导出
- 路线说明页只承担“解释未来增强方向”，不承接支付或留资。

## 停止做的事情

- 不再新增 Popup 问卷、表单、候补入口
- 不再把“你最想先解决什么”作为默认增长动作
- 不再把 `/pricing` 当作收款页
- 不再用问卷样本数代替真实激活、复用和口碑指标
