# V1-17 Prompt 管理器排序能力（最常用 / 最近使用）+ 修正“最近使用”时间显示 测试用例

## 目标
验证 V1-17 达到可发布：
- Options -> Prompts Tab 工具条新增排序下拉框（DOM：`#prompt-sort-select`），支持 `default/most_used/recent_used`
- 切换排序后列表顺序立即生效，且规则确定可回归
- `recent_used` 严格按 `lastUsedAt` 倒序：使用过的 Prompt 在前，未使用过（无 `lastUsedAt`）在末尾
- Prompt 卡片底部“最近使用”不再误用 `createdAt`：已使用展示 `lastUsedAt` 的相对时间；未使用展示 i18n 占位文案 `promptNeverUsed`
- 刷新 Options 页面后排序模式保持（localStorage：`copylot_prompt_sort_mode`）
- `bash scripts/test.sh` 全量通过

## 预置条件
- 已在 Chrome/Chromium 加载本地扩展（开发包或生产包均可）
- 能打开 Options 页面（`src/options/options.html`）
- 至少存在 3 个可见 Prompt（可通过新建或导入构造）

## 用例 A（控件可见：可切换三种模式）
步骤：
1. 打开 Options -> Prompts Tab。
2. 在工具条 `.toolbar .filter-options` 内找到排序下拉框（DOM：`#prompt-sort-select`）。
3. 依次切换 `default / most_used / recent_used`。

期望：
- `#prompt-sort-select` 可见且可用。
- 每次切换后 Prompt 列表立即按对应规则重新排序（无需刷新页面）。

## 用例 B（最常用排序：usageCount 倒序 + title 稳定次排序）
步骤：
1. 构造 3 个 Prompt：标题分别为 `Alpha`、`Beta`、`Gamma`（标题仅用于验证稳定次排序）。
2. 在任意网页触发 Prompt 使用（右键菜单或悬浮按钮均可）：
   - 使用 `Beta` 3 次
   - 使用 `Alpha` 1 次
   - `Gamma` 不使用
3. 回到 Options -> Prompts Tab，切换排序为 `most_used`。

期望：
- 列表顺序为：`Beta`（usageCount=3）在前，`Alpha`（usageCount=1）其后，未使用过的 `Gamma` 在末尾。
- 若存在 usageCount 相同的多个 Prompt，其相对顺序按标题（忽略大小写）升序稳定排序。

## 用例 C（最近使用排序：lastUsedAt 倒序 + 未使用的在末尾）
步骤：
1. 构造 3 个 Prompt：`P1`、`P2`、`P3`（其中至少保留一个“从未使用过”的 Prompt，例如 `P3`）。
2. 在任意网页依次使用：
   - 先使用 `P1` 一次
   - 再使用 `P2` 一次（确保 `P2.lastUsedAt > P1.lastUsedAt`）
3. 回到 Options -> Prompts Tab，切换排序为 `recent_used`。

期望：
- `P2` 置顶，`P1` 次之（按 `lastUsedAt` 倒序）。
- 未使用过（无 `lastUsedAt`）的 Prompt（如 `P3`）必须排在列表末尾。

## 用例 D（时间显示正确：已使用=lastUsedAt，相对时间；未使用=promptNeverUsed）
步骤：
1. 确保存在一个“已使用过”的 Prompt（存在 `lastUsedAt`）和一个“未使用过”的 Prompt（无 `lastUsedAt`）。
2. 打开 Options -> Prompts Tab，观察两张卡片底部右侧时间区域（DOM：`.prompt-card .last-used`）。

期望：
- 已使用 Prompt：显示 `lastUsedAt` 对应的相对时间（例如“几分钟前/几小时前/日期时间”）。
- 未使用 Prompt：显示 i18n 文案 `promptNeverUsed`（而不是使用 `createdAt` 冒充“最近使用”）。

## 用例 E（选择记忆：刷新后排序模式保持）
步骤：
1. 在 Options -> Prompts Tab 将 `#prompt-sort-select` 切换为 `most_used` 或 `recent_used`。
2. 刷新 Options 页面（重新加载页面）。

期望：
- 刷新后 `#prompt-sort-select` 仍保持上一次选择的模式（localStorage：`copylot_prompt_sort_mode`）。
