# 问题修复和功能优化（V1-50 Pro 意向验证节奏固化：每周“意向证据摘要”一键生成 + 问卷模板渠道化）

## 目标
基于 `docs/roadmap_status.md` 的收入优先 Top3，本轮由于 Top1「真实 CWS 发布 + 商店端取证」与 Top2「真实发布后跑数取证 + 24h 复盘」仍受“网络可达性/商店可达”阻塞，按规则顺延选择 Top3「基于问卷留资证据推进 Pro 下一步验证（离线可推进）」作为 1~3 天最小可交付增量主题。

收入第一：把现有本地匿名事件与 Pro 漏斗取证能力（`pro_waitlist_survey_copied/pro_waitlist_opened/pro_waitlist_copied` 等）进一步沉淀为“每周可复制的意向证据摘要”与“可复用的问卷/分发模板”，让付费意向与留资动作在真实发布被阻塞期间仍能持续获客、持续取证、持续复盘，并在网络恢复后可直接接入真实商店跑数对比（不新增权限、不引入支付/订阅、不联网发送数据、不采集用户复制内容）。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、每周“Pro 意向证据摘要（7d window）”一键生成（可复制、可审计、可复盘）
  - 在 Options -> 隐私页（增长统计/证据导出区域）新增一键动作：`复制本周 Pro 意向证据摘要`（建议稳定 DOM：`#copy-pro-intent-weekly-digest`）。
  - 摘要内容格式（Markdown，便于粘贴到 GitHub Issue/邮件/社群）至少包含：
    - 时间窗：过去 7 天（本地时间，写清 `from/to`）
    - 关键事件计数（overall，必要时含 bySource）：`pro_entry_opened/pro_waitlist_opened/pro_waitlist_copied/pro_waitlist_survey_copied`
    - 关键转化率（可从摘要直接验证）：`waitlist_opened_per_entry_opened`、`waitlist_copied_per_waitlist_opened`、`survey_copied_per_entry_opened`（若分母为 0 则输出 `null`/`N/A`）
    - 环境信息：extension version、exportedAt、匿名开关状态
    - 隐私声明：不包含网页内容/复制内容/URL/标题；仅来源于本地匿名事件统计与扩展环境信息
  - 隐私与开关行为：
    - 匿名开关 OFF：不读取/不推断历史，不生成“伪统计”；摘要应明确提示“匿名使用数据关闭（无可用事件）”，并仅输出环境信息与说明
    - 匿名开关 ON：仅使用本地 telemetry（白名单 + FIFO window）计算 7d 统计；不新增任何网络请求

2、问卷/摘要模板渠道化（可转化获客增长，形成可复用运营资产）
  - 新增并固化 2 份可复用模板（固定路径）：
    - `docs/monetization/pro-waitlist-survey-template.md`：对外收集 Pro 需求/定价意向的“最小问卷模板”（可复制到 Issue/邮件/社群）
    - `docs/growth/pro-intent-weekly-digest-template.md`：每周意向证据摘要的“复盘模板”（包含指标口径、粘贴区、结论区、下一步假设）
  - 模板要求：
    - 与产品隐私边界一致：明确不收集用户复制内容；建议用户不要粘贴敏感信息
    - 与现有事件口径一致：模板字段与 `docs/growth/telemetry-events.md`、Options -> 隐私页导出字段可一一对应（避免“写了但无法验证”）
  - 形成“可量化证据”的最小动作链路说明（写入模板末尾）：如何触发事件 -> 如何导出/复制摘要 -> 如何落盘证据索引

3、补齐测试/用例与证据落盘（确保可执行、可回归、可用于商业化取证）
  - 新增测试用例文档（固定路径）：`docs/test-cases/v1-50.md`
    - 覆盖：匿名开关 OFF/ON 两种；7d window 边界（跨天/无事件/有事件）；摘要字段断言（时间窗、计数、转化率、隐私声明）
  - 自动化测试加固（优先纯函数单测）：覆盖“7d 过滤 + 统计 + 转化率”核心逻辑稳定性；并保证 `bash scripts/test.sh` 全量通过
  - 新增证据目录与索引（固定路径）：`docs/evidence/v1-50/` + `docs/evidence/v1-50/index.md`
    - 落盘：复制出的「本周 Pro 意向证据摘要」示例（ON/OFF 各 1 份）+ 截图索引 + 导出文件清单与命名规范
  - 新增交付简报（固定路径建议）：`docs/reports/v1-50-report.md`

## 验收标准
* 修复上述所有问题，并测试通过，达到可发布状态
* 实现以上所有需求，并测试通过，达到可发布状态
* 可验证的商业化推进证据（必须可审计/可导出/可复用）：
  - Options -> 隐私页存在“复制本周 Pro 意向证据摘要（7d window）”入口与稳定 DOM；复制内容为 Markdown，包含时间窗、关键事件计数、关键转化率、环境信息与隐私声明
  - 匿名开关 OFF：摘要明确提示“匿名使用数据关闭（无可用事件）”，且不读取/不推断历史；复制内容不包含事件统计明细
  - 匿名开关 ON：摘要统计仅基于本地 telemetry（7d window），且可与 Options -> 隐私页导出的「Pro 意向漏斗摘要/证据包」口径互相印证
  - 模板可复用可验证：`docs/monetization/pro-waitlist-survey-template.md` 与 `docs/growth/pro-intent-weekly-digest-template.md` 可直接复制到外部渠道使用，且字段与事件口径一致
  - 证据落盘可复核：`docs/evidence/v1-50/index.md` 至少包含（1）摘要示例（ON/OFF）与截图索引（2）导出文件清单（3）最小复盘口径（本周问卷复制次数/候补打开次数/候补复制次数/入口打开次数与对应转化率）
