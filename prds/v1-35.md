# 问题修复和功能优化（V1-35 全页转换正文优先：语义 main/article 作为默认根节点 + 用例/回归闭环）

## 目标
在不新增任何扩展权限、不新增任何联网发送、且不改变既有“选区转换/智能区块复制/表格转换/Prompt 选区优先”等主流程行为的前提下，优化“整页转换（Convert Page）/整页 Prompt（PROCESS_PAGE_*）”的默认处理根节点：当页面存在语义明确且足够大的正文容器（`<main>` / `<article>` / `[role="main"]`）时，默认仅处理该正文容器而非 `document.body`，从而显著减少导航/页脚/侧边栏等噪音进入复制结果，提升新用户首次成功复制质量与可复用性，并通过可执行测试用例与自动化回归验证达到可发布状态。

## 具体任务
1、为“整页处理”增加语义正文根节点选择（仅在输入根为 `document.body` 时生效）
  - 修改：`src/shared/content-processor.ts`
  - 在 `processContent(root, settings)` 内增加一个极小且可审计的“整页根节点选择”逻辑（不引入新依赖）：
    - 仅当 `root.tagName === 'BODY'` 时尝试选择正文容器；其它场景（选区 fragment、点击块、表格元素等）完全不受影响
    - 候选选择器：`main, article, [role="main"]`
    - 选择策略（MVP，避免误选小容器）：
      - 计算每个候选元素的 `innerText.trim().length`（或等价稳定指标）
      - 取文本长度最大的候选作为 `bestCandidate`
      - 仅当 `bestCandidateTextLen >= 200`（阈值可在实现中定义为常量，便于未来调整）时，才用 `bestCandidate` 替代 `root` 进入后续 `createVisibleClone()` 与转换链路
      - 若无候选/候选过小：保持现状，继续处理 `document.body`
  - 行为约束：
    - 不新增任何用户配置项（先以“默认改进整页质量”为目标）
    - 不新增任何持久化字段；不新增任何可能包含网页内容/URL/标题的落盘
    - 不改变 `CONVERT_PAGE_WITH_SELECTION`、`PROCESS_SELECTION_WITH_PROMPT`、悬浮按钮 Prompt（块级/选区优先）等路径的根节点判定逻辑

2、补齐 DOM 回归用例（自动化可执行，验证“正文优先”确实生效且无回归）
  - 新增用例 HTML：`test/cases/semantic-main-content.html`
    - 页面必须同时包含：`<header>/<nav>/<aside>` 噪音区域 + `<main>` 或 `<article>` 正文区域
    - 正文区域文本应足够长（确保命中文本阈值），且噪音区域包含“可被错误复制”的干扰文本，便于断言
  - 更新 DOM runner manifest：`test/test-manifest.json`
    - 新增一条用例，期望输出仅包含正文区域内容（不得包含 nav/aside 的干扰文本）
  - 回归要求：
    - `bash scripts/test.sh` 必须全量通过（包含 unit-tests + build:prod + verify-prod-build）

3、补齐测试/用例文档闭环（让验收可执行、可审计）
  - 新增测试用例文档：`docs/test-cases/v1-35.md`
  - 文档必须覆盖（至少）：
    - 用例 A（整页转换正文优先）：在“有 `<main>/<article>` 的文章页”无选区触发 Convert Page，结果不包含 header/nav/aside 的干扰文本
    - 用例 B（整页 Prompt 正文优先）：无选区通过右键 Prompt 触发 `PROCESS_PAGE_WITH_PROMPT`，结果同样仅包含正文（并正确拼装 Prompt）
