# Roadmap 状态

## 当前阶段

- 阶段名称：S0 / G0 `安装与激活面重构`
- 阶段目标：把官网、商店、Popup、Options 全部统一回 `安装 -> 首次成功复制 -> 再次复用 -> 分享 / 评价` 这条增长闭环。
- 阶段反思：前一轮把 Popup 与 Pro 链路导向问卷/候补，直接稀释了“安装后尽快完成第一次干净复制”的核心路径。
- 优化方向：优先把官网首页、`/pricing`、`/privacy` 变成真实安装承接面，并同步清掉扩展内与文档中的旧命名、旧 CTA 和旧投放口径。
- 经营边界：当前版本不做表单、问卷、候补收集，也不做支付承接。

## 当前进度

- 阶段进度：S0 已完成 `2/6`；全局已完成 `2/18`。
- 已完成事项：
  - 已回退 Popup 中“你最想先解决什么”与相关预填问卷链路。
  - 已把 Options Pro 页调整为“路线说明 + 分享工具”，不再承接表单或候补。
- 进行中事项：
  - 官网首页、`/pricing`、`/privacy` 正在切换到安装承接与信任承接口径。
  - 扩展内剩余显性表单/问卷/候补入口、旧命名与增长文档口径仍需统一清理。
- 当前经营判断：
  - Chrome Web Store 已公开可安装，官网与商店可以并行承担获客与转化承接。
  - 本轮最小可交付增量必须优先服务真实获客转化，而不是继续收集需求文本。

## 下一步最重要的 3 件事（收入优先）

1. 完成官网首页、`/pricing`、`/privacy` 的安装承接与信任承接改造，把所有 CTA 统一为安装、上手、隐私与分享。
2. 清理扩展内剩余显性表单、问卷、候补入口与旧命名，把 Pro 入口统一为路线说明与分享/评价承接。
3. 基于当前已上架状态并行开始增长循环：同步无表单版测试口径、投放包、商店/官网截图基线与漏斗证据导出路径。

## 阻塞与需要的人类输入

- 当前 Top1 不依赖账号、凭据、支付或上架权限，暂无阻塞。
- 实施官网改造时，站点页面需落在实际站点工程中；当前扩展仓可直接同步的基线包括 `src/shared/external-links.ts`、`e2e/options-pro-flow.spec.ts`、`e2e/popup-growth-flow.spec.ts`、`docs/test-cases/` 与增长文档。
