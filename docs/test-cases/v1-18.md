# V1-18 本地漏斗指标补齐：GrowthStats 记录关键里程碑时间戳 测试用例

## 目标
验证 V1-18 达到可发布：
- `copilot_growth_stats` 扩展后仍可在 Options -> 隐私页的“本地增长统计”面板（`#growth-stats-panel`）正确展示与复制导出
- 打开 Popup 至少 1 次后写入 `firstPopupOpenedAt`，且后续不覆盖
- 任意成功复制后：
  - `successfulCopyCount` 正确 +1
  - `firstSuccessfulCopyAt` 仅在首次成功复制写入
  - `lastSuccessfulCopyAt` 每次成功复制都会更新
- 当第二次成功复制发生在 `firstSuccessfulCopyAt` 后 7 天内时写入 `reusedWithin7DaysAt`（仅一次）；超过 7 天不写入
- 当通过 Prompt 触发且复制成功时写入 `firstPromptUsedAt`（仅一次）；普通复制不写入该字段
- 隐私校验：`copilot_growth_stats` 仅包含时间戳/计数/枚举；不包含网页内容、URL、标题、复制内容
- `bash scripts/test.sh` 全量通过

## 预置条件
- 已在 Chrome/Chromium 加载本地扩展（开发包或生产包均可）
- 能打开 Options 页面（`src/options/options.html`）
- 可在 Options -> 隐私页找到“本地增长统计”面板：
  - 重置按钮：`#growth-stats-reset`
  - 刷新按钮：`#growth-stats-refresh`
  - 展示区域：`#growth-stats-view`
- Prompt 用例需至少存在 1 个 Prompt（Options -> Prompts Tab 新建即可）

## 用例 A（首次打开 Popup：firstPopupOpenedAt 写入且不覆盖）
步骤：
1. 打开 Options -> 隐私页，点击 `#growth-stats-reset` 重置。
2. 点击 `#growth-stats-refresh`，确认 `successfulCopyCount=0` 且无 `firstPopupOpenedAt`（或为 `undefined`）。
3. 点击扩展图标打开 Popup，然后关闭 Popup。
4. 回到 Options -> 隐私页，点击 `#growth-stats-refresh`。
5. 记录 `firstPopupOpenedAt` 的值；再次打开/关闭 Popup 1 次。
6. 再次 `#growth-stats-refresh`。

期望：
- 第 4 步刷新后：`firstPopupOpenedAt` 已写入且为毫秒时间戳（`number`，>0）。
- 第 6 步刷新后：`firstPopupOpenedAt` 保持与第 5 步一致（不被覆盖）。

## 用例 B（首次成功复制：firstSuccessfulCopyAt/lastSuccessfulCopyAt/count）
步骤：
1. 在 Options -> 隐私页点击 `#growth-stats-reset` 重置。
2. 打开任意普通网页（非 `chrome://` 页面），右键页面，点击菜单项“Convert Page to AI-Friendly Format”（会将内容复制到剪贴板）。
3. 回到 Options -> 隐私页点击 `#growth-stats-refresh`，查看 `#growth-stats-view`。

期望：
- `successfulCopyCount = 1`
- `firstSuccessfulCopyAt` 已写入
- `lastSuccessfulCopyAt` 已写入，且与 `firstSuccessfulCopyAt` 相等或非常接近（同一次复制）

## 用例 C（二次成功复制：7 天内复用写入 reusedWithin7DaysAt）
步骤：
1. 延续用例 B 的状态（当前 `successfulCopyCount=1`）。
2. 在同一网页再次执行一次“Convert Page to AI-Friendly Format”成功复制。
3. 回到 Options -> 隐私页点击 `#growth-stats-refresh`。

期望：
- `successfulCopyCount = 2`
- `firstSuccessfulCopyAt` 不变（仍为首次复制时间）
- `lastSuccessfulCopyAt` 更新为第二次复制时间
- `reusedWithin7DaysAt` 已写入（仅一次），且等于第二次复制的时间戳（通常与 `lastSuccessfulCopyAt` 相等）

## 用例 D（二次成功复制：超过 7 天不写入 reusedWithin7DaysAt）
说明：通过手动构造本地存储时间戳，模拟“首次复制发生在 8 天前，当前为第二次复制”。

步骤：
1. 打开 `chrome://extensions`，找到该扩展的 Service Worker（或 Background）并点击“Inspect”打开控制台。
2. 在控制台执行（会将 growth stats 调整为：已成功复制 1 次，且首次复制发生在 8 天前）：

```js
const key = 'copilot_growth_stats';
const days = 24 * 60 * 60 * 1000;
const firstAt = Date.now() - 8 * days;
const existing = (await chrome.storage.local.get(key))[key] || {};
await chrome.storage.local.set({
  [key]: {
    ...existing,
    installedAt: existing.installedAt || Date.now() - 9 * days,
    successfulCopyCount: 1,
    firstSuccessfulCopyAt: firstAt,
    lastSuccessfulCopyAt: firstAt
  }
});
```

3. 回到任意普通网页，再次执行一次“Convert Page to AI-Friendly Format”成功复制（此时视为第 2 次复制）。
4. 回到 Options -> 隐私页点击 `#growth-stats-refresh`。

期望：
- `successfulCopyCount = 2`
- `reusedWithin7DaysAt` 不存在或为 `undefined`（不应写入）

## 用例 E（Prompt 复制成功：firstPromptUsedAt 写入；普通复制不写入）
步骤：
1. 在 Options -> 隐私页点击 `#growth-stats-reset` 重置。
2. 先执行一次普通复制（右键“Convert Page to AI-Friendly Format”），然后 `#growth-stats-refresh`。
3. 确保当前 `firstPromptUsedAt` 不存在或为 `undefined`。
4. 在同一网页右键，进入“Magic Copy with Prompt”子菜单，选择任意 Prompt 执行一次复制成功。
5. 回到 Options -> 隐私页点击 `#growth-stats-refresh`，记录 `firstPromptUsedAt`。
6. 再重复一次 Prompt 复制成功，然后再次刷新查看。

期望：
- 第 2-3 步：普通复制不会写入 `firstPromptUsedAt`
- 第 5 步：Prompt 复制成功后，`firstPromptUsedAt` 写入且为毫秒时间戳（`number`，>0）
- 第 6 步：再次 Prompt 复制成功不会覆盖 `firstPromptUsedAt`（保持第一次的值）

## 用例 F（隐私校验：仅时间戳/计数/枚举）
步骤：
1. 在 Options -> 隐私页点击 `#growth-stats-refresh`，查看 `#growth-stats-view` 的 JSON 内容。
2. 检查所有字段：
   - 允许：时间戳（number）、计数（number）、枚举（`ratingPromptAction` 的 `'rate' | 'later' | 'never'`）
   - 禁止：任何网页内容、URL、标题、复制内容、用户输入文本等（例如出现 `http`、长文本段落、HTML 片段）

期望：
- `copilot_growth_stats` 不包含任何网页/复制内容、URL、标题或用户输入文本字段。
