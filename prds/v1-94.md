# 问题修复和功能优化（V1-94 Pro 意向样本量提升：问卷区“计入样本”提示 + 一键开启匿名使用数据 + 证据回归）

## 目标
在不引入支付/订阅、不新增权限、不上传用户复制内容的前提下，优先解决「Pro 候补问卷样本可计入率」问题：当用户处于匿名使用数据 OFF 状态时，在 `#pro-waitlist-survey` 明确提示“本次不计入样本”并提供“一键开启并计入样本”入口，使 `survey_intent = count(pro_waitlist_survey_copied)` 的可计入样本量可持续提升，且可通过 v1-90 证据包与漏斗摘要稳定复核，为 S4「订阅 MVP go/no-go」提供可审计、可复盘的商业化证据输入。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、在 Options 的问卷区补齐“计入样本”提示 + 一键开启匿名使用数据（提升可计入率）
  - 修改：`src/options/options.html`、`src/options/options.ts`、`_locales/*/messages.json`
  - 交互要求（透明、可逆、零打扰）：
    - 当「匿名使用数据」开关为 OFF 时，在 `#pro-waitlist-survey` 区域展示提示（i18n 文案）：
      - “未开启匿名使用数据：本次填写/复制不会计入 `survey_intent` 统计（仍可继续填写）”
    - 提示内提供 1 个按钮（i18n 文案）：
      - “开启并计入样本”（点击后仅修改本地设置为 ON，并立刻刷新相关面板）
    - 当开关为 ON 时隐藏该提示与按钮（避免噪音）
  - 隐私与门禁：
    - 不得新增权限；不得联网发送；不得记录/导出 URL/标题/网页内容/复制内容/联系方式明文
    - 仅复用现有匿名开关逻辑：OFF 时不记录/不缓存/不补发，且关闭会清空本地匿名事件日志

2、漏斗口径补齐 + 证据包可复核（让“样本可计入率提升”可被量化验证）
  - 修改：`src/shared/pro-funnel.ts`、`src/options/options.ts`、`scripts/unit-tests.ts`
  - 漏斗口径补齐（稳定字段）：
    - 在 `ProFunnelSourceStats.rates` 新增稳定字段：
      - `survey_copied_per_prompt_shown = pro_waitlist_survey_copied / pro_prompt_shown`
    - 分母为 0 时该字段必须为 `null`（避免 NaN/Infinity 与口径误读）
    - Options -> 隐私与可观测性 ->「Pro 意向漏斗摘要」与 v1-90 的证据包导出中必须可观察该字段
  - 自动化门禁（必须可执行）：
    - 更新/新增单测：`scripts/unit-tests.ts`
      - 断言 `survey_copied_per_prompt_shown` 字段存在、类型稳定、且在分母为 0 时为 `null`
    - `bash scripts/test.sh` 全量 PASS（若 `check-i18n` 误报复现，优先修复直到通过）

3、补齐测试用例文档与回归记录（确保可发布、可复盘）
  - 新增测试用例文档（固定路径）：`docs/test-cases/v1-94.md`
    - 必须覆盖（离线可复现）：
      - 匿名开关 OFF：问卷区出现“不计入样本”提示与“开启并计入样本”按钮
      - 点击“一键开启”后：开关变为 ON，提示隐藏；面板刷新无残留旧口径
      - 在匿名开关 ON 并完成 1 次问卷复制后：导出 v1-90 证据包并验证 `survey_intent >= 1`，且包含新增漏斗字段
    - 必须包含一次回归记录（PASS/BLOCKED + 导出文件名/截图索引）
