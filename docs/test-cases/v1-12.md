# V1-12 隐私页匿名事件日志可视化与一键导出（可审计）测试用例

## 目标
验证 V1-12 达到可发布：
- Options -> 隐私与可观测性（`#privacy-tab`）新增“本地匿名事件日志”面板：可刷新/复制/清空
- 面板展示数据严格为 `copilot_telemetry_events` 的白名单过滤结果：仅 `name/ts/props(白名单)`
- “匿名使用数据”开关关闭：不记录、关闭即清空、不缓存、不补发
- `bash scripts/test.sh` 全量通过

## 预置条件
- 已在 Chrome/Chromium 加载本地扩展（开发包或生产包均可）
- 可打开 Options 页面

## 用例 A（开关关闭：不记录 + 面板可见）
前置：
- Options -> 隐私与可观测性：`匿名使用数据` 开关为 **关闭**

步骤：
1. 在 Options -> 隐私与可观测性找到“本地匿名事件日志”面板（默认折叠），展开。
2. 点击“刷新”。
3. 执行若干会产生匿名事件的动作（示例）：
   - 打开扩展 Popup（触发 `popup_opened`）
   - 执行一次复制成功动作（触发 `copy_success`）
   - 在 Popup 内进入分享并复制分享文案（触发 `wom_share_copied`）
4. 回到 Options 面板再次点击“刷新”。

期望：
- 面板可见且不影响其它功能。
- 事件展示始终为空数组（`[]`），事件数为 `0`。

## 用例 B（开关开启：记录 + 白名单结构校验）
前置：
- Options -> 隐私与可观测性：`匿名使用数据` 开关为 **开启**

步骤：
1. 重复执行至少 3 类事件动作（示例同用例 A）：`popup_opened` / `copy_success` / `wom_share_copied`。
2. 回到 Options -> “本地匿名事件日志”面板点击“刷新”。

期望：
- 事件数 > 0，事件列表按 `ts` **降序**（新到旧）展示。
- 每条事件结构仅包含：
  - `name`（字符串，事件名）
  - `ts`（数字，时间戳）
  - `props`（可选对象，仅包含白名单字段，值为 string/number/boolean）
- 面板内容不包含任何 URL/标题/复制内容等敏感信息（可在文本中搜索 `http`、`title`、`content` 等关键字应不存在）。

## 用例 C（复制导出：JSON 可解析 + 与面板一致）
前置：
- 用例 B 已产生事件且面板可见

步骤：
1. 点击“复制”按钮。
2. 粘贴到文本编辑器。

期望：
- 粘贴内容为可解析 JSON（pretty print）。
- 内容与面板展示一致（同一份过滤后的事件数组）。

## 用例 D（清空：立刻移除 key + UI 同步为空）
前置：
- 用例 B 已产生事件且面板可见

步骤：
1. 点击“清空”按钮。
2. 点击“刷新”。

期望：
- 面板展示为空数组（`[]`），事件数为 `0`。

可选兜底校验（用于最终验收）：
- 在 Options 页面打开 DevTools Console，执行：
  - `chrome.storage.local.get('copilot_telemetry_events')`
- 期望：返回对象中该 key 不存在或值为空数组。

## 用例 E（关闭即清空：不缓存、不补发）
前置：
- Options -> 隐私与可观测性：`匿名使用数据` 开关为 **开启**

步骤：
1. 产生若干匿名事件后，在面板点击“刷新”确认事件存在。
2. 将 `匿名使用数据` 开关切换为 **关闭**。
3. 观察面板是否立刻刷新为空；再点击“刷新”二次确认。
4. 在开关关闭状态下，再执行若干会产生事件的动作（如打开 Popup / 复制成功）。
5. 回到面板点击“刷新”。
6. 再将开关切换为 **开启**，点击“刷新”。

期望：
- 第 2 步关闭后：本地日志被清空，面板立刻变为空数组（`[]`）。
- 第 4-5 步：关闭状态不记录，刷新后仍为空。
- 第 6 步重新开启：不会补发关闭期间或关闭前历史事件（仍为空，除非重新触发新事件）。
