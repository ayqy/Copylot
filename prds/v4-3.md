# Shift 追加模式与多段收集工作流强化

## 目标
实现 Shift 追加模式与多段收集工作流的状态承接、清理复用与证据闭环，让用户在不记录复制内容的前提下获得明确的连续复制理由，并沉淀可验证的复用/商业化推进证据，使其达到可发布验收标准。

本轮只做 `docs/roadmap_status.md` Top3 第 1 项，并沿用 `docs/test-cases/v4-3.md`、`docs/reports/v4-3-report.md` 与 `docs/evidence/v4-3/*` 作为同编号验收产物路径；实现范围优先收敛到 `src/content/content.ts`、`src/background.ts`、`src/popup/popup.html`、`src/popup/popup.ts` 与相关测试入口，不新增支付、问卷、候补、留资或其他偏航承接。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、更新 `src/content/content.ts`、`src/background.ts`、`src/shared/growth-stats.ts` 与必要的共享状态/文案文件，补齐 Shift 追加模式与多段收集工作流的状态承接，至少覆盖“追加中提示、已收集段数反馈、完成后清理、下一次复用可继续开始”四类关键动作，确保用户在长文、多表格、跨段参考资料场景下能稳定完成连续复制，同时全程不记录复制内容、页面正文或 URL。
2、更新 `src/popup/popup.html`、`src/popup/popup.ts`、`src/options/options.ts` 与相关 UI 文案/导出逻辑，把追加收集状态的清理与复用动作接入扩展内可见主路径，至少提供 1 个明确的“继续追加/清空后重来/查看当前追加状态”承接入口，并把与连续复制相关的匿名计数、状态枚举和审计摘要导出到 `docs/evidence/v4-3/append-mode-audit.md`、`docs/evidence/v4-3/append-mode-funnel-sample.json` 与 `docs/evidence/v4-3/append-mode-commercialization-proof.md`，用于证明该工作流推动了真实复用质量而非只增加按钮曝光。
3、创建并维护 `docs/test-cases/v4-3.md`、`docs/reports/v4-3-report.md`，同步更新 `e2e/content-output-flow.spec.ts`、`scripts/content-interaction-tests.ts`、`scripts/unit-tests.ts` 与 `bash scripts/test.sh` 的覆盖范围，确保 Shift 追加、多段收集、状态清理、隐私约束和证据导出都有可执行回归，且整体任务规模控制在 1-3 天内可完成。

## 验收标准
* 修复上述所有问题，并测试通过，达到可发布状态
* 实现以上所有需求，并测试通过，达到可发布状态
* `docs/test-cases/v4-3.md` 必须明确覆盖 Shift 追加、多段收集、状态清理、继续复用与隐私约束的手工/自动化用例，且 `bash scripts/test.sh` 退出码为 `0`
* 用户在至少 1 条长文或多段内容路径中，能够完成“开始追加 -> 连续收集多段 -> 完成复制 -> 清理或继续下一轮追加”的闭环，Popup 或其他扩展内承接入口可明确反馈当前追加状态，不出现脏状态残留到下一轮
* `docs/evidence/v4-3/append-mode-audit.md` 与 `docs/evidence/v4-3/append-mode-funnel-sample.json` 必须可复核追加次数、完成次数、清理次数、当前状态枚举等匿名证据，且不包含复制文本、网页正文、页面 URL 或其他用户原文
* `docs/evidence/v4-3/append-mode-commercialization-proof.md` 必须提供至少 1 组可验证的商业化证据口径，例如“连续复制完成率”“完成追加后再次 `copy_success` 的样例漏斗”“追加模式带动的复用场景样本”，并说明该证据如何支撑真实复用增长、口碑样本质量或后续商店转化判断
* `docs/reports/v4-3-report.md` 必须逐项说明具体任务完成状态、实现效果、涉及目录文件、测试结果与是否完成 commit/push，以便中断后可由同一轮继续闭环
