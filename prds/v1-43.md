# WOM 小实验：分享/评价/反馈入口低打扰优化 + 可量化证据导出（v1-43）

## 目标
由于 `docs/roadmap_status.md` Top1「真实 CWS 发布 + 商店端取证」仍受网络可达性阻塞（需人类提供可用代理/VPN），本轮选择 Top2「WOM 小实验」作为 1~3 天最小可交付增量主题。在“匿名使用数据默认关闭、仅本地可审计、不影响复制体验”的前提下，完成一轮可交付的 WOM（分享/评价/反馈）低打扰优化，并提供可导出的量化口径与证据包，用于验证可转化获客增长（商店访问/评价引导点击/反馈入口点击）是否被推进，为后续真实上架跑数与 Pro 意向验证提供更强证据链。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、WOM 入口与 UTM 口径统一（Popup / Options / 评分引导）
  - 统一所有打开 Chrome Web Store 的入口使用同一套 UTM 规则：`utm_source=copylot-ext`、`utm_campaign=v1-43`，并通过 `utm_medium` 精确区分入口（至少区分 popup/options/rating_prompt 三类）。
  - 对「分享」「去评价」「评分引导（rating prompt）」三类入口的打开链接与“复制分享文案”中的商店链接做一致性校验，确保可在新标签正常打开，且不引入新增权限。

2、WOM 行为摘要 + 证据包导出（可审计、可对比）
  - 基于本地匿名事件（`wom_*`、`rating_prompt_*`）新增一份可复制的 WOM 摘要 JSON（按 source 分组统计 count/lastTs，并包含最关键的转化率口径，例如 `share_copied_per_share_opened`）。
  - 在 Options -> 隐私与可观测性新增「WOM 摘要」面板：支持刷新、复制摘要、复制 WOM Evidence Pack（`meta/settings/womSummary/events`），匿名开关关闭时必须明确提示“未开启则不记录/不导出”。
  - 落盘本轮可验证商业化证据资产目录：`docs/evidence/v1-43/`（至少包含 `wom-summary.json`、`wom-evidence-pack.json`、截图索引 `index.md`，并保留 telemetry off 的对照导出）。

3、测试与用例（可执行、可回归）
  - 新增测试用例文档：`docs/test-cases/v1-43.md`，覆盖：匿名开关开/关、Popup/Options 各入口点击、UTM 参数校验、摘要/证据包字段白名单与脱敏规则、截图命名规范。
  - 补齐自动化断言（单元测试/集成回归）：覆盖 WOM 摘要/证据包结构稳定性、匿名开关关闭时“无事件/不可导出”的行为，以及 `bash scripts/test.sh` 全量回归通过。

## 验收标准
* 修复上述所有问题，并测试通过，达到可发布状态
* 实现以上所有需求，并测试通过，达到可发布状态
* WOM 入口的商店链接具备统一且可核验的 UTM 参数（`utm_source/utm_medium/utm_campaign=v1-43`），并在 Popup/Options/评分引导三处一致生效
* Options -> 隐私与可观测性可生成并复制「WOM 摘要」与「WOM Evidence Pack」：字段白名单过滤可审计；匿名开关关闭时导出为空且 UI 明确解释
* 商业化证据可验证落盘：`docs/evidence/v1-43/` 中存在 `wom-summary.json`、`wom-evidence-pack.json`、`index.md`（含截图索引与口径说明），可用于与 `docs/evidence/v1-42/` 做前后对比
