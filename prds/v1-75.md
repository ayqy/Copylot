# 问题修复和功能优化（V1-75 并行增长循环最小闭环：对外入口一致化证据包 + 投放资产样例自动生成）

## 目标
基于 `docs/roadmap_status.md` 的收入优先 Top3：本轮选择 Top1「（v1-75）并行增长循环启动（最小闭环）」作为 1~3 天最小可交付增量主题（承接 v1-74 的目标，但收敛为更小步可落地交付）。

以收入优先为原则，把“官网/商店/Pro 候补 CTA”与对外投放资产收敛到单一事实来源，并自动生成可审计、可导出、可复用的证据包，使增长循环可以开始跑数并可复盘。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、增长循环入口“单一事实来源”与样例导出（收入优先、可审计）
  - 以 `src/shared/external-links.ts` 作为唯一真实来源，固化并覆盖：
    - 官网入口 / Chrome Web Store 入口 / Pro 候补入口（`#pro`）
    - `utm_source=copylot-ext`；`utm_medium` 至少覆盖 `popup` / `options` / `distribution_toolkit`
    - 支持可选 `utm_campaign`（样例至少包含 `campaign=twitter`）
    - Pro 候补 URL 仅允许 `ext_version` / `ext_id` / `nav_lang` / `ui_lang` 等白名单字段；不得出现网页 URL/标题/复制内容等字段
  - 新增证据目录与索引（固定路径，便于审计与复用）：
    - `docs/evidence/v1-75/`
    - `docs/evidence/v1-75/index.md`（记录 `src/shared/external-links.ts` sha256、输出文件清单、PASS/BLOCKED 结论）
    - `docs/evidence/v1-75/official-links.json`（覆盖无 campaign 与 `campaign=twitter` 两类样例）

2、对外投放资产样例自动生成（可直接复制投放，避免口径漂移）
  - 新增脚本：`scripts/build-growth-loop-evidence-pack.ts`
    - 输入（单一事实来源 + 与 UI 文案一致）：
      - `src/shared/external-links.ts`
      - `_locales/en/messages.json`
      - `_locales/zh/messages.json`
    - 输出（固定路径，便于审计与复用）：
      - `docs/evidence/v1-75/official-links.json`
      - `docs/evidence/v1-75/pro-distribution-pack.sample.md`
      - `docs/evidence/v1-75/share-copy.sample.txt`
      - `docs/evidence/v1-75/index.md`（写入 sha256 与生成结论）
    - 要求：脚本输出必须确定性（同输入多次运行 sha256 不变）
  - `docs/evidence/v1-75/pro-distribution-pack.sample.md` 必须包含：官网/商店/候补三类入口与招募文案，且包含 `campaign=twitter`
  - `docs/evidence/v1-75/share-copy.sample.txt` 必须至少包含商店安装链接（可选包含官网/候补）

3、门禁与测试/用例闭环（可执行、可复现、可追责）
  - 将证据包生成脚本纳入门禁：在 `bash scripts/test.sh` 中增加“生成 + 确定性 hash 断言”（参考 v1-66/v1-67 的 determinism 校验模式）
  - 新增测试用例文档（固定路径，必须可逐条执行）：`docs/test-cases/v1-75.md`
    - 覆盖：证据包生成脚本 -> 入口 URL 断言（utm + medium + campaign + env 白名单）-> 投放资产样例可复制性 -> 门禁 determinism 校验
    - 必须包含一次回归记录（日期 + PASS/BLOCKED + 证据文件名）
  - 新增简报（固定路径建议）：`docs/reports/v1-75-report.md`
    - 只写结论：本轮形成的“可投放资产”是什么、可审计证据是什么、下一步如何衔接 v1-70/v1-72
