# 官网安装承接与信任承接改造（无表单闭环版）

## 背景

当前 roadmap 已切回浏览器插件增长闭环优先，且 Chrome Web Store 已公开可安装。现阶段最小可交付增量不能再继续围绕表单、问卷、候补或支付承接，而应优先把官网首页、`/pricing`、`/privacy` 三个核心页面改造成真实安装承接面与信任承接面，让官网能够直接承接商店曝光、分享流量和自然获客，并为后续复用、评价与增长投放提供统一口径。

## 目标

把官网首页、`/pricing`、`/privacy` 改造成以安装、上手、隐私信任和分享为核心的浏览器插件承接面，同时同步扩展内外链、增长文档与测试口径，使当前版本能够用可验证的安装入口、信任说明与增长证据证明商业化主线已经从“收集意向”切换到“推动真实安装与激活”。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 结合项目代码的需求澄清

当前仓库能直接观察到的官网承接相关基线主要位于两类位置：一是扩展侧的外链与分发链路，例如 `src/shared/external-links.ts`、`src/options/options.ts`、`src/shared/pro-waitlist-distribution.ts`、`e2e/options-pro-flow.spec.ts`、`e2e/popup-growth-flow.spec.ts`；二是增长与验收文档，例如 `docs/roadmap.md`、`docs/roadmap_status.md`、`docs/growth/plan.md`、`docs/growth/blocked.md`、`docs/test-cases/v1-108.md`。仓库内没有直接可见的官网源码目录，但已有报告明确提到官网改造目标是首页、`/pricing`、`/privacy` 三页，因此本轮需求应把站点改造限定为这三个承接页面，并同步修正扩展端生成链接、E2E 断言、测试用例与增长文案基线，避免页面内容、链接目标、分享包和验收口径再次分叉。所有改动都必须坚持当前边界：不新增表单、问卷、候补收集，不新增支付承接，只允许安装、上手、隐私、分享和 Pro 路线说明。

## 具体任务

1、完成官网首页安装承接改造
  - 首页首屏必须明确回答“为什么不用原生复制”“安装后 30 秒能完成什么”“现在去哪里安装”。
  - 主 CTA 只允许围绕 Chrome Web Store 安装、使用演示/上手说明、分享传播，不允许出现问卷、候补、留资或支付 CTA。
  - 首页内容必须补齐至少 3 类高频场景承接：长文干净复制、表格转 Markdown/CSV、代码块清洗，并与扩展实际能力保持一致。

2、完成 `/pricing` 与 `/privacy` 的信任承接改造
  - `/pricing` 改成 Pro 路线说明页：只解释 Free 与未来 Pro 的能力边界、当前为什么不做支付、哪些增强方向会在真实复用数据成立后推进。
  - `/privacy` 必须正式解释“本地处理、不上传复制内容、匿名使用数据默认关闭”的边界，并给出用户能在扩展内哪里查看或关闭相关设置。
  - 这两个页面都必须提供明确的安装回流入口，并保证文案与首页、扩展内链接、分享包一致。

3、同步扩展链接、测试口径与增长投放基线
  - 更新扩展内官网相关外链与分享文案生成逻辑，使其全部指向新的安装承接页面与 Pro 路线说明页，不再出现表单、问卷、候补语义。
  - 更新自动化测试与用例文档，至少覆盖 `e2e/options-pro-flow.spec.ts`、`e2e/popup-growth-flow.spec.ts` 和测试说明文档 `docs/test-cases/v1-109.md`，验证首页、`/pricing`、`/privacy` 的 CTA 语义、分享复制内容和无表单边界。
  - 更新增长文档与投放基线，包括 `docs/growth/plan.md`、相关分享包/投放包口径以及 `docs/roadmap_status.md` 的阶段说明，确保并行增长循环的入口、指标与证据路径统一。

## 验收标准

* 实现以上所有需求，并测试通过，达到可交互、可运行或可使用的状态。
* 如果本轮任务涉及部署、发布、上线、同步到目标环境或其他交付落地动作，则在测试通过后完成对应动作，并对目标环境效果进行回归验证。

* 官网首页、`/pricing`、`/privacy` 三页都可作为真实分发落地页使用：存在可点击的 Chrome Web Store 安装入口，且文案不包含表单、问卷、候补或支付承接。
* 扩展内所有指向官网的主链路、分享文案和分发包与这三页的新定位一致；复制出的链接/文案可人工复核为“安装 -> 上手 -> 隐私信任 -> 分享”闭环。
* `docs/test-cases/v1-109.md` 明确覆盖首页、`/pricing`、`/privacy`、Options Pro 分享工具和 Popup 增长入口的回归步骤，并能直接对应以下命令：`npm run type-check`、`npm run check-i18n`、`node --no-warnings=ExperimentalWarning --loader=ts-node/esm scripts/unit-tests.ts`、`npm run build:e2e`、`COPYLOT_E2E_SKIP_BUILD=1 npx playwright test --config=playwright.config.ts --project=main e2e/popup-flow.spec.ts e2e/options-pro-flow.spec.ts e2e/popup-growth-flow.spec.ts`。
* 本轮必须产出可验证的商业化推进证据，至少包括以下任一可审计结果并落到仓库文档路径中：官网/商店安装入口清单、无表单版分享投放包、可导出的安装承接/分享漏斗口径、首页与 `/pricing` / `/privacy` 的截图或文案基线索引。
