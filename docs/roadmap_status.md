# Roadmap 状态

## 当前阶段

- 阶段名称：S1 / G1 `安装与首次成功`
- 阶段目标：把安装后的第一条主路径收敛成“看到价值主张 -> 触发第一次干净复制 -> 愿意再次复用”。
- 阶段反思：S0 已经把扩展仓内能直接控制的安装、隐私、路线说明和分享口径统一了，但“安装后为什么马上试第一次复制”这一步仍缺少更强的价值主张、截图顺序和证据闭环。
- 优化方向：下一轮优先对齐官网、商店、Popup 的价值主张与截图顺序，再把 `copy_success` 的事件口径和回归证据补齐。
- 经营边界：当前版本继续不做表单、问卷、候补或支付承接，所有动作都服务第一次干净复制与后续复用。

## 当前进度

- 阶段进度：S0 已完成 `6/6`；S1 已完成 `0/4`；全局已完成 `6/18`。

  - 已回退 Popup 中“你最想先解决什么”与相关预填问卷链路。

## 下一步最重要的 3 件事（收入优先）

1. 对齐官网、商店、Popup 的价值主张、截图顺序与安装指引，确保所有入口都围绕“第一次干净复制”而不是泛功能罗列。
2. 固化安装 -> `copy_success` 的事件口径、可导出证据与对账方式，证明首次成功路径正在变短。
3. 为首次成功路径补齐自动化回归与手工用例，确保上架版本能稳定复现安装后的第一次成功复制。
