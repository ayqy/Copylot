# V1-13 设置页口碑入口补齐（分享/评价/反馈）

## 目标
在不新增权限、不新增联网埋点、且不影响现有复制体验的前提下，将“口碑与获客闭环”的关键入口补齐到 Options 设置页：提供可审计的“分享给朋友/一键复制分享文案/去评价/反馈与建议”入口，并复用现有 URL 构造与反馈模板能力；同时补齐可执行的测试用例文档，确保 `bash scripts/test.sh` 通过，达到可发布状态。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、Options 页新增“口碑与支持”区块（稳定 DOM、低打扰、可 i18n）
  - 位置：`src/options/options.html` 的 `#pro-tab`（复用现有卡片风格，避免大改版）
  - 新增一个区块容器（建议 card 样式），并提供稳定 DOM 标识（用于用例与未来自动化）：
    - 容器：`#wom-actions-panel`
    - 按钮（均为用户点击触发，不做任何自动弹窗/自动复制）：
      - 打开分享：`#wom-share-open`
      - 复制分享文案：`#wom-share-copy`
      - 打开去评价：`#wom-rate-open`
      - 打开反馈：`#wom-feedback-open`
  - i18n：
    - 复用现有 key：`shareWithFriends` / `copyShareText` / `rateUs` / `feedback`
    - 新增区块标题/说明文案 key（如 `womActionsTitle` / `womActionsHint`），并补齐 `_locales/en/messages.json` 与 `_locales/zh/messages.json`
  - 约束：
    - 不新增任何权限
    - 不新增任何第三方依赖
    - 不影响既有 Pro Tab（候补名单）功能与样式

2、复用 WOM 链路实现 Options 页行为（UTM 可审计 + 模板化反馈）
  - 逻辑落地：`src/options/options.ts`
  - 分享链接（必须可审计）：
    - 复用 `src/shared/word-of-mouth.ts` 的 `buildChromeWebStoreDetailUrl(...)`
    - UTM 参数固定为：
      - `utm_source=copylot-ext`
      - `utm_medium=options-entry`
      - `utm_campaign=v1-13`
    - 打开方式：`chrome.tabs.create({ url })`
  - 复制分享文案：
    - 复用 `src/shared/word-of-mouth.ts` 的 `buildShareCopyText(getMessage, storeUrl)`
    - 仅在用户点击 `#wom-share-copy` 后写入剪贴板；失败需提示但不得影响 Options 其它功能
  - 去评价：
    - 复用 `buildChromeWebStoreReviewsUrl(chrome.runtime.id)`，用 `chrome.tabs.create` 打开
  - 反馈与建议：
    - 复用 `buildFeedbackIssueUrl(...)` + `buildFeedbackSettingsSnapshot(currentSettings)`
    - env 字段仅允许：扩展版本/扩展 ID/userAgent/navigator.language/UI language（`chrome.i18n.getUILanguage()`）
    - 严禁拼装任何网页内容、URL、标题、复制内容
  - 本地匿名事件（可选但建议，便于验证漏斗；仍遵循开关默认关闭）：
    - 当 `Settings.isAnonymousUsageDataEnabled=true` 时，点击对应入口记录事件（复用既有事件名，不新增事件）：
      - `wom_share_opened` / `wom_share_copied` / `wom_rate_opened` / `wom_feedback_opened`
    - 当开关关闭时：不记录、不缓存、不补发（由 `recordTelemetryEvent` 现有逻辑保证）

3、同步埋点清单与补齐测试用例文档（可执行、可回归）
  - 更新“埋点清单”文档：`docs/growth/telemetry-events.md`
    - 为 `wom_*` 事件补充 Options 侧触发位置（文件/交互），保证可审计
  - 新增并维护测试/用例文档：`docs/test-cases/v1-13.md`
    - 文档必须覆盖（至少）：
      - 用例 A（入口可见）：打开 Options -> Pro Tab，`#wom-actions-panel` 可见且按钮可点击，不影响候补名单功能
      - 用例 B（分享链接 UTM）：点击 `#wom-share-open` 打开商店页，URL 含 `utm_source=copylot-ext&utm_medium=options-entry&utm_campaign=v1-13`
      - 用例 C（复制分享文案）：点击 `#wom-share-copy` 后粘贴验证文案含商店链接且可读
      - 用例 D（去评价）：点击 `#wom-rate-open` 打开 reviews 页面
      - 用例 E（反馈模板）：点击 `#wom-feedback-open` 打开 GitHub `issues/new`，title/body 已预填且仅包含环境信息 + settings 快照占位；不包含任何复制内容/网页内容/URL/标题
      - 用例 F（匿名事件开关）：开关关闭时上述动作不产生 `copilot_telemetry_events`；开启后会记录对应 `wom_*` 事件（可在隐私页事件面板刷新验证）
      - 自动化回归：记录一次 `bash scripts/test.sh` 执行结果（命令 + 结论）
