# V1-13 设置页口碑入口补齐（分享/评价/反馈）测试用例

## 目标
验证 V1-13 达到可发布：
- Options -> Pro Tab（`#pro-tab`）新增“口碑与支持”区块 `#wom-actions-panel`，包含分享/复制文案/去评价/反馈四个入口
- 分享入口打开的商店详情页 URL 携带固定 UTM：`utm_source=copylot-ext&utm_medium=options-entry&utm_campaign=v1-13`
- 反馈入口打开的 GitHub Issue 页面已预填 title/body，且 body 仅包含环境信息 + settings 快照 + 复现模板；不包含任何网页内容/复制内容/URL/标题
- “匿名使用数据”开关关闭时不记录 `copilot_telemetry_events`；开启后会记录 `wom_*` 事件（本地，不联网）
- `bash scripts/test.sh` 全量通过

## 预置条件
- 已在 Chrome/Chromium 加载本地扩展（开发包或生产包均可）
- 可打开 Options 页面（扩展详情页 -> 扩展选项）

## 用例 A（入口可见：不影响候补名单）
步骤：
1. 打开 Options 页面。
2. 点击顶部 Tab：Pro（`#pro-tab`）。
3. 检查“口碑与支持”区块 `#wom-actions-panel` 是否可见。
4. 检查 4 个按钮是否存在且可点击：
   - `#wom-share-open`
   - `#wom-share-copy`
   - `#wom-rate-open`
   - `#wom-feedback-open`
5. 同时检查原有候补区块的按钮仍可点击：
   - `#pro-waitlist-button`
   - `#pro-waitlist-copy`

期望：
- `#wom-actions-panel` 可见，按钮可点击；不会自动弹窗、不会自动复制。
- 原有候补名单功能与样式不受影响。

## 用例 B（分享链接 UTM 可审计）
步骤：
1. 在 Pro Tab 点击 `#wom-share-open`。
2. 浏览器应打开新标签页进入 Chrome Web Store 详情页。
3. 在地址栏检查 URL 查询参数。

期望：
- URL 包含：`utm_source=copylot-ext&utm_medium=options-entry&utm_campaign=v1-13`（顺序不限）。

## 用例 C（复制分享文案：仅用户点击后写入剪贴板）
步骤：
1. 在 Pro Tab 点击 `#wom-share-copy`。
2. 打开任意文本输入框（如记事本/聊天框），粘贴。

期望：
- 粘贴内容为可读分享文案，且包含 Chrome Web Store 链接（链接包含用例 B 的固定 UTM）。
- 除非用户点击 `#wom-share-copy`，否则不会自动写入剪贴板。

## 用例 D（去评价：打开 reviews）
步骤：
1. 在 Pro Tab 点击 `#wom-rate-open`。

期望：
- 打开新标签页进入 reviews 页面：`https://chrome.google.com/webstore/detail/<EXT_ID>/reviews`。

## 用例 E（反馈模板：仅环境信息 + settings 快照 + 复现模板）
步骤：
1. 在 Pro Tab 点击 `#wom-feedback-open`。
2. 浏览器应打开 GitHub `issues/new` 页面（带 title/body 预填）。
3. 检查 body 内容。

期望：
- title/body 已预填。
- body 仅包含：
  - 扩展版本、扩展 ID
  - User Agent
  - `navigator.language`
  - UI language（`chrome.i18n.getUILanguage()`）
  - settings 快照（仅关键开关白名单字段）
  - 复现步骤/期望结果/实际结果模板
- body 不包含任何网页内容、复制内容、URL、标题（可在 body 中搜索 `http`/`https`/`title`/`url`/`content` 等关键字应不存在）。

## 用例 F（匿名事件开关：开关关闭不记录，开启记录 wom_*）
前置：
- Options -> 隐私与可观测性（`#privacy-tab`）“匿名使用数据”开关可用

步骤（关闭态）：
1. 将“匿名使用数据”开关切换为 **关闭**。
2. 在隐私页展开“本地匿名事件日志”面板，点击“清空”后再点击“刷新”，确认事件为空数组 `[]`。
3. 进入 Pro Tab，依次点击：
   - `#wom-share-open`（会打开新标签页，可关闭或忽略）
   - `#wom-share-copy`
   - `#wom-rate-open`（会打开新标签页，可关闭或忽略）
   - `#wom-feedback-open`（会打开新标签页，可关闭或忽略）
4. 回到隐私页点击“刷新”。

期望（关闭态）：
- 面板仍为 `[]`，不会出现 `wom_*` 事件；不会缓存、不会补发。

步骤（开启态）：
1. 将“匿名使用数据”开关切换为 **开启**。
2. 在隐私页点击“清空”后再点击“刷新”，确认事件为空数组 `[]`。
3. 进入 Pro Tab，重复步骤（关闭态）的四个点击动作。
4. 回到隐私页点击“刷新”。

期望（开启态）：
- 事件列表出现对应的 `wom_*` 事件：
  - `wom_share_opened` / `wom_share_copied` / `wom_rate_opened` / `wom_feedback_opened`
- 事件结构仅包含 `name/ts/props(白名单)`，且 `wom_*` 事件无 props。
