- `src/shared/growth-stats.ts`
- `src/shared/wom-summary.ts`

## 自动化入口

```bash
bash scripts/test.sh
```

专项覆盖：

- `scripts/unit-tests.ts`：WOM 资格判断、Growth Funnel 新字段、WOM evidence pack 审计字段。
- `scripts/ui-integration-tests.ts`：Popup / Options 在资格前后的解锁状态、Options 完整 WOM 报告包导出。
- `e2e/popup-growth-flow.spec.ts`：浏览器级验证锁前不可触发、解锁后可触发。

## 用例 A：Popup 在第二次成功复制前锁住 WOM 入口

步骤：

1. 准备 `successfulCopyCount=1`、`firstSuccessfulCopyAt` 已存在的本地增长统计。
2. 打开 Popup。
3. 检查分享链接、复制分享文案、评价链接的状态。

期望：

- `#share-link` 与 `#rate-link` 带有禁用态属性。
- `#copy-share-button` 处于 disabled。
- `#wom-status-hint` 显示还差几次成功复制。
- 强制点击也不会产生 `wom_share_opened`、`wom_rate_opened`、`wom_share_copied`。

## 用例 B：Options 在第二次成功复制前锁住 WOM 按钮

步骤：

1. 准备 `successfulCopyCount=1` 的本地增长统计。
2. 打开 `src/options/options.html#pro`。
3. 检查 WOM 按钮与提示文案。

期望：

- `#wom-share-open`、`#wom-share-copy`、`#wom-rate-open` 为 disabled。
- `#wom-actions-status` 显示还差几次成功复制。
- `#wom-feedback-open` 仍可正常打开反馈链接。

## 用例 C：第二次成功复制后 WOM 入口自动解锁

步骤：

1. 准备 `successfulCopyCount>=2` 且存在 `secondSuccessfulCopyAt` 的本地增长统计。
2. 打开 Popup 并点击分享、评价入口。
3. 打开 Options 并点击复制分享文案。

期望：

- Popup / Options 中 WOM 入口均恢复可用。
- 可正常打开商店详情页、评价页并复制分享文案。
- 仅在解锁后才会写入 `wom_share_opened`、`wom_share_copied`、`wom_rate_opened`。

## 用例 D：WOM 完整报告包包含资格审计

步骤：

1. 在 Options 中点击“复制完整报告包”。
2. 检查 JSON 内 `womQualificationAudit` 字段。
3. 对照 `docs/evidence/v4-3/wom-evidence-pack-sample.json`。

期望：

- 报告包包含 `successfulCopyCount`、`secondSuccessfulCopyAt`、`isEligibleForWomActions`、`gatedEventsBeforeQualificationCount`。
- 合格样本的 `gatedEventsBeforeQualificationCount` 为 `0`。
- 报告包不包含网页正文、页面 URL 或复制内容。
