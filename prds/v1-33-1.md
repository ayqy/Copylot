# V1-33-1 拆分：CWS 发布流程防呆（publish:cws 强制生产回归 + 产物校验 + dry-run）+ 文档/用例闭环

## 范围（MVP）
在不新增任何扩展权限、不改变现有复制/Prompt/引导/口碑/商业化功能逻辑的前提下，将 `npm run publish:cws` 改造成**安全、可重复、可审计**的 Chrome Web Store 发布入口：

- 发布前置门禁：强制执行 `bash scripts/test.sh`（lint/type-check/check-i18n/unit-tests/build:prod/verify-prod-build）
- 产物一致性校验：`dist/manifest.json` 的 `version === manifest.json.version`
- 重新打包策略：基于当前生产 `dist/` 生成/覆盖 `plugin-${version}.zip`（同名 zip 必须先删除再生成）
- 支持 `--dry-run`：无网络副作用、允许无 `.env` 凭据演练完整发布前置流程
