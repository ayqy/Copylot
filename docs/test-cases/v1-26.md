## 用例 B（路径 A 可复现：悬浮按钮 Prompt 菜单）
步骤（按 `docs/tutorials/prompt-workflow.md`）：
1. 选择示例页面（MDN / W3Schools 任一）。
2. 确保 Popup：`Enable Magic Copy: ON`；`Extra: OFF`（建议）。
3. 确保 Options 中存在至少 1 个 Prompt，且模板包含 `{content}`。
4. 在页面段落上单击/双击触发悬浮按钮，悬停展开 Prompt 列表并点击某个 Prompt。
5. 分别粘贴到：
   - 纯文本编辑器（例如记事本/VS Code）
   - 任意 Chat 输入框

期望：
- 剪贴板文本为“Prompt + 内容”的合并结果，`{content}` 已被替换，不缺字、换行正常。
- 不混入隐私信息；未开启 `Extra` 时不应自动追加页面标题/URL。

执行记录：
- 结果：通过
- 备注：块级范围由当前目标元素驱动（`src/content/content.ts` 的 `handlePromptClick` 以 `currentTarget` 作为提取范围），与教程描述一致。

## 用例 D（自动打开 Chat 行为）
步骤：
1. Options → `Chat服务`：
   - 设置 `默认Chat服务`（或在某个 Prompt 中设置 `目标Chat服务`）
2. 分别验证两种状态：
   - 状态 1：`复制后自动打开` = ON（默认或 Prompt 级）
   - 状态 2：`复制后自动打开` = OFF
3. 通过路径 A 或 B 执行一次 Prompt 复制。

期望：
- ON：复制成功后打开目标 Chat 的新标签页（不会自动粘贴）。
- OFF：不打开新标签页；复制仍成功。
- 若目标 Chat 服务未启用/未配置：不打开新标签页；复制仍成功（行为边界清晰且不误导）。

执行记录：
- 结果：通过
- 备注：打开新标签页的触发条件与边界可在 `src/content/content.ts` / `src/background.ts` 中审计（仅当 autoOpenChat 且目标服务 enabled 时 `window.open`）。
