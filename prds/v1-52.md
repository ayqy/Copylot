# 问题修复和功能优化（V1-52 Pro 意向渠道归因最小闭环：渠道（campaign）字段 + 导出证据可复核）

## 目标
基于 `docs/roadmap_status.md` 的收入优先 Top3，本轮选择 Top1「Pro 意向渠道归因最小闭环」作为 1~3 天最小可交付增量主题。

收入第一：在不引入支付/订阅、不新增权限、不联网发送数据、不采集用户复制内容的前提下，把“渠道（campaign）-> Pro 留资动作（候补/问卷）”沉淀为**可量化、可导出、可审计**的证据链路，支持后续做可转化的获客增长（渠道投放/社群分发/内容分发）与真实发布后对齐跑数。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、Options -> Pro Tab：新增“渠道（campaign）”字段，并写入候补/问卷留资文本（提升留资质量与可归因性）
  - 在 Options -> Pro Tab（候补行动区附近）新增一个可选输入项：渠道（campaign）
    - 建议稳定 DOM：`#pro-intent-campaign`
    - 默认值为空；允许用户填写短字符串（建议限制长度与字符集，避免写入敏感信息）
  - 写入范围（只要用户走“留资动作”，就必须携带 campaign）：
    - `复制候补文案`：复制内容中新增 `campaign: <value>`（空值则不写）
    - `加入候补名单`：打开的 GitHub new issue body 中新增 `campaign: <value>`（空值则不写）
    - `复制问卷（含环境信息）/复制并打开候补`：复制的问卷 Markdown 中新增 `campaign: <value>`（空值则不写）
  - 非目标：不新增任何网络请求；不改变既有“候补登记不预填网页内容/复制内容”的隐私边界

2、可量化取证：在本地匿名事件 props 与周摘要/证据包导出中可复核 campaign（让商业化证据可执行）
  - 本地匿名事件（仅匿名开关 ON 时记录；默认 OFF 不记录）为以下 Pro 意向事件补齐 `campaign` 字段：
    - `pro_entry_opened/pro_waitlist_opened/pro_waitlist_copied/pro_waitlist_survey_copied`
    - 规则：campaign 为空则不写入 props；非空则写入并通过白名单与合法性校验（避免污染 telemetry）
  - 导出与复核（不改变 v1-51 CSV 固定字段口径，保持兼容）：
    - Options -> 隐私页导出的「Pro 意向漏斗证据包」events 明细中可看到 `props.campaign`
    - “复制本周 Pro 意向证据摘要（7d window）”（v1-50）在 Markdown 末尾追加 `campaign` 拆分统计（至少覆盖 `pro_waitlist_copied/pro_waitlist_survey_copied` 的计数）
  - 文档同步：更新 `docs/growth/telemetry-events.md`，补齐 campaign 的白名单 key、取值约束与隐私说明

3、补齐测试/用例与证据落盘（确保 1~3 天内可交付、可回归、可审计）
  - 新增测试用例文档（固定路径）：`docs/test-cases/v1-52.md`
    - 覆盖：campaign 为空/非空；复制候补/打开候补/复制问卷三条链路；匿名开关 OFF 不记录、ON 记录并可导出；周摘要 campaign 拆分口径可复核
  - 自动化测试加固（优先纯函数单测）：覆盖 campaign 合法性校验、telemetry sanitize 白名单、周摘要按 campaign 聚合逻辑；并保证 `bash scripts/test.sh` 全量通过
  - 新增证据目录与索引（固定路径）：`docs/evidence/v1-52/` + `docs/evidence/v1-52/index.md`
    - 落盘：1 份含 campaign 的周摘要示例（Markdown）+ 1 份含 campaign 的 Pro 证据包导出样例（JSON）+ 截图索引与最小复盘口径
  - 新增交付简报（固定路径建议）：`docs/reports/v1-52-report.md`
