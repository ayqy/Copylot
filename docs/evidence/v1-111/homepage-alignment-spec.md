# v1-111 官网首页承接规范

## 目标

即使官网源码不在当前仓库，本文件也能直接交给站点仓落地，实现与商店、Popup 同口径的“第一次干净复制”承接页。

## Hero 区

- 主标题：`30 秒完成第一次干净复制`
- 副标题：先复制长文、表格或代码块，再把结果直接贴到 ChatGPT、Claude、Notion 或文档。
- 主 CTA：`安装 Copylot`
  - 目标链接：Chrome Web Store 安装页
- 次 CTA 1：`查看安装指引`
  - 目标链接：`first-copy-install-guide.md` 对应站点页或官网首页锚点
- 次 CTA 2：`隐私说明`
  - 目标链接：隐私说明页
- 次 CTA 3：`Pro 路线`
  - 目标链接：Pro 路线页

## Hero 下方的三个价值块

1. 长文
   - 文案：去掉导航、广告和杂项说明，直接得到干净 Markdown / 纯文本。
2. 表格
   - 文案：一键转成 Markdown / CSV，粘贴后结构仍然清楚。
3. 代码块
   - 文案：保留缩进与空行，尽量去掉行号和复制按钮噪声。

## 首屏三步指引

1. 安装并固定扩展。
2. 打开一个长文、表格或代码块并触发复制。
3. 把结果粘贴到 AI 或文档，确认第一次干净复制成功。

## 截图位顺序

1. Popup 首屏
2. 长文复制结果
3. 表格复制结果
4. 代码块复制结果
5. 粘贴结果

## 页面禁区

- 不新增表单、问卷、候补、收款或支付承接。
- 不把首页第一 CTA 指向 Pro 路线页。
- 不把“安装成功”与“第一次干净复制成功”混为一个概念。

## 交付核对

- 首页主标题、前三个卖点、首屏三步指引必须与 Popup 首屏及商店说明一致。
- 所有对外固定入口只允许官网首页、商店安装页、隐私说明页、Pro 路线页四类。
