## 用例 A：Pro Tab 出现收费评估审计区块

步骤：

1. 打开 `src/options/options.html#pro`。
2. 检查 `#pro-payment-evaluation-audit-panel`、`#copy-pro-payment-evaluation-audit-summary`、`#download-pro-payment-evaluation-audit-json`。

期望：

- 新区块可见。
- 两个动作按钮存在且默认可点击。
- 隐私提示明确说明不包含页面正文、复制内容、URL、标题或联系方式。

## 用例 B：复制出的审计摘要同时包含状态、阻塞项和边界

步骤：

1. 在 `Options -> Pro` 点击“复制收费评估审计包”。
2. 读取最近一次剪贴板文本。

期望：

- 文本包含 `V4-12 收费评估审计包`。
- 文本包含 `audit_status=`、`route_stability_ready=`、`gate_allows_payment_evaluation=`。
- 文本包含“继续停留在验证阶段”或 `stay in validation` 的审计结论。
- 文本包含至少一条阻塞项和一条边界说明。

## 用例 C：下载出的 JSON 可复核 go/no-go 与证据链

步骤：

1. 点击“下载收费评估审计 JSON”。
2. 打开生成文件并检查关键字段。

期望：

- 文件名符合 `copylot-pro-payment-evaluation-audit-v4-12.json`。
- 文件内包含 `auditStatus`、`readyForPaymentEvaluation`、`checks`、`blockers`、`boundaries`、`nextActions`、`evidence`。
- 当前样例数据下，结果仍回到 `auditStatus = hold_validation`。

## 用例 D：Roadmap 与边界同步收口

步骤：

1. 复核 `docs/roadmap.md`、`docs/roadmap_status.md`、`docs/reports/v4-12-report.md`。

期望：

- Roadmap 明确 `S3 Pro 路线验证` 已完成 `4/4`。
- 状态文档明确收费评估审计包已完成，但当前结论仍停在 `stay_validation`。
- 文档继续明确本轮没有进入支付/收款实现。
