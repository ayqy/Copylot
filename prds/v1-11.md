# V1-11 含表格混合内容转换一致性修复（复用 convertToMarkdown/PlainText 链路）

## 目标
修复“页面/选区包含表格时走混合内容路径，导致链接/图片预处理与 Markdown 清理缺失，从而出现同页不同区域、含表格 vs 不含表格输出不一致”的问题（对应 `review-issues.md` ISSUE-009），确保一键转换输出稳定、可读、尽量不丢内容；并补齐可执行的自动化回归与用例文档，使之达到验收标准、可发布。

## 具体任务
1、统一“含表格混合内容”的转换链路（修复 `review-issues.md` ISSUE-009）
  - 现状（需在 PR 实现中保留可追溯性）：
    - `src/shared/content-processor.ts` 的 `processContent()` 在检测到 `tables.length > 0` 时走 `processElementWithMixedContent(...)`
    - `processElementWithMixedContent(...)` 当前通过 `convertHtmlToMarkdown((element as HTMLElement).outerHTML)` 做整体 turndown，并直接返回结果
    - 该路径会绕过 `convertToMarkdown(...)` 中的关键处理：
      - `<a>` 链接规范化与“含 img/svg 链接不丢正文”的收敛逻辑（ISSUE-001/002 相关能力在该路径不一致）
      - `cleanMarkdownLinksPreservingCode(...)`（例如 `[](#)` 清理、无效链接规范化、且必须跳过 code block / inline code）
  - 改造要求（保持范围可控、可回归）：
    - 在 `src/shared/content-processor.ts` 重构 `processElementWithMixedContent(...)`：
      - 保持现有“表格 → token 占位符 → turndown → token 替换”的整体思路不变
      - 将“整体 turndown”从 `convertHtmlToMarkdown(...)` 切换为复用既有导出函数，按 `settings.outputFormat` 分流：
        - `settings.outputFormat === 'markdown'`：调用 `convertToMarkdown(...)` 生成整体 Markdown（确保 `<a>/<img>` 预处理与 `cleanMarkdownLinksPreservingCode` 生效）
        - `settings.outputFormat === 'plaintext'`：调用 `convertToPlainText(...)` 生成整体纯文本（行为需与“无表格路径”一致）
      - token 替换时，将每个表格替换为：
        - `settings.tableOutputFormat === 'csv'`：使用 `convertTableToCSV(table)`
        - 否则：使用 `toGfmMarkdownOrFallback(table)`（保持当前规则不变）
    - token 设计约束：
      - token 必须保证在 turndown/纯文本提取后仍可稳定匹配（避免被转义/折行破坏）
      - 必须确保 token 在输出中与相邻段落/行有稳定分隔（例如保证 token 前后至少各有一个换行），避免替换后表格与文本粘连
    - 兼容性/安全性约束：
      - 不引入新依赖
      - 不改变“纯表格元素”（`workingRoot instanceof HTMLTableElement`）路径的输出规则
      - 不增加 `createVisibleClone()` 次数（全链路仍应仅裁剪一次）
      - 不得修改原页面 DOM（只允许修改可见性 clone）
