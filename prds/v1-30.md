## 具体任务
1、右键菜单 Prompt 支持“选区优先”（Selection > Page），并保持与现有 Chat 跳转策略一致
  - 修改：`src/background.ts`
  - ContextMenu 可用性（使 background 能识别“是否存在选区”）：
    - `updateContextMenu()` 中 `PARENT_MENU_ID` 及其子菜单项的 `contexts` 必须包含 `selection`（同时保留 `page`），确保在用户划词后右键触发时，`chrome.contextMenus.onClicked` 的 `info.selectionText` 可用
  - 点击行为（不新增事件名、不新增依赖、不新增权限）：
    - 当用户点击 `PARENT_MENU_ID` 下的某个 Prompt（`info.parentMenuItemId === PARENT_MENU_ID`）时：
      - 若 `info.selectionText` 存在且 `trim()` 非空：发送消息 `type='PROCESS_SELECTION_WITH_PROMPT'`
      - 否则：保持现有逻辑（整页）发送 `type='PROCESS_PAGE_WITH_PROMPT'` / `type='PROCESS_PAGE_WITH_PROMPT_AND_CHAT'`
    - “自动打开 Chat”对齐：
      - 若应打开 Chat 且目标 Chat 服务可用：在“选区优先”路径下也必须传递 `chatServiceUrl/chatServiceName`（由 content script 在复制成功后执行 `window.open`）
  - 约束：
    - 不改变既有 Prompt 的 `usageCount/lastUsedAt` 更新与保存逻辑（仍在 background 内更新并 `saveSettings`）
    - 不新增任何 telemetry 事件名；仍复用既有 `copy_success/prompt_used`（由 content script 负责记录；且受“匿名使用数据”开关控制）

2、选区 Prompt 补齐“自动打开 Chat”（仅在复制成功后触发；失败不触发）
  - 修改：`src/content/content.ts`
  - 扩展 `message.type === 'PROCESS_SELECTION_WITH_PROMPT'` 分支：
    - 当选区存在时：仍按现有逻辑生成 `finalText = combinePromptWithContent(promptTemplate, content)` 并写入剪贴板
    - 在写入剪贴板成功后：
      - 仍记录 `copy_success` + `prompt_used`，并 `reportSuccessfulCopy({ isPromptUsed: true })`（保持漏斗一致）
      - 若 message 携带 `chatServiceUrl/chatServiceName`：展示一次 `showChatRedirectNotification(chatServiceName)`，并在固定延迟后 `window.open(chatServiceUrl, '_blank')`
    - 失败兜底：任意异常只应导致本次失败提示，不得影响后续复制/Prompt 使用
  - 约束：
    - 不新增任何联网发送；不新增任何包含网页内容/URL/标题/复制内容的落盘字段
    - 延迟与既有页级路径 `PROCESS_PAGE_WITH_PROMPT_AND_CHAT` 保持一致（避免“选区/整页”体验不同步）

3、文档口径与测试/用例闭环（可执行、可回归、可发布）
  - 更新教程口径（避免误导用户）：
    - 修改：`docs/tutorials/prompt-workflow.md`
    - 要求：将“右键 Prompt（页级）”口径更新为“**选区优先**：有选区处理选区，无选区处理整页”，并在步骤中明确如何验证两种路径
  - 新增并维护测试/用例文档：`docs/test-cases/v1-30.md`
    - 文档必须覆盖（至少）：
      - 用例 A（右键 Prompt 选区优先）：在任意页面划词后右键选择某个 Prompt，剪贴板结果仅包含该选区（且包含 Prompt 拼装规则），不应出现整页内容
      - 用例 B（右键 Prompt 无选区处理整页）：取消选区后右键选择同一 Prompt，剪贴板结果为整页提取内容（与 `PROCESS_PAGE_WITH_PROMPT` 行为一致）
      - 用例 C（选区 Prompt 自动打开 Chat）：为该 Prompt（或默认）开启 `复制后自动打开` 且目标 Chat 服务启用；执行用例 A 后应在复制成功后打开新标签页（不自动粘贴）；关闭开关后不应打开
      - 用例 D（隐私校验）：无论选区/整页路径，均不应产生任何联网发送；且本需求不引入新的存储字段
      - 用例 E（匿名使用数据开关回归）：开关关闭时不记录 `copilot_telemetry_events`；开启后选区/整页 Prompt 均记录 `copy_success/prompt_used`（且不包含任何网页内容/URL/标题/复制内容）
      - 自动化回归：记录一次 `bash scripts/test.sh` 结论（命令 + 通过/失败）
