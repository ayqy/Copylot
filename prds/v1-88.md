# 问题修复和功能优化（V1-88 Pro 意向跑数取证闭环：证据包一键导出 + 落盘复盘材料）

## 目标
在不引入支付、不新增权限、不上传用户复制内容的前提下，把“Pro 意向跑数取证”从“需要多次手工导出/拼接”升级为“一键导出可复盘证据包 + 可落盘的复盘材料模板”，让团队可以稳定、可审计地累积 `survey_intent` 样本与漏斗证据，为 S4「收款/订阅链路打通」提供可验证的 go/no-go 决策输入。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、产品内“一键导出 Pro 意向跑数证据包”（收入证据优先）
  - 修改：`src/options/options.html`、`src/options/options.ts`、`_locales/*/messages.json`、（新增）`src/shared/pro-intent-run-evidence-pack.ts`
  - 在 Options -> 隐私与可观测性 ->「Pro 意向漏斗摘要」动作区新增按钮（i18n 文案）：
    - 「下载 Pro 意向跑数证据包（JSON）」
  - 证据包（JSON）必须是可审计、可复盘、可离线分享的单文件，至少包含：
    - `env`：`exportedAt`、`extensionVersion`、`isAnonymousUsageDataEnabled`
    - `proFunnelSummary`：复用现有汇总口径（按 `source=popup|options` 分组 + overall）
    - `proWaitlistSurveyIntentDistribution`：复用现有“问卷意向分布”口径（`survey_intent = count(pro_waitlist_survey_copied)` 等）
    - `proIntentEvents7dCsv`：复用现有“Pro 意向明细（CSV）”导出口径（作为字符串内嵌到 JSON，避免多文件散落）
    - `proIntentWeeklyDigestMarkdown`：复用现有“本周 Pro 意向证据摘要”口径（作为字符串内嵌到 JSON）
  - 隐私与门禁：
    - 匿名开关 OFF 时：证据包 `enabled=false`，必须给出 `disabledReason=anonymous_usage_data_disabled`，且不得包含事件明细/导出 CSV 内容（只允许空串或空数组），避免“看似导出成功但实际泄露/误导”
    - 严格复用 `src/shared/telemetry.ts` 的白名单清洗逻辑；证据包内不得出现 URL/标题/网页内容/复制内容/用户输入的联系方式明文

2、离线落盘与复盘材料（可执行、可复核）
  - 新增脚本（固定入口）：`scripts/build-pro-intent-run-evidence-pack.ts`
    - 输入：第 1 项生成的证据包 JSON（文件路径参数）
    - 输出：生成/更新 `docs/evidence/v1-88/`，落盘至少包含：
      - `index.md`：口径说明、文件清单、sha256（可复算）、“无 PII”断言结论
      - `pro-intent-run-evidence-pack.json`：原样落盘（或带文件名规范化后的等价内容）
      - `sha256.json`：对 `index.md` 与证据包文件进行 sha256 固化（便于后续复盘对比）
    - （可选但推荐）脚本内自动调用 `scripts/build-pro-intent-decision-pack.ts`（以证据包中的 distribution 为输入），并将决策摘要一并落盘到 `docs/evidence/v1-88/decision-pack/`
  - 目录结构必须稳定，且可被后续自动化门禁/复盘脚本复用

3、测试/用例/门禁（必须可执行）
  - 自动化门禁：
    - 更新/新增单测：`scripts/unit-tests.ts`
      - 断言证据包 `disabledReason` 与 `enabled` 的一致性（OFF 时不得包含明细）
      - 断言证据包的关键字段存在且类型稳定（避免口径漂移导致后续脚本/复盘失效）
    - `bash scripts/test.sh` 全量 PASS
  - 新增测试用例文档（固定路径）：`docs/test-cases/v1-88.md`
    - 必须覆盖：匿名开关 ON -> 触发一次 `pro_waitlist_survey_copied` -> 一键下载证据包 -> 离线执行 `scripts/build-pro-intent-run-evidence-pack.ts` -> `docs/evidence/v1-88/` 生成并可复核 sha256
    - 必须覆盖：匿名开关 OFF -> 一键下载证据包 -> 证据包标记 disabled 且不包含明细/CSV
    - 必须包含一次回归记录（PASS/BLOCKED + 导出文件名/证据文件名/截图索引）
