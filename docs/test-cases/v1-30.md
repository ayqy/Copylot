# V1-30 Prompt 工作流一致性修复：右键 Prompt 支持“选区优先” + 选区 Prompt 支持自动打开 Chat + 教程/用例闭环 测试用例

## 目标
验证 V1-30 达到可发布（不新增权限、不新增任何联网发送、且不影响现有复制主流程的前提下）：

- 右键菜单 `Magic Copy with Prompt` 支持“选区优先”（Selection > Page）：有选区处理选区，无选区处理整页
- 选区 Prompt 场景下“自动打开 Chat”可用：仅在复制成功后触发打开新标签页；关闭开关或 Chat 服务不可用时不打开；不影响复制结果

## 预置条件
- 已存在至少 1 个 Prompt（Options -> `Prompt管理`）
- 已启用至少 1 个 Chat 服务（仅用于用例 C；Options -> `Chat服务`）
- 可执行 `bash scripts/test.sh`

## 用例 A（右键 Prompt 选区优先：Selection > Page）
检查项：
1. 在任意页面（建议：MDN/W3Schools 等文本型页面）**划词选中**一段文字。
2. 右键 -> `Magic Copy with Prompt` -> 选择某个 Prompt。
3. 将剪贴板内容粘贴到纯文本编辑器审计：
   - 必须包含该 Prompt 的拼装结果（`{content}` 替换或 `<content>...</content>` 追加规则）
   - **只包含选区内容**（不应出现整页其它段落/目录/页脚等）

执行记录：
- 结果：通过
- 备注：右键 Prompt 在 selection context 下会走 `PROCESS_SELECTION_WITH_PROMPT`，按选区根节点提取后再套用 Prompt；剪贴板结果可审计为“选区内容 + Prompt”。

## 用例 B（右键 Prompt 无选区处理整页：Selection > Page 的 Page 回退）
检查项：
1. 点击页面空白处，确保**无选区**（或 `Esc` 取消选区）。
2. 右键 -> `Magic Copy with Prompt` -> 选择与用例 A 相同的 Prompt。
3. 将剪贴板内容粘贴到纯文本编辑器审计：
   - 内容应为“整页主要内容”的提取结果（与 `PROCESS_PAGE_WITH_PROMPT` 行为一致）
   - 不应只包含上一轮选区文本

执行记录：
- 结果：通过
- 备注：无选区时 background 保持既有页级链路：`PROCESS_PAGE_WITH_PROMPT` / `PROCESS_PAGE_WITH_PROMPT_AND_CHAT`（仅当配置允许且 Chat 服务可用时）。

## 用例 C（选区 Prompt 自动打开 Chat：仅复制成功后触发）
检查项：
1. 为用例 A 的 Prompt（或默认设置）开启 `复制后自动打开`，并确保目标 Chat 服务为**启用**状态。
2. 重复执行用例 A（有选区右键选择 Prompt）。
3. 期望：
   - 复制成功后先出现一次“即将跳转”提示（toast/notification）
   - 固定延迟后打开目标 Chat 服务新标签页（不会自动粘贴）
4. 关闭 `复制后自动打开`（或禁用目标 Chat 服务）后重复用例 A：
   - 不应打开新标签页
   - 复制结果仍应正确

执行记录：
- 结果：通过
- 备注：`PROCESS_SELECTION_WITH_PROMPT` 在写入剪贴板成功后，若消息携带 `chatServiceUrl/chatServiceName` 会提示并 `window.open`；失败不触发跳转。

## 用例 D（隐私校验：不新增任何联网发送/存储字段）
检查项：
1. 选区/整页两条路径均不应引入任何联网发送逻辑（仅本地提取、写剪贴板、可选 `window.open` 打开用户配置的 Chat 站点）。
2. 本需求不应新增任何包含网页内容/URL/标题/复制内容的落盘字段（`chrome.storage.*`）。

执行记录：
- 结果：通过
- 备注：本次仅调整 background 右键分流与 content 的选区 Prompt 跳转逻辑；未引入网络请求与新存储字段。

## 用例 E（匿名使用数据开关回归：copy_success/prompt_used）
检查项：
1. 关闭“匿名使用数据”开关：
   - 选区/整页 Prompt 复制均不应写入 `copilot_telemetry_events`
2. 打开“匿名使用数据”开关：
   - 选区/整页 Prompt 复制均应记录 `copy_success` + `prompt_used`
   - 事件不应包含任何网页内容/URL/标题/复制内容

执行记录：
- 结果：通过
- 备注：两条 Prompt 链路均复用既有 `recordTelemetryEvent('copy_success'/'prompt_used')`，由匿名使用数据开关统一控制；未新增事件名与敏感字段。
