## 用例 A：从 Popup 进入问卷并产生 `source=popup` 的问卷事件

前置：
- 在 Options -> 隐私页，开启「匿名使用数据」开关（ON）。

步骤：
1. 打开 Popup，在 Pro 区点击 “Pro 问卷（1 分钟）”。
2. 验证打开的 Options 页面自动定位到 `#pro-waitlist-survey` 问卷区。
3. 在问卷中任意选择（可不填写自由文本/联系方式），点击「复制问卷（含环境信息）」。
4. 回到 Options -> 隐私页，展开「本地匿名事件日志」与「Pro 意向漏斗摘要」，点击刷新。

期望：
- 事件日志中出现 `pro_waitlist_survey_copied`，且 `props.source === 'popup'`。
- 「Pro 意向漏斗摘要」中 `bySource.popup.counts.pro_waitlist_survey_copied >= 1`。

## 用例 B：一次性归因仅消费 1 次（防口径漂移）

前置：
- 已完成用例 A 的步骤 1-2（通过 Popup 入口打开 Options）。

步骤：
1. 在问卷区连续点击两次「复制问卷（含环境信息）」。
2. 回到 Options -> 隐私页刷新「本地匿名事件日志」与「Pro 意向漏斗摘要」。

期望：
- 第 1 次复制：`pro_waitlist_survey_copied.props.source === 'popup'`
- 第 2 次复制：`pro_waitlist_survey_copied.props.source === 'options'`

## 用例 C：在 Options 内直接复制问卷并产生 `source=options` 的问卷事件

前置：
- 在 Options -> 隐私页，开启「匿名使用数据」开关（ON）。

步骤：
1. 直接打开 Options，并滚动到 `#pro-waitlist-survey`（不通过 Popup 入口）。
2. 点击「复制问卷（含环境信息）」。
3. 展开「Pro 意向漏斗摘要」，点击刷新。

期望：
- `bySource.options.counts.pro_waitlist_survey_copied` 增长，且 `bySource.popup` 不应因为本用例增长。

## 用例 D：匿名开关 OFF 时不记录/不导出

前置：
- 在 Options -> 隐私页，关闭「匿名使用数据」开关（OFF）。

步骤：
1. 重复用例 A 或 C 的操作，尝试复制问卷。
2. 展开「Pro 意向漏斗摘要」并刷新；尝试点击「复制证据包」/「导出过去 7 天 Pro 意向明细（CSV）」等导出动作。

期望：
- 面板提示“未开启，不记录/不导出”（以 UI 文案为准）。
- `pro_waitlist_survey_copied` 计数不增长。

## 用例 E：导出复盘证据并做“无 PII”复核（落盘证据）

前置：
- 完成用例 A，确保至少产生 1 条 `source=popup` 的 `pro_waitlist_survey_copied`。

步骤：
1. 在「Pro 意向漏斗摘要」点击「复制证据包」，将 JSON 保存到 `docs/evidence/v1-87/`。
2. 对落盘文件计算 sha256，并在 `docs/evidence/v1-87/index.md` 记录文件清单与 sha256：
   - `shasum -a 256 docs/evidence/v1-87/*`
3. 用 `rg` 或人工复核导出内容不包含：联系方式原文、使用场景自由文本、其他能力自由文本、网页 URL/标题/复制内容。

期望：
- 证据文件可复盘：能看到 `pro_waitlist_survey_copied` 且 `source=popup`，并可复算 sha256。
- `docs/evidence/v1-87/index.md` 存在，且包含：口径、文件清单、sha256、“无 PII”断言结论。
