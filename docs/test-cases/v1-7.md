# V1-7 商店转化与合规文案 MVP（ASO 话术对齐 + 隐私/数据声明更新）测试用例

## 目标
验证 V1-7 的“商店文案 + 合规材料”最小闭环可交付、可上架复用、且与仓库真实能力一致：
- 话术基准统一且可审计（Slogan/卖点/差异化/隐私口径）
- 短描述（i18n `appDescription`）可直接用于商店展示且口径一致
- 长描述（中英文）可直接粘贴到 Chrome Web Store，且不包含 Markdown 标题语法
- 隐私政策命名/日期/数据口径一致，可支撑 CWS “数据使用披露”填写
- `bash scripts/test.sh` 全量通过，可发布

## 用例 1：话术基准文档完整性与一致性
步骤：
1. 打开 `docs/aso/value-prop.md`。
2. 检查是否包含：
   - 1 句主 Slogan（中英文对照）
   - 3 条核心卖点（中英文对照）
   - 3 条差异化卖点（中英文对照）
   - 隐私声明口径（中英文对照）
3. 抽查表述是否与下列来源一致（禁止夸大/未实现能力）：
   - `README.md`
   - `docs/roadmap.md`
   - `manifest.json`

期望：
- 文档存在且内容齐全；
- 功能描述与仓库一致（不出现“悬停复制图片/视频”等未实现能力）；
- 隐私口径包含：默认本地处理；不收集/不上传复制内容；匿名使用数据默认关闭、开启后仅本地记录且不联网发送。

## 用例 2：短描述（i18n appDescription）口径对齐
步骤：
1. 打开 `_locales/en/messages.json`，检查 `appDescription.message`。
2. 打开 `_locales/zh/messages.json`，检查 `appDescription.message`。
3. 对照 `docs/aso/value-prop.md` 的 Slogan/卖点口径，确认中英文均可直接用于商店短描述展示。

期望：
- 中英文短描述均简洁、聚焦价值，不堆砌；
- 不出现“收集数据/联网分析”等歧义；
- 与话术基准一致；
- `npm run check-i18n` 可通过（在回归脚本中覆盖）。

## 用例 3：长描述（中英文）格式与能力一致性
步骤：
1. 打开 `docs/ChromeWebStore-Description-EN.md`。
2. 打开 `docs/ChromeWebStore-Description-ZH.md`。
3. 校验格式：
   - 不包含 Markdown 标题语法（例如以 `#`/`##` 开头的标题行）
   - 排版适合直接粘贴到 CWS（纯文本分段、信息结构清晰）
4. 校验内容：
   - 功能点均为已实现能力，且与 `README.md` 描述一致
   - 隐私段落包含：匿名使用数据默认关闭且仅本地记录、不联网发送

期望：
- 两份长描述结构一致且可直接用于上架；
- 不出现与实现不符的能力描述；
- 隐私与权限说明明确、无歧义。

## 用例 4：关键词矩阵与标题建议
步骤：
1. 打开 `docs/aso/keywords.md`。
2. 检查是否至少包含：
   - EN：关键词组 + 3 套标题/副标题组合
   - ZH：关键词组 + 3 套标题/副标题组合

期望：
- 文档存在且内容满足最低要求；
- 关键词与标题口径与 V1-7 话术一致（不过度承诺）。

## 用例 5：隐私政策命名/日期/开关语义校验
步骤：
1. 打开 `docs/privacy-policy.md`。
2. 检查：
   - 历史命名已统一为 Copylot
   - “Last Updated” 日期已更新
   - 明确说明本地/浏览器存储范围（settings/prompts/growth_stats/telemetry_events）
   - “匿名使用数据”开关语义正确：默认关闭；开启后仅本地记录；关闭后清空；不联网发送
   - 不出现“上传复制内容/联网分析复制内容”等冲突表述

期望：
- 文案可支撑 CWS 数据披露填写，且与实际实现一致。
