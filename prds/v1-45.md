# 问题修复和功能优化（V1-45 发布取证准备包（离线）：上架前核对清单 + 上架后 24h/7d 复盘模板 + 可审计落盘路径）

## 目标
基于 `docs/roadmap_status.md` 的收入优先 Top3：Top1「真实 CWS 发布 + 商店端取证」与 Top2「真实跑数取证与对比」当前仍被“网络可达性（需可用代理/VPN）”阻塞，因此本轮选择 Top3「商店可达前的发布取证准备」作为 1~3 天最小可交付增量主题。收入第一：把“真实发布/取证/跑数对比”从临场发挥变成**可复制的最短路径**——在不新增权限、不引入支付/订阅、不改动现有用户侧功能逻辑的前提下，固化上架前核对清单、上架后 24h/7d 复盘模板、以及可执行的测试用例与证据落盘路径，确保一旦代理/VPN 可用即可在最短时间拿到“商店端 + 转化漏斗”的可审计商业化证据。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 具体任务
1、固化 CWS 上架前核对清单（离线可执行、可审计、可复用）
  - 新增并填写发布前核对清单文档（固定路径建议）：`docs/publish/cws-preflight-checklist.md`
  - 核对清单必须覆盖并给出“可执行命令/可核验文件路径”：
    - 版本与产物一致性：`manifest.json`、`dist/manifest.json`、`plugin-*.zip` 版本一致；`bash scripts/test.sh` 通过；`bash scripts/verify-prod-build.sh` 通过（权限白名单）。
    - 发布脚本门禁：`npm run publish:cws -- --dry-run` 可通过，并包含稳定的 Proxy Diagnostic Block（可复制、无敏感信息）。
    - 商店物料一致性：`docs/ChromeWebStore-Description-EN.md` / `docs/ChromeWebStore-Description-ZH.md`、`docs/aso/store-assets.md`（截图脚本）、`docs/aso/keywords.md`（关键词）之间的口径一致；无“未实现能力”夸大。
    - 隐私口径一致：`docs/privacy-policy.md` 与 Options/商店描述中“本地处理/不上传复制内容/匿名使用数据默认 OFF”一致。
    - 转化入口可核验：商店描述与扩展内 Pro 候补入口均存在且不暗示“Pro 已上线”；所有 CWS/评价入口 UTM 参数可核验（以当前实现口径为准，至少包含 `utm_source/utm_medium/utm_campaign`）。

2、准备“真实上架后 24h/7d”对比复盘模板（收入证据链：转化 + 口碑 + 可观测性）
  - 新增复盘模板文档（固定路径建议）：`docs/growth/post-release-review-template.md`
  - 模板必须能直接复用在“真实发布完成后”的 24 小时与 7 天复盘，并包含：
    - 商店端取证清单：版本号、发布时间、核心 CTA（Pro 候补/升级入口）可见性的截图命名规范与索引位置。
    - 可量化指标口径（与本地导出一致）：Pro 意向漏斗（entry/opened/copied + 转化率）、WOM 摘要（share/rating prompt 的分来源计数与转化率）、以及“对比基线”的引用（`docs/evidence/v1-42/`、`docs/evidence/v1-44/`）。
    - 数据采集动作：从商店安装回归后如何在 Options -> 隐私页导出「Pro 意向漏斗摘要/证据包」「WOM 摘要/证据包」，并把导出文件落盘为证据目录（见任务 3）。

3、用例/证据/简报落盘闭环（确保一旦网络可达即可最短路径取证跑数）
  - 新增测试用例文档（固定路径）：`docs/test-cases/v1-45.md`
    - 用例必须覆盖：离线可执行项（核对清单逐条核验、`npm run publish:cws -- --dry-run`、`bash scripts/test.sh` 回归记录）+ 网络可达后的真实动作（`npm run publish:cws`、商店端截图取证、从商店安装回归、导出摘要/证据包并落盘）。
  - 新增证据目录（固定路径）：`docs/evidence/v1-45/`
    - 至少包含：证据索引 `docs/evidence/v1-45/index.md`（含截图命名规范、导出文件清单、以及与 v1-42/v1-44 的对比口径说明）。
  - 新增交付简报（固定路径建议）：`docs/reports/v1-45-report.md`
    - 简报必须说明：本轮把哪些“发布/取证/复盘”动作固化为可执行模板；当前阻塞点（代理/VPN）；以及下一步一旦网络可达的最短执行路径。
