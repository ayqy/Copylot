# V1-9-1 可见性克隆纯空白文本节点保留修复（避免文本粘连）

## 目标
修复 `src/shared/dom-preprocessor.ts` 的 `createVisibleClone()` 在非 `pre/code` 场景丢弃“纯空白文本节点”导致的词语粘连问题（例如 `Hello world` 变 `Helloworld`），并在保持输出干净的前提下仅在必要时补回单个空格。

## 具体任务
1. 在 `src/shared/dom-preprocessor.ts` 中为非 `pre/code` 场景的“纯空白 TEXT_NODE（text.trim()===''）”引入上下文判断：
   - 仅当该空白节点位于需要分隔的上下文时，保留为单个空格 `' '`（不保留原始缩进/换行）。
   - 其它场景继续丢弃，避免引入噪音空白。
2. 新增 helper，便于 code review 与未来单测：
   - `isBlockSeparatorTag(tagName)`
   - `shouldPreserveWhitespaceTextNode(prev, next)`
3. 保持 `pre/code` 内逻辑不变：继续保留所有空白与换行（现有行为）。

## 验收标准
- 非 `pre/code` 场景不再发生“因缩进/换行空白节点被删除导致词语粘连”
- 仅在必要时补回单个空格，不引入明显脏空白
- 不引入新依赖；不为每个节点新增 `getComputedStyle` 调用
