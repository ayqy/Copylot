# V1-107 Popup Pro Intent Prefill Funnel

## 背景

基于 Copylot 最近一轮真实增长与 SEO 复盘，当前增长主问题之一已从“Google 数据不可得”转移为“站内 Pro 承接过于空白”。现有 Popup 只有升级 Pro / 问卷 / 候补三个入口，用户需要自己从零填写问卷，导致高意向样本信号偏弱。本轮在不引入联网埋点、不改官网代码、不增加权限的前提下，补一个 Popup 内的高意向直达链路：用户先表达最想要的能力，再带着预选场景与能力标签落到 Options -> Pro 问卷，降低空表单流失。

## 目标

在扩展内实现 Popup -> Pro survey 的预填直达链路，并把预填样本沉淀为本地可审计证据字段，用于后续判断哪一类 Pro 需求最值得优先实现。

## ULTRA IMPORTANT 第一原则：任务开始后，不要停下来，直到完成目标
**所有细节，都由你来定，判断依据是根据核心目标决定应该如何做，而不要停下来问我**

## 结合项目代码的需求澄清

1. 仓库是浏览器扩展仓库，不直接承载 copy.useai.online 官网页面；本轮只改扩展内 Popup / Options / shared / tests / docs。
2. 现有 Pro 承接链路已经具备 Popup survey deep link、Options survey copy/open、v1-100 漏斗导出与 telemetry 白名单约束。
3. 本轮必须沿用现有本地 telemetry 体系，不新增联网发送，不记录网页内容、URL、标题或用户原文。
4. 新增预填能力需要兼容 E2E build、prod build、zip 安装回归和 i18n 校验。

## 具体任务

1. 新增共享预填协议模块，定义 `pro_survey_prefill` query 参数的编码/解码/清洗逻辑，允许携带 use case 和受限能力枚举。
2. 在 Popup 中新增“你最想先解决什么”意向卡片，至少覆盖 advanced_cleaning / batch_collection / prompt_pack / note_export 四类能力；点击后打开 `options.html#pro-waitlist-survey`，并携带一次性 `pro_survey_source=popup` 与预填参数。
3. 在 Options 中解析预填参数并自动写回 Pro survey 表单，只允许写入 use case 与能力复选框；解析后清理 URL，避免后续误归因。
4. 扩展 `pro_waitlist_survey_copied` telemetry 白名单，新增 `prefill_used` 与 `prefill_capability_count` 字段，用于标记样本是否来自 Popup 预填直达。
5. 补齐中英文 i18n 文案、单元测试、Popup/Options E2E 与 telemetry 文档说明，确保 `npm run type-check`、`npm run check-i18n`、`node --loader=ts-node/esm scripts/unit-tests.ts`、`npm run build:e2e`、目标 Playwright spec 通过。
6. 同步更新版本与简报，确保产物版本、roadmap 状态与本轮交付主题一致。

1. Popup 新增意向卡片不影响现有 Convert、Manage Prompts、Share/Feedback/Rate 主路径。
2. 通过 Popup 意向卡片进入 Pro 问卷时，use case 与对应能力复选框会被自动预填，且 `pro_waitlist_survey_copied` 可导出 `prefill_used=true`。
