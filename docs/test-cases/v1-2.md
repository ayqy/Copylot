# V1-2 测试用例（Popup 新手引导 MVP）

## 范围
- Popup：首次打开三步新手引导（可跳过、可回看、状态持久化）
- 引导内：一键应用推荐配置（仅基础开关，不影响 Prompt/Chat 等存量数据）
- 引导内：跳转 Options（`chrome.runtime.openOptionsPage()`）
- i18n：中英文展示
- 零打扰：不影响 Popup 现有入口可用性（AI Convert / 管理 Prompts / 反馈 / 分享 / 去评价）

## 运行自动化检查
```bash
./scripts/test.sh
```

## 手工用例

### TC-01 首次打开 Popup 自动展示三步引导（未完成态）
前置：
- 新装扩展，或清空 `chrome.storage.sync` 中的 `copilot_settings`
步骤：
1. 打开插件 Popup
期望：
- Popup 内自动展示引导 Modal（不新开 Tab）
- 显示步骤进度 `1/3`
- 可见：下一步 / 跳过 / 关闭（X）

### TC-02 跳过/完成后不再自动弹出（完成态持久化）
步骤：
1. 在引导中点击“跳过”或走到第 3 步点击“完成”
2. 关闭 Popup
3. 再次打开 Popup
期望：
- 引导不再自动展示

### TC-03 重新查看引导入口可用（可回看）
步骤：
1. 打开 Popup
2. 点击“新手引导 / Quick Start Guide”（稳定入口：`#popup-onboarding-reopen`）
期望：
- 强制打开引导 Modal
- 可正常上一页/下一页切换

### TC-04 Step1：一键应用推荐设置立即生效（不影响存量数据）
步骤：
1. 打开引导第 1 步
2. 点击“应用推荐设置 / Apply recommended settings”
3. 关闭引导（跳过或完成均可）
4. 在 Popup 中核对相关开关状态
期望（必须生效）：
- 交互模式为 `click`
- 悬停触发为关闭（Hover Trigger off）
- 输出格式为 `markdown`
- 表格格式为 `markdown`
- 附带标题为关闭
- 附带 URL 为关闭
期望（回归/不破坏）：
- Options 页中的 Prompt 列表、Chat 列表等存量数据不被覆盖/清空

### TC-05 Step2：追加模式说明与对应开关位置不冲突
步骤：
1. 打开引导第 2 步
2. 确认文案提到 Shift 追加与 Popup 内 “Append Mode” 开关
3. 关闭引导后，手动切换 “Enable Append Mode” 开关
期望：
- 文案存在且中英文可切换
- 开关切换可正常保存，不受引导影响

### TC-06 Step3：打开管理 Prompts 跳转 Options 页
步骤：
1. 打开引导第 3 步
2. 点击“打开管理 Prompts / Open Prompt Manager”
期望：
- 打开 Options 页（`chrome.runtime.openOptionsPage()`）

### TC-07 i18n 展示（至少中英文）
步骤：
1. 将浏览器 UI 语言切换为中文，打开 Popup
2. 检查引导入口、引导标题/按钮文案为中文
3. 将浏览器 UI 语言切换为英文，打开 Popup
4. 检查上述文案为英文
期望：
- 引导入口与引导内容随 UI 语言切换

### TC-08 零打扰校验：现有入口可用性不受影响
步骤：
1. 首次自动弹出引导时，点击“跳过”
2. 依次验证 Popup 内现有入口可用：
   - “AI Convert”
   - “管理 Prompts”
   - “反馈与建议 / Feedback & Suggestions”
   - “分享 / Share”
   - “去评价 / Rate”
期望：
- 引导可随时关闭/跳过
- 关闭后不影响任何既有按钮/入口正常使用
