# v4-9 领先路线回写包

## 背景

`v4-8` 已经把三条 Pro 验证路线的真实打开与复制样本拉到同一张记分板上，但判断仍停留在内部摘要里，没有直接变成路线页、商店说明和下一轮汇总可复用的对外文案。

## 本轮目标

1. 新增共享纯逻辑，把当前领先路线整理成“路线页 / 商店说明 / 汇总”三段回写包。
2. 在 `Options -> Pro` 新增回写包入口，支持复制 Markdown 与下载 JSON。
3. 保持当前边界：只做路线验证与文案回写，不做支付、候补、问卷或收款实现。

## 验收口径

1. `Options -> Pro` 可见“领先路线回写包”区块、复制按钮和下载按钮。
2. 回写包基于当前领先路线生成，且输出包含路线页 headline、proof、boundary，商店说明 shortDescription、bullet、boundary，以及 summary judgement / nextStep。
3. `bash scripts/test.sh` 通过，覆盖 shared 逻辑、Options 入口、UI 集成、Playwright 和 determinism 门禁。
