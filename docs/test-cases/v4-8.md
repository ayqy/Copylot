- `scripts/ui-integration-tests.ts`：`Options -> Pro` 中“三条路线样本比较”摘要复制动作。

## 用例 A：Pro Tab 出现三路线样本比较区块

步骤：

1. 打开 `src/options/options.html#pro`。
2. 检查 `#pro-route-comparison-panel`、`#copy-pro-route-validation-comparison-summary`、`#download-pro-route-validation-comparison-json`。

期望：

- 新区块可见。
- 两个动作按钮存在且默认可点击。
- 隐私提示明确说明不包含页面正文、复制内容、URL、标题或联系方式。

## 用例 B：复制出的摘要包含三路线记分板

步骤：

1. 在 `Options -> Pro` 点击“复制路线比较摘要”。
2. 读取最近一次剪贴板文本。

期望：

- 文本包含 `V4-8 三条路线样本比较摘要`。
- 文本包含三条路线标题中的至少一条，以及 `route_opened`、`validation_route`、`validation_brief`、`validation_checklist`、`total_signals`。
- 文本不包含页面正文、复制内容、URL、标题或联系方式。

## 用例 C：下载出的 JSON 可复核

步骤：

1. 点击“下载路线比较 JSON”。
2. 打开生成文件并检查关键字段。

期望：

- 文件名符合 `copylot-pro-route-validation-comparison-v4-8.json`。
- 文件内包含 `campaigns`、`tracks`、`leadingTrackId`、`signalGap`。
- 若匿名数据关闭，结果仍可导出，但必须回到 `leadingTrackId = none`。

## 用例 D：状态文档与当前边界一致

步骤：

1. 复核 `docs/roadmap.md`、`docs/roadmap_status.md`、`docs/reports/v4-8-report.md`。

期望：

- Roadmap 明确最后一项已进入“路线样本比较导出已完成，后续只剩最清晰样本回写”的状态。
- 状态文档明确当前仍未进入支付/收款实现，只是把三路线样本比较变成了可导出的产品能力。
