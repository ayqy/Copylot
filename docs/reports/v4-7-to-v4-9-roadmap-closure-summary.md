# v4-7 到 v4-9 三轮闭环总结

## Roadmap 进展

- 当前阶段：S3 / G3 `Pro 路线验证`
- 当前进度：S0 `6/6`，S1 `4/4`，S2 `4/4`，S3 `3/4`，全局 `17/18`
- 这 3 轮的实质性里程碑：
  - `v4-7`：把收费前 A/B/C 门槛判断从离线脚本推进成 `Options -> Pro` 内可导出的产品能力
  - `v4-8`：把三条验证路线的真实打开与复制信号拉到同一张比较记分板上
  - `v4-9`：把当前领先路线整理成可直接回写到路线页、商店说明和下一轮汇总的三段文案包

## 每轮进展、结果、观察

### 第 1 轮：`v4-7`
- 提交：`91cc04e`
- 结果：`Options -> Pro` 新增“收费前门槛判断”区块，可直接复制 Markdown 摘要或下载 JSON。
- 观察：此前是否该进入收费评估只能靠脚本或人工读样本；这一轮把判断口径产品化之后，后续每轮都能复用同一条门槛线。

### 第 2 轮：`v4-8`
- 提交：`fd4f68d`
- 结果：`Options -> Pro` 新增“三条路线样本比较”区块，把 `advanced_cleaning / bulk_collection / structured_export` 的真实打开与复制样本放到同一张记分板。
- 观察：这一步第一次明确看清当前领先路线是 `advanced_cleaning`，说明路线验证不能再只看“功能愿望”，必须回到真实样本强弱。

### 第 3 轮：`v4-9`
- 提交：`49e12ce`
- 结果：`Options -> Pro` 新增“领先路线回写包”区块，可直接复制 Markdown 回写包或下载 JSON，把领先路线变成路线页、商店说明和下一轮汇总三段对外文案。
- 观察：这一步把“内部判断”推进成“可对外复用的表达”，但仍然只是验证能力，不等于已经进入支付或收费实现。

## 复盘

- 做对的事：
  - 三轮都沿着同一条闭环推进，没有扩散到表单、候补、支付或其他偏航入口。
  - 每一轮都把上一轮的判断继续向前推进一层，从门槛判断，到路线比较，到领先路线回写，形成了连续的 milestone。
  - 全量回归已经覆盖 shared 逻辑、Options 入口、UI 集成、Playwright 和 determinism 门禁，产物可复核。
- 暴露的问题：
  - 软件工厂入口每轮都卡在 `sub_prd_plan`，这让“自动闭环”没有真正恢复，仍需人工接管实现与验证。
  - 当前领先路线仍只是最近样本领先，样本规模还不足以支持直接进入收费实现。

## 重新思考后的优化方向

1. 停止继续扩新路线，改为持续放大 `advanced_cleaning` 的真实任务样本，验证领先是否稳定。
2. 把“路线样本比较摘要 + 领先路线回写包 + A/B/C 门槛摘要”作为固定三联口径，避免单一指标误判。
3. 继续把所有对外文案锁在“路线验证中”边界内，直到真实样本与门槛摘要同时支持下一步收费评估。

## 接下来最重要的 3 件事

1. 继续收集 `advanced_cleaning` 的真实打开与复制样本，确认领先不是短期波动。
2. 用三联口径重新复核当前是否仍应停在验证阶段，而不是提前推进支付或收款。
3. 把当前领先路线的回写包继续同步到下一轮 roadmap 汇总和商店话术审计里，保持内外口径一致。
